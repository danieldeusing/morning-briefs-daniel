vLLM 0.21.0 (heute) — CUDA 13 / PyTorch 2.11 baseline

0.20.0 letzte Woche brachte TurboQuant (2-Bit KV-Cache, 4× Kapazität) und FA4 als Default für MLA-Prefill. 0.21 stabilisiert den CUDA-13-Pfad und schließt den DeepSeek-V4-Support ab. Für Self-Host-Setups die solideste Release seit Monaten.

TurboQuant: Wie der KV-Cache (also das „Kurzzeitgedächtnis" eines LLM für die laufende Conversation) auf 2 Bits pro Wert geschrumpft wird. Statt 16 Bits pro Eintrag → 2 Bits → 8× kleiner; vLLM gewinnt damit 4× mehr Kontext pro GPU-RAM-Einheit.

Bench-Update · skeptisch lesen

Die Wahrheit zu SWE-Bench Verified ist seit Februar bekannt: OpenAI berichtet es nicht mehr, Trainings-Kontamination ist real, Top-Modelle reproduzieren Gold-Patches wörtlich. Die ehrlichere Sicht heute steht in zwei anderen Leaderboards.

Scaffold-Varianz Scale AIs SEAL-Scaffold liefert 10+ Punkte weniger als die Custom-Scaffolds der Modell-Anbieter (Opus 4.5: 45,9 % auf SEAL vs. 50,2–55,4 % mit eigenem Scaffold auf SWE-Pro). Die Bench misst nicht das Modell — sie misst Modell + Scaffold + Tool-Stack. Bei jedem Vergleich zuerst nach Scaffold fragen.

Auf SWE-Bench Pro (Stand 13.05., quelle labs.scale.com) ist die Reihenfolge:

SWE-Bench Pro — Top 6, Stand 13.05.2026 (Scale AI)

Mythos Preview ist 13,5 Pkt. vor Opus 4.7, dann ein Pulk von GPT-5.4/5.5/5.3-Codex in ein paar Punkten Abstand. Mythos ist nicht self-serve — Invitation-only für Defense-Cyber. Die spannende Zahl bleibt die Lücke Verified vs Pro beim selben Modell: Mythos 93,9 % vs 77,8 % (16 Pkt.), Opus 4.7 87,6 % vs 64,3 % (23 Pkt.). Verified sagt dir, wie gut das Modell Gold-Patches imitiert; Pro sagt dir, was bei echter Schwierigkeit übrig bleibt.

Nebius' SWE-rebench (contamination-aware, Time-Window-Filter) zeigt einen anderen Sieger: Opus 4.6 65,3 %, GLM-5 62,8 %, GLM-5.1 62,7 % — Top 3 in 2,6 Punkten. Saturation-Warnung. LiveCodeBench widerspricht sich diese Woche selbst je nach Mirror: LLM-Stats hat DeepSeek-V4-Pro-Max bei 0,935, PricePerToken hat Gemini 3 Pro bei 91,7 %, v6 hat Kimi K2.6 bei 0,896. Die Geschichte ist nicht „wer führt", sondern „dass die Tabellen sich widersprechen". Wenn dir jemand mit einer einzelnen LiveCodeBench-Zahl winkt, frag nach Mirror und Datum.

Ab 15.06.: Pro-User ($20/Monat) bekommt einen separaten Credit-Topf für Agent-SDK, claude -p, Claude-Code-GitHub-Actions, Third-Party-Agents. Max 5× ($100), Max 20× ($200), Enterprise per-Seat. MCP-Ökosystem · Sicherheits-Druck trifft Spec-Momentum

AgentSeal-Doppelschlag: Registry-Scan + Runtime-Validierung

Vor zwei Tagen: 1.808 von 8.013 gescannten MCP-Servern haben Findings — 1.067 Critical, 1.795 High, 1.240 Medium. Häufigste Critical-Klasse: Command Injection (114), dann Output Instruction Injection (32), Path Traversal (5), SSRF via unvalidated Connection URLs (4), Sandbox-Escape via Tool-Bridge (4). Gestern dann der Hammer: bei sechs hochpopulären Servern (zusammen 68 K Stars) reproduzierten sie 27 von 28 testbaren Critical/High-Findings real exploitable in isolierten Docker-Containern mit Test-Daten. Beispiel-Kette: lokaler Server mit .env-Credentials + permissive CORS + SSRF-Tool → bösartige Webseite zieht AWS-IMDS-Credentials.

Achtung Heute Mittag durchgehen, welche MCP-Server du im PATH hast — und welche davon Credentials in .env halten und Outbound-HTTP machen dürfen. Quick-Audit: claude mcp list, dann jeden Server gegen die AgentSeal-Findings-Liste.

Standing Threads

Harness-Vergleich: was hat sich verschoben?

Vor einer Woche war der Pitch noch: Claude Code = Default-Terminal-Tool, Cursor = beste IDE-UX, Aider = Open-Source-Klassiker, Codex = serverseitiges Async. Jetzt: Cursor schiebt sich Richtung Cloud-Plattform (Envs, Teams, Bugbot), Codex landet auf dem Handy, GitHub Copilot will die eigene App sein, Devin 2.0 verkauft sich für $20. Der breite Trend ist deutlich: weg vom Inline-Autocomplete, hin zum delegierbaren Agent mit eigener Sandbox. Drei der vier großen Tools (Cursor, Codex, Devin) haben diese Woche „eigene Cloud-Env" oder „eigene IDE" geshippt.

Open-Weights-Coding: Saturation auf Verified, Lücke auf Pro

Qwen3.6-27B (77,2 % Verified) liegt 3,7 Pkt. unter Opus 4.6. Aber auf SWE-Bench Pro (53,5 % vs 64,3 % Opus 4.7) klafft eine 11-Punkte-Lücke. Das ist der ehrliche State: für „passt-in-meinen-Mac"-Workflows ist Qwen weit gekommen; für „kann selbst harte Tasks lösen" sind Closed-Frontier-Modelle weiter vorne. Ein offener 1,6 T-Modell-Drop wie DeepSeek-V4-Pro schließt die Lücke auf Verified — aber zum Self-Host-Preis, den niemand außerhalb von Mega-Clustern bezahlen kann.
