Korrektur · JetBrains ACP ist Client, nicht Control

Eine wichtige Berichtigung zu gestern. Im Samstags-Brief war die Rede von JetBrains „Agent Control Protocol" als „vendor-neutraler Agent-Anbindung, konkurriert mit MCP". Beides ist falsch.

Was ACP tatsächlich ist

ACP steht für Agent Client Protocol. Es ist co-entwickelt mit Zed, nicht JetBrains-only und nicht als Rivale gegen MCP positioniert. Der Punkt: MCP regelt die Kommunikation zwischen Agent und Tools/Daten („hole mir aus dieser Datenbank das Schema"). ACP regelt die Kommunikation zwischen Editor und Agent („User hat in Zeile 42 cmd-K gedrückt, hier sind Cursor-Position und ausgewählter Code"). Die zwei Protokolle leben auf verschiedenen Layers, sie konkurrieren nicht.

JetBrains ist Teilnehmer in der ACP-Registry seit Januar 2026; Cursor ist im März dazugekommen; der eigentlich frische Punkt diese Woche ist die ReSharper-2026.2-EAP-Ankündigung vom 11. Mai: ACP läuft jetzt in Visual Studio via Junie — der erste Host ausserhalb Zed und JetBrains-IDEs.

agentclientprotocol.com · blog.jetbrains.com/dotnet
User in Editor → ACP → Agent → MCP → Tools / Daten
Lesson learned Bei Drei-Buchstaben-Protokollnamen mit „Agent" davor: einmal zuviel doppelchecken. ACP, A2A, MCP, ANP, AGNTCY — alle drei Wochen erfindet jemand ein neues. Im Zweifel auf die Spec-Site gehen statt auf einen Recap.

Benchmark-Realität · Verified vs Pro [1][2][3]

Die Anbieter zitieren SWE-Bench Verified, weil dort Modelle 80–93 % erreichen und das auf der Bühne gut aussieht. OpenAI selbst hat Verified im Februar zurückgegeben, mit der Begründung: „alle Frontier-Modelle sind kontaminiert" und „59,4 % der Hard-Tasks haben fehlerhafte Tests". Wer realistisch vergleichen will, schaut auf SWE-Bench Pro — die Nachfolger-Suite, die schwerer zu memorieren ist.

Benchmark-Kontamination: wie wenn ein Kandidat die Klassenarbeits-Aufgaben aus dem Test-Vorjahr beim Lernen versehentlich mitgenommen hat. Er löst die Aufgaben „mit System", aber das System ist Auswendiglernen, nicht Können. Bei LLMs heisst das: die Bench-Aufgaben sind ins Trainings-Datensatz gerutscht. Hohe Scores in Verified, magere Scores in Pro = wahrscheinlich Kontamination.

Zweitens: eine Korrektur zum Samstag. Im gestrigen Brief stand, JetBrains habe ein Agent Control Protocol vorgestellt, das mit MCP konkurriert. Beides ist falsch. ACP heisst Agent Client Protocol, ist mit Zed co-entwickelt, und sitzt auf einer anderen Schicht als MCP. MCP regelt, wie ein Agent mit Tools und Daten redet. ACP regelt, wie ein Editor mit einem Agent redet. Die zwei Protokolle ergänzen sich, statt sich zu konkurrieren. Das aktuell relevante Stück: JetBrains hat am elften Mai mit ReSharper 2026.2 EAP angekündigt, dass ACP jetzt auch in Visual Studio läuft, via Junie. Das ist der erste Host ausserhalb Zed und JetBrains-eigenen IDEs.

OpenAI selbst hat Verified im Februar zurückgegeben, weil sie ihre Frontier-Modelle als kontaminiert eingestuft haben.
