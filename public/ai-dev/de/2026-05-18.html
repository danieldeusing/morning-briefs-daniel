OpenAI · Codex zieht aufs Handy, Remote-SSH wird GA [1][2][3]

OpenAIs Donnerstag-Drop (14.05.) ist die strukturell wichtigste Bewegung der Woche. Codex landet als Steuersurface in der ChatGPT-App auf iOS und Android — über alle Pläne hinweg, einschliesslich Free und Go. Wichtige Einschränkung: das Handy verbindet sich zur Codex-Session, die auf deinem Mac (Codex-Desktop-App) läuft. Windows-Support angekündigt, undatiert.

Was das praktisch ändert

Bisher war die Approval-Schleife für autonome Agenten ein Argument gegen lange Laufzeiten — man wollte nicht eine Stunde vor dem Terminal sitzen, um „yes" zu tippen. Mit der Mobile-Anbindung kannst du den Agent auf der Mini im Büro laufen lassen, im Zug ein „run the migration" bestätigen, einen Diff genehmigen, Test-Output durchscrollen. Live-Project-Context, Plugins, Screenshots, Terminal-Output, Diffs, Test-Resultate — alles auf dem Handy.

Skeptische Note: bei Free/Go läuft das auf OpenAI-Infrastruktur, bei Plus/Pro lokal — das Privacy-Modell zwischen den Tiers ist nicht gut dokumentiert. Wer für Kunden arbeitet, klärt das vor dem ersten Run.

Anthropic · /goal macht aus der CLI einen Long-Horizon-Agent

Claude Code v2.1.139 ist seit Mo 12.05. draussen, hat sich übers Wochenende verbreitet und ist das Counter-Story zu Codex-Mobile: statt am Workflow drumherum (Steuerung, Approval) zu schrauben, schraubt Anthropic am Agent-Loop selbst.

Was /goal tut

Du schreibst eine Erfolgs-Bedingung — „alle CI-Tests im Repo grün" oder „Endpoint /orders hat eine OpenAPI-Spec". Der Agent arbeitet dann über mehrere Turns hin, und ein kleines Prüf-Modell (Haiku per Default) liest jeweils das Transcript und prüft, ob die Bedingung erfüllt ist. Wenn ja, stoppt. Wenn nein, weiter. Funktioniert in interaktivem Modus, in -p und in Remote-Control. Live-Overlay zeigt Elapsed-Time, Turn-Count und Token-Verbrauch.

Das ist der gleiche Loop, den Devin und OpenHands seit Monaten haben — Anthropic kommt spät, aber sauber. Der Trick ist die Trennung von Worker und Judge: das billige Haiku prüft, das teure Opus arbeitet.

github.com/anthropics/claude-code · code.claude.com/docs/en/goal

Agent-View — endlich das Dashboard, das gefehlt hat

claude agents öffnet eine einzige Liste aller Claude-Code-Sessions im System: laufend, auf User-Input wartend, fertig. Sortierbar, durchklickbar. Wenn du drei Tasks parallel auf drei Repos laufen lässt — exakt der Workflow, den die /goal-Logik erst sinnvoll macht — ist das Dashboard die Voraussetzung, dass du den Überblick behältst.

Try-this Wenn du Claude Code nutzt: npm i -g @anthropic-ai/claude-code@latest, dann auf einem überschaubaren Bug-Fix-Task /goal "alle tests grün und keine flake-warnings" probieren. Schaut ihm 15 Min zu, ob das Judge-Modell vernünftig stoppt — die kritische Frage ist nicht „kann der Worker das", sondern „weiss der Judge, wann fertig fertig ist".

Cursor 3.4 · Dockerfile-Environments für Cloud-Agenten

Cursor hat am Di 13.05. die Hauptbewegung der Woche im Tool-Layer gemacht: v3.4 bringt development environments for cloud agents — Dockerfile-basierte Env-Konfiguration mit Build-Secrets, Multi-Repo-Environments (bauen auf den bestehenden Multi-Root-Workspaces auf), 70 % schnelleres Layer-Caching, Per-Environment-Audit-Logs + Version-History, Scoped-Egress/Secrets. Zusätzlich am 11.05.: Microsoft Teams-Integration — @Cursor in einem Channel mentionen, der Cloud-Agent übernimmt.

Warum das die wichtige Cursor-Bewegung 2026 ist

Cloud-Agents waren bis vor Quartalen ein „magic black box"-Layer mit unklarer Reproducibility. Mit Dockerfile-Definition wird die Env zur First-Class-Citizen — du kannst die gleiche Env lokal builden, in CI verwenden, im Cloud-Agent laufen lassen. Das senkt die Risk-Hürde, von Local zu Cloud zu wechseln. Wer eine Flotte paralleler Cloud-Agents fahren will (Code-Reviews, Test-Generierung, Migration-Helper), bekommt damit das Audit-Log obendrauf.

Zed 1.2.6 · ChatGPT-Abo als Agent-Provider — die kleine grosse Idee

Cline · der Agent-Runtime wird OSS — @cline/sdk

Was am 14.05. passiert ist

Cline hat seinen internen Agent-Harness als Apache-2.0-TypeScript-SDK extrahiert (@cline/sdk). Cline-CLI, -Kanban und die IDE-Extensions werden jetzt darauf umgebaut. Das SDK liefert den vollen Loop: LLM-Calls, Tool-Orchestration, Session-Persistence, Multi-Agent-Koordination, Scheduling. Plus: --worktree-Flag, das automatisch ein frisches git-worktree unter ~/.cline/worktrees/ anlegt — sauber isolierte Task-Läufe ohne Working-Tree-Konflikte.

Damit hat das OSS-Lager jetzt drei ernstzunehmende Agent-Runtimes: Anthropic Claude Agent SDK (Sept-2025-Pivot), OpenHands SDK (All-Hands), Cline SDK. Wer als Drittentwickler einen domain-spezifischen Coding-Agent bauen will, hat damit echte Wahl statt vendor-Lock-in.

marktechpost.com

MCP-Security · drei Datenbank-Server-CVEs auf einmal [7][8][9]

Bug-Hunter Konstantin Burov (Akamai) hat am Mi 13.05. drei MCP-Datenbank-Server gleichzeitig gemeldet. Pattern: alle drei führen Tool-Argumente in SQL-/Query-Kontext, ohne sauber zu validieren. Klassisches LLM-Tool-Validation-Loch — der Agent kann „freundliche" Argumente bekommen, die wie Daten aussehen und wie Query-Code wirken.

Was genau betroffen ist

  • Apache Doris MCP Server — CVE-2025-66335, SQL-Injection via unvalidiertem db_name in exec_query. Kein Auth nötig. Fix in 0.6.1, jetzt upgraden.
  • Alibaba RDS MCP — Metadata-Exfiltration. Patch in Arbeit.
  • Apache Pinot MCP — Takeover-Pfad. Vendor lehnt Patch vorerst ab, Empfehlung: server abdrehen oder isolieren.
theregister.com · akamai.com · nvd.nist.gov
Achtung Wer einen MCP-Server vor eine Datenbank gehängt hat: heute überprüfen. Die Burov-Befunde sind das deutlichste Signal dieses Quartals, dass MCP-Server denselben Threat-Modellierungs-Aufwand brauchen wie öffentliche APIs. Auth, Input-Validation, Least-Privilege-DB-User — alles non-negotiable.

Weiterhin im Hintergrund relevant: die OX-Security-„Mother of All AI Supply Chains"-Welle (April) berichtet von ~7.000 öffentlich erreichbaren Servern und 150 Mio.+ Downloads im MCP-STDIO-Pfad — Anthropic nennt das öffentlich „expected behavior". Plus offene CVEs: CVE-2026-32211 (Azure MCP), CVE-2026-33032 (nginx-ui MCP, CVSS 9.8), CVE-2026-26118 (Microsoft MCP tool hijacking).
