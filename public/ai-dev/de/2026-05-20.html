Hangzhou kontert · Qwen 3.7 Max — Flagship ja, Gewichte nein [1][2][3]

Während Google gestern Abend die Bühne hatte, lieferte Alibaba heute den Konter: Auf dem Cloud Summit in Hangzhou (20.–21.05.) wurde Qwen 3.7 Max offiziell enthüllt — gebaut für agentic Workloads, „sustained, multi-step operations", ausgeliefert über Bailian (außerhalb Chinas: Model Studio). Damit ist aus dem „Preview-Sighting" der letzten Tage ein echter Launch geworden — mit einer wichtigen Einschränkung.

Die Einschränkung: kein Open-Release

Stand heute liegen keine offenen Gewichte für Qwen 3.7 vor: HuggingFace und ModelScope tragen weiterhin nur die 3.6-Generation (Qwen3.6-27B, Qwen3.6-35B-A3B unter Apache 2.0), kein QwenLM/Qwen3.7-Repo.[2] Qwen 3.7 Max ist heute API-only. Das passt zum Muster: Alibaba hält — wie schon bei 3.6-Max/Plus — die stärksten Modelle proprietär, die offenen Mid-Tier-Varianten kommen (wenn überhaupt) später und kleiner. Skeptische Lesart: ein Flagship-Reveal ist noch kein Geschenk an die Self-Hosting-Community. Wer auf offene Qwen-3.7-Gewichte für den lokalen Stack wartet, wartet weiter.

alibabacloud.com · scmp.com

Der Hardware-Teil: Zhenwu M890

Substanzieller als das Modell selbst ist vielleicht der Chip: Alibabas T-Head-Tochter zeigte den Zhenwu M890 — 3× Performance des Vorgängers Zhenwu 810E, 144 GB On-Chip-Speicher, Präzision von FP32 bis FP4. Dazu der Panjiu-AL128-Supernode (128 Accelerators) und ein ICN-Switch-1.0-Netzwerkchip (25,6 Tbit/s). Einordnung für Daniel: das ist Chinas Antwort auf die Export-Kontrollen — eine vertikale Stack-Strategie (eigener Chip + eigenes Modell + eigene Cloud), die mittelfristig beeinflusst, wie billig agentic-Inferenz außerhalb der NVIDIA-Welt wird.

business-standard.com
FP4 (4-Bit-Präzision): Wie stark ein Modell beim Rechnen „rundet". FP32 = volle Genauigkeit, viel Speicher; FP4 = nur 4 Bit pro Zahl, viel kleiner und schneller, aber gröber. Ein Chip, der bis FP4 runtergeht, kann große Modelle billiger laufen lassen — solange die Qualität bei der groben Rundung nicht zu sehr leidet.

Google I/O · Gemini 3.5 Flash wird Default [4][5][6]

Die eigentliche I/O-Geschichte ist eine Default-Entscheidung. Google macht Gemini 3.5 Flash sofort zum Standard-Modell in der Gemini-App, im Search-AI-Mode, in der API, in AI Studio und Antigravity. Das Frontier-Pro-Modell (3.5 Pro) kommt erst „nächsten Monat". Die Wette: ein schnelles, billiges Modell auf Frontier-Niveau schlägt den letzten Benchmark-Punkt.

Zahlen & Preis — die skeptische Lesart

Laut Googles I/O-Material schlägt 3.5 Flash den Vorgänger 3.1 Pro auf der Agentic-Suite: Terminal-Bench 2.1 76,2 % (3.1 Pro 70,3 %), MCP Atlas 83,6 % (78,2 %), Finance Agent v2 57,9 % (43,0 %), GDPval-AA 1656 Elo (1314). Die unabhängigere Zahl: SWE-Bench Pro Public 55,1 % (54,2 %) — nur +0,9 Punkte.[5] Preis: $1,50/M Input, $9,00/M Output, gecacht $0,15/M; Kontext 1.048.576 In / 65.536 Out; vier Thinking-Level. Skeptisch: Terminal-Bench, MCP Atlas und GDPval sind Googles ausgewählte Bühnen-Benchmarks, der Vergleich geht gegen den eigenen alten Vorgänger — nicht gegen Opus 4.7 oder GPT-5.5. Der agentic-Sprung ist real; „schlägt die Frontier" ist eine Schlagzeile.

llm-stats.com · latent.space
Try-this Try Gemini 3.5 Flash steckt seit gestern als Default in API und AI Studio. Wer agentic Loops mit vielen Tool-Calls fährt (Test-Sweeps, Refactor-Batches), sollte einen Lauf timen — bei $1,50/$9,00 rechnet sich „billig + schnell" gegen „ein Bench-Punkt mehr". Ehrliche Frage wie immer: wo bricht es in deinem Code weg.

Antigravity 2.0 · die Plattform wird ernst [4][7]

Auf der I/O kam Antigravity in Version 2.0 — der Schritt von der IDE zur Plattform: eine Desktop-App mit Agent-first-Interface, eine CLI mit spezialisierten Subagents, ein SDK für programmatische Harness-Kontrolle, und Managed Agents in der Gemini API mit hosted Linux-Sandbox.

Der 12-Stunden-OS-Stunt

Die Bühnen-Demo: Antigravity + Gemini 3.5 Flash baute in 12 Stunden ein funktionierendes Betriebssystem from scratch — mit 93 parallelen Subagents, 15.000+ Modell-Requests, 2,6 Mrd. Token, unter $1.000 Credits. Lies es als das, was es ist: ein maximal parallelisierter Brute-Force-Lauf für die Bühne. Die Botschaft an Entwickler ist „massive Parallelität wird bezahlbar", nicht „dein nächstes OS schreibt sich selbst".

developers.googleblog.com
Managed Agents (hosted Sandbox): Statt dass dein Agent auf deinem Rechner Code ausführt, läuft er in einer abgeschotteten Linux-Box in Googles Cloud — wie ein Wegwerf-Linux nur für diese eine Aufgabe. Geht etwas schief, ist die Box weg, dein Rechner bleibt sauber. Praktisch für parallele Agents, riskant fürs Daten-Vertrauen (dein Code liegt kurz bei Google).

MCP-Ökosystem · WebMCP als Standard-Vorschlag [6]

Browser-Agents bekommen einen offenen Tool-Standard

Eine der substantielleren Dev-Ankündigungen der I/O: WebMCP, ein vorgeschlagener offener Web-Standard, mit dem Websites strukturierte Tools — JS-Funktionen und HTML-Formulare — so exponieren, dass browser-basierte Agents komplexe Tasks darauf ausführen. Der experimentelle Origin-Trial startet in Chrome 149. MCP hat sich serverseitig durchgesetzt; WebMCP zieht dieselbe Idee in den Browser. Noch früh, aber die Richtung ist klar.

developers.googleblog.com
MCP (Model Context Protocol): Ein einheitlicher Stecker, über den ein KI-Modell mit externen Werkzeugen redet — wie USB für KI-Tools. Statt für jede App eine eigene Anbindung zu basteln, spricht jeder Server „MCP", und jeder Agent versteht ihn. WebMCP bringt diesen Stecker in die Website selbst.
