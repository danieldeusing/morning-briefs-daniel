Claude Code 2.1.149 · Usage-Breakdown und drei Sicherheits-Fixes [1]

An einem Tag, an dem die meisten Anbieter Wochenende machen, ist Claude Code 2.1.149 (22.05., 22:09 UTC) der einzige substanzielle Stable-Release — und er hat zwei Gesichter: ein nettes Feature und einen ungewöhnlich dichten Sicherheits-Block.[1]

Das sichtbare Feature: /usage mit Aufschlüsselung

/usage zeigt jetzt eine Aufschlüsselung pro Kategorie — wie viel Token-Budget auf Skills, Subagents, Plugins und auf jeden einzelnen MCP-Server entfällt. Für jeden, der mehrere MCP-Server angeschlossen hat, ist das die erste Möglichkeit zu sehen, welcher Server wie teuer ist, statt nur eine Gesamtsumme zu lesen. Dazu: die /diff-Detailansicht ist jetzt per Tastatur scrollbar (Pfeile, j/k, PgUp/PgDn), Markdown rendert GFM-Task-Checkboxen, und Enterprise bekommt ein allowAllClaudeAiMcps-Managed-Setting.

Der eigentliche Inhalt: die Sicherheits-Fixes

Gewichtiger als das Feature sind drei Fixes, die direkt Daniels Setup betreffen. Zwei PowerShell-Permission-Bypässe wurden geschlossen: eingebaute Befehle wie cd.. oder cd\ konnten aus dem freigegebenen Arbeitsverzeichnis ausbrechen, und Prefix-/Wildcard-Allow-Regeln ließen sich umgehen. Dazu ein Worktree-Sandbox-Fix (die Schreib-Allowlist griff bei Git-Worktrees nicht sauber) und — der unangenehmste — ein find-Bug, der die macOS-vnode-Tabelle erschöpfte und den ganzen Rechner zum Absturz brachte.[1]

vnode-Tabelle: macOS führt intern eine Liste aller gerade „angefassten" Dateien und Ordner — wie eine Garderobe mit begrenzten Haken. Ein außer Kontrolle geratenes find hängte so viele Haken voll, dass das ganze System keinen freien mehr hatte und stehenblieb. Nicht nur das Tool stürzte ab, sondern der Host.

Einordnung: Permission-Bypässe sind in einem Agenten, der Shell-Befehle ausführt, keine Kosmetik — sie sind genau die Klasse Bug, bei der ein Sandbox-Versprechen heimlich löchrig wird. Praktisch für Daniel: direkt auf 2.1.149 (oder 150) ziehen, besonders wer Claude Code mit angebundenen MCP-Servern oder unter Windows/PowerShell betreibt.
