Antigravity 2.0 · eine Woche nach dem Knall [1][2][3]

Vor neun Tagen hat Google auf der I/O sein agent-first Desktop-Werkzeug in Version 2.0 vorgestellt — standalone Anwendung für macOS/Linux/Windows, parallele Sub-Agenten, geplante Hintergrund-Tasks, ein agy-CLI, ein SDK, Anbindung an AI Studio/Android/Firebase. Die Architektur, das ist die Quintessenz aus den unabhängigen Reviews der letzten Tage, ist real: das bekommt man heute weder von Claude Code noch von Cursor.[1]

Die Auslieferung war ein Problem. Google hat am Launch-Tag automatisch auf bestehende v1-Installationen gepatcht — und dabei den eingebauten Editor entfernt, Konfigurationen an einen neuen Pfad verschoben (= weg), Source-Control und Remote-Verbindungen aus dem Default-Bild geworfen. Wer Antigravity in CI/CD hatte, fand am Mittwoch eine Chat-Oberfläche statt einer IDE vor. Das Ökosystem zerfiel an dem Tag in drei separate Downloads: Antigravity 2.0 (Agenten-Shell), Antigravity IDE (klassisches Coden) und Antigravity CLI. Wer den Editor zurück wollte, musste die IDE eigenhändig nachinstallieren.[2]

Free-Tier und CLI-Lücke

Zwei Zahlen, die zur Stimmung beigetragen haben: der Free-Tier ist von 250 auf 20 Requests/Tag gefallen — zu wenig, um das Tool seriös zu evaluieren ohne ein zahlendes Konto. Und die agy-CLI, die die Gemini CLI am 18.06. ablöst, war noch am 21.05. nicht über einen öffentlichen Package-Manager zu bekommen. Das hat in CI-Pipelines, die sich auf npm i -g @google/gemini-cli verlassen hatten, blanken Migrationsdruck ohne Zielsystem erzeugt.[2]

Update seit gestern: Der offizielle Installer ist inzwischen live. curl -fsSL https://antigravity.google/cli/install.sh | bash legt das Native-Binary agy nach ~/.local/bin (Unix) bzw. %LOCALAPPDATA%\Antigravity\ (Windows). Kein npm-Paket — Google ist explizit vom Node-Workflow weg.[3]

Für Daniel: nicht heute installieren

Antigravity 2.0 ist kein Drop-in-Ersatz für Claude Code; „Revisit in 60–90 Tagen" (so der Reviewer) ist der ehrliche Rat. Was sich heute aus der Antigravity-Welt lohnt, ist der MCP-Server, der drinsteckt — Chrome-DevTools-for-agents, ohne Antigravity nutzbar (nächste Sektion).

Chrome DevTools for agents · stiller Standard [4][5]

Während Antigravity die Schlagzeile macht, läuft im Hintergrund etwas Substanzielleres: Chrome DevTools for agents 1.0 ist am 19.05. stabil gegangen, das npm-Paket chrome-devtools-mcp stand gestern bei 1.1.1 (release-Tag 27.05.). Das gibt deinem Coding-Agenten echte Browser-Augen: Console-Logs, Network-Requests, Performance-Traces, Memory-Snapshots, Lighthouse-Audits, Geräte-Emulation — alles per MCP statt per Screenshot-Glaskasten.[4]

Chrome DevTools for agents (per MCP): Stell dir vor, dein KI-Agent sitzt blind vor dem Bildschirm und schreibt Frontend-Code — bisher musste er raten, ob die Seite überhaupt lädt. Mit diesem MCP-Server bekommt er das gleiche Inspektions-Werkzeug, das du in Chrome unter F12 öffnest: er sieht, welche Requests fehlschlagen, welches Bild 3 Sekunden braucht, welche Eingabe-Felder das Screen-Reader-Label vermissen. Statt „ich glaube der Bug ist hier" → „Lighthouse sagt Z, fix Y."

Die Liste der Clients ist die eigentliche Überraschung: vor-installiert in Antigravity, als Marketplace-Plugin in Claude Code, MCP-Extension in Gemini CLI, dazu offiziell gelistet für Cursor, Copilot (VS-Code + CLI), Codex, Cline, Command Code, Factory CLI, JetBrains AI Assistant, Junie, Katalon, Mistral Vibe, OpenCode, Qoder, Visual Studio, Warp, Windsurf.[5] Das ist kein Google-Werkzeug mehr in jeder relevanten Hinsicht außer Branding — es ist gemeinsames Substrat.

Was du heute hinzufügen kannst

In Claude Code: das offizielle Plugin aus dem Anthropic-Marketplace; Codex/Cursor: per MCP-Server-Config (npx chrome-devtools-mcp@latest). Der LY-Corporation-Case-Bericht, den Google zitiert: ein automatisches Audit-System, das die manuelle Performance-Analyse um bis zu 98 % reduziert — die Zahl kommt aus Googles Blog, Reportage-Sekundärquelle bestätigt sie ungebrochen, bleibt aber Anbieter-zitiert. Try Niedrige Einstiegshürde, hoher Hebel — gerade für Front-End-Sessions, wo der Bug optisch ist und das Modell ihn ohne Augen rät.

Managed Agents + AI-Studio Android [8]

Managed Agents in der Gemini API (Preview)

Ein API-Call startet eine isolierte Linux-Sandbox, in der ein Agent reasoniert, Tools nutzt und Code ausführt — dieselbe Harness, die hinter Antigravity läuft. Eigene Agenten definierst du als versionierbare AGENTS.md und SKILL.md-Dateien. Verfügbar via Interactions API + AI Studio, „Public Preview". Das ist Googles Antwort auf OpenAIs Codex-as-a-Service und Anthropics Agent-SDK — derselbe Konzept-Block, anderer Anbieter.[8]

Managed Agents: Statt eine Agent-Loop selbst zu bauen (Modell-Call → Tool-Call → Resultat → nächster Schritt), kriegst du einen fertigen Agent-Container vom Anbieter, dem du nur sagst „mach X". Vorteil: weniger Boilerplate, automatische Sandbox-Isolation. Nachteil: weniger Kontrolle, Anbieter-Lock-in, Sicherheits-Boundary liegt bei Google.

Native Android-Apps in AI Studio

Ein Prompt → Kotlin/Jetpack-Compose-Projekt, Live-Vorschau im Browser-Emulator oder direkt aufs Gerät via ADB, mit einem Klick in den Google Play Internal Test Track gepusht. Das ist das Stück I/O, das am stärksten unter dem Radar lief, und das praktischste: niemand musste Android Studio installieren, niemand musste signieren, niemand musste publishen. Für Hobby-Projekte und Prototyping interessant; für Daniels Hauptarbeit (TypeScript/Python-Stack) tangential.

Es ist Donnerstag, der achtundzwanzigste Mai. Neun Tage nach Google I/O setzt sich der Staub, und das Bild ist gemischt. Eine große Geschichte zerfällt in zwei Hälften: eine raue und eine substantielle.

Die raue Hälfte ist Antigravity zwei-Punkt-null, Googles neue agent-first Desktop-App. Die Architektur ist real: paralleler Sub-Agent-Orchestrator, geplante Hintergrund-Aufgaben, ein Browser-Agent. Das gibt es heute weder von Claude Code noch von Cursor. Aber Google hat am Launch-Tag ein automatisches Update auf bestehende Installationen geschoben — und dabei den eingebauten Editor entfernt, Konfigurationen verloren, Continuous-Integration-Pipelines gebrochen. Wer Antigravity in Produktion hatte, fand am Mittwoch eine Chat-Oberfläche statt einer Entwicklungsumgebung. Reviews stehen bei drei von fünf. Der Reviewer-Rat: in sechzig bis neunzig Tagen wieder hinschauen. Für Daniel heute: nicht installieren.

Die substantielle Hälfte ist Chrome DevTools for agents in Version eins-Punkt-null. Das ist der Model-Context-Protocol-Server, der deinem Agenten echte Browser-Augen gibt — Console-Logs, Network-Requests, Performance-Traces, Lighthouse-Audits. Stell dir vor, dein Agent sitzt sonst blind vor dem Bildschirm und rät, ob die Seite überhaupt lädt; dieser Server ist die F-Zwölf-Inspektion, die du selbst öffnest, nur maschinenlesbar. Vor-installiert in Antigravity — aber auch als Plugin in Claude Code, Cursor, Copilot, JetBrains, Codex und einem Dutzend weiterer. Das ist still zum gemeinsamen Substrat geworden. Daniels Empfehlung: heute in Claude Code haken, niedrige Einstiegshürde, hoher Hebel bei Frontend-Arbeit.

Gemini drei-Punkt-fünf Flash ist seit dem neunzehnten Mai allgemein verfügbar. Anbieter-Zahlen sind stark: Terminal-Bench bei sechsundsiebzig Komma zwei, M-C-P-Atlas bei dreiundachtzig Komma sechs. Bei reinem abstrakten Denken liegt es allerdings hinter Gemini drei-Punkt-eins Pro. Simon Willison rechnet einen drei-fachen Preissprung gegenüber dem Vorgänger vor — die großen Labs probieren die Preis-Toleranz ihrer Kunden aus. Und entscheidend: kein einziger Eintrag für drei-Punkt-fünf Flash oder Antigravity zwei-Punkt-null im prüfbaren SWE-Bench-Verified-Datensatz. Bis es einen gibt, sind die Anbieter-Zahlen Marketing.

Heute selbst, abseits der I/O-Stories: Claude Code ist auf zwei-Punkt-eins-Punkt-hundertdreiundfünfzig gesprungen, getrieben von Bug-Fixen — ein Model-Context-Protocol-Reconnect-Loop ist geschlossen, ein OAuth-Gateway-Bug auch, der Befehl Slash-model speichert seine Auswahl jetzt als Default.
