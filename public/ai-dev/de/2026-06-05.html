Microsoft trainiert sein eigenes Copilot-Modell

Das ist die Story des Tages, und sie ist strategischer als sie klingt. Bei der Build-2026-Keynote am 2. Juni stellte Microsoft sieben hauseigene MAI-Modelle vor — Bild, Sprache, Transkription, ein Reasoning-Modell (MAI-Thinking-1) — und mittendrin das einzige, das uns Entwickler direkt betrifft: MAI-Code-1-Flash, Microsofts erstes eigenes Coding-Modell, das seit dem 2. Juni in GitHub Copilot ausgerollt wird.[1] Es taucht im Model-Picker von VS Code auf und kommt über alle Pläne hinweg — Free, Pro, Pro+, Max.

Der eigentliche Punkt: Bisher lief Copilot auf fremden Modellen — OpenAI, Anthropic, Google. MAI-Code-1-Flash ist das erste, das Microsoft selbst besitzt, end-to-end. Trainiert wurde es laut Microsoft auf „sauberen und ordentlich lizenzierten Daten", ausdrücklich ohne Distillation aus OpenAI-Modellen[2] — eine Spitze gegen den Partner, von dem Microsoft sich gerade unabhängiger macht.

Distillation: Ein kleines Modell lernt, indem es die Antworten eines großen, teuren Modells nachahmt — wie ein Azubi, der dem Meister über die Schulter schaut. Microsoft sagt: Wir haben unserem Azubi nicht OpenAI über die Schulter schauen lassen. Wichtig, weil distillierte Modelle juristisch und vertraglich an ihrem „Lehrer" hängen.

Architektur: ein sparse Mixture-of-Experts mit 137 Mrd. Parametern gesamt, aber nur 5 Mrd. aktiv pro Token, 256K Kontext.[3] (Die offizielle Modellkarte nennt 137B/5B; Microsofts Familien-Post spricht verkürzt von einem „5B-Modell" — gemeint ist die aktive Parameterzahl. Das hat für Verwirrung gesorgt.[4])

Mixture-of-Experts (sparse): Statt für jede Anfrage das ganze riesige Modell zu rechnen, hat das Modell viele „Experten" und schickt jedes Token nur an die paar passenden — wie ein Krankenhaus, das dich nicht von allen 137 Ärzten gleichzeitig untersuchen lässt, sondern nur von den 5 relevanten. Ergebnis: die Wucht eines großen Modells zum Rechenpreis eines kleinen.

Die Benchmarks — skeptisch gelesen

Microsofts Modellkarte stellt MAI-Code-1-Flash direkt gegen Claude Haiku 4.5, die naheliegende Vergleichsklasse (kleines, schnelles Modell). Die Zahlen sind, und das ist entscheidend, Hersteller-eigene Messungen — MAI-Code-1-Flash steht (Stand 05.06.) noch nicht im unabhängigen Coding-Index von Artificial Analysis.[5] Solange die Replikation fehlt, gilt: interessant, aber nicht bestätigt.

SWE-Bench Verified — Hersteller-Zahlen, gemischte Quellen, Stand 02.06.2026

Auf SWE-Bench Verified meldet die Karte 71,6 % für MAI-Code-1-Flash gegen 66,6 % für Haiku 4.5 — und, fast spannender als der Score: bei rund 10,8K statt 27,3K Token pro Aufgabe.[3] Auf der härteren SWE-Bench Pro ist der Abstand größer (51,2 % vs. 35,2 %), bei SWE-Bench Multilingual knapper (65,5 % vs. 62,7 %). Quer über die harten Tasks spricht Microsoft von bis zu 60 % weniger Token.[4]

SWE-Bench Verified: Eine Sammlung echter GitHub-Bugs mit echten Test-Suiten. Das Modell muss den Fehler im Repo selbst finden und einen Patch schreiben, der die Tests grün macht — wie ein Mechaniker, der ein fremdes Auto reparieren soll und am Ende eine echte Probefahrt besteht. „Verified" heißt: ein Mensch hat geprüft, dass die Aufgaben lösbar und die Tests fair sind.
Skepsis-Filter Zwei Vorbehalte. Erstens: Hersteller misst sich selbst — Microsoft hat MAI-Code-1-Flash genau auf den Copilot-Harness trainiert, gegen den hier gebencht wird; ein Heimvorteil, der real ist und sich in der echten Welt anders verteilen kann. Zweitens: die Token-Effizienz ist der glaubwürdigste Teil der Story. Weniger Token = niedrigere Kosten und Latenz für Copilots Millionen Nutzer — und genau das (nicht der letzte SWE-Bench-Prozentpunkt) ist Microsofts eigentliches Motiv.

Copilot wird agentic: Agent HQ try

Das zweite große Build-Thema fürs Coding: Agent HQ. GitHub lässt dich jetzt mehrere Coding-Agenten parallel laufen — direkt in GitHub, GitHub Mobile und VS Code — und zwar nicht nur Copilot selbst, sondern auch Claude von Anthropic und OpenAI Codex (beide in Public Preview) nebeneinander.[6] Für Pro+- und Enterprise-Nutzer.

Das ist mehr als Marketing. Ein Orchestrator-Agent kann Subagenten auf getrennte Workstreams ansetzen — Linting, Test-Generierung, Doku, Security-Review laufen gleichzeitig statt nacheinander. Wer mit Claude Code oder einem eigenen Harness arbeitet, kennt das Muster; neu ist, dass GitHub es als First-Party-Feature und herstellerübergreifend baut, mit Kontext, History und Review an einem Ort.

Warum probieren Wenn du ohnehin Copilot Pro+ hast: Agent HQ ist der erste Ort, an dem du Claude und Codex im selben Workflow gegeneinander antreten lassen kannst, ohne drei Tools offen zu haben. Laut Berichten (techtimes.com) soll für Enterprise ein Autonomous Agent Mode folgen, der ganze Feature-Branches schreibt, testet und committet — Merge weiter nur mit menschlicher Freigabe; GitHub hat dieses Feature im offiziellen Agent-HQ-Blog aber weder benannt noch terminiert.[7]

Codex jetzt regulär auf Amazon Bedrock

Parallel, einen Tag vor Build: OpenAIs Coding-Agent Codex ist zusammen mit GPT-5.5 und GPT-5.4 seit dem 1. Juni generell verfügbar auf Amazon Bedrock.[8] Zugriff über Codex App, CLI und IDE-Integrationen (VS Code, JetBrains, Xcode). OpenAI nennt über 5 Mio. Entwickler pro Woche, die Codex nutzen.

Der Reiz für AWS-Häuser: die gesamte Inferenz läuft über Bedrock und damit unter den AWS-Governance-Knöpfen, die man schon hat — IAM, VPC/PrivateLink-Isolation, KMS-Verschlüsselung, CloudTrail-Audit. Prompts werden nicht zum Training verwendet. Preise „matchen OpenAIs First-Party-Raten", pay-per-token, keine Seat-Lizenzen.

Einordnung Drei Tage, drei Labs, ein Muster: Microsoft besitzt jetzt sein Copilot-Modell, OpenAI verteilt Codex über die Cloud-Schiene des Konkurrenten, GitHub macht alle drei in Agent HQ austauschbar. Das Coding-Modell ist 2026 kein Produkt mehr, sondern eine Komponente, die du je nach Plattform, Preis und Compliance steckst.

Gemini 3.5 Flash als Coding-Modell

Etwas älter (Google I/O, 19.05.), aber im Coding-Kontext gehört es hierher: Gemini 3.5 Flash ist laut Google sein bisher stärkstes Modell fürs Programmieren und für autonome Agenten — nicht fürs Chatten. Google verkauft es offen als Agent-Modell: planen, bauen, iterieren mit minimalem menschlichem Eingriff, rund 4× schneller als vergleichbare Frontier-Modelle.[9]

Die (wieder Hersteller-)Zahlen: Terminal-Bench 2.1 76,2 %, MCP Atlas 83,6 %, GDPval-AA 1656 Elo — jeweils über Gemini 3.1 Pro. Verfügbar via Antigravity, Gemini API und Enterprise. Gemini 3.5 Pro soll „nächsten Monat" (Juni) folgen, intern schon im Einsatz. Ein 2M-Token-Kontext kursiert nur in sekundären Berichten und ist von Google bislang nicht offiziell bestätigt.[10] Für uns interessant ist Flash vor allem als günstiges, schnelles Agent-Backend in Harnesses, die Modelle frei steckbar machen — siehe Agent HQ oben.

Open-Weights: MiniMax M3 vorn, Qwen effizient Local

Für Daniels lokalen Ollama-/Qwen-Stack bewegt sich auch etwas. Zwei Notable Releases:

MiniMax M3 (Juni) führt die Open-Weight-Rangliste auf SWE-Bench Pro mit 59,0 % an und ist laut kilo.ai das erste Open-Weight-Modell, das Frontier-Coding, 1M-Kontext und native Multimodalität verbindet.[11]

Qwen3-Coder-Next bleibt der Effizienz-König: 80B MoE / nur 3B aktiv, 256K Kontext, läuft auf einem 64-GB-MacBook oder einer RTX 5090. Hersteller-Zahlen: SWE-Bench Verified 70,6 / Pro 44,3.[12] Spannend: das ist Sonnet-4.5-nahe Coding-Leistung lokal — ohne Token-Rechnung.

DeepSeek V4 (24.04.) rundet das Feld ab — V4-Pro (1,6T/49B) und V4-Flash (284B/13B), beide 1M Kontext, MIT-Lizenz. Für die meisten lokal zu groß, aber als selbst-hostbares Frontier-Modell relevant.

Tool-Roll-Call

Cursor 3.6 (29.05.) bringt Auto-review: ein Run-Modus, der Cursor länger ohne Nachfrage arbeiten lässt. Shell-, MCP- und Fetch-Calls gehen entweder sofort durch (Allowlist), laufen sandboxed — oder ein Klassifizierer-Subagent entscheidet, ob er den Call erlaubt, anders löst oder doch nachfragt.[13] Das ist derselbe „weniger Approval-Friction"-Trend, den auch die berichteten Copilot-Autonomous-Agent-Pläne treiben — Agenten dürfen länger allein laufen, Menschen kontrollieren nur noch die Grenzen.

Cline v3.85 (25.05.) hat GPT-5.5 und DeepSeek V4 Flash/Pro nachgezogen; 61k+ Sterne.[14]

Still im Schatten Aider hat weiter kein natives MCP — nur Community-Bridges, Stand Mai 2026; das ist selbst eine Information (Stagnation). Continue und OpenHands ohne nennenswerte Bewegung diese Woche. JetBrains AI und Plandex ruhig.

Was das für Daniels Toolbox heißt

Meine Lesart: Der Tag bestätigt einen Trend, der für uns gut ist — Modell-Lock-in löst sich auf. Agent HQ macht Claude, Codex und Copilot im selben Editor austauschbar; MAI-Code-1-Flash drückt die Kosten am unteren Ende; Codex auf Bedrock entkoppelt OpenAI von einer einzigen Cloud. Wer wie Daniel schon mit steckbaren Harnesses und lokalen LLMs arbeitet, gewinnt: mehr Auswahl, weniger Abhängigkeit von einem Anbieter.

Konkret diese Woche try-worthy: MAI-Code-1-Flash im VS-Code-Picker für günstige, schnelle Routine-Edits (wenn der Rollout dich erreicht), und Cursor 3.6 Auto-review, wenn dich Approval-Prompts bei langen Agent-Läufen nerven. Auf die unabhängige SWE-Bench-Replikation der MAI-Zahlen würde ich warten, bevor ich Microsofts 71,6 % glaube.

Es ist Freitag, der fünfte Juni. Drei Tage nach Microsofts Build-Konferenz dreht sich im Coding-Tooling fast alles um dieselbe Idee: Das KI-Modell, mit dem du programmierst, wird zur austauschbaren Komponente.

Die Hauptgeschichte kommt von Microsoft. Zum ersten Mal hat Microsoft ein eigenes Coding-Modell in GitHub Copilot eingebaut, es heißt MAI-Code-1-Flash. Bisher lief Copilot auf fremden Modellen von OpenAI, Anthropic und Google. Dieses hier hat Microsoft selbst trainiert, nach eigenen Angaben auf sauber lizenzierten Daten und ausdrücklich ohne bei OpenAI abzuschauen. Es ist klein und schnell, ein sogenanntes Mixture-of-Experts-Modell, das pro Anfrage nur einen Bruchteil seiner Größe rechnet.

Die Benchmark-Zahlen lese ich vorsichtig. Microsofts eigene Modellkarte meldet auf dem Test SWE-Bench Verified einen Wert von einundsiebzig Komma sechs Prozent gegen sechsundsechzig Komma sechs bei Claude Haiku, und das bei deutlich weniger verbrauchten Token. Aber das ist eine Hersteller-Messung. Eine unabhängige Stelle wie Artificial Analysis hat das Modell noch nicht nachgemessen. Glaubwürdiger als der letzte Prozentpunkt ist für mich die Token-Effizienz, denn weniger Token heißt niedrigere Kosten, und genau das ist Microsofts eigentliches Motiv.

Das zweite Thema von Build heißt Agent HQ. GitHub lässt dich jetzt mehrere Coding-Agenten gleichzeitig laufen, und zwar nicht nur Copilot, sondern auch Claude und Codex nebeneinander, im selben Editor. Ein Orchestrator verteilt Aufgaben wie Tests, Dokumentation und Security-Review parallel an Unter-Agenten. Für Firmenkunden soll laut Berichten ein Modus folgen, der ganze Feature-Branches selbst schreibt und committet, gemerged wird aber nur mit menschlicher Freigabe; GitHub selbst hat das offiziell noch nicht angekündigt.

Dazu passt OpenAI. Einen Tag vor Build ist der Coding-Agent Codex regulär auf Amazon Bedrock verfügbar geworden, zusammen mit den neuen GPT-Modellen. Über fünf Millionen Entwickler nutzen Codex pro Woche, und auf Bedrock läuft alles unter den vertrauten AWS-Sicherheitskontrollen. Drei Tage, drei Labs, ein Muster: das Modell wird zur Infrastruktur, die du je nach Plattform und Preis steckst.

Bei Google ist Gemini 3.5 Flash inzwischen Googles stärkstes Modell fürs Programmieren und für autonome Agenten, rund viermal schneller als die Konkurrenz. Die größere Version, Gemini 3.5 Pro, soll noch im Juni folgen.

Für Daniels lokalen Stack zwei Notizen: Das Open-Weight-Modell MiniMax M3 führt jetzt die offene Rangliste an, und Qwen3-Coder-Next bleibt der Effizienz-König, der nahe an die Qualität teurer Modelle herankommt und trotzdem auf einem MacBook läuft. Bei den Tools hat Cursor in Version drei Komma sechs einen Modus eingebaut, der weniger oft nachfragt, und Cline hat die neuen Modelle nachgezogen. Aider bleibt ohne natives MCP, was selbst eine Aussage ist.

Mein Fazit: Modell-Lock-in löst sich auf, und das ist gut für jeden, der mit steckbaren Werkzeugen arbeitet. Nächste Woche schaue ich vor allem auf zwei Dinge: ob eine unabhängige Stelle Microsofts SWE-Bench-Zahlen bestätigt, und ob Google sein Gemini 3.5 Pro wirklich noch im Juni liefert.
