Das Feld ist jetzt vierfach besetzt

Die Geschichte des Tages ist keine einzelne Meldung, sondern ein Muster, das diese Woche fertig geworden ist: Bis vor Kurzem war „KI fürs Coden" faktisch ein Zweikampf zwischen Anthropic (Claude Code/Opus) und OpenAI (Codex/GPT). Mit Microsofts MAI-Modellen und dem Copilot-App auf der Build (02.06.) und Googles Gemini 3.5 Flash plus Antigravity (I/O, 19.05.) stehen jetzt alle vier großen Labs mit einem eigenen, vertikal integrierten Coding-Stack auf dem Platz.

CNBC fasst die Lage am 1. Juni so zusammen: Microsoft und Google seien „spät dran, aber es sei absolut entscheidend, dass sie um dieses Wachstum mitkämpfen".[1] Der Grund steht in den Adoptions-Daten dieser Woche: Claude Code ist der am schnellsten wachsende Coding-Agent — laut JetBrains-Entwickler-Umfrage 2026 von ~3 % (Apr. 2025) auf 18 % (Jan. 2026) gestiegen, etwa 6× in neun Monaten — und unter erfahrenen Entwicklern (10+ Jahre) der klare Favorit: 46 % wählen Claude Code für die tägliche Arbeit gegenüber 9 % Copilot.[2] GitHub Copilot bleibt aber bei der reinen Verbreitung führend (Stack-Overflow-Umfrage Ende 2025: ~51 % der Profi-Entwickler, wenn auch von 67 % gefallen; Cursor neu bei ~18 %, Claude Code bei ~10 % im Erst-Auftritt). Der Run-Rate-Umsatz von Claude Code lag im Feb. 2026 bei über 2,5 Mrd. $. Genau dieses Revier — Entwickler, die täglich einen Agenten bezahlen — wollen Microsoft und Google nicht Anthropic und OpenAI überlassen.

Vertikal integrierter Coding-Stack: Wenn ein Anbieter alles aus einer Hand liefert — das Modell, das Tool drumherum (IDE oder Agent-Harness) und die Cloud, auf der es läuft. Wie ein Autohersteller, der nicht nur den Motor baut, sondern auch das Auto und die Tankstellen. Vorteil: Modell und Werkzeug sind aufeinander trainiert (MAI-Code-1 wurde direkt auf Copilots Produktions-Harness trainiert). Nachteil für dich: Lock-in — der Stack zieht dich in ein Ökosystem.

Für Daniels Toolbox ist die Lehre nüchtern: Der Wettbewerb ist gut für Preise und Tempo, aber vier Stacks heißt nicht vier echte Optionen. Die Newcomer müssen erst beweisen, dass ihre Modelle außerhalb der eigenen Marketing-Folien tragen — und dort steht der Skepsis-Filter (nächste Sektion).

Microsoft MAI: erstes eigenes Coding- und Reasoning-Modell Bench

Auf der Build hat Microsoft seine MAI-Familie vorgestellt — der bemerkenswerte Punkt: Es sind die ersten von Microsoft selbst trainierten Modelle für diese Rolle, ein bewusster Schritt weg von der reinen OpenAI-Abhängigkeit.[3] Zwei sind für uns relevant:

MAI-Code-1-Flash — das kleine, schnelle Coding-Modell, das direkt im GitHub Copilot und VS Code landet (Model-Picker, Rollout über Free/Student/Pro/Pro+/Max, schrittweise „über die kommenden Wochen").[4] Der eigentliche Trick: Es wurde direkt auf Copilots Produktions-Harness trainiert und löst Aufgaben mit „bis zu 60 % weniger Tokens" dank adaptiver Antwortlänge — kurz bei einfachen Anfragen, mehr Reasoning-Budget bei schweren.[5]

MAI-Thinking-1 — Microsofts erstes Reasoning-Modell, von Grund auf trainiert ohne Distillation, auf „sauberen, kommerziell lizenzierten" Daten. Ein MoE mit ~35 Mrd. aktiven Parametern, 256K-Kontext.[6]

Distillation (Modell-Destillation): Ein kleines Modell lernt, indem es die Antworten eines großen „Lehrer"-Modells nachahmt — wie ein Lehrling, der dem Meister über die Schulter schaut. Schneller und billiger, aber das kleine Modell erbt die Macken des Lehrers. Microsoft betont „ohne Distillation, von Grund auf" — Marketing-Botschaft: „wir haben kein OpenAI-Modell abgeschrieben". Für dich relevant v. a. wegen sauberer Lizenz-Herkunft der Trainingsdaten.
Bench-Skepsis Die Zahlen sind durchweg Hersteller- oder Sekundärquellen, kein unabhängiges Leaderboard. Gemeldet: MAI-Code-1-Flash ~51 % SWE-Bench Pro (+16 Punkte gegen Claude Haiku 4.5 mit 35,2 %, etwa GPT-5.3-Tier); MAI-Thinking-1 ~53 % SWE-Bench Pro und damit „Kopf-an-Kopf mit Claude Opus 4.6", 97 % AIME 2025.[6][7] Zur Einordnung: Opus 4.8 liegt bei 69,2 % Pro — die MAI-Modelle spielen also eine Klasse tiefer, treffen aber ihr Preis-/Größen-Segment. Und Vorsicht: Die Parameterzahl von MAI-Code-1-Flash ist widersprüchlich gemeldet (Microsoft-Seite/mehrere Sekundärquellen: 5B; eine Quelle: „137B MoE") — ich würde keine der beiden als gesichert behandeln, bis ein Modell-Card-Detail vorliegt.

GitHub-Copilot-App: vom Editor-Plugin zum Agent-Leitstand try

Die für mich praktisch relevanteste Build-Neuigkeit ist nicht ein Modell, sondern ein Werkzeug: Die GitHub-Copilot-App ist eine eigenständige Desktop-Anwendung (Win 11, macOS, Linux), getrennt von der VS-Code-Extension. Sie macht aus Copilot „einen Leitstand für parallele KI-Agenten-Sessions — jede in ihrem eigenen, isolierten Git-Worktree".[8]

Git-Worktree: Mehrere Arbeitskopien desselben Repos nebeneinander, jede auf ihrem eigenen Branch — ohne ständiges Hin- und Her-Wechseln. Wie mehrere Werkbänke in derselben Werkstatt: An jeder läuft ein anderer Umbau, ohne dass sie sich ins Gehege kommen. Genau das nutzt die App, damit drei Agenten gleichzeitig an drei Aufgaben arbeiten, ohne sich die Dateien zu überschreiben.

Aus einer „My Work"-Ansicht siehst du laufende Sessions, Issues, PRs und Hintergrund-Automationen über mehrere Repos. Das ist exakt das Orchestrator-Muster, das Daniel von Claude Code bereits kennt — GitHub gießt es in eine native App. Dazu ist das Copilot-SDK jetzt GA in sechs Sprachen (Node/TS, Python, Go, .NET, Rust, Java); derselbe Agent-Runtime, auf dem die App läuft, ist programmierbar.[9]

Warum probieren Wenn du ohnehin im GitHub-Ökosystem bist und mehrere Agenten parallel fahren willst: Die App ist ein ernsthafter Claude-Code-/Cursor-Konkurrent für Multi-Agent-Workflows — und das SDK macht eigene Orchestrierung möglich. Technical Preview für Pro/Pro+/Business/Enterprise. Für reine Solo-Single-Agent-Arbeit überspringen.

Google Gemini 3.5 Flash + Antigravity 2.0

Googles Antwort ist seit der I/O (19.05.) breit ausgerollt und gehört in dieses Landschafts-Bild — gestern unter „Pro-Warten" nur gestreift, heute mit Substanz: Gemini 3.5 Flash ist GA über Antigravity, die Gemini API in AI Studio und Android Studio, Gemini Enterprise sowie die Gemini-App.[10] Das Verkaufsargument ist Tempo: ~289 Tokens/Sekunde, rund 4× schneller als Opus oder GPT-5.5, in Antigravity bis 12×.[11]

Bench-Skepsis Die gemeldeten Coding-Werte sind Google-eigene Zahlen auf teils Google-kuratierten Suiten: Terminal-Bench 2.1 76,2 %, MCP Atlas 83,6 %, GDPval-AA 1656 Elo.[10] Unabhängig ist nur grob einzuordnen: Artificial Analysis verortet Gemini 3.5 Flash auf dem Intelligence-Index bei 55 — solides Mittelfeld, nicht an der Coding-Spitze (dort GPT-5.5 xhigh mit Coding-Index 59, Opus 4.8 vorne im Intelligence-Index mit 61,4).[12] Flash spielt also nicht gegen Opus 4.8 auf Qualität, sondern auf Qualität pro Sekunde pro Dollar — das ist sein Revier.

Antigravity 2.0 ist der eigentliche Dev-Hebel: eine Standalone-Desktop-App plus CLI, agent-first gebaut, mit Managed Agents in der Gemini API — ein einziger API-Call liefert einen Agenten samt gehostetem Linux-Sandbox (Bash/Python/Node, Datei-IO, Browsing, Markdown-definierte Skills, Repo-/GCS-Mounts).[13] Das ist Googles direkter Angriff auf die IDE-Agent-Ebene, auf der Claude Code und Cursor sitzen.

SWE-Bench Pro: Eine Sammlung echter, schwerer GitHub-Bugs — das Modell muss selbst den Patch schreiben, der die Tests grün macht, wie ein Mechaniker, der ein fremdes Auto reparieren und die Probefahrt bestehen muss. „Pro" ist neuer und härter als das ältere „Verified" und weniger im Training „verbrannt" — deshalb das ehrlichere Signal, wenn Verified oben flach wird (alle nahe 90 %).

Was das für Daniels Toolbox heißt

Meine Lesart: Der Vier-Labs-Markt ist gut für Daniel — mehr Wettbewerb drückt Preise und treibt Tempo. Aber „vier Stacks" ≠ „vier echte Optionen". Heute bleibt Opus 4.8 die Coding-Referenz, und Claude Code ist nicht ohne Grund der am schnellsten wachsende Agent und der Favorit erfahrener Entwickler — auch wenn Copilot bei der reinen Verbreitung noch vorn liegt. Die Newcomer adressieren andere Achsen: MAI das günstige In-Copilot-Segment, Gemini 3.5 Flash Geschwindigkeit.

Konkret try-worthy: die GitHub-Copilot-App, falls du Multi-Agent-Workflows im GitHub-Ökosystem fährst, und ein Blick auf Antigravity 2.0, wenn dich Managed Agents per API reizen. Bei den MAI-Modellen und Geminis Coding-Werten würde ich vor jedem Stack-Wechsel die unabhängigen Benchmarks abwarten — geliefert sind bislang fast nur Hersteller-Zahlen. Und der nächste echte Termin bleibt Gemini 3.5 Pro im Juni-Fenster.

Es ist Sonntag, der siebte Juni, und der Nachrichten-Tag selbst ist ruhig. Das ist ein guter Moment, um einen Schritt zurückzutreten, denn diese Woche hat sich der Markt für KI-Coding-Modelle vervollständigt.

Bis vor Kurzem war das im Kern ein Zweikampf zwischen Anthropic mit Claude Code und OpenAI mit Codex. Diese Woche sind die anderen beiden Großen dazugekommen. Microsoft hat auf seiner Build-Konferenz am zweiten Juni eigene Modelle vorgestellt, und Google hat sein Gemini drei Komma fünf Flash samt der Entwicklungsumgebung Antigravity breit ausgerollt. Damit stehen jetzt alle vier großen Labs mit einem eigenen Coding-Stack auf dem Platz. CNBC nennt Microsoft und Google spät dran, aber es sei zwingend, dass sie mitkämpfen. Der Grund ist einfach: Claude Code ist acht Monate nach dem Start der am schnellsten wachsende Coding-Agent und unter erfahrenen Entwicklern der klare Favorit, laut JetBrains-Umfrage wählen ihn sechsundvierzig Prozent gegenüber neun Prozent für Copilot. GitHub Copilot bleibt aber bei der reinen Verbreitung führend, mit rund einundfünfzig Prozent in der Stack-Overflow-Umfrage.

Microsoft hat zwei Modelle, die uns angehen. MAI-Code-1-Flash ist ein kleines, schnelles Coding-Modell, das direkt in GitHub Copilot und Visual Studio Code landet und auf dem echten Copilot-Produktions-Harness trainiert wurde. MAI-Thinking-1 ist Microsofts erstes Reasoning-Modell, von Grund auf trainiert ohne Distillation. Die Botschaft dahinter: weniger Abhängigkeit von OpenAI. Wichtig ist mein Skepsis-Filter: Alle gemeldeten Zahlen sind Hersteller-Zahlen, noch nicht unabhängig nachgemessen. Auf dem härteren Test SWE-Bench Pro liegen die MAI-Modelle bei rund einundfünfzig bis dreiundfünfzig Prozent, während Anthropics Opus vier Komma acht bei neunundsechzig Prozent steht. Sie spielen also eine Klasse tiefer, treffen aber ihr Preis- und Größen-Segment.

Praktisch am interessantesten ist kein Modell, sondern ein Werkzeug: die neue GitHub-Copilot-App. Das ist eine eigenständige Desktop-Anwendung, getrennt von der Editor-Erweiterung, die mehrere KI-Agenten parallel orchestriert. Jeder Agent arbeitet in seinem eigenen, isolierten Git-Worktree, also einer eigenen Arbeitskopie des Projekts, sodass sie sich nicht in die Quere kommen. Das ist genau das Orchestrator-Muster, das Daniel von Claude Code kennt, nun als native App. Dazu ist das Copilot-Entwickler-Kit jetzt in sechs Sprachen verfügbar.

Google setzt mit Gemini drei Komma fünf Flash auf Tempo: rund vierfache Geschwindigkeit gegenüber Opus oder GPT, in Antigravity bis zwölffach. Auch hier gilt der Skepsis-Filter, denn die Coding-Werte stammen von Google selbst und teils aus Google-eigenen Test-Suiten. Unabhängig eingeordnet liegt Flash im soliden Mittelfeld, nicht an der Coding-Spitze. Sein Revier ist Qualität pro Sekunde und pro Dollar, nicht der direkte Qualitäts-Vergleich mit Opus. Antigravity zwei Komma null selbst ist Googles direkter Angriff auf die Agent-Ebene von Claude Code und Cursor, mit gehosteten Agenten direkt über die Programmierschnittstelle.

Heute selbst gab es nur Kleinigkeiten: Anthropic hat sein Partner-Netzwerk erweitert, OpenAI hat in der Enterprise-Variante von ChatGPT das Teilen von Codex-Plugins standardmäßig aktiviert. Bei den offenen Modellen tut sich seit MiniMax M3 vom ersten Juni nichts Neues, und Werkzeuge wie Aider, Cline und OpenHands bleiben still.

Mein Fazit: Vier Stacks bedeuten nicht vier echte Optionen. Opus vier Komma acht bleibt die Coding-Referenz, die Newcomer adressieren andere Achsen. Probierenswert sind die GitHub-Copilot-App für Multi-Agent-Arbeit und ein Blick auf Antigravity. Vor jedem Stack-Wechsel würde ich aber die unabhängigen Benchmarks abwarten. Nächste Woche schaue ich vor allem darauf, ob Google endlich Gemini drei Komma fünf Pro liefert.
