Fable 5 ist jetzt ein Bezahl-Modell [1][2][3]

Das ist die Nachricht, die deinen Workflow direkt trifft. Anthropic hat Fable 5 (Release 09.06.2026) am 23.06. aus den Plan-Limits genommen — Weiternutzung läuft jetzt über bezahlte API-Credits.[2] Die Preise: $10/M Input-Tokens, $50/M Output.[1] Das ist laut ayautomate „das teuerste generell verfügbare Frontier-Modell, das Anthropic ausliefert, doppelter Preis von Opus 4.8".

$/M Tokens: Du bezahlst pro Million Wörter-Bausteine, die rein- und rausgehen — wie ein Taxameter, das beim Lesen und beim Antworten tickt. Output ist teurer, weil das Modell dafür „denkt". Bei $50/M raus summiert sich ein agentic Lauf mit viel Tool-Output schnell.

Praktische Folge: Wer Fable 5 über das Plan-Kontingent zieht, verbrennt es laut future-stack-reviews mit „roughly double the rate of Opus 4.8".[3] Für Daniels Setup heißt das: Fable 5 ist ab jetzt das Modell für die schweren 10 % — die Architektur-Entscheidung, der knifflige Refactor —, nicht der Default-Fahrer. Opus 4.8 bleibt für den Alltag das bessere Preis-Leistungs-Verhältnis, und für Routine-Code lohnt es weiter, auf günstigere Modelle (oder lokal: Qwen/Gemma) zu routen. Ein „Fable 5 für alles"-Reflex wird jetzt teuer.

SpaceX will Cursor kaufen — gemeldet, nicht besiegelt [4][5]

Großer Aufschlag, aber mit Vorsicht zu lesen: Laut TechCrunch (16.06.2026) hat SpaceX eine Übernahme von Cursor (Anysphere) vereinbart, die das Coding-Startup mit $60 Mrd. bewertet — komplett in Aktien.[4] Wichtig: Das ist eine gemeldete, noch nicht abgeschlossene Transaktion; der Abschluss wird erst für Q3 2026 erwartet, kann also platzen oder sich verschieben.

Die Zahlen dahinter: Cursor habe „roughly $4 billion in annualized revenue in under four years" erreicht, davon laut CNBC ~$2,6 Mrd. aus Enterprise-B2B — der Deal entspreche dem ~15-fachen Umsatz.[5] Warum das für die Toolbox zählt: Wenn ein Raketenbauer den meistgenutzten AI-Editor übernimmt, ist das ein Signal, dass Coding-Tools zur strategischen Infrastruktur werden — und ein Erinnerungsposten, sich nicht zu fest an einen einzelnen proprietären Editor zu binden, dessen Eigentümer (und Roadmap) sich gerade ändern könnte. Cursor selbst lieferte parallel weiter: v3.9 (22.06.) bündelt Plugins, Skills und MCPs in einer Customize-Page, v3.8 (18.06.) brachte Automations mit GitHub-/Slack-Triggern.[5]

Google Managed Agents: Agent per einem API-Call try [6]

In der Gemini API rollt Google Managed Agents als Preview aus (Gemini API + AI Studio Playground).[6] Idee: ein Agent, der „reasons, uses tools and executes code in an isolated, ephemeral Linux environment with a single API call". Befeuert wird das vom Antigravity-Agenten auf Gemini 3.5 Flash ($1,50/$9,00 pro M Tokens). Definiert werden die Agenten über Markdown — AGENTS.md + SKILL.md, also versionierbar und deklarativ.

Warum probieren: Du musst keine eigene Sandbox-Infrastruktur betreiben — der Linux-Container kommt mit dem API-Call. Für ein schnelles „Agent baut und testet etwas isoliert" ohne eigenes Harness-Setup ist das ein günstiger Einstieg. Abgrenzung: Die Gemini Enterprise Agent Platform (das Industrie-/Business-Paket drumherum) gehört in den ai-usecases-Brief — hier nur das, was für Dev-Hände relevant ist: der API-Baustein selbst.

Benchmarks: starke Zahlen, schwache Beweislage [7][8][9]

Das Leaderboard-Feld bewegt sich — aber die wichtigste Zahl ist nicht ein Score, sondern eine Methoden-Warnung. Auf dem SWE-Bench-Verified-Board von llm-stats.com führt Fable 5 mit 95,0 % (über 102 evaluierte Modelle), gefolgt von Claude Mythos Preview (93,9 %) und Opus 4.8 (88,6 %); Gemini 3.1 Pro und DeepSeek-V4-Pro-Max teilen sich 80,6 %.[7] Stand: Leaderboard-Scrape 24.06.2026.

Skeptisch lesen Laut digitalapplied waren von 100 Modellen auf diesem Board (Stand 16.06.) nur 1 Ergebnis unabhängig verifiziert — die anderen 99 reichten die Anbieter selbst ein.[8] Das heißt: Diese 95 % sind ein von Anthropic gemeldeter Wert, kein neutral gemessener. Bei selbstberichteten Scores in diesem Bereich (95–99) ist Vorsicht Pflicht — Benchmark-Gaming, Trainings-Kontamination und Scaffolding (das Agent-Gerüst um das Modell) machen oft mehr aus als das rohe Modell.

Zur Einordnung „mit Gerüst": Auf SWE-Bench Pro (Anthropics agentic Benchmark) postet Fable 5 80,3 % vor Opus 4.8 (69,2 %)[9] — derselbe Sprung, andere Messlatte. Lies die 95 % also nicht als „macht 95 von 100 echten PRs", sondern als „führt auf einem Board, dessen Zahlen die Hersteller selbst eintragen".

Claude Code & Codex: dichte Tool-Woche [11][12]

Claude Code lief von 17.–23.06. in hoher Kadenz. v2.1.187 (23.06.) bringt MCP-Login/Logout per CLI, ein sandbox.credentials-Setting (blockt sandboxed Befehle vom Lesen von Credential-Dateien) und Org-Modell-Restriktionen.[11] Dazwischen: blockierte destruktive Git-Befehle im Auto-Mode, ein /cd-Kommando zum Wechseln des Arbeitsverzeichnisses mitten in der Session, Sub-Agent-Ketten bis 5 Ebenen tief, und Artefakte als teilbare Live-Webseiten (Versionshistorie als Beta auf Team/Enterprise).

MCP: Ein Standard-Stecker, über den der Coding-Agent externe Werkzeuge und Datenquellen anspricht — wie ein USB-Port, an den du Datenbank, Ticketsystem oder Browser hängst, ohne für jedes ein eigenes Kabel zu löten. Das neue Login/Logout-CLI verwaltet die Zugänge zu diesen Steckern sauberer.

Codex CLI 0.142.0 (22.06.) bringt Usage-Credit-Einlösung, Plugin-Organisation nach Kategorie und konfigurierbare Rollout-Token-Budgets.[12] Schon am 15.06. kam /import zur Migration aus Claude Code, am 18.06. „Record & Replay" für macOS (macht aus vorgeführten Workflows wiederverwendbare Skills). Der /import-Pfad ist bemerkenswert: OpenAI senkt aktiv die Wechsel-Hürde von Claude Code zu Codex.

Codex vs. Claude Code: zwei Philosophien [10]

Der Harness-Vergleich ist gerade aufschlussreich. Codex setzt auf 8 parallele Subagenten (GA) und ein 200K-Kontextfenster; Claude Code auf ein 1M-Token-Fenster — verbraucht dafür aber „3–4× more tokens" für identische Aufgaben (im Figma-Plugin-Beispiel 6,2M vs. 1,5M Tokens).[10] Das ist der Trade-off in Reinform: Claude Code sieht mehr Kontext auf einmal, kostet aber mehr pro Aufgabe — was nach der heutigen Fable-5-Preiserhöhung doppelt ins Gewicht fällt. Codex ist token-sparsamer, muss aber den engeren Kontext aktiver managen. Zur Verbreitung: Claude Code soll laut morphllm „326K+ daily commits (~10% of all public commits)" erzeugen, Codex steht bei 91.000 GitHub-Stars.[10]

Konkret für die Kosten-Rechnung: Wenn Claude Code 3–4× mehr Tokens zieht und man darin das teurere Fable 5 fährt, multiplizieren sich die beiden Faktoren. Für token-intensive Routine-Läufe ist der günstigere Pfad ab heute klarer denn je — ein günstigeres Modell im Harness oder, wo es passt, ein lokales (Qwen-Coder, Gemma) für Aufgaben, die kein Frontier-Reasoning brauchen. Frontier-Tokens nur dort ausgeben, wo das Problem sie wirklich verlangt.
