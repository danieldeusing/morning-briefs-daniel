Claude Sonnet 5 — der neue Default im Terminal

Anthropic hat gestern (30.06.) Claude Sonnet 5 veröffentlicht, und heute macht Claude Code v2.1.197 es zum Standard-Modell. Das ist die praktisch relevanteste Nachricht des Tages: Wer Claude Code offen hat, arbeitet ab dem Update ohne eigenes Zutun mit einem neuen Modell.[1][2]

Die Kern-Zahlen: 85,2 % auf SWE-bench Verified und 63,2 % auf dem härteren SWE-bench Pro (von Sonnet 4.6s 58,1 %). Dazu ein natives 1M-Token-Kontextfenster (bis 128k Output) — das erste Mal, dass Sonnet in Claude Code mit der langen Kontextlänge läuft, die man vorher Opus vorbehalten hatte.[3][1][2]

SWE-bench Verified vs. Pro: Beide messen, ob ein Modell echte GitHub-Bugs eigenständig fixt. „Verified" ist die von Menschen geprüfte, sauber lösbare Auswahl (~500 Fälle); „Pro" ist eine bewusst schwerere, kontaminations-resistentere Variante. Denk an TÜV vs. Rallye: dieselbe Prüfung, aber Pro ist die Strecke, auf der die Reifen wirklich glühen — deshalb liegen die Pro-Zahlen aller Modelle 20+ Punkte tiefer.

Try this Der Einführungspreis von $2 pro Mio. Input- / $10 pro Mio. Output-Token (bis 31.08., danach $3/$15) macht Sonnet 5 sogar günstiger als Sonnet 4.6 — bei besseren Bench-Zahlen. Für den Alltags-Loop in Claude Code lohnt sich das Ausprobieren schlicht, weil du eh schon draufgeschaltet wirst; einen bewussten Vergleichslauf gegen deinen bisherigen Default würde ich trotzdem machen, bevor du 1M-Kontext-Prompts baust (langer Kontext kostet real).[3][1]

Claude Code: der Harness-Sprint der letzten Woche

Neben dem Modellwechsel hat Claude Code in einer Woche gleich mehrere spürbare Features bekommen. Der Reihe nach, mit Datum:

v2.1.196 (30.06.) bringt zwei Dinge, die man ernst nehmen sollte. Erstens ein MCP-Sicherheits-Gate: In .mcp.json definierte Server starten nicht mehr automatisch, sondern brauchen ausdrückliche Freigabe. Zweitens dürfen Subagents jetzt 5 Ebenen tief geschachtelt werden — plus Org-weite Default-Modell-Steuerung.[2]

MCP (Model Context Protocol): Ein offener Standard, über den ein Agent externe Werkzeuge und Datenquellen anspricht — wie ein USB-Stecker für KI-Tools: einmal definiert, spricht jeder Client dieselbe Buchse an. Das neue Gate ist der Grund, warum das wichtig ist: Ein .mcp.json, das ein Repo mitbringt, könnte sonst beim Öffnen ungefragt einen Server starten — jetzt fragt Claude Code erst nach.

v2.1.191 (25.06.) lieferte den /rewind-Befehl (Konversation an einen früheren Punkt zurückspulen — praktisch, wenn ein Agent-Lauf entgleist), MCP-Retry-Logik und eine ~37 % CPU-Reduktion durch gebündelte Text-Updates (100-ms-Fenster). Letzteres merkst du vor allem auf dem Laptop im Akku-Betrieb.[2]

Einordnung: Das 5-Ebenen-Subagent-Nesting ist die Ansage im Wettlauf um „Agenten, die Agenten koordinieren" — Cursor geht mit Cloud-VMs denselben Weg (unten). Das MCP-Gate ist die reifere Antwort: erst Kontrolle, dann Autonomie.

GLM-5.2 — das freie Feld hat einen neuen Anführer

Aus dem Open-Weights-Lager kommt der stärkste Beitrag: Z.ai hat GLM-5.2 zunächst an Coding-Plan-Abonnenten (13.06.) und dann als offene Gewichte unter MIT-Lizenz (16./17.06.) veröffentlicht. MIT heißt: du darfst es lokal laufen lassen, kommerziell nutzen, feintunen — ohne Nutzungs-Klausel.[7][8]

Die Zahlen, die zählen: 62,1 % SWE-bench Pro (von 58,4 %) — damit führt GLM-5.2 die offenen Modelle an — und 81,0 % Terminal-Bench 2.1 (von 63,5 %). Auf dem Intelligence Index v4.1 steht es mit 51 vorn (MiniMax-M3 44, DeepSeek V4 Pro 44). Architektur: 753B Parameter total, ~40B aktiv als Mixture-of-Experts, 1M-Kontext.[8][7][9]

Mixture-of-Experts (MoE): Statt für jedes Token das ganze 753-Milliarden-Netz zu rechnen, aktiviert das Modell nur ein passendes Teil-Netz (~40 Mrd.). Wie ein Krankenhaus, das nicht alle Ärzte zu jeder Untersuchung ruft, sondern nur die Fachrichtung, die gebraucht wird — volle Kompetenz im Haus, aber pro Fall arbeitet nur ein Bruchteil. Das macht ein sonst riesiges Modell bezahlbar im Betrieb.

Try this Für den lokalen/souveränen Einsatz ist das der interessanteste Kandidat der Woche: offene Gewichte, führende freie SWE-bench-Pro-Zahl, 2,9× weniger FLOPs pro Token bei 1M-Kontext (IndexShare-Attention). Über OpenRouter liegt der Preis bei ~$1,40/$4,40 — deutlich unter den proprietären Flaggschiffen.[7]

Ein Haken GLM-5.2 ist gesprächig: 43k Output-Token pro Aufgabe (Intelligence-Index) gegenüber 26k bei GLM-5.1. Bei Output-basierter Abrechnung frisst das einen Teil des günstigen Token-Preises wieder auf — im Agenten-Loop mit vielen Schritten spürbar. Rechne das in deinen Vergleich ein.[7]

Das öffentliche SWE-bench-Verified-Leaderboard wurde heute (01.07.) aktualisiert. Die Spitze im Überblick — und gleich die Warnung dazu:

SWE-bench Verified — Top 4, Stand 01.07.2026

Fable 5 steht mit 95,0 % vorn, Opus 4.8 bei 88,6 %, der neue Sonnet 5 bei 85,2 %, GLM-5 (5.1) bei 77,8 %. Auf dem härteren SWE-bench Pro dreht sich das Feld: Opus 4.8 führt aktiv mit 69,2 %, GLM-5.2 mit 62,1 % als bestes offenes Modell, Sonnet 5 bei 63,2 %.[10][11]

Skepsis-Pflicht Das sind überwiegend anbieter-gemeldete Zahlen. Scale/SEAL misst auf einem standardisierten, kontaminations-kontrollierten Set deutlich niedriger: Dort führt GPT-5.4 (xHigh) mit 59,1 %, Opus 4.6 thinking mit 51,9 %. Die Faustregel aus der Praxis: Anbieter-Scores liegen 15–30 Punkte über den standardisierten. Nimm die 95 % also nicht als „fast gelöst" — nimm sie als Reihenfolge-Hinweis, nicht als Absolutwert.[11][12]

Warum das für dich zählt: Ein Modell, das auf Verified glänzt, aber im eigenen Repo mit ungewöhnlichem Build-Setup patzt, ist nicht überraschend — Benchmark-Gaming und Trainingsdaten-Nähe sind real. Der einzige verlässliche Test bleibt dein eigener Code.

Am Rande

Anthropics Fable 5 und Mythos 5 tauchen im Verified-Board mit 95,0 % auf, waren aber seit 12.06. wegen einer US-Export-Kontroll-Direktive zeitweise nicht verfügbar (Rückkehr um den 01.07. erwartet). Die Modell-Verfügbarkeits- und Export-Story gehört ins Nachbar-Revier — Details im ai-usecases-Brief (dort auch GeneBench-Pro). Hier zählt nur: Wenn Fable 5 als Board-Referenz auftaucht, ist es kein Alltags-Tool für dich, sondern die Messlatte am oberen Ende.

Es ist Mittwoch, der erste Juli, und es ist ein voller Tag im KI-Coding-Feld. Gleich drei Modell-Klassen bewegen sich zur selben Zeit — ein neues Standard-Modell im Terminal, eine Preview vom großen Konkurrenten, und ein starker offener Herausforderer.

Die praktisch wichtigste Nachricht: Anthropic hat gestern Claude Sonnet fünf veröffentlicht, und ab heute macht Claude Code in Version zwei-Punkt-eins-Punkt-einhundertsiebenundneunzig es zum Standard-Modell. Wer Claude Code offen hat, arbeitet nach dem Update automatisch damit. Sonnet fünf erreicht fünfundachtzig Komma zwei Prozent auf SWE-bench Verified und dreiundsechzig Komma zwei auf der härteren Pro-Variante, hat jetzt ein Kontextfenster von einer Million Token — und kostet im Einführungspreis nur zwei Dollar für Eingabe und zehn für Ausgabe je Million Token, bis Ende August. Damit ist es sogar billiger als der Vorgänger.

Bei OpenAI läuft seit vergangenem Freitag eine Preview der GPT-fünf-Punkt-sechs-Reihe: Sol, Terra und Luna, drei Stufen wie bei den anderen Anbietern auch. Aber Zugang haben nur etwa zwanzig Partner-Organisationen, und — das ist entscheidend — es gibt noch keinen einzigen öffentlichen Coding-Benchmark dafür. Nur einen Terminal-Bench-Wert von rund achtundachtzig Prozent, der aber etwas anderes misst als SWE-bench. Ich nenne hier bewusst keine Vergleichszahl gegen Sonnet, die es schlicht nicht gibt. Bis zum breiten Release ist GPT fünf-Punkt-sechs nur ein Eintrag auf der Beobachtungsliste.

Aus dem Lager der offenen Modelle kommt der stärkste Beitrag: GLM fünf-Punkt-zwei von Z.ai, veröffentlicht unter der freien MIT-Lizenz. Es führt mit zweiundsechzig Komma eins Prozent auf SWE-bench Pro das gesamte offene Feld an, hat ein Kontextfenster von einer Million Token und läuft dank Mixture-of-Experts-Architektur trotz seiner Größe bezahlbar. Für den lokalen oder souveränen Einsatz ist das der interessanteste Kandidat der Woche. Ein Haken: Es ist gesprächig und verbraucht spürbar mehr Ausgabe-Token pro Aufgabe — das solltest du in die Kostenrechnung einbeziehen.

Bei den Werkzeugen hat Claude Code selbst einen Sprint hingelegt: Subagents dürfen jetzt fünf Ebenen tief geschachtelt werden, ein neues Sicherheits-Gate verhindert, dass MCP-Server ungefragt starten, es gibt einen Rewind-Befehl zum Zurückspulen entgleister Läufe, und die CPU-Last sank um rund siebenunddreißig Prozent. Cursor zieht mit Cloud-Subagents und einer neuen iOS-Beta nach, mit der man Agenten vom Handy aus steuert.

Zu den Benchmarks noch ein Wort Vorsicht: Das aktualisierte Verified-Board zeigt oben Werte über neunzig Prozent, aber das sind überwiegend Anbieter-Zahlen. Auf standardisierten, kontaminations-kontrollierten Sets liegen dieselben Modelle fünfzehn bis dreißig Punkte tiefer. Nimm die Reihenfolge, nicht den Absolutwert — und teste am eigenen Code.

Morgen dürfte vor allem Praxis-Feedback zu Sonnet fünf im echten Einsatz kommen, und irgendwann in den nächsten Wochen der breite GPT-fünf-Punkt-sechs-Release mit den ersten echten Coding-Zahlen. Bis dahin.
