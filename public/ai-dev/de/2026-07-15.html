Continue geht in Cursor auf — heute stirbt die Cloud [1][2]

Continue.dev, der quelloffene Coding-Agent für VS Code, CLI und JetBrains, ist von Cursor (Anysphere) übernommen worden — angekündigt um den 16.06.2026 herum, ohne Pressemitteilung, nur als Zeile auf der Homepage („Continue has been acquired by Cursor"). Der letzte Release v2.0.0-vscode lief am 19.06. auf GitHub.[1]

Der für Daniel entscheidende Teil ist datiert und heute: Die gehostete Plattform wird abgewickelt, und die dort liegenden Cloud-Daten — Chatverlauf, gespeicherte Configs, Team-Settings — werden nach dem 15.07.2026 gelöscht.[2] Wer Continue nur lokal in der IDE nutzte, ist nicht betroffen; wer die Hosted-Plattform mit Team-Einstellungen fuhr, sollte heute exportieren.

Heute handeln Nutzt du Continues gehostete Plattform? Configs und Team-Settings jetzt exportieren — nach heute sind sie weg. Das lokale Open-Source-Continue bleibt auf GitHub bestehen (letzter Stand v2.0.0-vscode, 19.06.).

Einordnung: Es ist die zweite große Konsolidierung im Agent-Tooling binnen weniger Wochen — der Käufer-Kreis um Cursor zieht offene Projekte ein, statt sie als Konkurrenz laufen zu lassen. Für die Tool-Landschaft heißt das weniger Vielfalt bei den gehosteten Agenten; das lokale, self-hosted Lager (Cline, Aider, OpenHands) gewinnt dadurch an relativer Bedeutung.

GPT-5.6 vs. Muse Spark vs. Fable 5 — wer kann wirklich coden? [3][4][5]

Zwei Modell-Familien kamen am selben Tag, dem 09.07. — mit unterschiedlichem Reifegrad. OpenAIs GPT-5.6 wurde allgemein verfügbar (drei Stufen — Sol, Terra, Luna — nach einer Limited Preview seit dem 26.06.). Metas Muse Spark 1.1, ein agentisches Modell mit 1-Mio-Token-Kontext, das MCP und eigene Skills spricht, ist dagegen erst in öffentlicher Vorschau: die Meta Model API läuft vorerst nur in den USA (noch nicht in der EU), allgemein live ist bislang nur der Consumer-„Thinking-Mode" in der Meta-AI-App.[3][4][5]

MCP / Tool-Use: Model Context Protocol ist ein einheitlicher „Stecker", über den ein KI-Agent externe Werkzeuge anspricht — Dateisystem, Datenbank, API. Wie USB-C für Tools: einmal implementiert, überall nutzbar. „Tool-Use" heißt, das Modell ruft diese Werkzeuge selbst auf, statt nur Text zu produzieren.

Die Marketing-Zahlen: GPT-5.6 Sol führt den Artificial-Analysis-Coding-Agent-Index mit 80 Punkten und meldet auf Terminal-Bench 2.1 88,8 % sowie auf DeepSWE v1.1 72,7 %.[3] Muse Spark 1.1 wiederum führt bei reiner Werkzeug-Nutzung (MCP Atlas 88,1), vor Opus 4.8 und GPT-5.5.[5]

Die Skepsis: Auf SWE-Bench Pro — dem härteren, näher an echter Arbeit liegenden Test — dreht sich das Bild. Dort schlägt Anthropics Claude Fable 5 mit rund 80 % das GPT-5.6 Sol (64,6 %) deutlich, und Muse Spark 1.1 (61,5) liegt hinter Opus 4.8 (69,2). Auf DeepSWE 1.1 und Terminal-Bench 2.1 landet Muse Spark ebenfalls hinter GPT-5.5 und Opus 4.8.[4][5] Pikant: OpenAI hat keinen SWE-Bench-Verified-Wert für GPT-5.6 veröffentlicht — und reagierte auf die Pro-Niederlage mit einer öffentlichen Kritik, „~30 % der SWE-Bench-Pro-Aufgaben" seien kaputt.[4]

SWE-Bench Pro vs. Verified: SWE-Bench misst, wie viele echte GitHub-Bugfixes ein Modell selbstständig löst — jede Aufgabe hat einen Test, der beweist, ob der Fix sitzt. „Verified" ist die von Menschen geprüfte, saubere 500er-Auswahl (deshalb hohe Zahlen); „Pro" ist ein größerer, härterer Satz — deshalb liegen die Werte dort niedriger und die Modelle weiter auseinander.

Was Daniel mitnimmt: Anbieter-Zahlen sind kein Ranking, sondern ein Marketing-Cherry-Pick — jeder führt genau das Board an, das er zeigt. Für tägliches Coding zählt weniger der Index-Sieg als die Pro-/Verified-Realität, und die spricht diese Woche für die Claude-Tier. Wichtige Einschränkung zur Quelle: OpenAIs eigene GPT-5.6-Seiten waren nicht erreichbar (403), die GA-/Preis-/Bench-Zahlen stammen aus zwei sich deckenden Sekundärquellen (marktechpost, simonwillison.net), nicht von OpenAI direkt.

Preise (pro 1M Tokens): GPT-5.6 Sol 5 $/30 $, Terra 2,50 $/15 $, Luna 1 $/6 $.[3] Muse Spark 1.1 laut Sekundärberichten 1,25 $/4,25 $ (plus 20 $ Startguthaben) — nicht primär bestätigt.

Claude Code: eingebauter Browser + schnellere CLI [7][8]

Claude Code hat im Release-Fenster 06.–10.07. (v2.1.202–206) einen eingebauten, getabbten Sandbox-Browser bekommen, der sich mit Cmd+Shift+B (macOS) bzw. Ctrl+Shift+B (Windows) öffnet. Sicherheits-Klassifizierer prüfen alle Schreib-Aktionen auf fremden Seiten; Käufe oder Account-Erstellungen brauchen ausdrückliche Freigabe.[7] Try Für „lies diese Doku / prüf diese Web-App"-Schleifen spart das den externen Browser-Umweg.

Sandbox-Browser: Eine Sandbox ist ein abgeschotteter Spielplatz. Der Browser läuft in einem isolierten Bereich, in dem der Agent klicken und tippen darf, ohne an dein echtes System oder deine echten Konten zu kommen.

Die CLI selbst steht bei v2.1.210 (14.07.) — gleich drei Releases (v2.1.208–210) liefen an diesem Tag als großes Bugfix-/Performance-Bündel: Screen-Reader-Modus (Plain-Text-Rendering), bis zu 7× schnellere Tool-Runden bei vielen MCP-Tools und eine bis zu 79× kleinere Session-Transkript-Größe in edit-lastigen Sessions.[8] Ein reiner Coding-Feature-Sprung ist nicht dabei — es ist Wartung, aber spürbare.

GitHub Copilot — drei kleine Schritte [9][10]

Copilot rollt in kleinen Schritten aus. 01.07.: Agentische Browser-Tools in VS Code erreichen GA — standardmäßig aktiv, der Agent kann Seiten navigieren, Inhalte inspizieren, Screenshots machen und Web-Apps direkt in der IDE prüfen. BYOK: „Bring Your Own Key" — du hängst deinen eigenen API-Schlüssel eines Modell-Anbieters ins Tool, statt das Kontingent des Tool-Herstellers zu verbrauchen. Wie beim Streaming den eigenen Account mitbringen statt einen fremden.

Interessant ist die Konvergenz: Copilot-Browser-Tools (01.07.) und Claude-Code-Sandbox-Browser (Woche 28) landen im selben Fenster — „der Agent bedient einen Browser selbst" wird zur Standard-Fähigkeit statt zum Alleinstellungsmerkmal.
