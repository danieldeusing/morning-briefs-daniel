While Anthropic is still putting out the credit-pool fire since Tuesday and Google deliberately says nothing over the weekend, OpenAI repainted an org chart on Friday at 5 PM PT that matters directly for devs. TechCrunch and Benzinga confirm: ChatGPT, Codex, and the developer API now sit under one shared product lead — Greg Brockman permanently (vs. interim, while Fidji Simo is on medical leave). Why this matters to you

Until now the API ran on its own roadmap cadence, Codex on its own, ChatGPT on its own — which explains why the same GPT-5.5 logic landed in the three surfaces at three different times. A shared lead means a shared release sequence: features that ship first in consumer ChatGPT should drip into the API faster. In return, pressure rises to slip API features into the Codex workflow earlier.

That's the reading Brockman himself would prefer — the reporter reading is more pointed: "three products with one stack rank for what gets tuned first." Both can be true at once. Practically: API stability SLAs will get more demanding in the coming months because consumer features can grab backlog priority. If you have production pipelines on the responses API, read the changelog more closely.

Correction · JetBrains ACP is Client, not Control

One important correction to yesterday. Saturday's brief talked about JetBrains' "Agent Control Protocol" as a "vendor-neutral agent connector, competing with MCP." Both are wrong.

What ACP actually is

ACP stands for Agent Client Protocol. It's co-developed with Zed, not JetBrains-only and not positioned as a rival to MCP. The point: MCP governs communication between agent and tools/data ("fetch me the schema from this database"). ACP governs communication between editor and agent ("user hit cmd-K on line 42, here's cursor position and selected code"). The two protocols live on different layers; they don't compete.

JetBrains has been a participant in the ACP registry since January 2026; Cursor joined in March; the genuinely fresh point this week is the ReSharper 2026.2 EAP announcement of May 11: ACP now runs inside Visual Studio via Junie — the first host outside Zed and JetBrains IDEs.

agentclientprotocol.com · blog.jetbrains.com/dotnet
User in editor → ACP → Agent → MCP → Tools / data
Lesson learned With three-letter protocol names prefixed by "Agent": double-check once more than feels necessary. ACP, A2A, MCP, ANP, AGNTCY — every three weeks someone invents another. When in doubt, hit the spec site, not a recap.

Benchmark reality · Verified vs Pro [1][2][3]

Vendors quote SWE-Bench Verified because there, models hit 80–93%, which looks good on stage. OpenAI itself dropped Verified in February, citing: "all frontier models are contaminated" and "59.4% of hard tasks have flawed tests." If you want a realistic comparison, look at SWE-Bench Pro — the successor suite that's harder to memorize.

Benchmark contamination: like when a student accidentally brought last year's exam questions home while studying. He "systematically" solves the problems — but the system is memorization, not skill. With LLMs that means: the benchmark questions slipped into the training set. High scores on Verified, mediocre scores on Pro = likely contamination.
