Hangzhou counters · Qwen 3.7 Max — flagship yes, weights no [1][2][3]

While Google had the stage last night, Alibaba delivered the counter today: at the Cloud Summit in Hangzhou (05-20–05-21), Qwen 3.7 Max was officially unveiled — built for agentic workloads, "sustained, multi-step operations," shipped via Bailian (outside China: Model Studio). That turns the "preview sighting" of recent days into a real launch — with one important caveat.

The caveat: no open release

As of today there are no open weights for Qwen 3.7: Hugging Face and ModelScope still carry only the 3.6 generation (Qwen3.6-27B, Qwen3.6-35B-A3B under Apache 2.0), no QwenLM/Qwen3.7 repo.[2] Qwen 3.7 Max is API-only today. That fits the pattern: Alibaba keeps — as it did with 3.6 Max/Plus — its strongest models proprietary, with the open mid-tier variants arriving (if at all) later and smaller. Skeptical reading: a flagship reveal is not yet a gift to the self-hosting community. Anyone waiting for open Qwen 3.7 weights for the local stack is still waiting.

alibabacloud.com · scmp.com

The hardware half: Zhenwu M890

More substantial than the model itself may be the chip: Alibaba's T-Head subsidiary showed the Zhenwu M890 — 3x the performance of its predecessor, the Zhenwu 810E, 144 GB on-chip memory, precision from FP32 down to FP4. Plus the Panjiu AL128 supernode (128 accelerators) and an ICN Switch 1.0 networking chip (25.6 Tbit/s). What it means for Daniel: this is China's answer to the export controls — a vertical stack strategy (own chip + own model + own cloud) that, over the medium term, shapes how cheap agentic inference becomes outside the NVIDIA world.

business-standard.com
FP4 (4-bit precision): how much a model "rounds" when computing. FP32 = full accuracy, lots of memory; FP4 = only 4 bits per number, much smaller and faster, but coarser. A chip that goes all the way down to FP4 can run large models more cheaply — as long as quality doesn't suffer too much from the coarse rounding.

Google I/O · Gemini 3.5 Flash becomes the default [4][5][6]

The real I/O story is a default decision. Google makes Gemini 3.5 Flash the default model immediately in the Gemini app, Search AI Mode, the API, AI Studio and Antigravity. The frontier Pro model (3.5 Pro) won't arrive until "next month." The bet: a fast, cheap model at frontier level beats the last benchmark point.

Numbers & price — the skeptical reading

Per Google's I/O material, 3.5 Flash beats its predecessor 3.1 Pro on the agentic suite: Terminal-Bench 2.1 76.2% (3.1 Pro 70.3%), MCP Atlas 83.6% (78.2%), Finance Agent v2 57.9% (43.0%), GDPval-AA 1656 Elo (1314). The more independent number: SWE-Bench Pro Public 55.1% (54.2%) — only +0.9 points.[5] Price: $1.50/1M input, $9.00/1M output, cached $0.15/1M; context 1,048,576 in / 65,536 out; four thinking levels. Skeptical note: Terminal-Bench, MCP Atlas and GDPval are Google's hand-picked stage benchmarks, and the comparison is against its own old predecessor — not against Opus 4.7 or GPT-5.5. The agentic jump is real; "beats the frontier" is a headline.

Antigravity 2.0 · the platform gets serious [4][7]

At I/O, Antigravity arrived in version 2.0 — the step from IDE to platform: a desktop app with an agent-first interface, a CLI with specialized sub-agents, an SDK for programmatic harness control, and Managed Agents in the Gemini API with a hosted Linux sandbox.

The 12-hour OS stunt

The stage demo: Antigravity + Gemini 3.5 Flash built a functioning operating system from scratch in 12 hours — with 93 parallel sub-agents, 15,000+ model requests, 2.6B tokens, under $1,000 in credits. Read it as what it is: a maximally parallelized brute-force run for the stage. The message to developers is "massive parallelism becomes affordable," not "your next OS writes itself."

developers.googleblog.com
Managed Agents (hosted sandbox): instead of your agent running code on your machine, it runs in a sealed-off Linux box in Google's cloud — like a throwaway Linux just for this one task. If something goes wrong, the box is gone and your machine stays clean. Handy for parallel agents, risky for data trust (your code briefly sits with Google).

MCP ecosystem · WebMCP as a standard proposal [6]

Browser agents get an open tool standard

One of the more substantial dev announcements at I/O: WebMCP, a proposed open web standard that lets websites expose structured tools — JS functions and HTML forms — so browser-based agents can run complex tasks on them. The experimental origin trial starts in Chrome 149. MCP won on the server side; WebMCP pulls the same idea into the browser. Still early, but the direction is clear.

developers.googleblog.com
MCP (Model Context Protocol): a single plug through which an AI model talks to external tools — like USB for AI tools. Instead of building a custom connection per app, every server speaks "MCP" and every agent understands it. WebMCP brings that plug into the website itself.
