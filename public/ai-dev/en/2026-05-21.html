Antigravity 2.0 · the reviews are in — and rough [1][2][3]

Yesterday's shiny I/O stage story, today the first reality check: with Antigravity 2.0 (GA since May 19) the first serious hands-on reviews are online — and they paint a split picture. The platform idea convinces; the delivery doesn't.

What gets praised

The Agent Manager — the surface that coordinates several autonomous agents in parallel — is described in several reviews as the "star of the show."[1] Parallel multi-agent workflows (two agents building concurrently), artifact generation (task lists, plans, screenshots as verification evidence) and one-shot generation all work in testing. That's the real move: away from the IDE-with-chat, toward an Agent-Manager surface.

What breaks

The catch is the delivery. Reviews from May 20 report that the automatic update to 2.0 removed the integrated code editor — leaving broken terminals, missing sidebars, overwritten configs and lost project settings.[2] Source control, local terminals and Remote-SSH/WSL2 dropped out; macOS auth loops; namespace conflicts when both versions coexist. Recommended workaround: roll back to 1.23.2 and disable auto-update. An honest dev.to review adds: buttons stop working mid-session, sidebar icons vanish, no Git Worktrees support, input lag while agents generate, heavy battery drain.[3]

piunikaweb.com · dev.to · scalablepath.com

The takeaway for Daniel: this is no reason to write off the idea — the Agent Manager is exactly the direction Cursor (Agents window) and GitHub (cloud agents) are heading too. But as a daily driver, 2.0 isn't ready today. If you want to test it, test it alongside your existing setup, not as a replacement, and with auto-update off. Price right now: free in preview, quota resets every 5 hours.

Gemini CLI → Antigravity CLI · a forced migration with a date [4]

Free + AI Pro/Ultra lose the Gemini CLI on June 18

In the shadow of the Antigravity story sits an announcement with an expiry date: Google is retiring the Gemini CLI for Free users and AI Pro/Ultra subscribers on 2026-06-18 — after that only the new, Go-written Antigravity CLI remains, sharing the agent harness with Antigravity 2.0 (async background orchestration; Gemini CLI extensions become Antigravity plugins). Enterprise licenses keep the Gemini CLI indefinitely.[4] In practice: if you maintain a Gemini CLI script or a CI hook and you're not Enterprise, you now have a migration ticket with a deadline — and you should test the Antigravity CLI early, while the platform (see above) is still bumpy.

developers.googleblog.com
Origin Trial (browser): A time-limited test program that lets a browser maker try out a not-yet-standardized feature on real websites — developers register their domain and enable the feature for their users without it being permanently baked into the browser. Like a test drive with temporary plates before the car is officially registered.

Browser agents get an open tool standard

The WebMCP announcement from I/O now has substance: it's a draft from the W3C Web Machine Learning Community Group (repo: webmachinelearning/webmcp), with two APIs — imperative (JS functions) and declarative (annotated HTML forms) — plus tool discovery, JSON-schema I/O and shared page state. Benchmarks · what's actually verifiable today [18][19][20]

As of 2026-05-21. There's no new, independently verifiable benchmark movement today — and that's precisely the story.

SWE-Bench Verified — the last independently verifiable standing
Source: github.com/SWE-bench/experiments (open repo). Academic submissions only since 2025-11-18 — commercial systems (Opus 4.7, GPT-5.5 …) are absent. Top: Opus 4.5 / 79.2%. The 87–94% figures making the rounds are vendor claims, not verifiable here.

Frozen, stale, JS-blocked

Three of the four coding boards don't qualify as "current" today: SWE-Bench Verified has been frozen in the open repo since November (top Opus 4.5 / 79.2%), and the live page loads via JS and returns no numbers to a crawl.[18] Aider Polyglot still reads 2025-11-20 (GPT-5-high 88.0%) — over six months old.[19] LiveCodeBench loads its table via JS; no statically verifiable score — so no number here rather than a guess. What is available is a preference source: the arena.ai WebDev arena (as of May 17) leads with Opus 4.7-thinking (1567) ahead of Opus 4.7 (1560), then GLM-5.1 (1532) — no Gemini 3.5 Flash in the top 5.[20]

github.com/SWE-bench/experiments · aider.chat · arena.ai
Heads up Google's "Gemini 3.5 Flash beats frontier models" is a headline, not a measured comparison. Arena Elo: Like a chess ranking, but for models: two answers compete anonymously, humans pick the better one, and a score emerges from that. More points = preferred more often. It measures the voters' taste, not whether the code objectively runs correctly — unlike a pass-rate benchmark such as SWE-Bench.
