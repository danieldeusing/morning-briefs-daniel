Codex 0.133.0 · Goal mode becomes the default [1][2]

Yesterday the racing alpha train (0.133.0-alpha.4 at 02:02 UTC), today the landed stable: Codex 0.133.0 (05-21, 16:48 UTC) is the day's most substantive drop. The headline isn't a new model but a workflow decision: Goal mode leaves beta and is now on by default — simultaneously in the Codex app, the IDE extension and the CLI.[1]

Goal mode: Instead of explaining the overall point to the agent anew at every step, it gets a persisted goal with its own storage that it carries across turns and checks its progress against. Like a sticky note with the end goal that the agent keeps re-reading — instead of you repeating yourself in every message.

What else is in it

Beyond Goal mode: better remote-control UX (now waits for readiness, daemon start/stop, machine status), a tougher Windows sandbox with permission profiles, and a marketplace-aware plugin list. The same day brought companion-app build 26.519 with Appshots (macOS) and remote computer use.[2] Context: this is OpenAI's answer to the same question Claude Code and Cursor are working on — how does an agent hold on to intent across many steps without the human constantly steering. That Goal mode is now the default is the real statement.

Claude Code patches itself the same day [3]

2.1.147 — the useful additions

Building on yesterday's /code-review rename, 2.1.147 (05-21) adds the substance: a --comment flag that posts review findings directly as inline comments on a GitHub PR, plus effort levels for the review. Pinned background sessions now survive idle and a restart-in-place during updates; the auto-updater retries transient failures and reports OS error codes. Also an MCP pagination fix (paginated tools/list responses only returned the first page, silently dropping tools) and several Windows/PowerShell fixes.[3]

2.1.148 — the self-hotfix

A few hours later, already on 05-22, came 2.1.148 with a single fix: after 2.1.147 the Bash tool returned exit code 127 on every command — a regression that 147 had introduced itself. Exactly the flip side of a high release cadence: ship daily (or several times a day) and you catch regressions daily. Practically for Daniel: move straight to 2.1.148, but skip 2.1.147 as a mandatory stepping stone — that goes for any CLI with multiple tags per day.
