Claude Code 2.1.149 · usage breakdown and three security fixes [1]

On a day when most vendors are taking the weekend off, Claude Code 2.1.149 (May 22, 22:09 UTC) is the only substantive stable release — and it has two faces: a nice feature and an unusually dense block of security fixes.[1]

The visible feature: /usage with a breakdown

/usage now shows a per-category breakdown — how much token budget goes to skills, subagents, plugins, and each individual MCP server. For anyone running several connected MCP servers, this is the first way to see which server costs what, instead of reading one lump sum. Plus: the /diff detail view is now keyboard-scrollable (arrows, j/k, PgUp/PgDn), Markdown renders GFM task-list checkboxes, and enterprise gets an allowAllClaudeAiMcps managed setting.

The real substance: the security fixes

Weightier than the feature are three fixes that directly touch Daniel's setup. Two PowerShell permission bypasses were closed: built-in commands like cd.. or cd\ could escape the granted workspace, and prefix/wildcard allow rules could be circumvented. Plus a worktree-sandbox fix (the write allowlist didn't apply cleanly to git worktrees) and — the nastiest — a find bug that exhausted the macOS vnode table and crashed the whole machine.[1]

vnode table: macOS keeps an internal list of every file and folder currently "touched" — like a coat check with a limited number of hooks. A runaway find hung so many coats that the system had no free hook left and froze. Not just the tool crashed — the host did.

Context: permission bypasses in an agent that runs shell commands aren't cosmetic — they're exactly the class of bug where a sandbox promise quietly springs a leak. Practical for Daniel: pull straight to 2.1.149 (or 150), especially anyone running Claude Code with connected MCP servers or under Windows/PowerShell.
