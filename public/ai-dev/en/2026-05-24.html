Open weights · Qwen Conference Monday, 3.7 still closed [1]

The real date is ahead of us: the Qwen Conference 2026 on Monday (May 26) is the most plausible stage for the open-weights drop local users have been waiting on for weeks. As of today the picture is unchanged from yesterday — so I re-checked exactly the question that resolves on Monday.[1]

Re-checked today: no open 3.7

huggingface.co/Qwen still carries only the 3.5 and 3.6 families — no 3.7 repo, no GGUFs, no date. The newest items there (~11 days old) are research artifacts on the 3.x base models, not a new chat model. Qwen 3.7-Max was unveiled May 20 at the Apsara/Alibaba Cloud Summit, but runs preview/API-only on chat.qwen.ai and LMArena — no open weights, no API pricing, no GGUFs. The promised open 27B/35B variants remain "forthcoming."[1]

preview/API-only vs. open weights: "Preview" means you may use the model through the website or a cloud endpoint — like renting a car. "Open weights" means you get the model file itself and can run it on your own machine — like buying the car. For Daniel's local Ollama/llama.cpp stack only the latter counts; a preview-only model isn't touchable locally.

The qwenlm.github.io and qwen.ai blogs are JS-rendered and returned only stale caches today — a pre-conference teaser could not be confirmed, so I'm not claiming one. Same for the Twitter page (@Alibaba_Qwen): unreadable today, no live signal.

MCP · one doc merge, otherwise weekend quiet [6][7]

PR #2769 · docs aligned to the schema

The spec repo saw exactly one notable move over the weekend: PR #2769 ("Update Documentation to match Schema," merged May 22, 19:48 UTC). Pure doc work, no breaking changes — the docs now name subscribe as an optional Resources feature, the mandatory _meta block in tool-call examples, and resource links in Prompts. After that, silence: no spec PRs on May 23 or 24.[6]

_meta (MCP): A side pocket in every MCP message for extra info that isn't part of the actual content — like the address label on a package: not the goods, but what the transport needs. The spec now makes it requirable in more responses.

Server side and the RC date

The GitHub MCP Server stays at 1.0.5 (May 18), and the MCP registry has had no new commit since May 14. The set date for the next spec release candidate stays 2026-07-28 (PR #2750), the published spec unchanged at 2025-11-25.[7]
