Open weights · Qwen Conference tomorrow, 3.7 still closed [1]

The real date is ahead of us — this time genuinely tomorrow: the Qwen Conference 2026 on Tuesday (May 26) in Singapore (Sands Expo, Marina Bay Sands) is the most plausible stage for the open-weights drop local users have been waiting weeks for. Correction to yesterday: I wrongly listed May 26 as "Monday" — the date is right, the weekday is Tuesday. As of today I re-checked exactly the question that lands tomorrow.[1]

Checked today: no open 3.7

huggingface.co/Qwen still carries only the 3.5 and 3.6 families — the qwen3.7 filter returns 0 hits, no repo, no GGUFs, no date. The newest entries there (~12 days old) are interpretability/SAE artifacts on the 3.x base models, not a new chat model. Qwen 3.7-Max debuted on LMArena May 14, hit the API May 19, and went official at the Alibaba Cloud Summit May 20 — but it runs preview/API-only (1M-token context, extended thinking), with no open weights.[1]

preview/API-only vs. open weights: "Preview" means you're allowed to use the model via the website or a cloud endpoint — like renting a car. "Open weights" means you get the model file itself and run it on your own machine — like buying the car. For Daniel's local Ollama/llama.cpp stack, only the latter counts.

Reporting says the Plus tier will be open-sourced while Max stays closed — so a locally runnable GGUF, if any, would arrive as a Plus or Air variant. So far that's a promise, not an artifact: until a HF repo exists, I treat it as unconfirmed. We'll know more tomorrow.

Speculative decoding (MTP): A small, fast model guesses the next few tokens on a hunch, the big model checks them all at once and waves the correct ones through — like an intern proposing sentences and the boss just nodding them off. Result: more tokens/second with identical output. "MTP" = multi-token prediction, the technique behind it.
