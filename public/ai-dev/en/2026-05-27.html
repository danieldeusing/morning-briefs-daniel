Claude Code · 2.1.152 brings auto-fix and skill control [1]

After last week, when Claude Code sat at 2.1.150 (2026-05-23), 2.1.152 is here today — and it's the most substantial build in the series in days. Three things are worth a look.[1]

/code-review --fix writes back

Until now /code-review was a read-only assessment. Now --fix writes the findings straight into the working tree — reuse, simplification and efficiency suggestions are applied instead of just listed. /simplify now calls the same --fix internally. This is the step from "the reviewer tells you" to "the reviewer does it." Try For Daniel's workflow, the most relevant point: approve a review pass and see the mechanical changes right in the diff, instead of porting them by hand.

code.claude.com/docs/changelog

Skills can now take tools away

Skills and slash commands can set disallowed-tools in their frontmatter — tools removed from the model while the skill is active. Add /reload-skills (re-scan skill directories without restarting the session) and a SessionStart hook that can return reloadSkills: true, so a skill installed by the hook is available in the same session.

disallowed-tools in a skill: Picture a toolbox from which you deliberately remove the sharp saw for a given task, so nobody grabs it by accident. A skill can now say "while I run, no Bash" — a tighter frame means fewer ways to break something.
code.claude.com/docs/changelog

Small print with consequences

Three lines that matter day-to-day: a new MessageDisplay hook can rewrite or hide assistant text as it's shown; when the primary model isn't found, Claude Code switches to --fallback-model for the rest of the session instead of failing every request; and "Auto mode no longer requires opt-in consent" — autonomous mode runs without an up-front consent prompt. That last one is a convenience with an aftertaste: one click fewer, but also one confirmation fewer before unattended runs.

Codex · 0.134.0 from alpha to stable [2]

The alpha is through

What ripened since 2026-05-22 as 0.134.0-alpha.1…3 shipped yesterday (2026-05-26, 19:13 UTC) as stable 0.134.0 — up from stable 0.133.0 (2026-05-21). The notable changes: searchable local conversation history (case-insensitive), --profile is now the primary profile selector (legacy profile configs rejected), per-server MCP env targeting plus OAuth for streamable HTTP, and read-only MCP tools now run concurrently instead of serially.[2]

The last two points are the real signal: Codex is visibly reworking its MCP plumbing this week — matching the MCP spec surge on the other side (see right). Running read-only tools in parallel is exactly the kind of latency win that makes agentic runs noticeably faster, without the risk of parallel writes.

MCP · governance surge in the spec repo [7][8]

While the CLIs shipped new builds, the modelcontextprotocol repo had its busiest day in weeks — and the focus is striking: security, auditability, governance.

Merged: Codex/ChatGPT read server instructions

PR #2790 (merged 2026-05-26) records that Codex and ChatGPT now support MCP server instructions — the operating-hint text a server ships is actually read by the client and fed to the model. The newest merge in the repo is now #2800 (2026-05-27, release-workflow plumbing); the published spec stays 2025-11-25, the RC target 2026-07-28 (PR #2750 merged).[7]

MCP server instructions: Like the leaflet on a device — the server tells the AI client on its own "here's how to use me right" (which tools for what, in what order). Some clients used to ignore the leaflet; Codex and ChatGPT now read it.
MCP PR #2790

Three new SEPs in one day

On 2026-05-26 three proposals landed at once: SEP "Tool Risk Metadata" (#2793) extends ToolAnnotations with machine-readable risk info, so a client can automatically decide what may run without a prompt and what may not; #2792 adds per-request language negotiation (i18n); #2778 "Adding constraints." The SEP-2787 attestation proposal (signed tool-call receipts, EU AI Act Art. 12) stays open but on 2026-05-26 picked up first test vectors with #2789. All open, all Extensions track — but the direction is clear.[8]

Tool Risk Metadata: A label on each tool saying how dangerous it is — "read only" (green) vs. "deletes files" (red). The agent can then use the green ones without asking and pause only on the red ones. Today that call is often a hand-kept list; the label makes it machine-readable.

Community · the flip side of agents

Simon Willison documents this week a prompt-injection exfiltration via Microsoft Copilot "Cowork" — an attacker smuggles instructions into data the agent processes and uses that to leak information out. Plus Daniel Stenberg's "The Pressure" on the flood of AI-generated, voiceless security reports overwhelming curl maintainers. Two stories, one punchline: the more autonomous the agents, the more audit trails and risk labels matter — exactly what MCP is reworking right now.

Prompt injection: Like a forged note in the inbox of an overly obedient assistant — hidden commands ("send everything to this address") sit inside the text the agent is meant to process. The agent can't tell harmless data from smuggled instructions, and executes them.
