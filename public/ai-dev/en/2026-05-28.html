Antigravity 2.0 · a week after the bang [1][2][3]

Nine days ago, Google unveiled its agent-first desktop tool in version 2.0 at I/O — standalone application for macOS/Linux/Windows, parallel sub-agents, scheduled background tasks, an agy CLI, an SDK, integration with AI Studio/Android/Firebase. The architecture, that's the consensus from independent reviews over the past few days, is real: you get nothing like it today from Claude Code or Cursor.[1]

The delivery was a problem. Google automatically patched existing v1 installations on launch day — removing the built-in editor, moving configs to a new path (= lost), dropping source control and remote connections from the default image. Anyone running Antigravity in CI/CD found a chat interface instead of an IDE on Wednesday. The ecosystem fractured that day into three separate downloads: Antigravity 2.0 (agent shell), Antigravity IDE (traditional coding), and Antigravity CLI. Whoever wanted the editor back had to manually install the IDE.[2]

Free tier and CLI gap

Two numbers that contributed to the mood: the free tier dropped from 250 to 20 requests/day — too little to seriously evaluate the tool without a paid account. And the agy CLI, replacing the Gemini CLI on 18.06., wasn't available via a public package manager as of 21.05. That created bare migration pressure in CI pipelines relying on npm i -g @google/gemini-cli, with no target system in sight.[2]

Update since yesterday: The official installer is now live. curl -fsSL https://antigravity.google/cli/install.sh | bash puts the native binary agy into ~/.local/bin (Unix) or %LOCALAPPDATA%\Antigravity\ (Windows). No npm package — Google explicitly moved away from the Node workflow.[3]

chatforest.com (Review, 3/5) · antigravity.google/docs/cli-features

For Daniel: don't install today

Antigravity 2.0 is not a drop-in replacement for Claude Code; "revisit in 60–90 days" (the reviewer's honest advice) is the call. What's worth your time today from the Antigravity world is the MCP server built into it — Chrome DevTools for agents, usable without Antigravity (next section).

Chrome DevTools for agents · quiet standard [4][5]

While Antigravity grabs headlines, something more substantial is running in the background: Chrome DevTools for agents 1.0 went stable on 19.05., the npm package chrome-devtools-mcp was at 1.1.1 yesterday (release tag 27.05.). This gives your coding agent real browser eyes: console logs, network requests, performance traces, memory snapshots, lighthouse audits, device emulation — all via MCP instead of a screenshot sandbox.[4]

Chrome DevTools for agents (via MCP): Imagine your AI agent sitting blind in front of the screen, writing frontend code — it used to have to guess whether the page even loaded. With this MCP server, it gets the same inspection tool you open in Chrome via F12: it sees which requests fail, which image takes 3 seconds, which input fields miss a screen-reader label. Instead of "I think the bug is here" → "Lighthouse says Z, fix Y."

The list of clients is the real surprise: pre-installed in Antigravity, marketplace plugin in Claude Code, MCP extension in Gemini CLI, plus officially listed for Cursor, Copilot (VS Code + CLI), Codex, Cline, Command Code, Factory CLI, JetBrains AI Assistant, Junie, Katalon, Mistral Vibe, OpenCode, Qoder, Visual Studio, Warp, Windsurf.[5] This is no longer just a Google tool in every meaningful sense except branding — it's shared infrastructure.

What you can add today

In Claude Code: the official plugin from the Anthropic Marketplace; Codex/Cursor: via MCP server config (npx chrome-devtools-mcp@latest). The LY Corporation case study Google cites: an automated audit system reducing manual performance analysis by up to 98% — the figure comes from Google's blog, secondary reporting confirms it unchallenged, but remains vendor-cited. Try Low friction entry, high leverage — especially for front-end sessions where the bug is visual and the model guesses it blind.

Managed Agents + AI-Studio Android [8]

Managed Agents in the Gemini API (Preview)

One API call spins up an isolated Linux sandbox where an agent reasons, uses tools, and executes code — the same harness behind Antigravity. You define your own agents as versionable AGENTS.md and SKILL.md files. Available via Interactions API + AI Studio, "Public Preview." That's Google's answer to OpenAI's Codex-as-a-Service and Anthropic's Agent SDK — same conceptual block, different vendor.[8]

Managed Agents: Instead of building an agent loop yourself (model call → tool call → result → next step), you get a ready-made agent container from the vendor where you just say "do X." Upside: less boilerplate, automatic sandbox isolation. Downside: less control, vendor lock-in, security boundary is the vendor.

Native Android apps in AI Studio

A prompt → Kotlin/Jetpack Compose project, live preview in a browser emulator or straight to device via ADB, one click into the Google Play Internal Test Track. This is the I/O piece that flew most under the radar, and the most practical: nobody had to install Android Studio, nobody had to sign, nobody had to publish. Interesting for hobby projects and prototyping; tangential to Daniel's main work (TypeScript/Python stack).
