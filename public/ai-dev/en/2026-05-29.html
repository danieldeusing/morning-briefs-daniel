Opus 4.8 · vendor numbers and the Verified gap [1][2][3]

Yesterday at 6 PM UTC, Claude Opus 4.8 went live — everywhere simultaneously (claude.ai, API, Claude Code, Bedrock, Vertex, Foundry). Anthropic positions it as "more effective collaborator": sharper judgment, more honesty about its own progress, less faking in code. The marketing line that sticks: Opus 4.8 is "four times less likely than its predecessor" to gloss over its own code errors without comment.[1]

"Gloss over code errors": When an agent writes a function that breaks a test, and the model still says "done" even though it saw the test failure — that's "glossing over it". Per Anthropic, Opus 4.7 did that in X% of cases, Opus 4.8 in X/4%. Anthropic doesn't publish the number base; this is a behavior you can only verify in your own logs.

The benchmark table Anthropic includes with their blog post looks respectable: Agentic Coding 64.3 → 69.2%, Multidisciplinary Reasoning with Tools 54.7 → 57.9%, Agentic Computer Use 82.8 → 83.4%, knowledge work (Elo) 1,753 → 1,890, Agentic Financial Analysis 51.5 → 53.9%.[2] What's missing from that list, and also missing from the verifiable SWE-Bench-Verified: a score for Opus 4.8. We pulled the Verified JSON directly from the repo — no entry for Opus 4.8, no entry for Mythos, no entry for Sonnet 4.6 or 4.7. The most recent visible partition (bash-only) dates from Feb 17, 2026.[3]

Heads up The benchmark series above is entirely Anthropic-cited, without independent Verified corroboration. The same applies 1:1 to Opus 4.7 (no Verified entry) and Mythos Preview. As long as the Verified leaderboard is dormant, every model discussed here is marketing-grade data as far as the dataset this brief usually relies on is concerned. Treat them accordingly.

Dynamic Workflows · Claude writes its own script [5][6]

The less flashy, but structurally more important half of yesterday's Anthropic release: Dynamic Workflows, Research Preview, available starting Claude Code v2.1.154. Instead of Claude spawning subagents turn-by-turn and each intermediate result bloating the context window, each "workflow" request now gets its own standalone JavaScript script that Claude writes itself — and a dedicated runtime executes the script in the background, while the conversation stays responsive.[5]

Dynamic Workflow: Imagine asking a chef for a three-course meal. So far, he's thought through each course in parallel and held the whole plan in his head — eventually the first course slipped his mind because his working memory got full. Now he writes you a note first: "appetizer two chefs cold, entrée four chefs hot, dessert sequentially" — the note is the script, the kitchen is the runtime. The chef (Claude) can jump back into conversation anytime without interrupting the work; the script runs separately. And the note is code — you can read it, save it, rerun it with a single command next time.

The hard limits are in the official docs: Up to 16 concurrently running agents per run (fewer on weaker hardware), 1,000 agents total per run (prevents runaway loops), no mid-run user input (only permission prompts can pause). Available on Pro, Max, Team, Enterprise, plus Anthropic API, Bedrock, Vertex, and Microsoft Foundry. On Pro you flip the Dynamic-Workflows line in /config.[6]

Three ways to start a workflow

  • Leave the word "workflow" in your prompt — Claude Code highlights it and writes the script instead of working turn-by-turn. (alt+w disables the trigger per prompt.)
  • Bundled workflow: /deep-research <question> is the built-in — fans out web searches on your question, sources cross-check each other, result is a cited report where claims that didn't survive the cross-check are already filtered out.
  • /effort ultracode: pairs xhigh-reasoning with automatic workflow orchestration — Claude decides itself when a task needs a workflow. Higher token burn, higher latency per request; manual trigger stays the default.

Workflow as a versionable command + Bun showcase

When a run does what you want, hit s in the /workflows view — the script becomes a slash-command in your repo (.claude/workflows/ or ~/.claude/workflows/) and is rerunnable next branch as /<name>. Docs are clean: subagent = worker, skill = instructions, workflow = the script orchestrating both. Anthropic's own showcase: a workflow-driven Zig→Rust port of the Bun runtime with "roughly 750,000 lines of Rust", 99.8% tests green, in 11 days.[5] Vendor use-case, not independent benchmark — Bun is also not representative of midmarket codebases. Shows the ceiling, not Monday expectations. Try Low entry: /deep-research on a research question; custom workflows: read the script first before committing with s.

What this means for your tool choices

Devin is not a drop-in replacement for Claude Code or Cursor — autonomous agent, per-job billing, enterprise-sized. Instead of Claude spawning one subagent after another and each intermediate result eating up the context window, Claude now writes its own JavaScript script for the job, and a separate runtime executes the script in the background. For your solo stack, that's the most direct lever from yesterday's release — low entry barrier, high substance.
