Gemini 3.5 Flash · the rare case of a vendor number that holds [1][2][3]

At Google I/O on May 19 — eleven days ago, lost in the Anthropic Opus 4.8 week — Gemini 3.5 Flash went GA and has been Google's default model since. Google positions it as an agentic coding model: a Flash model (the cheap, fast tier) that according to Google beats its predecessor 3.1 Pro on coding benchmarks — and, measured against frontier models generally, runs "four times faster" in output tokens per second and "often at less than half the cost" (Google's comparison is against frontier models generally, not specifically 3.1 Pro).[1]

The numbers Google shows: Terminal-Bench 2.1 76.2%, MCP Atlas 83.6%, GDPval-AA 1.656 Elo, CharXiv-Reasoning 84.2%. Against 3.1 Pro these are noticeable jumps (Terminal-Bench 70.3 → 76.2; MCP Atlas 78.2 → 83.6; Finance Agent v2 43.0 → 57.9).[2]

Terminal-Bench / MCP Atlas / GDPval: Three tests that measure agentic capability, not just code snippets. Terminal-Bench puts the model in a real shell and has it solve a task over multiple commands — like an intern working through your terminal alone. MCP Atlas checks whether the model correctly invokes external tools via the Model Context Protocol (MCP = a USB-like standard through which a model calls tools like GitHub or a database). GDPval-AA is an Elo score over realistic knowledge work — like a chess rating, just for professional tasks.

Normally this is where this brief writes "vendor number, not independently verified." This time it's different: Artificial Analysis has independently measured these exact values — Terminal-Bench 2.1, MCP Atlas, GDPval-AA, Humanity's Last Exam, MRCR 128k — and arrives at zero deviation from Google's figure. That's rare and worth noting.[3]

But Confirmed single-benchmark scores are not the same as top-tier performance. That same Artificial Analysis Intelligence Index v4.0 places Gemini 3.5 Flash at rank 8 out of 148 — behind GPT-5.5 and Claude Opus 4.7 (each ~3 points ahead). And on SWE-Bench Pro Flash sits at 55.1%, 9.2 points behind Opus 4.7 (64.3%) — these Pro numbers are, however, via digitalapplied.com aggregated, not a native Artificial Analysis run. On long context, MRCR drops from 77.3% (128k) to 26.6% at 1 M tokens — finding a needle in a haystack over full contexts is a genuine weakness.[3]

For Daniel's toolbox: Gemini 3.5 Flash is not the best coding model — it's the best price-to-performance model for agentic loops where latency and token cost dominate (test loops, refactor swarms, CI bots). Heavy architecture work stays Opus/GPT-5.5; for a hundred small parallel agent tasks, Flash is the more rational default. Try Via Google AI Studio / Gemini API, "high"-thinking level (the benchmark variant).

Managed Agents · the agent as an API primitive [4][5]

The more interesting I/O announcement for developers is not the model, but how Google packages it: Managed Agents in the Gemini API. A single API call starts an agent that operates in an isolated, ephemeral Linux sandbox, thinking, planning, calling tools, executing code, managing files, and browsing the web — you don't have to run a runtime, container, or tool loop yourself.[4]

Managed Agent: Until now, an "agent" was something you built yourself — model call, tool invocation, result back to the model, loop, plus a sandbox where the code can run. Google takes away all the plumbing: you tell the API "do this" and a ready-made Linux machine runs in the background that the agent controls. It's like the difference between running your own server (you manage power, cooling, OS) and a cloud function (you just upload the code). You describe what, not how to operate.

You define behavior in Markdown — AGENTS.md and SKILL.md — and register these files as a Managed Agent. The underlying harness (the control loop around the model) is Google's Antigravity agent, built on Gemini 3.5 Flash, reachable via the Interactions API. Each interaction creates or inherits an environment that you reuse in the next call — files and state persist, you can continue the session.[5]

Familiar pattern Anyone who knows the AGENTS.md/SKILL.md schema sees it immediately: this is the same convention as OpenAI Codex and Claude Code. Three vendors converge on Markdown-defined agents + hosted sandbox. That's good for Daniel — skills he writes for Claude Code are conceptually portable. Google, however, names neither pricing nor limits in the announcement post; before production use, pricing needs separate review.

Antigravity 2.0 · from IDE helper to agent platform [6]

Antigravity was previously Google's agentic IDE layer. With Antigravity 2.0 (also I/O, May 19), it becomes a standalone desktop app built entirely around agent orchestration — plus an Antigravity CLI and an Antigravity SDK. The key: CLI, SDK, and desktop app share the same harness that powers Google's own products. Default model everywhere: Gemini 3.5 Flash.[6]

The strategic point is the same as Anthropic's Dynamic Workflows last week and Cognition's "orchestration layer" thesis: the orchestration layer above the model becomes its own product category — Antigravity, Dynamic Workflows, the Codex service, one per vendor. For Daniel: the question increasingly shifts from "which model" to "which harness orchestrates it, and can I bring my skills along."

Honest framing

2.0 is enterprise-oriented (scheduled/async tasks, Cloud Run/Firebase integration). For a solo developer, the CLI is the relevant piece — same harness as the desktop app, without GUI overhead. Whether it matches Claude Code / Codex CLI / Grok Build is not yet supported by independent reviews three weeks post-launch — watchlist, not a judgment call today.

xAI Grok Build · the fourth player, honestly measured [7][8]

While Google held center stage, xAI quietly entered the same category: Grok Build, a terminal-native coding agent CLI built on the purpose-built model grok-build-0.1. Launch as Early Beta on May 15 (SuperGrok Heavy only, $300/Mo), open to all SuperGrok ($30) and X Premium+ ($40) since May 25 — a steep price drop designed to buy reach.[7]

Technically interesting: up to 8 parallel subagents, each in its own Git worktree on a separate branch; an orchestrator then merges the results. Before that, a Plan→Search→Build flow — for anything from a single line, Grok Build first writes a structured plan that you approve or revise step by step. 256K context, MCP native (GitHub, Linear, Slack without reconfiguration). API pricing circulates in conflicting third-party sources — buildfastwithai.com cites $0.20/1 M input + $1.50/1 M output, others $1.00/$2.00 (the $0.20 figure is likely the cached input price); the official xAI pricing page was unreachable, so unconfirmed.[8]

Git worktree per subagent: A worktree is a second working directory from the same repo, on its own branch. If eight agents worked on the same code at once, they'd overwrite each other's files — like eight chefs at one stove. With a worktree each, every agent gets its own cooking station; only at the end do they merge. Clean and conflict-free — that's the technically most elegant part of Grok Build.

The benchmark number — and its limits

xAI reports SWE-Bench Verified 70.8% for grok-build-0.1 — a vendor number, not on the official Verified board (see right column). For context: on the real board GPT-5.5 stands at 88.7% and Opus 4.7 at 87.6% — grok-build-0.1 plays, if its own number even holds, a class below. Independent reviews were mixed: strong on parallelism and privacy, weak on benchmark performance and ecosystem depth. For Daniel: don't switch today — Grok Build is locked behind X Premium+ and ecosystem-light; the parallel-worktree idea is what to watch.
