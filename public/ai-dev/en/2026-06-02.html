Claude Opus 4.8 — the only major release you can touch today

On 05-28, Anthropic shipped Opus 4.8 — no preview, no waitlist, API from day one. For a dev, that's the story of the day because it's the only one of the four "big" models that actually exists right now (the other three below are announcements). The jump hits exactly where Daniel works: the agentic-coding score (SWE-Bench Pro) jumps from 64.3% to 69.2% — measured by Anthropic, so read it with the usual skepticism.[1][2]

Agentic coding: Not "write me this function," but "work through the whole task on your own" — read files, run tests, see errors, iterate, across many steps. Like an intern who doesn't ask for guidance after every line. That's exactly what the agentic score measures: success across a full task chain, not a single prompt.

The actually nice part: the price stays the same as Opus 4.7 — $5 per million input tokens, $25 per million output.[1] More power at the same rate is rare; usually you pay for the better model. Plus two Claude Code relevant additions: Dynamic Workflows (Research Preview) let Claude plan a task then orchestrate hundreds of parallel subagents in one session — and an Effort Control slider next to the model selector to dial in how much "thinking" the model puts into an answer.[1]

Fast Mode now runs at ~2.5x speed and is three times cheaper than the earlier Fast Mode ($10/$50 vs previously more expensive).[1] try it Why to try: For long agentic runs in Claude Code, 4.8 is a quasi-free upgrade — same rate, and Fast Mode drops costs for the fast intermediate steps noticeably. Just bump the default model and run a real task against it.

SWE-Bench Verified: Tied at the top — and one entry you can't use

With the 4.8 release, looking at the leaderboard is worth it — but with the skepticism the spec demands. As of 06-02, per llm-stats.com, Opus 4.8 stands at 88.6% and GPT-5.5 (OpenAI-reported) at 88.7% — within measurement tolerance, so tied, no lead.[3][4] Opus 4.7 holds 87.6%, Gemini 3.1 Pro 80.6%. The official SWE-Bench page renders its table via JavaScript and yielded no numbers on fetch — hence the independent cross-check source, verified against andrew.ooo (as of 05-11).

SWE-Bench Verified: 500 real GitHub bugs from actual open-source projects, verified as solvable by humans. The model gets the repo and the ticket, writes a patch — it passes only if the real tests turn green afterward. "88%" means: nearly nine out of ten of these bugs, the model actually fixed.
Skepticism At the very top, llm-stats lists an entry "Claude Mythos Preview" at 93.9%.[3] This one is real: Anthropic announced it on 04-07 with a full system card — but deliberately not generally available, restricted to a small set of "Project Glasswing" partners for defensive security work. So the score is genuine, but you can neither call the model from the public API nor reproduce the number. Treat the GA models below as the actionable signal — Mythos is a capability preview, not something you can ship on.

What the number doesn't say: Gemini 3.5 Flash doesn't appear on the independent Verified board yet — Google cites ~78% (self-measured), but only the older 3.1 Pro is listed.[4] Vendor number ≠ independently measured; until Flash is on the board, that 78% is a promise.

Gemini 3.5 Pro: announced, not shipped

Search snippets like to claim Gemini 3.5 Pro is "slated for June" — that's only true as a promise. In the official Gemini API changelog, there's still no entry for 3.5 Pro as of today (06-02), neither GA nor preview; Flash alone is listed, GA since 05-19.[5] On the I/O stage, Pichai said roughly "give us until next month" — this month is running now.[6] That means for Daniel: nothing to try yet, regardless of what blog aggregators suggest with spec tables (2M context, Deep Think — all unconfirmed).

What's actually usable today is Gemini 3.5 Flash: Google reports 76.2% on Terminal Bench 2.1 — solid for a Flash tier, but just behind GPT-5.5 (78.2%) and also vendor-measured.[7] For agentic coding, solid and cheap; it doesn't reach Opus 4.8 / GPT-5.5 on the hard tasks.

MiniMax M3 — the open-weights thread moves, but halfway

Yesterday (06-01) launched: MiniMax M3, a Chinese open-weights model with 1-million-token context, built for long coding-agent runs.[8] The vendor numbers are impressive — SWE-Bench Pro 59.0%, Terminal Bench 2.1 66.0% — but two caveats the spec demands: (1) all benchmarks are vendor-measured, and (2) the weights aren't out yet — API is live, open-weights and technical report follow in "~10 days".[9]

Open-weights: The finished model weights are published — you can download the model and run it on your own hardware without a cloud API. Like a recipe you take home instead of eating only in the restaurant. But as long as the weights aren't out, "open-weight" is just a promise — you can't self-host or independently measure anything.

For local setups (Daniel's Ollama/Qwen world), M3 is not an option yet — wait until the weights land on Hugging Face and the first independent re-runs come through. On the independent Verified board, open-weights leaders remain DeepSeek-V4-Pro-Max (~80.6%) and Qwen 3.7 Max (~80.4%), closely trailed by the older MiniMax M2.5.[3]

Tools: Cursor pulls ahead, Claude Code gets cautious

Cursor 3.6 — agent runs longer without a click

On 05-29 came "Auto-review run mode": the agent's shell, MCP, and fetch calls are no longer individually approved — allowlisted calls run instantly, sandboxable ones run in the sandbox, the rest go to a classifier subagent that decides: allow, try differently, or ask.[11] That's the same trend as Opus' Dynamic Workflows — less approval friction for long runs. Before that (05-20) came /loop for repeated local agent runs and Composer 2.5 ($3/$15 Fast).[11] try it Anyone running long refactors in Cursor saves themselves constant clicking with auto-review.

Claude Code v2.1.160 — security prompts

Recent builds are tightening security: a prompt before writing to shell init files (.zshenv, .zlogin, .bash_login) and before build-tool configs that unlock code execution — even in acceptEdits mode.[12] A sensible response to the attack surface "agent silently writes to your shell init". Also, plugins from .claude/skills now load automatically, without the marketplace.[13]

Heads up · 06-15 Claude Code billing splits: interactive terminal/IDE use stays subscription, but programmatic use (Agent SDK, claude -p, GitHub Actions integration) draws from a new monthly pool, billed at full API rates.[14] If you use Claude Code in scripts/CI, calculate first.
