Microsoft trains its own Copilot model

This is the story of the day, and it's more strategic than it sounds. At the Build 2026 keynote on June 2, Microsoft unveiled seven proprietary MAI models — image, speech, transcription, a reasoning model (MAI-Thinking-1) — and right in the center, the one that affects us developers directly: MAI-Code-1-Flash, Microsoft's first proprietary coding model, rolling out in GitHub Copilot since June 2.[1] It appears in the VS Code model picker and comes across all plans — Free, Pro, Pro+, Max.

The real point: until now, Copilot ran on other companies' models — OpenAI, Anthropic, Google. MAI-Code-1-Flash is the first that Microsoft owns outright, end-to-end. According to Microsoft, it was trained on "clean and properly licensed data," explicitly without distillation from OpenAI models[2] — a jab at the partner from which Microsoft is now distancing itself.

Distillation: A small model learns by imitating the answers of a large, expensive model — like an apprentice watching the master's shoulder. Microsoft says: we didn't let our apprentice watch OpenAI over the shoulder. It matters because distilled models are legally and contractually tied to their "teacher."

Architecture: a sparse mixture-of-experts with 137 billion parameters total, but only 5 billion active per token, 256K context.[3] (The official model card cites 137B/5B; Microsoft's family post shorthand calls it a "5B model" — meaning the active parameter count. That's caused confusion.[4])

Mixture-of-experts (sparse): Instead of computing the entire giant model for every query, the model has many "experts" and routes each token to just the relevant few — like a hospital that doesn't examine you with all 137 doctors at once, only the 5 relevant ones. Result: the muscle of a large model at the compute cost of a small one.

The benchmarks — skeptically read

Microsoft's model card pits MAI-Code-1-Flash directly against Claude Haiku 4.5, the obvious comparison class (small, fast model). The numbers are, and this is crucial, vendor's own measurements — MAI-Code-1-Flash is not yet (as of 06-05) in the independent coding index from Artificial Analysis.[5] Until replication comes, the verdict is: interesting, but not confirmed.

SWE-Bench Verified — vendor numbers, mixed sources, as of 06-02-2026

On SWE-Bench Verified, the card reports 71.6% for MAI-Code-1-Flash vs. 66.6% for Haiku 4.5 — and, almost more interesting than the score: across roughly 10.8K instead of 27.3K tokens per task.[3] On the harder SWE-Bench Pro, the gap is larger (51.2% vs. 35.2%), on SWE-Bench Multilingual tighter (65.5% vs. 62.7%). Microsoft claims up to 60% fewer tokens across hard tasks.[4]

SWE-Bench Verified: A collection of real GitHub bugs with real test suites. The model must find the bug in the repo itself and write a patch that makes the tests pass — like a mechanic who has to fix a stranger's car and by the end pass a real test drive. "Verified" means: a human checked that the tasks are solvable and the tests are fair.
Skepticism filter Two caveats. First: vendor measures itself — Microsoft trained MAI-Code-1-Flash specifically on the Copilot harness against which it's benched here; that's a home-field advantage that's real and may distribute differently in the wild. Second: token efficiency is the most credible part of the story. Fewer tokens = lower cost and latency for Copilot's millions of users — and that (not the last SWE-Bench percentage point) is Microsoft's real motive.

Copilot becomes agentic: Agent HQ try

The second major Build theme for coding: Agent HQ. GitHub now lets you run multiple coding agents in parallel — directly in GitHub, GitHub Mobile, and VS Code — and not just Copilot itself, but also Claude from Anthropic and OpenAI Codex (both in Public Preview) side by side.[6] For Pro+ and Enterprise users.

This is more than marketing. An orchestrator agent can dispatch subagents to separate workstreams — linting, test generation, documentation, security review run simultaneously instead of sequentially. If you work with Claude Code or your own harness, you know that pattern; what's new is that GitHub is building it as a first-party feature and cross-vendor, with context, history, and review in one place.

Why try it If you already have Copilot Pro+: Agent HQ is the first place you can pit Claude and Codex in the same workflow against each other without three tools open. According to reports (techtimes.com), Enterprise is expected to get an Autonomous Agent Mode that writes, tests, and commits entire feature branches — merge still requires human approval; GitHub has neither named nor scheduled this feature in the official Agent HQ blog.[7]

Codex now regularly on Amazon Bedrock

Parallel, one day before Build: OpenAI's coding agent Codex is, along with GPT-5.5 and GPT-5.4, generally available on Amazon Bedrock as of June 1.[8] Access via Codex App, CLI, and IDE integrations (VS Code, JetBrains, Xcode). OpenAI cites over 5 million developers per week using Codex.

The appeal for AWS shops: all inference runs through Bedrock and thus under AWS governance levers you already have — IAM, VPC/PrivateLink isolation, KMS encryption, CloudTrail audit. Prompts are not used for training. Pricing "matches OpenAI's first-party rates," pay-per-token, no seat licenses.

Context Three days, three labs, one pattern: Microsoft now owns its Copilot model, OpenAI distributes Codex through the cloud arm of its rival, GitHub makes all three interchangeable in Agent HQ. The coding model is no longer a product in 2026 — it's a component you plug in depending on platform, price, and compliance.

Gemini 3.5 Flash as a coding model

Slightly older (Google I/O, May 19), but in a coding context it belongs here: Gemini 3.5 Flash is, according to Google, its strongest model for programming and autonomous agents — not for chat. Google sells it openly as an agent model: plan, build, iterate with minimal human intervention, roughly 4× faster than comparable frontier models.[9]

The (again, vendor) numbers: Terminal-Bench 2.1 76.2%, MCP Atlas 83.6%, GDPval-AA 1656 Elo — each an improvement over Gemini 3.1 Pro. Available via Antigravity, Gemini API, and Enterprise. Gemini 3.5 Pro is expected "next month" (June), already in internal use. A 2M-token context circulates only in secondary reports and has not yet been officially confirmed by Google.[10] For us, Flash is interesting mainly as cheap, fast agent backend in harnesses that make models pluggable — see Agent HQ above.

Open-Weights: MiniMax M3 ahead, Qwen efficient Local

For Daniel's local Ollama/Qwen stack, things are moving too. Two notable releases:

MiniMax M3 (June) leads the open-weight leaderboard on SWE-Bench Pro with 59.0% and according to kilo.ai is the first open-weight model to combine frontier coding, 1M context, and native multimodality.[11]

Qwen3-Coder-Next remains the efficiency king: 80B MoE / only 3B active, 256K context, runs on a 64GB MacBook or RTX 5090. Vendor numbers: SWE-Bench Verified 70.6 / Pro 44.3.[12] Striking: that's Sonnet-4.5-class coding performance locally — with no token bill.

DeepSeek V4 (April 24) rounds out the field — V4-Pro (1.6T/49B) and V4-Flash (284B/13B), both 1M context, MIT license. Too large for most local setups, but as a self-hostable frontier model, relevant.

Tool roll call

Cursor 3.6 (May 29) brings Auto-review: a run mode that lets Cursor work longer without prompting. Shell, MCP, and fetch calls either go straight through (allowlist), run sandboxed — or a classifier subagent decides whether to permit the call, solve it differently, or still ask.[13] This is the same "less approval friction" trend driving the reported Copilot autonomous-agent plans — agents are allowed to run longer alone, humans control only the perimeter.

Cline v3.85 (May 25) has caught up with GPT-5.5 and DeepSeek V4 Flash/Pro; 61k+ stars.[14]

Languishing in the shadows Aider still has no native MCP — only community bridges, as of May 2026; that's itself a signal (stagnation). Continue and OpenHands with no meaningful motion this week. JetBrains AI and Plandex quiet.

What this means for Daniel's toolbox

My reading: the day confirms a trend that's good for us — model lock-in is dissolving. Agent HQ makes Claude, Codex, and Copilot interchangeable in the same editor; MAI-Code-1-Flash pushes costs down at the bottom end; Codex on Bedrock decouples OpenAI from a single cloud. If you, like Daniel, already work with pluggable harnesses and local LLMs, you win: more choice, less dependence on one vendor.

Concretely this week, worth trying: MAI-Code-1-Flash in the VS Code picker for cheap, fast routine edits (if the rollout reaches you), and Cursor 3.6 Auto-review if approval prompts annoy you during long agent runs. I'd wait for independent SWE-Bench replication of Microsoft's numbers before I believe the 71.6%.

It's Friday, June the fifth. Three days after Microsoft's Build conference, nearly everything in coding tools revolves around the same idea: the AI model you code with is becoming a pluggable component.

The main story comes from Microsoft. For the first time, Microsoft has embedded its own coding model in GitHub Copilot, called MAI-Code-1-Flash. Until now, Copilot ran on other companies' models from OpenAI, Anthropic, and Google. This one Microsoft trained itself, using, by its account, cleanly licensed data and explicitly without looking over OpenAI's shoulder. It's small and fast, a so-called mixture-of-experts model that computes only a fraction of its size for each request.

I read the benchmark numbers with caution. Microsoft's own model card reports on the test SWE-Bench Verified a figure of seventy-one point six percent versus sixty-six point six for Claude Haiku, and that's using significantly fewer tokens. But that's a vendor measurement. An independent body like Artificial Analysis hasn't re-measured the model yet. More credible than the last percentage point, in my view, is the token efficiency, because fewer tokens means lower costs, and that's exactly Microsoft's real motivation.

The second Build topic is Agent HQ. GitHub now lets you run multiple coding agents simultaneously, not just Copilot, but also Claude and Codex side by side, in the same editor. An orchestrator distributes tasks like testing, documentation, and security review in parallel to sub-agents. For enterprise customers, according to reports, there's supposed to be a mode that writes and commits entire feature branches itself; merge only with human approval; GitHub itself hasn't officially announced that yet.

That fits with OpenAI. One day before Build, the coding agent Codex became regularly available on Amazon Bedrock, together with the new GPT models. Over five million developers use Codex each week, and on Bedrock everything runs under the familiar AWS security controls. Three days, three labs, one pattern: the model becomes infrastructure, something you plug in depending on platform and price.

At Google, Gemini 3.5 Flash is now Google's strongest model for programming and autonomous agents, roughly four times faster than the competition. The larger version, Gemini 3.5 Pro, is expected to follow still in June.

For Daniel's local stack, two notes: the open-weight model MiniMax M3 now leads the open leaderboard, and Qwen3-Coder-Next remains the efficiency king, delivering quality close to expensive models and still running on a MacBook. Among tools, Cursor in version three point six added a mode that asks less often, and Cline has caught up with the new models. Aider remains without native MCP, which is itself a statement.

My conclusion: model lock-in is dissolving, and that's good for anyone working with pluggable tools. Next week I'm watching mainly two things: whether an independent body confirms Microsoft's SWE-Bench numbers, and whether Google really delivers Gemini 3.5 Pro still in June.
