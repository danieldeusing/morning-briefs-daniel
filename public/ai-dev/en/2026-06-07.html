The field is now quadruple

Today's story is not a single announcement, but a pattern that finished this week: Until recently, "AI for coding" was essentially a two-way fight between Anthropic (Claude Code/Opus) and OpenAI (Codex/GPT). With Microsoft's MAI models and Copilot App at Build (06/02) and Google's Gemini 3.5 Flash plus Antigravity (I/O, 05/19), all four major labs now have their own, vertically integrated coding stack on the field.

CNBC sums up the situation on June 1st like this: Microsoft and Google are "late, but it is absolutely essential that they compete for this growth".[1] The reason sits in this week's adoption data: Claude Code is the fastest-growing coding agent — per JetBrains' 2026 developer survey up from ~3% (Apr 2025) to 18% (Jan 2026), roughly 6× in nine months — and among experienced developers (10+ years) the clear favorite: 46% choose Claude Code for daily work versus 9% Copilot.[2] GitHub Copilot remains adoption leader in raw numbers (Stack Overflow end-2025 survey: ~51% of professional developers, though fallen from 67%; Cursor new at ~18%, Claude Code at ~10% in first appearance). Claude Code's run-rate revenue was over $2.5 billion in Feb 2026. Exactly this turf — developers who pay daily for an agent — Microsoft and Google don't want to cede to Anthropic and OpenAI.

Vertically integrated coding stack: When a vendor supplies everything from one hand — the model, the tool around it (IDE or agent harness) and the cloud it runs on. Like a car manufacturer that not only builds the engine but also the car and the gas stations. Advantage: model and tool are trained on each other (MAI-Code-1 was trained directly on Copilot's production harness). Disadvantage for you: lock-in — the stack pulls you into an ecosystem.

For Daniel's toolbox, the lesson is sober: competition is good for prices and tempo, but four stacks doesn't mean four real options. The newcomers have to prove their models work outside their own marketing slides — and that's where the skepticism filter comes in (next section).

Microsoft MAI: its first own coding and reasoning model Bench

At Build, Microsoft unveiled its MAI family — the noteworthy point: these are Microsoft's first self-trained models for this role, a deliberate step away from pure OpenAI dependence.[3] Two are relevant for us:

MAI-Code-1-Flash — the small, fast coding model that lands directly in GitHub Copilot and VS Code (model picker, rollout across Free/Student/Pro/Pro+/Max, gradually "over the coming weeks").[4] The actual trick: It was trained directly on Copilot's production harness and solves tasks with "up to 60% fewer tokens" thanks to adaptive response length — short for simple queries, more reasoning budget for hard ones.[5]

MAI-Thinking-1 — Microsoft's first reasoning model, trained from scratch without distillation, on "clean, commercially licensed" data. A MoE with ~35 billion active parameters, 256K context.[6]

Distillation (model distillation): A small model learns by imitating the answers of a large "teacher" model — like an apprentice looking over the master's shoulder. Faster and cheaper, but the small model inherits the teacher's quirks. Microsoft emphasizes "without distillation, from scratch" — marketing message: "we didn't copy an OpenAI model". Relevant for you mainly for clean license origins of training data.
Bench skepticism The numbers are consistently vendor or secondary sources, not independent leaderboard. Reported: MAI-Code-1-Flash ~51% SWE-Bench Pro (+16 points vs. Claude Haiku 4.5 at 35.2%, roughly GPT-5.3 tier); MAI-Thinking-1 ~53% SWE-Bench Pro and thus "neck-and-neck with Claude Opus 4.6", 97% AIME 2025.[6][7] For context: Opus 4.8 is at 69.2% Pro — so the MAI models play a class lower, but they hit their price/size segment. And beware: the parameter count of MAI-Code-1-Flash is reported contradictorily (Microsoft site/multiple secondary sources: 5B; one source: "137B MoE") — I wouldn't treat either as settled until a model card detail surfaces.

GitHub Copilot App: from editor plugin to agent command center try

The practically most relevant Build news for me is not a model, but a tool: The GitHub Copilot App is a standalone desktop application (Win 11, macOS, Linux), separate from the VS Code extension. It turns Copilot into "a command center for parallel AI agent sessions — each in its own, isolated Git worktree".[8]

Git worktree: Multiple working copies of the same repo side-by-side, each on its own branch — without constant switching back and forth. Like multiple workbenches in the same workshop: each one has a different overhaul running, without them getting in each other's way. That's exactly what the app uses so three agents can work simultaneously on three tasks without overwriting each other's files.

From a "My Work" view you see running sessions, issues, PRs and background automations across multiple repos. That's exactly the orchestrator pattern Daniel already knows from Claude Code — GitHub bakes it into a native app. Plus the Copilot SDK is now GA in six languages (Node/TS, Python, Go, .NET, Rust, Java); the same agent runtime the app runs on is programmable.[9]

Why try If you're already in the GitHub ecosystem and want to run multiple agents in parallel: the app is a serious Claude Code/Cursor competitor for multi-agent workflows — and the SDK makes custom orchestration possible. Technical Preview for Pro/Pro+/Business/Enterprise. Skip it for pure solo single-agent work.

Google Gemini 3.5 Flash + Antigravity 2.0

Google's answer has been broadly rolled out since I/O (05/19) and belongs in this landscape picture — grazed yesterday under "Pro wait", today with substance: Gemini 3.5 Flash is GA across Antigravity, the Gemini API in AI Studio and Android Studio, Gemini Enterprise and the Gemini app.[10] The sales pitch is speed: ~289 tokens/second, about 4× faster than Opus or GPT-5.5, in Antigravity up to 12×.[11]

Bench skepticism The reported coding values are Google's own numbers on partly Google-curated suites: Terminal Bench 2.1 76.2%, MCP Atlas 83.6%, GDPval-AA 1656 Elo.[10] Independent is rough: Artificial Analysis locates Gemini 3.5 Flash on the Intelligence Index at 55 — solid midfield, not at the coding top (there GPT-5.5 xhigh with Coding Index 59, Opus 4.8 forward in Intelligence Index at 61.4).[12] So Flash doesn't compete against Opus 4.8 on quality, but on quality per second per dollar — that's its turf.

Antigravity 2.0 is the real dev lever: a standalone desktop app plus CLI, built agent-first, with Managed Agents in the Gemini API — a single API call delivers an agent plus hosted Linux sandbox (Bash/Python/Node, file I/O, browsing, Markdown-defined skills, repo/GCS mounts).[13] That's Google's direct strike at the IDE-agent tier where Claude Code and Cursor live.
