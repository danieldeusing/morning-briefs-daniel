Claude moves into Excel

At Build 2026, Microsoft yesterday (June 8) unlocked Claude as a model option in Excel Agent Mode. You pick Claude in the model selector and let it write and explain formulas, clean and transform dirty data, generate written analysis of the sheet, and build automated workflows — all without leaving the spreadsheet.[1] Microsoft puts the reach at roughly 750 million Excel users — the real lever of the story is not the model, but where it now sits.

Agent Mode (Excel): Instead of answering a single question, the agent plans multiple steps on its own — like an intern you tell "pull together the quarterly report" and who then creates columns, types formulas, hunts for errors, and annotates the result. The model picker decides whose brain this intern uses: Microsoft's Copilot model or Claude.
techtimes.com · support.microsoft.com

The fine print for dev teams

Before you roll it out: the model switcher in Excel requires, per Microsoft Support, a commercial Microsoft 365 Copilot license or Microsoft 365 Premium subscription.[2] And for enterprise tenants, the admin must first approve Anthropic as a Microsoft sub-processor in the M365 Admin Center — a data-governance step, not a one-click. Means: technically trivial, organizationally a compliance question that in regulated environments has to go through IT/privacy.

Take For you as a coder, Claude-in-Excel is less a tool than a signal: Anthropic is landing its model exactly where the business departments sit — not in the IDE, but in the sheet. The dev side of that is Foundry (next section); the office side is an end-user play and belongs more in the ai-usecases brief. By the way: Apple's WWDC announcement about Claude/Siri on iPhone runs there too, not here.

Foundry: the full Opus 4.8 lineup under one bill

The dev-relevant part of the same week: Microsoft Foundry now lists the complete current Claude generation — Opus 4.8, 4.7, 4.6, 4.5, 4.1, Sonnet 4.6 and 4.5, Haiku 4.5, all via Global Standard Deployment.[3] Opus 4.8 (since May 28) is per Microsoft's own docs "the most intelligent Opus model and the best available model for coding and agents". You call them via the Anthropic Messages API, but with Entra ID auth and an Azure bill instead of a separate Anthropic account — relevant if your customer is already Azure-committed (MACC-chargeable).

Catch On Foundry, Opus 4.8 has only 200K context instead of the 1M tokens of the native Claude API.[4] If you need 1M context for codebase-wide tasks, stick with the direct API. Also: Claude on Foundry is currently only available for Enterprise and MCA-E subscriptions — no pay-as-you-go for individual devs.
learn.microsoft.com · platform.claude.com

The special case: Mythos Preview

Interesting in the Foundry catalog is a model you can't simply book: Claude Mythos Preview, a "gated research preview" that Anthropic releases at its discretion, prioritized for defensive cybersecurity (part of "Project Glasswing", invite-only, not self-service).[3] Anthropic markets it as "the world's best model for cybersecurity, autonomous coding, and long-running agents" with 1M context. It's not really available to most — but it shows where frontier providers steer their strongest coding models: into tightly curated security channels, not open sale.

What that means in practice In agentic workflows, the output price almost always dominates the bill (the model writes more than it reads). There, Gemini Flash at 9 $ looks excellent, followed by Sonnet 4.6 at 15 $ — both solid default picks for volume. Opus 4.8 and GPT-5.5 you reserve for the really hard, multi-step tasks where the quality justifies the 25–30 $ output. Watch long context: GPT-5.5 gets pricier at 272K tokens (2× input, 1.5× output), Gemini 3.1 Pro at 200K (4/18 $).[5][7]

Gemini 3.5 Pro: the wait continues

Google hasn't shipped Gemini 3.5 Pro yet. Sundar Pichai said in effect "give us until next month" — the window is June 2026, no concrete date, and the rumored "late June" estimate is speculation, not a Google target.[8] More interesting: what Gemini 3.5 Flash (since May 19) already does: it beats the old 3.1 Pro on coding and agentic benchmarks — Terminal-Bench 2.1 76.2 % (vs. 70.3 %), MCP Atlas 83.6 % (vs. 78.2 %), Finance Agent v2 57.9 % (vs. 43.0 %).[8][9]

Enterprise agent platforms are opening

Two big low-/pro-code houses sharpened their agentic plays this week — both now betting on open dev standards instead of closed proprietary frameworks. SAP Joule Studio 2.0 (Sapphire) brings pro-code support for LangChain, Pydantic AI, and LlamaIndex plus embedded n8n for visual multi-agent orchestration; first customers in June, GA in H2 2026.[11]

OutSystems unveiled its Open Agentic Systems Platform on June 3 at the ONE conference — with A2A and MCP tools, an "Enterprise Context Graph", and native integration of Kiro, AWS's agentic dev environment.[12] The trend that matters to you: MCP is shifting from Anthropic's house protocol to the common table — whoever builds MCP servers increasingly also serves SAP and OutSystems agents.

MCP & A2A: MCP (Model Context Protocol) is the standard socket by which an AI model taps external tools and data. A2A (Agent-to-Agent) is the protocol two agents use to talk to each other. Together: a plug standard for tools, a language standard for teams — so agents from different makers can work together without each reinventing the wheel.
