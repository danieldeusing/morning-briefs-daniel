Claude Fable 5: the Mythos tier goes public

Anthropic announced two models simultaneously yesterday, June 9: Claude Fable 5 and Claude Mythos 5.[1] They share the same weights — the same trained model foundation. The difference is the leash: Fable 5 is the guardrailed, publicly available variant; Mythos 5 is the same model with relaxed safeguards, available only to whitelisted customers in "Project Glasswing."

"Mythos tier": Anthropic sorts its models into tiers — Haiku (small/fast), Sonnet (mid-range), Opus (flagship). "Mythos" is a new tier above Opus. Think of the model lineup as engines: Mythos is the race engine that lived only in the lab until now. Fable 5 is that exact engine, but with a rev limiter for the road.[1]

The bare specs from the official model docs:[2] API ID claude-fable-5, 1M token context window, up to 128k token output, priced at $10/$50 per million tokens (input/output). Extended Thinking isn't a switch — instead, Adaptive Thinking is always on: the model decides how long to reason. GA as of 9 June across Claude API, AWS Bedrock, Vertex, and Microsoft Foundry.

The price is the real story for the toolbox: Fable 5 costs twice Opus 4.8 ($5/$25), but Anthropic says it's "less than half" the previous Mythos preview.[1] One tier above Opus, at double the Opus price — whether that pays for itself depends entirely on whether you trust the benchmarks. And that's where it gets interesting.

The numbers — and the asterisk [3][4]

Anthropic reports a leap for Fable 5 that nearly maxes the SWE-Bench-Verified scale: 95.0 %, versus 88.6 % for Opus 4.8. On harder SWE-Bench Pro, 80.0 % (Opus 4.8: 69.2 %; GPT-5.5: 58.6 %; Gemini 3.1 Pro: 54.2 %).[3]

SWE-Bench Verified: a test of real GitHub bugs — the model has to fix a bug in a live repo such that the project's test suite passes. "Verified" means a human has confirmed each task is solvable. "Pro" is the tougher sibling with gnarly multi-file tasks.

Why you need to look twice here — three things:

(1) All numbers are vendor-reported. They come from Anthropic's own announcement and system card, not independent measurement.[3] The independent SWE-Bench-Verified board (last updated May 2026) lists neither Fable 5 nor Mythos 5 — and not even Opus 4.8. GPT-5.5 leads there at 88.7 %, followed by Opus 4.7 at 87.6 %.[4] Until Fable 5 appears verified on the board, the 95 % is a factory claim, not a measured result.

(2) Safeguards drag the real score. This is the critical caveat: requests touching cybersecurity, biology, chemistry, or model distillation automatically fall back to Opus 4.8.[1] So the Mythos numbers apply only to the unsafeguarded variant. On Terminal-Bench 2.1, 20.9 % of attempts triggered a refusal and fell back — Fable 5's effective score dropped from 88.0 % (Mythos 5) to 84.3 %.[3] If you write security-adjacent code — auth, crypto, pen-test tooling — you don't get the Mythos model for parts of that; you get Opus.

(3) The new tokenizer. Fable 5 uses the tokenizer from Opus 4.7 and later — the same text produces ~30 % more tokens than older models.[2] At the $10/$50 rate, that means the effective cost per line of code is higher than the per-token price alone suggests. Against Opus 4.8 (same tokenizer) it washes out; against Sonnet or older models, it doesn't.

Bottom line: even with skepticism, the leap is real — from 88.6 to somewhere north of 90 % even conservatively. But the "95 %" headline is best-case without safeguard friction. For Daniel's day-to-day (web apps, tooling, no bio/cyber domain), Fable 5 should land close to the reported number; for security work, probably closer to Opus parity.

Fable 5 everywhere day-one [5][6][7]

What's remarkable is less the model than the rollout. Unlike earlier frontier releases, there was no preview bottleneck: Fable 5 was available day-one across the harnesses Daniel actually uses — immediate via API and consumption-based Enterprise, while subscription tiers (Pro/Max/Team and seat-based Enterprise) roll out through 22 June.[1]

Claude Code v2.1.170 (09.06.) ships Fable 5 support — the changelog puts it plainly: "Fable's capabilities surpass those of any model we've ever made generally available."[5] A day earlier, in v2.1.169 (08.06.), two small but useful CLI improvements landed: --safe-mode boots Claude Code with zero customizations (helpful when a hook or skill misbehaves), and /cd switches the working directory mid-session without invalidating the prompt cache.[5] try it /cd alone saves the tedious restart when jumping between repos.

GitHub Copilot released Fable 5 as GA for Pro+, Max, Business, and Enterprise — across all modes (Chat, Ask, Edit, Agent).[6] One detail teams need to know: Fable 5 enforces data retention — prompts are kept up to 30 days for Anthropic's safety systems. Other Claude models in Copilot don't have this requirement. For client code under NDA, that's a real criterion.

Heads up 30-day retention on Fable 5 in Copilot applies regardless of Enterprise tier. If you work on highly confidential client code, verify this before switching — or stick with Opus 4.8 for sensitive repos.

Cursor reports Fable 5 leads "every other model" on CursorBench (Cursor's own real-world coding benchmark — number not disclosed, so treat with caution).[7] There's a Thinking variant for tasks that reward extra reasoning; it costs roughly 2× Opus 4.8. The batch price ($5/$25 per million tokens) is Anthropic's standard 50 % batch discount passed through — no Cursor-specific pricing. AWS Bedrock and Databricks (via Unity AI Gateway) added two more governance-safe paths the same day.

Fable 5 vs. Opus 4.8 — when to reach for which

Choose Fable 5 when …
  • … you face a long, async agent task where Opus asks for clarification too often — that's Anthropic's pitch.
  • … the domain is not security/bio/chemistry (else it falls back to Opus).
  • … you truly need large context (1M tokens).
  • … double the cost is offset by fewer iterations.
Stick with Opus 4.8 when …
  • … cost matters (half the price; 88.6 % SWE-Bench often suffices).
  • … security-adjacent code falls back to Opus anyway.
  • … 30-day retention (Copilot) is a dealbreaker.
  • … you want independent board confirmation first.

My take: for Daniel's typical stack (web, tooling, local experiments), a test run of Fable 5 on a real task is worth it — but Opus 4.8 stays the default workhorse until the 95 % is independently verified.

The headlines are the benchmarks. Anthropic reports ninety-five point zero percent on SWE-Bench Verified, compared to eighty-eight point six for Opus four point eight. First, it's vendor-reported — on the independent board last updated in May, Fable five doesn't appear at all; G-P-T five point five still leads there at eighty-eight point seven.

On one of the tests, nearly twenty-one percent of attempts triggered a refusal, which dropped the effective score from eighty-eight to eighty-four point three percent. If you write security-adjacent code, you don't get the new top-tier model for parts of that — you get Opus.
