Copilot now bills on consumption

Since June 1, GitHub has completely overhauled Copilot's billing: the old Premium Requests (PRUs) are gone, replaced by GitHub AI Credits.[1] Instead of "you get X premium requests per month," the new model is: each plan comes with a monthly credit budget, consumed based on actual token use — input, output, and cache — at the published API rates for each model.[1]

Consumption-based billing: Copilot used to be like a phone plan with "300 free calls per month" — short or long, each call counted as one. Now it's like a power meter: charged for how much actually flows through. A long agent run across a big repo eats many tokens (= many credits), a quick question eats few. 1 AI Credit = $0.01.[2]

The plan details: Pro+ is $39/month with $39 in AI Credits; Business $19/user with $19 credits; Enterprise $39/user with $39 credits.[1] Key point: code completions and next-edit suggestions stay unlimited and cost no credits — only the expensive stuff (chat, agent, edit with large models) is metered.[1]

Migration: If you have a monthly subscription (Pro/Pro+), you were switched on June 1. If you hold an annual subscription, you stay on the old request model until it expires.[1] Developer reaction was predictably mixed — the consensus in the community: "you get less, but pay the same," because pricey models drain the budget faster than the old request quota.[3]

Take: This is not cosmetic. It shifts model choice from "which do I like?" to "which token budget do I have left?". That's exactly why the arrival of an ultra-cheap model (MAI-Code-1-Flash) and an ultra-pricey one (Fable 5) in the same picker is now an economic decision — see next.

What a model costs you in Copilot now

With token billing, the raw price per million tokens becomes the control lever. Three points from the official price list, all per 1 million tokens (input / output):[2]

  • MAI-Code-1-Flash: $0.75 / $4.50 (cached input $0.075).
  • Claude Fable 5: $10 / $50 (cached input $1).

In numbers: Fable 5 costs roughly 13 times MAI-Code-1-Flash per input token, about 11 times per output token. On consumption billing, that means: a day of Fable-5 agent runs can drain a monthly budget that would last two weeks on MAI-Code-1-Flash. The old logic "always pick the strongest model" no longer works automatically — for routine work (boilerplate, tests, small refactors), the cheap model is often the economically right choice.

Microsoft's first in-house coding model [4][5][6]

Microsoft has shipped MAI-Code-1-Flash, its first in-house coding model — unveiled at Build 2026, in the Copilot model picker since June 2 in VS Code, available for Free, Student, Pro, Pro+ and Max.[4] What's remarkable is less its existence than its pedigree: Microsoft trained it directly inside the live Copilot harness — from day one, the model learned what a Copilot diff looks like, what "Apply Edit" means, and how terminal commands route.[5]

Trained in the harness: A "harness" is the scaffolding around a model — the layer that applies file edits, calls tools, executes commands. Most models learn to fit into one after the fact. MAI-Code-1-Flash is like an apprentice trained from day one in the exact workshop where they'll work — instead of learning elsewhere then adapting.[5]

The spec is sparsely documented — and in one place contradictory: the model card lists a sparse mixture-of-experts architecture with 137B parameters and 256k token context, while Microsoft's family post mentions a 5B-parameter model. Whether the 5B refers to active parameters (typical for MoE) and 137B to the sum, Microsoft hasn't clarified.[6] I cite both numbers instead of guessing.

Sparse Mixture-of-Experts (MoE): Instead of running the whole model for each request, there are many small "experts" and a router that activates only a few per token. Like a hospital: not every doctor treats every patient; the right specialist gets paged. So active parameters (speed, cost) are much smaller than total (knowledge).

The numbers Microsoft reports: on SWE-Bench Pro, 51.2% vs. 35.2% for Claude Haiku 4.5 — a 16-point lead in the small-model tier.[5] This is explicitly a small-fast-model matchup, not against Fable 5 or GPT-5.5. As always: vendor-reported, no independent board has verified it yet. But for a $0.75 input model, that's the right yardstick — and the right question is not "does it beat Fable 5?" (no) but "is it good enough for 80% of routine work at a fraction of the price?".

OpenCode bumps Cursor off the top [7][8]

While the big labs drop models, the tool landscape has reshuffled: OpenCode ranks #1 for the first time in LogRocket's power rankings for June 2026, bumping Cursor to #2. Claude Code holds #3, Windsurf drops two spots to #4, Google's Antigravity slides to #5.[7]

OpenCode: An open-source coding agent for the terminal (from the SST team, written in Go). Unlike Cursor, it's not its own editor — it's an agent you attach to any model — over 75 providers via models.dev, including local models. It "sees" code via Language Server Protocol (real type info, signatures, compiler errors), not just as text. MIT-licensed, runs fully air-gapped.[8]

The adoption numbers LogRocket cites: 160k+ GitHub stars and 7.5M monthly users — per LogRocket "the most-adopted open-source coding agent ever built".[7] A cross-check at the repo directly showed ~173k stars, though under a renamed/forked repo path — the star count is likely even higher, but exact lineage is in flux; I take LogRocket's conservative 160k+ as the solid baseline.[9]

Why it matters: The rise of a model-agnostic, locally-runnable OSS agent fits the Copilot billing story above perfectly. If you're dodging consumption billing or running your own/local models, OpenCode is now a first-class, vendor-free tool — exactly Daniel's profile (Ollama/Qwen-Box, no appetite for token meters). try it A run against your own local Qwen3-Coder box costs nothing but electricity.
