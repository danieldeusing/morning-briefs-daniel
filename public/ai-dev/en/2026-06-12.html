The 80 % That's Really 59 % [1][2]

Since Fable 5's launch, one number has circulated: 80.3 % on SWE-Bench Pro, more than 20 points ahead of GPT-5.5 (58.6 %) and Gemini 3.1 Pro (54.2 %). Sounds like a generational leap. But it's halfway a statement about the testing setup, not just the model — and that's precisely what makes today relevant for Daniel beyond any release: it's about learning to read such numbers before choosing a tool.

SWE-Bench Pro: A collection of real GitHub bug-fix tasks — the model opens a real repo, finds the bug, and writes a patch that makes tests pass. Unlike "write me a function," this measures agentic work in code: navigating, reading, modifying, testing.
Scaffold (Harness): The framework around the model — how it sees files, what tools it gets, how many attempts it has. Picture: two chefs with the same knife, but one has an organized kitchen with a sous-chef and the other an empty board. The dish says something about the kitchen too, not just the cook.

And here's the rub. On the only standardized board — Scale's SEAL leaderboard, which runs every model through the same scaffold on 731 public tasks — the rankings look completely different:[1]

GPT-5.4 xHigh
59.1 % › Muse Spark
55.0 % › Opus 4.6 (think)
51.9 % › Gemini 3.1 Pro
46.1 %

59.1 % at the top — not 80. And Fable 5 simply isn't there yet (Scale needs days for a new model to run through the standard pipeline). The rule of thumb Scale itself articulates: "If you see an SWE-Bench Pro score 10–30 points above the SEAL board, it's a vendor-scaffold number."[1] Fable 5, at 80.3 %, sits exactly 21 points above SEAL's current leader — textbook.

Take This is not an accusation of manipulation. Three "best scores" are out there — 80.3 % (Anthropic's scaffold), 59.1 % (GPT-5.4 on SEAL), 47.1 % (Opus 4.6 on Scale's private set) — and all three are real. The spread comes from scaffold and task splits, not cheating. You just can't lay them side by side as if they're the same test.

What this means in practice: The vendor-reported order (Fable 5 ≫ GPT ≫ Gemini) and the standardized one (GPT-5.4 ≫ Muse ≫ Opus) don't just differ in magnitude — they sometimes differ in ranking itself. Picking a model by a press-release bar chart means you might be buying mostly a well-tuned harness. For your own work, only one benchmark matters: a run against your own repo — the only one with no vendor skin in the game.

Google Kills the Gemini CLI — 6 Days [3]

Hard deadline instead of benchmark debate: on 06-18-2026, the Gemini CLI (and the Gemini Code Assist IDE extension) shuts down for all individual users — Google AI Pro, Ultra, and the free tier. If you use the terminal tool daily, you have exactly six days starting today. The move was announced on 05-19 at I/O; a 30-day window is in effect.[3]

Who's affected Individual plans only. Enterprise customers are unaffected — nothing changes for them. If you're on a company subscription, you can skip this section.

The successor is Antigravity CLI, the terminal component of Google's agent platform Antigravity (unveiled as 2.0 at I/O). Critical features move over — Agent Skills, Hooks, Subagents, and Extensions (now "Antigravity Plugins") — but Google itself is clear: no 1:1 parity on day one.[3] For most setups, migration takes under ten minutes; still, don't wait until the last day if a custom MCP server or skill might cause friction.

Antigravity: Google's "agent-first" developer platform — idea in, working app out, built by an agent rather than by hand. The CLI is the terminal version, the counterpart to Claude Code or OpenCode. Server-side harness included.

Why Google is doing this (the coding angle): The fresh Gemini 3.5 Flash (now GA) beats the larger 3.1 Pro on agentic benchmarks — Terminal-Bench 2.1 at 76.2 %, MCP Atlas at 83.6 %.[4] A smaller, faster, cheaper model as the default makes the old CLI tool, built atop the bigger generation, economically obsolete. The shutdown is less housekeeping than model-shift with tool swap.

Claude Code: Subagents in Subagents [5]

This week at Anthropic, a quiet but harness-relevant change shipped. Claude Code v2.1.172 (06-10) now permits subagents to spawn their own subagents — up to 5 levels deep. Amazon Bedrock now also reads AWS region automatically from config.[5] v2.1.173 (06-11) is pure maintenance: a Fable 5 name bug with [1m] suffix and a false Windows sandbox warning.[5]

Subagent: Its own Claude run with its own context window, which the main agent spawns for a subtask (e.g., "search these 40 files") and gets back a short result — without flooding its own context with all the search noise. Nesting means: the sub-run can now subdivide further.

Opinion: Five levels sounds like a lot and is a double-edged sword in practice — each level costs tokens and latency, and deeply nested agent trees are notoriously hard to debug. Useful for clearly decomposable batch work (migrations, repo-wide refactors), overkill for daily work. Good that it's there; no reason to use it reflexively.

The Tools Are Converging [6]

A meta-observation that made the rounds this week across several analyses and fits Daniel's situation well: Claude Code, Cursor, Codex, and Antigravity are technically converging on the same agent blueprint — terminal/IDE agent that reads a repo, calls tools, tests, iterates. The form is largely settled.[6]

The upshot: Differences are no longer "what can this tool do," but price and workflow — exactly the money question from yesterday's Copilot brief. When everyone does the same thing, the cheapest fitting model and the tool that forces least lock-in become the rational pick. That's precisely where OpenCode plays its vendor-neutral, model-agnostic edge.

OpenAI & Microsoft: Quiet Today

OpenAI drops nothing new for coding today. The Responses API remains the agent foundation (built-in tools: Web-Search, File-Search, Computer-Use; AgentKit builds on it) — but that's existing, not a 06-12 release. Codex got only minor updates this week: desktop handoff for /app on macOS/Windows and more flexible reasoning-effort controls. GPT-5.5 stays the recommended Codex model and holds 82.7 % on Terminal-Bench 2.0, remaining the agentic reference.[7]

Microsoft: no new model since MAI-Code-1-Flash (06-02). As a reminder for the benchmark story above: its reported 51.2 % on SWE-Bench Pro is also a vendor number (vs. Claude Haiku 4.5 at 35.2 %) — and it ranks 27th on llm-stats, clearly in the small-model tier, nowhere near Fable 5.[8]
