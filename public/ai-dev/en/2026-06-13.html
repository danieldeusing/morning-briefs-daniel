Kimi K2.7-Code — the open one-trillion model [1][2]

On 06.12, Moonshot AI released Kimi K2.7-Code and dropped the full weights on Hugging Face the same day — under Modified MIT License, so freely usable and self-hostable. For Daniel, that's the relevant class: an open coding model you could theoretically run against your own Ollama/Qwen box, no API cage.[1]

MoE (Mixture-of-Experts): Instead of running the whole model for each query, many small "experts" are built in, and only a handful fire per token. K2.7-Code has 1 trillion parameters total but only 32 billion active (384 experts, 8 per token plus 1 shared). Picture: a massive consulting firm, but for each question only the eight relevant people are called in — knowledge of a giant, compute cost of a mid-size model.

Technically it inherits from K2.6: 256k context window, MLA-attention, tuned for agentic long-running tasks (planning, editing, running tools, debugging across many steps). Moonshot cites two concrete improvements over K2.6: better real-world coding performance and ~30 % fewer reasoning tokens per task — that directly cuts cost and latency.[1] The Hugging Face file, though, is ~595 GB, honestly beyond what runs on a normal workstation; realistically you'd run it quantized or via the Kimi API.

Careful reading The headline making the rounds — +21.8 % — is a jump on Kimi Code Bench v2 (50.9 → 62.0). That's Moonshot's own benchmark. Same with the other winning numbers: Program Bench +11.0 %, MLS Bench Lite +31.5 %, Kimi Claw 24/7 Bench +9.3 %. In-house suites work for "better than our last version," not "better than GPT-5.5."

Why the 21.8 % tells little about the competition [2][3]

It's exactly yesterday's scaffold problem, this time with an open model. The only numbers with outside credibility would be on standardized boards — SWE-Bench Pro (Scale SEAL), Terminal-Bench 2.0, Aider. And that's exactly where K2.7-Code stands: nowhere. Multiple reports from release day explicitly note that until publication no independent leaderboard had rerun with K2.7-Code.[2]

The standardized picture has been unchanged since 06.09 (Scale-SEAL board, same framework for everyone, 731 public tasks): at the top GPT-5.4 xHigh with 59.1 %, then Muse Spark 55.0 %, Opus 4.6 (think) 51.9 %, Gemini 3.1 Pro 46.1 %.[3] The only open model with a standardized entry remains Qwen3-Coder-480B at 38.7 % — a good 20 points below the leader. For DeepSeek V4 Pro there is (as of 06.09) no SEAL entry; circulating 80.6 % SWE-Bench-Verified or ~55 % Pro are vendor-scaffold values.[3]

The only third-party source One non-vendor number exists: MCPMark Verified 81.1 % (K2.7-Code) versus 76.4 % for Claude Opus 4.8 — that measures tool/MCP calls, not end-to-end bug fixing.[2] Solid, but narrow. For the question "does it solve my GitHub issue?" it says little.

Opinion: K2.7-Code is a serious, honestly open release — and for Daniel the most exciting open-weights drop since DeepSeek V4. But the order of steps matters: only when Scale (or another neutral board) runs it through the same framework as GPT-5.5 will you know if the 62.0 on the house bench means real 50s on SWE-Bench Pro or real 35s. Until then: try it yes, rank it no. The only benchmark that counts today is an afternoon against your own repo.

Kimi Work: 300 agents on your own laptop [4]

Parallel to the model, Moonshot released Kimi Work on 06.12 — a local desktop app for macOS (Apple Silicon) and Windows. You state a goal in plain language; the app works on your own machine: file access, browser control (via a "WebBridge" extension on logged-in sessions), and scheduled tasks.[4]

Agent-Swarm: Instead of a single agent working through a task serially, an orchestrator spawns many small agents working in parallel on pieces and coordinating. Picture: not one mason for the whole wall, but a foreman setting thirty masons to work simultaneously on different sections.

Per Moonshot, Kimi Work scales to up to 300 sub-agents in parallel locally, backed by K2.6, whose swarm is documented to handle "4,000 coordinated steps."[4] Both are vendor claims — the reporting article verifies neither the 300 nor the underlying model independently.

Security reflex A local agent with file and browser access to your logged-in sessions is powerful and for that reason tricky: it's effectively a program with your rights and your cookies. For experiments use a throwaway VM or a fresh browser profile — not your main system with production logins.

Take: "300 agents locally" is a strong marketing line, but 300 parallel runs on a laptop mostly means 300-fold memory and context pressure. The interesting question isn't the number but whether coordination stays stable over long runs — and that only practice shows, not a datasheet.

The open-weights camp, honestly sorted [3]

With K2.7-Code, a sober inventory makes sense: where the open-weights coding models really stand in 2026 — separated by vendor number and standardized measurement:

  • Kimi K2.7-Code (06.12) — open, 1T-MoE, agentic. Shines on house benchmarks; not listed on any neutral board. The hottest new arrival. try it
  • DeepSeek V4 Pro (04.24) — open, 1.6T-MoE, 1M context. Vendor values up to 80.6 % SWE-Bench Verified, but no Scale-SEAL entry — the same gap as Kimi.
  • Qwen3-Coder-480B — the only open model with standardized SEAL entry: 38.7 % SWE-Bench Pro. Realistically the cleanest open reference point today.

Take-away for Daniel: The speed at which the open-weights camp throws new versions is impressive — so is the documented gap to the closed top (GPT-5.4/5.5, Opus). Choose open and you buy sovereignty and cost, not (yet) top performance. That's a perfectly legitimate trade — as long as you don't mistake headline percentages for top rankings.[3]
