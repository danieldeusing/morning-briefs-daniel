Anthropic billing split: Agent SDK gets its own bucket [1]

Today, June 15, the quarter's most important change for Daniel's own tool chain takes effect: programmatic Claude use is decoupled from subscription limit and draws from a separate monthly credit pool. Announced May 14, the cutover is now in effect.[1]

Credit bucket vs. subscription limit: Until now, a Max subscription had one "cup" per 5 hours, from which everything drank—the human at the TUI the same as a background script via claude -p. If a long-running agent emptied the cup overnight, the next morning interactive work was rate-limited too. From today there are two cups: the old subscription cup for interactive work (TUI, Cowork, claude.ai) and a separate, dollar-denominated cup for everything programmatic. Script users won't throttle themselves anymore—but the script bucket is finite, doesn't roll over to next month, and isn't poolable across a team.

What moves to the new bucket: Claude Agent SDK, claude -p (headless CLI), Claude Code GitHub Actions, third-party apps via Agent Client Protocol.[1] What stays on subscription limit: interactive Claude Code (Terminal TUI, IDE plugin), Claude Cowork, claude.ai (Web/Desktop/Mobile).[1]

Operationally for Daniel (Max 20×) $200 sounds generous, but it's per user (no team pool), expires monthly (no rollover), and is list price. From today, track background volume (scripts, briefing orchestrator, GitHub Action reviews)—one week of data is enough to see whether $200 is sufficient or whether a step down to Sonnet makes sense.

Fable 5 + Mythos 5 disabled worldwide—export control [2][3][4]

Anthropic's new Mythos class—Fable 5 (GA since 06/09) and the preview Mythos 5—have been globally unavailable since June 12, 2026, 5:21 PM ET.[3] US Commerce Secretary Howard Lutnick issued an export-control directive to Dario Amodei that blocks access for every foreign national—inside and outside the US, including Anthropic's own international staff.[3] Anthropic disabled access for all users because foreign-national status cannot be reliably verified in a public API.

In GitHub Copilot, Fable 5 was disabled on Friday, 06/12, the same day: "effective today, access to Claude Fable 5 has been suspended across all GitHub Copilot experiences"[2]—three days after GA launch on 06/09.[2]

Export control on an API model: Classically, the US Department of Commerce regulates the export of chips, crypto algorithms, or dual-use software. Here, for the first time, a generative AI model is being treated as dual-use tech—allegation: the ability to "find weaknesses in code" is repurposable for cyber attacks. Anthropic's counterpunch: the same capability exists in GPT-5.5; "this standard would halt industry-wide re-releases."[3]

Trigger, per Marktechpost: a "narrow and non-universal" jailbreak that a "highly credible, trusted partner" of Anthropic's and the US government demonstrated—the technique asks the model to scan a codebase for security holes.[3][4]

Operationally The auto-mode fallback in Claude Code v2.1.176 (06/12) correctly reverts to the best available Opus.[8] Manual pins (model: "claude-fable-5") must switch to Opus 4.8 today or they fail hard. No official lifting deadline—until then, Opus 4.8 is the best available Anthropic model (88.6 % SWE-Bench Verified, see next section).

Gemini CLI → Antigravity CLI: Thursday is cutover [6]

Google shuts down Gemini CLI on June 18, 2026, 11:59 PM PT (Fri 06/19, 08:59 CEST / 03:59 BRT).[6] Replacement is Antigravity CLI, generally available since May 19—Go-based (not Python), asynchronous multi-agent workflows without terminal lock, unified with the Antigravity 2.0 desktop harness.[6]

Switch this week, not in four weeks. Anyone who has Gemini CLI pinned in CI or local scripts will see "command not found" starting Friday. Exception: Gemini Code Assist Standard/Enterprise licensees keep their access—EOL only affects free/login users.[6]

Multi-agent workflow (CLI): A single agent loops: plan → tool call → result → next plan. On long tasks (indexing, running tests, writing patches), that blocks the terminal. Multi-agent: the main tool delegates to sub-agents that tick in parallel in the background, returning the result. Native in Antigravity CLI; in the old Gemini CLI, you had to build that yourself.

GitHub Copilot: usage-based billing live, new Max tier [7]

Since June 1, 2026, GitHub Copilot Premium requests are charged in AI Credits: 1 AI Credit = $0.01.[7] Code completions and next-edit suggestions remain free (not charged to credits); the limit kicks in only for chat, agent mode, pull-request reviews, and spaces.[7]

PlanPrice/MonthIncluded Credits
Pro$10$15
Pro+$39$70
Max (new)$100$200
Business$19/User$19/User
Enterprise$39/User$39/User

The Max tier is new—for "sustained, high-volume agent workflows": $100 flat fee, $200 credit budget, effectively double the volume per dollar. GitHub's rationale: "a quick chat question and a multi-hour autonomous coding session can cost the user the same amount … the current premium request model is no longer sustainable."[7] Visual Studio Magazine headlined on 04/27: "You Will Get Less, but Pay the Same Price"—power users see no volume boost, only higher effective cost per run.[7]

In parallel with Anthropic Striking: GitHub and Anthropic are moving in lockstep—both separating background consumption from interactive limit and switching it to $-credits. Agent workloads are exploding in token burn; a unified subscription limit is no longer tenable on the provider side. For Daniel: consumption monitoring is now tool hygiene.

OpenCode overtakes Claude Code by stars [9]

This week's most striking OSS data point: OpenCode—MIT-licensed terminal coding agent—stands at 172,198 GitHub stars and is now the most-starred open-source coding agent.[9] For comparison: Claude Code ~124,000, Gemini CLI ~105,000, openai/codex ~90,000.[9] OpenCode supports 75+ AI providers per byteiota (Claude, GPT, Gemini, Ollama)—model agnosticism is its central pitch.

Reality check Stars ≠ quality. A medium.com reviewer benchmarked OpenCode on identical tasks: ~78 % slower than Claude Code.[9] Single voice, no official benchmark—but it tracks the pattern that "75+ providers" often means a generic adapter layer. try-this unleash it on a pet project—the speed trade-off shows fast; if it's negligible, model choice is a real upside.

Open weights: MiniMax M3 and Qwen3-Coder-Next [11]

Two open-weight models are worth noting for local coding this week. MiniMax M3 (June release) is, per HuggingFace blog, the first open weight combining frontier coding, 1M context, and native multimodality; tops the open-weight field on SWE-Bench Pro at 59.0 %.[11] Qwen3-Coder-Next is, per promptquorum.com, currently the best local LLM for coding—highest HumanEval and SWE-Bench Verified scores in the open-weight space.[11]

Local vs. frontier 59 % on SWE-Bench Pro for an open model is genuine progress—but Opus 4.8 scores 88.6 % on SWE-Bench Verified, and these are two different benchmarks. Practically: for code suggestions on confidential code with no cloud exposure, the local variant (Qwen3-Coder-Next on Ollama) gets interesting; for critical edits on production code path, Opus 4.8 remains by far the better tool. The two worlds are converging, but they haven't merged.
