Google shuts down Gemini CLI for individual developers today

Starting today, June 18, Google is ending Gemini CLI and Gemini Code Assist for individual users — specifically everyone on Google AI Pro, AI Ultra, and the free Code Assist tier for individuals. Google is "consolidating" everything under Google Antigravity, the agent-centric platform whose command-line interface (Antigravity CLI) is meant to be the successor.[1] For Code Assist on GitHub: as of 06.18, no new organization installations; existing requests will dwindle weeks later.[2]

Two things make the transition rough. First, Google itself admits there is no one-to-one feature parity at launch — the Antigravity CLI "preserves critical features including Agent Skills, Hooks, Subagents, and Extensions,"[1] which means other things are missing for now. Second, the Antigravity CLI is closed source, while Gemini CLI was open and had accumulated over 6,000 community contributions.[3] The reaction was accordingly cool: the official GitHub Discussion on the transition collected roughly 143 thumbs-down against 4 thumbs-up within 24 hours.[4]

CLI vs. IDE agent: A CLI (command-line interface) is the terminal tool — you type, the agent works in an editor-free window, good for scripts and servers. The developers' concern here is less "new name" and more that an open, hackable toolbox is being replaced by a closed one you can't peek inside.

Takeaway: If you used Gemini CLI privately, you should check today what workflows depend on it — and plan for the possibility that extensions not in the "critical" list may be unavailable for now. For sustained use, the open alternatives (Cline, OpenHands, Aider) are the steadier harbor, since they don't depend on a platform consolidation.

Android CLI 1.0 — every agent can now enter Android Studio

Quieter but relevant for mobile devs: Google's Android CLI 1.0 has been stable since May 19 (Google I/O). The core is the android studio command — it lets any agent (Claude Code, Codex, Antigravity) use the deep Android Studio capabilities: inspections, Compose previews, symbol search — without opening the IDE at all.[5] Google cites over 70% fewer LLM tokens vs. agent operation inside the IDE and threefold faster task completion.[6]

Read with skepticism: The token and speed figures are vendor-reported and have not been independently verified — plausible because a lean CLI path carries less IDE context, but treat as marketing number, not measurement. What's more interesting is the pattern: Google is opening its IDE tooling to foreign agents instead of locking it to Gemini — the opposite of today's Gemini CLI consolidation.

Claude Fable 5 has been globally locked for six days

The leading model on the coding leaderboards is currently unavailable. According to Anthropic on June 12, access to Claude Fable 5 (and Mythos 5) was suspended: "We are suspending access to Claude Fable 5 and Claude Mythos 5 … and are working to restore access as soon as possible."[7] The trigger, according to Anthropic's own account, is a US export-control directive that stops any access "by any foreign national, whether inside or outside the United States."[7] There is no restoration date as of today (06.18).

Anthropic openly contests the severity of the grounds: they disagree that "the finding of a narrow potential jailbreak should be cause for recalling a commercial model deployed to hundreds of millions of people."[7] Important for context: the exact nature of this "potential jailbreak" is disputed and not publicly documented — the statements above are Anthropic's own; I am explicitly not representing the jailbreak claim here as established fact, only the lock itself and its date are source-confirmed.

In practice Opus 4.8 and Sonnet 4.6 are unaffected and running normally — if you work in Claude Code and haven't explicitly requested Fable 5, you notice nothing. It's not a Claude Code outage, just a locked top-tier model.

Why it matters: This is the first lock of this magnitude due to an export directive, yanking a globally leading coding model off the market — and it hits all customers, including US developers. For tool selection it means: a model can vanish overnight for regulatory reasons. Don't build your entire stack on a single frontier model.

Benchmarks: Fable 5 leads, but unavailable [8][9][10]

On the current vals.ai leaderboard (as of 06.17), Claude Fable 5 leads SWE-bench Verified at 95.00% — ahead of Opus 4.8 (88.60%), GPT-5.5 (82.60%), Opus 4.7 (82.00%), and Gemini 3.5 Flash (78.80%).[8] The ironic twist: the top model in the table has been locked since 06.12, so these scores are from runs before the lock — there's no way to remeasure right now.

SWE-bench Verified — Top 5, as of June 17, 2026 (vals.ai)
SWE-bench Verified ≠ SWE-bench Pro: These are two different benchmarks, not two scales of one. "Verified" is the canonical, human-filtered public set from real GitHub issues; "Pro" is a harder set with production-grade, trickier tasks. Holding a Pro score against a Verified score compares apples to oranges — always read the label.

On the tougher SWE-bench Pro (vendor-run, with Anthropic scaffolding at launch on 06.09), Fable 5 stands at 80.3%, ahead of Opus 4.8 (69.2%) and GPT-5.5 (58.6%).[9] On the very hard FrontierCode Diamond, Fable 5 reaches 29.3% — more than double Opus 4.8 (13.4%).[9] As a practical anchor, Anthropic cites a Stripe migration: a 50-million-line Ruby codebase migrated in one day vs. over two months by hand.[10]

Skeptical The Pro, FrontierCode, and Stripe figures are vendor-reported and measured with vendor scaffolding — the standard benchmark-gaming caveat applies: such setups are optimized for the test and don't generalize one-to-one to your everyday code. The independent vals.ai Verified figure (95.00%) is the more solid anchor. And again: Fable 5 is not usable right now anyway.

Cursor: Cloud subagents and a much faster Bugbot

Cursor expanded the cloud side on 06.17: environment setup in the cloud in "less than 10 minutes" and an /in-cloud command that spins up subagents in separate VMs — parallel agent runs without local load.[13] Earlier on 06.10, Bugbot (the automated code-review agent) was overhauled: review time down to ~90 seconds vs. ~5 minutes, plus 10% more bugs found (0.62 vs. 0.56 per review) and ~22% cheaper per run.[13]

Note Cursor names "Composer 2.5" as the Bugbot driver. What exactly that is remains unspecified publicly — there's no open model card, likely a Cursor-internal model. I don't treat it as a known quantity here.

Cline v3.0.27 — daily cadence, new skill command

Cline released v3.0.27 on 06.17 with a new cline skill command (install/manage skills) and a prefilled MCP-Install Wizard for faster setup; plus parallel tool-call optimization and the ClinePass subscription model.[14] Earlier on 06.09, Fable-5 support landed.[14] Try this Why try it: With nearly daily releases, Cline is the most active open tool right now — the skill command makes recurring workflows portable without binding to a platform.

MCP (Model Context Protocol): A kind of USB standard for AI tools — instead of wiring each model individually to your database, filesystem, or ticket system, they all speak the same plug-and-play language. An "MCP server" is such a socket; the wizard installs them preconfigured.
