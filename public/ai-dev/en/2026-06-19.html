Fable 5 & Mythos 5: one week dark [1][2][3]

The week's biggest story is an absence. Anthropic shut down its flagship models Fable 5 and Mythos 5 on June 12 at 17:21 ET following a U.S. government directive — citing "national security authorities," according to the agency.[1] One week later, on 06-19, no official restoration date has been announced.[3]

The stated trigger: the government became aware of a method to "jailbreak" Fable 5 — circumvent its safety guardrails — with concerns about cybersecurity misuse.[2] For Daniel's workbench, the concrete consequence is: Opus 4.8 is currently the best Claude model you can run in Claude Code — not because it's the best Anthropic has built, but because the two above it are locked.

Heads up This is a political-regulatory block, not a technical rollback. Plan your stack as if Opus 4.8 is the ceiling — and keep an open or third-party fallback ready in case the ban lasts longer. There's no ETA; that's the real uncertainty.

Gemini CLI is dead — the migration hurts [4][5]

On 06-18, Google permanently shut down the Gemini CLI and Gemini Code Assist IDE extensions for AI Pro, Ultra, and free individual users — they no longer process requests.[4] The forced successor is the Antigravity CLI: a Go rewrite, closed-source, without feature parity to the old CLI.

The painful part is the quota model. Where Gemini CLI had a clear daily allowance (1,000 requests/day), Antigravity runs on a weekly compute cap — heavy users report burning through it quickly and hitting multi-day cooldowns.[5] If you relied on Gemini CLI day-to-day, don't underestimate the switchover: different tool, different language underneath, different billing behavior.

Compute cap: Instead of "you can make X requests per day," Google now measures how much compute your agents burn and caps it per week. Like a capped plan: run the agent hard, the quota empties fast — and you're locked out for days.

Cursor 3.8: Automate in plain language try it [6]

On the same day Gemini CLI died, Cursor 3.8 shipped.[6] The centerpiece is /automate: a skill to describe an automation in natural language inline in your agent session, rather than configure it. Plus five new GitHub triggers and a Slack emoji trigger — react with an emoji, the agent goes to work.

The more interesting switch: Computer Use for Cloud Agents is enabled by default on all automations. Cloud Agents can now generate demos and work artifacts, not just code diffs.[6] Why try it: if you have recurring repo tasks (triage, release notes, PR checks), the natural-language path to automation is much closer to "describe, don't tinker" — and Cursor supports Opus 4.8, GPT-5.5, Gemini 3.1 Pro, and Grok 4.3 in parallel, so no model lock-in.[7]

Caution "Computer Use enabled by default" means: Cloud Agents can do things beyond pure code-writing. For sensitive repos, it's worth checking the automation permissions before you let it run.

Claude Code: small, useful tweaks [8]

Two drops on 06-18. v2.1.181 brings flexible config via prompt commands like /config thinking=false, macOS sandbox support for Apple Events, and bumps the Bun runtime to 1.4.[8] v2.1.179 delivers what matters most day-to-day: recovery from connection drops mid-stream — partial responses survive, the hung spinner is fixed.[8]

Still relevant from last week: v2.1.172 (06-10) enables nested sub-agents up to five levels deep and a searchable plugin marketplace — for anyone building multi-tier agent workflows, the more material jump.

Benchmarks: the board that contradicts itself [9][10]

On the SWE-Bench Verified leaderboard (llm-stats.com, as of 06-19), Fable 5 leads at 95.0%, ahead of Mythos Preview (93.9%) and Opus 4.8 (88.6%). Best non-Anthropic model: DeepSeek-V4-Pro-Max and Gemini 3.1 Pro tied at 80.6%.[9] Two things immediately undercut this ranking: (1) The top two finishers are disabled — you can't use them. (2) Of 101 listed models, zero are independently verified — all 101 scores are provider-submitted.[9]

"Provider-submitted": The company runs its own model through the test and reports the score — nobody independent double-checks it. Like a student taking an exam at home and grading it themselves. Could be honest, but could also hide training data leakage or benchmark overfitting.

Counter-bench for reality-check: on LiveCodeBench (llm-stats.com, 06-19), DeepSeek-V4-Pro-Max leads at 93.5%, followed by DeepSeek-V4-Flash-Max (91.6%).[10] The disabled Claude models don't appear here at all, and Opus 4.8 isn't in the top 10 — a sober counterweight to the SWE-Bench crown. Read both boards as signals, not verdicts.
