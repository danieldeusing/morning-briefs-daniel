Fable 5 shut down—via export directive [1][2][3]

The most important news of this week is one you won't find in a changelog. Anthropic has shut down both of its newest flagship models—Claude Fable 5 and Claude Mythos 5—for all customers on 06/12/2026, just three days after launch on 06/09. Reason: a US government export control directive that Anthropic says reached them "today at 5:21pm (ET)".

The critical part for you is in the language. The directive demands to "all access to Fable 5 and Mythos 5 by any foreign national, whether inside or outside the United States" be blocked.[1] You're a German citizen in Brazil—you fall explicitly under that ban. Even if Anthropic reopened the models tomorrow for US citizens, you couldn't access them. In practice, Anthropic has flipped the switch for everyone: "we must abruptly disable Fable 5 and Mythos 5 for all our customers to ensure compliance."[1]

Export control on a cloud model: Imagine the government forbids a workshop to lend a specialized tool to foreigners. For physical goods (chips, weapons) that's routine. What's new is a cloud-delivered software model being caught by it—and "foreign" applies even to a German using the tool remotely via API.

The alleged motive is awkward for our category: the government reportedly found a way to "jailbreak" the model—and per Anthropic, that method boils down to "asking the model to read a specific codebase and fix any software flaws."[1] That's exactly the agentic coding workflow we use these models for. Anthropic calls it a "narrow, non-universal jailbreak" whose capabilities are "widely available from other models," and disputes it publicly: they think it's wrong to "disable a commercial model deployed to hundreds of millions of people" over a narrow potential jailbreak.[1]

Heads up Treat this as reported and disputed, not settled. Anthropic's statement is the only detailed source; Fortune and the National Law Review confirm date, directive, and "foreign national" scope, but report Anthropic's version of the tech details, not an independently verified account. What matters for you: Fable 5 and Mythos 5 are unavailable as a tool, with an open end.

GitHub Copilot: usage-based with "AI Credits" [4][5]

Since 06/01/2026, GitHub Copilot bills usage-based across all plans: "usage-based billing for GitHub Copilot is now live for all users."[4] The billing unit is the AI Credit, and the exchange rate is nicely simple: per GitHub docs "1 AI credit = $0.01 USD"—one US cent per credit.[5]

Usage-based billing / AI Credits: Before, Copilot was like a flat-rate internet plan—fixed monthly price, no matter how much you use. Now it's more like electricity: a base allowance is in the subscription, then the meter runs. Each "credit" is a virtual cent; expensive models (large reasoning models) cost more credits per request than cheap ones.

Key for daily work: code completions and next-edit suggestions are not billed in AI credits—"Code completions and next edit suggestions are not billed in AI credits."[5] Classic tab autocomplete stays unlimited; the meter only runs for pricier agent/chat calls to larger models. Model prices are public in the docs: Claude Opus 4.x costs $5 input / $25 output per 1M tokens, Claude Sonnet 4.x $3 / $15.[5]

What GitHub doesn't cleanly list officially are the credit allowances per plan. Secondary sources report Pro ≈ 1,500, Pro+ ≈ 7,000, and Max ≈ 20,000 credits/month; Pro+ reportedly costs $39/month and includes $39 credits.[6] Treat those numbers as reported—the official GitHub changelog doesn't name them, only the cent rate and model prices are first-hand confirmed.

Terminal-Bench 2.1: leader leads—but with a ghost [7]

Fresh movement on the Terminal-Bench-2.1 leaderboard, the scoreboard for agentic work in a real shell. At the top: Codex CLI + GPT-5.5 at 83.4 % ± 2.2 (submitted 05/01). On #2 follows Claude Code + Claude 5 Fable at 83.1 % ± 2.0—submitted on 06/17.[7]

Terminal-Bench & the "± 2.0": The benchmark gives an agent real tasks in a command line (build packages, fix files, get scripts running) and measures what fraction it solves. The "± 2.0" is measurement uncertainty: at 83.4 % vs 83.1 %, the error bars completely overlap—rank 1 and 2 are statistically indistinguishable. A half-percentage-point gap is noise here, not a result.

And that's exactly where the leaderboard collides with reality: the #2 model Fable 5 has been globally shut down since 06/12 (see left)—the submission date 06/17 is five days after shutdown. Whether that's a pre-shutdown computation submitted late or a data artifact can't be determined from outside. Practically: the #2 entry is unreproducible today. Rank 3 (Terminus 2 + Fable 5, 80.4 %) also hangs on the dead model. The best usable Anthropic entry is Claude Code + Opus 4.8 at 78.9 % ± 2.5 (05/29).

Gemini CLI is dead, long live Antigravity CLI [10][11]

Google shut down the Gemini CLI for the free, Pro, and Ultra tiers on 06/18/2026: "On June 18, 2026, Gemini CLI and Gemini Code Assist IDE extensions will stop serving requests for Google AI Pro and Ultra, as well as those using it free of charge."[10] The successor is Antigravity CLI—a closed-source Go binary replacing the previously open-source Python Gemini CLI (Apache 2.0); "Built in Go for improved performance."[10]

The catch is in the free quota. The old Gemini CLI offered "60 model requests per minute and 1,000 requests per day at no charge" at launch.[11] For Antigravity, that free quota is reportedly dropping from 1,000 to around 20 requests per day—reported by The Register; Google didn't publish a concrete Antigravity free number in its official post, so reported, not officially confirmed.[12] If you use Gemini CLI via a Code Assist Standard/Enterprise license, Google says you're unaffected.

Open → Closed The real loss isn't the quota, it's the license: an open-source, hackable CLI gets replaced by a closed binary. If you've relied on scriptability and self-patching, that's a step backward—and an argument for agent-agnostic harnesses that don't depend on a vendor binary.

Windsurf → Devin Desktop, plus ACP [13][14]

Windsurf officially became Devin Desktop with the OTA update on 06/02/2026.[13] The previous local agent Cascade gets an end-of-life on 07/01/2026 ("Cascade remains available through July 1st") and is replaced by Devin Local—rewritten in Rust and per the vendor "up to 30% more token efficient."[13]

ACP (Agent Client Protocol): A common plug standard between editor and coding agent. Before, each agent was wired into its IDE; with ACP you can plug third-party agents into the same interface—Devin Desktop "supports Codex, Claude Agent, OpenCode, and any other ACP-compatible agents."[14] Think USB-C for agents: one port, many devices.

That's the genuinely interesting part: instead of locking you into a house agent, Devin Desktop opens to the competition. Together with the Gemini CLI move above, a pattern emerges—tool shell and model are decoupling. Good for you, because your workflow then survives the failure of any single model (see Fable 5).
