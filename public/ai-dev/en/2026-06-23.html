Cursor announces "Origin" — and gets bought by SpaceX [1][2][3]

At its first company conference Compile (06.16., Fort Mason, San Francisco), Cursor dropped two things on the same day. First, a new product: Origin, a Git-hosting platform not built as a web UI wrapped around Git, but from the ground up for coding agents. Second — the same day — Cursor itself was acquired: "SpaceX announced a formal agreement to buy the artificial intelligence startup Cursor for $60 billion worth of stock."[1] All stock, no cash — an announced agreement (06.16.) with closing expected in Q3 2026, not yet completed.

The numbers for Origin, which Cursor showed on a single demo repo, are unusually concrete: around 296,000 clones per hour, ~81,000 pushes and 22.6 commits per second.[2] The actual selling point is latency: "Global synchronization operates 'under 400 milliseconds', with automatic failover within 10 milliseconds."[2] Technically, Origin sits on the re-architected foundation of the Graphite acquisition from December 2025 — "Built on re-architected technology from its December 2025 acquisition of Graphite."[3]

"Agent-native" Git hosting: GitHub is a web interface for humans to operate — you click to make branches, PRs, reviews. Agent-native hosting flips that: it's optimized for software agents that clone, modify, and push back to hundreds of repos in parallel without a human clicking along. The 296,000 clones per hour aren't marketing fluff — they're the load you get when your agent swarm, not you, is the user.

For you as a solo developer, Origin is not yet reachable: "Origin is expected to officially launch in fall 2026, with a waitlist now open."[2] What matters is the direction. If the dominant Git host suddenly faces competition from a player who treats agent load as the primary use case, that shifts, long-term, where your code lives and what CI and review look like. I'd file the $60-billion ownership change separately: it says more about the valuation picture (~$4 billion ARR, of which ~$2.6 billion B2B[4]) than about your editor's roadmap.

Context Treat Origin's numbers as vendor demo, not independent load test — they come from Cursor's own stage show. And a fall 2026 launch with waitlist means: nothing you're migrating to this week. The strategic takeaway is more about not locking your workflow too tightly to one Git host.

Gemini for coding: Flash is here, Pro is still waiting [5][6]

With Google's coding angle you need to keep two models straight. Gemini 3.5 Flash has been live since 05.19.2026 and is tuned for agentic work: Google claims it beats the older 3.1 Pro on "challenging coding and agentic benchmarks like Terminal-Bench 2.1 (76.2%)" and MCP Atlas (83.6%), at "4 times faster than other frontier models" in output tokens/sec.[5] For fast, cheap agent loops, that's the more interesting of the two.

The big Gemini 3.5 Pro — the model you want for heavy reasoning during coding — is still not generally available: "Gemini 3.5 Pro remains in limited preview for select Vertex AI enterprise customers only" (as of 06.19.).[6] A GA is expected "in June" and could land between the 19th and 30th — but that's announced, not shipped. Expect a 2-million-token context window and a "Deep Think" reasoning mode.

"Limited Preview" vs. "GA": Preview means the model runs, but only for selected enterprise customers via Google's cloud platform (Vertex AI) — you as a solo developer can't reach it with a regular API key. GA (General Availability) is the point where it opens to everyone, with published pricing and stability guarantees. Until then: nice specs, but no tool you can plan around today.

The circulating price estimate — ~$15 per million input, $60 per million output — is a secondary estimate, not an official rate: "roughly ten times the cost of Flash".[6] Without an official GA post with real pricing, that's an estimate, not a budgeting number.

Try GitHub Copilot SDK is GA [7][8]

As of 06.02.2026, the Copilot SDK is generally available: "GitHub Copilot SDK is now generally available."[7] The core: you can embed the agentic Copilot engine into your own apps instead of just using it in the editor. GA comes in six languages — "Node.js/TypeScript, Python, Go, .NET, Rust (new at GA), and Java (new at GA)"[8] — Rust and Java are new at GA.

The most important part for you is in the fine print: access is "to all existing GitHub Copilot subscribers … and to non-Copilot users via BYOK."[8] BYOK means you can bring your own model credentials instead of being locked into Copilot's backend — and the SDK talks to MCP servers.

SDK + BYOK (Bring Your Own Key): An SDK is a toolkit that lets you pull someone else's functionality into your own code. Here: the Copilot agent logic becomes a library you can build into your tool. "Bring Your Own Key" means you use your own API credentials — say, to Claude or another model — rather than Copilot's built-in model. You're using Copilot's agent machinery but feeding it the model of your choice.

Why try it: If you've been building your own coding automations and used to have to hand-stitch the agent loop, this SDK gives you a tested engine — with MCP and BYOK on top. That's the most concrete "grab it today" building block of the week, no fall-2026 promise.

Claude lands as an agent in JetBrains [9][10]

In IDEs a pattern is solidifying: the editor shell and the agent are decoupling. The JetBrains AI Assistant (v262.8117.29, 06.19.) now bundles two foreign agents natively in chat — "Claude Agent, powered by Anthropic's Agent SDK, is now seamlessly integrated"[9] alongside OpenAI's Codex agent. Plus "Next Edit Suggestions" in Preview.

In parallel and on the same track: GitHub shipped "Claude as agent provider preview in JetBrains IDEs"[10] on 06.22. via the Copilot route. You're getting Claude in JetBrains right now on two paths — directly via JetBrains' own AI Assistant and via Copilot as a provider.

Pattern Together with Copilot's BYOK SDK (above), this shows the same motion as in CLIs: tool ≠ model. When you can pick the agent freely in the IDE, you survive the outage or block of any single model more smoothly — an argument for favoring model-agnostic shells.

SWE-Bench Verified: all scores self-reported [11]

Quick sanity check on the leaderboards because top numbers are circulating everywhere this week. On SWE-Bench Verified, 0 of 102 listed models are independently verified per llm-stats — every top score is vendor self-report.[11] The best usable Google coding model on the list, Gemini 3.1 Pro, sits at 80.6% (self-reported).

"Self-reported" on benchmarks: The vendor runs its own model through the test and reports the result itself — nobody independent reproduces it. That's not automatically false, but it invites training contamination (the model has seen the test tasks before) and cherry-picking. A "verified" badge would mean a third party reproduced the same run. That hasn't happened for any of the 102 entries here.

Claude Code hardens agent mode [12]

In the Claude Code line v2.1.179–v2.1.185 there's a detail that matters in agent work: auto-mode now blocks destructive Git and Terraform commands unless you explicitly ask — "blocking of destructive git commands and terraform operations unless explicitly requested" (v2.1.183, 06.19.).[12] Also: /config key=value as new syntax, "Artifacts" (shareable web pages from sessions, Team/Enterprise beta), and sub-agents can spawn sub-agents — capped at five levels deep.

The auto-mode guardrail is the practically relevant change: an agent running unsupervised shouldn't accidentally hand you a git reset --hard or terraform destroy. Smart default brake for exactly the workflows where you run the tool unattended.
