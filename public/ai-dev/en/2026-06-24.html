Fable 5 is now a paid model [1][2][3]

This is the news that hits your workflow directly. Anthropic removed Fable 5 (released 06-09-2026) from plan limits on June 23 — continued use now runs on paid API credits.[2] The pricing: $10/M input tokens, $50/M output.[1] According to ayautomate, this is "the priciest generally available frontier model Anthropic ships, double the Opus 4.8 rate".

$/M tokens: You pay per million word-building blocks going in and coming out — like a taxi meter that ticks as you read and as you respond. Output is pricier because the model has to "think" for it. At $50/M out, an agentic run with heavy tool output adds up fast.

Practical upshot: anyone running Fable 5 over the plan quota burns it per future-stack-reviews at "roughly double the rate of Opus 4.8".[3] For Daniel's setup, that means: Fable 5 is now the model for the hard 10 % — the architecture call, the tricky refactor — not the default driver. Opus 4.8 stays the better price-to-quality for everyday work, and for routine code it's still worth routing to cheaper models (or locally: Qwen/Gemma). A "use Fable 5 for everything" reflex just got expensive.

SpaceX wants to buy Cursor — reported, not sealed [4][5]

Big headline, but read with caution: Per TechCrunch (06-16-2026), SpaceX has agreed to acquire Cursor (Anysphere), valuing the coding startup at $60B — entirely in stock.[4] Key point: this is a reported, not-yet-closed transaction; close is expected only in Q3 2026, so it could fall apart or shift.

The numbers behind it: Cursor hit "roughly $4 billion in annualized revenue in under four years", per CNBC ~$2.6B from enterprise B2B — the deal values it at roughly 15x revenue.[5] Why this matters for your toolbox: if a rocket builder buys the most-used AI code editor, it signals that coding tools are becoming strategic infrastructure — and a reminder not to bet too hard on a single proprietary editor whose owner (and roadmap) might change tomorrow. Cursor kept shipping in parallel: v3.9 (06-22) bundles plugins, skills, and MCPs in a Customize Page; v3.8 (06-18) brought automations with GitHub/Slack triggers.[5]

Google Managed Agents: agent via a single API call try [6]

Google is rolling out Managed Agents as a preview in the Gemini API (Gemini API + AI Studio Playground).[6] Idea: an agent that "reasons, uses tools and executes code in an isolated, ephemeral Linux environment with a single API call". Powered by Gemini 3.5 Flash ($1.50/$9.00 per M tokens). Agents are defined in Markdown — AGENTS.md + SKILL.md, so they're versioned and declarative.

Why to try it: You don't have to run your own sandbox infrastructure — the Linux container ships with the API call. For a quick "agent builds and tests something isolated" without custom harness setup, it's a cheap entry point. Boundary note: the Gemini Enterprise Agent Platform (the industry/business wrapper around it) belongs in the ai-usecases brief — here just what's relevant for dev hands: the API building block itself.

Benchmarks: strong numbers, weak evidence [7][8][9]

The leaderboard field is moving — but the most important number isn't a score, it's a methods warning. On the SWE-Bench-Verified board at llm-stats.com, Fable 5 leads at 95.0 % (across 102 evaluated models), followed by Claude Mythos Preview (93.9 %) and Opus 4.8 (88.6 %); Gemini 3.1 Pro and DeepSeek-V4-Pro-Max tie at 80.6 %.[7] As of leaderboard scrape on 06-24-2026.

Read skeptically Per digitalapplied, of 100 models on that board (as of 06-16), only 1 result was independently verified — the other 99 vendors submitted themselves.[8] That means: this 95 % is an Anthropic-reported figure, not neutrally measured. For self-reported scores in this range (95–99 %), caution is required — benchmark gaming, training contamination, and scaffolding (the agent framework around the model) often matter more than raw model capability.

To put "with scaffolding" in perspective: on SWE-Bench Pro (Anthropic's agentic benchmark), Fable 5 posts 80.3 % vs Opus 4.8 (69.2 %)[9] — same gap, different yardstick. So read the 95 % not as "solves 95 of 100 real PRs" but as "leads on a board where vendors enter their own numbers".

On the agentic terminal benchmark, going by available snippets, Codex CLI with GPT-5.5 leads the 2.1 board at 83.4 %, Claude Code with Fable 5 follows close at 83.1 %, Claude Code with Opus 4.8 at 78.9 %.[10]

Status unclear The Fable figure (83.1 %) comes only from a search snippet, not a fresh leaderboard fetch. Terminal-Bench: measures whether an agent solves real shell tasks in a terminal end-to-end — closer to "can this thing actually build and test my repo" than isolated code snippets.

Claude Code & Codex: dense tool week [11][12]

Claude Code ran at high cadence from 06-17–06-23. v2.1.187 (06-23) brings MCP login/logout via CLI, a sandbox.credentials setting (blocks sandboxed commands from reading credential files), and org model restrictions.[11] In between: blocked destructive git commands in auto mode, a /cd command to switch working directory mid-session, sub-agent chains up to 5 levels deep, and artifacts as shareable live web pages (version history in beta for team/enterprise).

MCP: A standard plug, through which the coding agent talks to external tools and data sources — like a USB port you plug a database, ticket system, or browser into without needing a separate cable for each one. The new login/logout CLI manages these port connections more cleanly.

Codex CLI 0.142.0 (06-22) brings usage credit redemption, plugin organization by category, and configurable rollout token budgets.[12] Already on 06-15 came /import to migrate from Claude Code; on 06-18, "Record & Replay" for macOS (turns demonstrated workflows into reusable skills). The /import path is notable: OpenAI is actively lowering the switching cost from Claude Code to Codex.

Codex vs. Claude Code: two philosophies [10]

The harness comparison is timely. Codex runs on 8 parallel sub-agents and a 200K context window; Claude Code on a 1M-token window — but burns "3–4× more tokens" for identical tasks (in the Figma plugin example, 6.2M vs 1.5M tokens).[10] This is the trade-off in pure form: Claude Code sees more context at once, costs more per task — which cuts twice after today's Fable 5 price jump. Codex is token-sparse but has to actively manage its tighter context. On adoption: Claude Code claims "326K+ daily commits (~10% of all public commits)" per morphllm; Codex stands at 91,000 GitHub stars.[10]

Concrete for the cost math: if Claude Code pulls 3–4× more tokens and you run pricier Fable 5 in it, the two factors multiply. For token-heavy routine runs, the cheaper path is now clearer than ever — a cheaper model in the harness, or where it fits, a local one (Qwen-Coder, Gemma) for tasks that don't need frontier reasoning. Spend frontier tokens only where the problem really demands them.
