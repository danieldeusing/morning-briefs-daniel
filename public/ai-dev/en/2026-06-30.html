GPT-5.6: Big Announcement, Limited Rollout

On June 26, OpenAI unveiled the GPT-5.6 family — three models with heavenly names: Sol (flagship), Terra (balanced), and Luna (fast/cheap).[1] The pitch is agentic coding: long, autonomous tasks instead of single queries. Sol costs $5 input / $30 output per 1M tokens; Terra $2.50/$15; Luna $1/$6.[1] Sol also gets two new reasoning tiers.

Reasoning Mode ("max"/"ultra"): Imagine giving someone more time to think before answering. That's the switch here: at "ultra", the model computes longer and more thoroughly before outputting code — pricier and slower, but more accurate on tricky tasks.

The striking part is how it's rolling out: a limited preview for "a small group of trusted partners," coordinated with the US government; broader access only "in the coming weeks."[1] OpenAI publicly distances itself from the restriction — "We do not believe this kind of government-access process should become the long-term standard."[2] For Daniel in practice: not testable yet, maybe only through Codex and select partners.

And exactly there, it got concrete: According to TechTimes, OpenAI quietly activated GPT-5.6 in Codex for some users — it only surfaced because a leaked system prompt revealed the swap.[3] So if you're building on Codex, double-check which model is actually responding.

Bench Caveat The preview comes with only Terminal Bench 2.1: Sol reaches 88.8% (Ultra config 91.9%), ahead of Fable 5 at 83.4%.[4] OpenAI deliberately did not publish a SWE-Bench score — the exact benchmark where Claude leads is "simply absent."[5] Until that gap closes, GPT-5.6 vs. the Claude family on coding can't be cleanly compared. A single benchmark doesn't prove generalization.

Fable 5: Three Days in Copilot, Then Gone

A lesson in how fragile model availability has become in the enterprise. Anthropic's Fable 5 — per GitHub, the "first Mythos-class model, designed for long, autonomous coding and knowledge work" — went into GitHub Copilot GA on June 9.[6] On June 12, it was back out, "suspended across all GitHub Copilot surfaces."[7]

The reason is a US export-control directive: On June 12, Commerce Secretary Howard Lutnick — invoking export-control and national-security authority — ordered Anthropic to block Fable 5 and Mythos 5 for every foreign user worldwide.[7] The trigger was a reported "jailbreak" that could unlock the models' cyber capabilities. Anthropic publicly disputes that the jailbreak is that serious.[7] Older Claude versions are unaffected and remain available in Copilot.

Export Control: The US government can prohibit some technologies from reaching foreign users if it sees a security risk — like an arms embargo, but for AI models. Here the government feared a "jailbreak" could unlock Fable 5 and Mythos 5's cyber capabilities. Result: Anthropic had to shut both models off worldwide for all foreign users — not for poor quality, but by government order.

Separately, as operational background: Fable 5 also required storing prompts and outputs for up to 30 days for safety classifiers[8] — a unique requirement among Copilot models that clashes with Microsoft's zero-data-retention stance. But that's a separate quirk, not the reason for suspension.

Qualitatively, it's a double shame: Fable 5 currently tops the SWE-Bench Verified leaderboard (95.0% per llm-stats, as of 06/30) — but it's just not usable right now.[9] The best actively available model on the list is Claude Opus 4.8 at 88.6%.[10] So if you're in Copilot today, you're fine on Opus — don't treat that as a step backward.

MCP-RC: The Infrastructure Gets a Rebuild [11][12][13]

The most consequential entry today — even if it sounds undramatic. The Model Context Protocol Release Candidate for version 2026-07-28 has been out since May 21; the final spec lands on July 28, with a 10-week RC window for Tier-1 SDKs to catch up.

MCP (Model Context Protocol): The universal socket through which AI agents talk to tools — filesystem, database, API. Instead of building a custom adapter for each tool, everyone speaks the same language. Daniel's Claude Code setup is entirely plugged into this socket; if the plug changes, the servers have to follow.

Three changes that matter:

  • Stateless core: The initialize/initialized handshake and the Mcp-Session-Id header disappear — every request now carries its own context. This decouples a client from a single server instance and makes horizontal scaling (load balancers, multiple replicas) finally trivial.
  • Full JSON Schema 2020-12: Tool inputSchema and outputSchema can now use the whole standard — stricter, more expressive tool contracts.
  • OAuth 2.0 / OIDC hardening plus an Extensions Framework (Tasks + MCP Apps). In exchange, Roots, Sampling, and Logging are deprecated — with 12-month windows.
What It Means for Daniel If you run your own MCP servers, test them against the RC right now — especially if you use Sampling or Roots. The 12-month deprecation gives breathing room, but stateless is a real architectural shift, not a flag.

Microsoft MAI-Code-1-Flash: Small, Native, in the Picker

Microsoft released MAI-Code-1-Flash on June 2 — a native coding model with just 5 billion parameters and 256k context, emphasizing it was "trained from scratch on clean, traceable data, without distillation from third-party models."[14]

Distillation: A small model learns by watching a big one and mimicking its answers — like an apprentice copying the master. Microsoft explicitly says MAI-Code is not that — relevant because "learned from OpenAI/Anthropic" has licensing and legal thorns.

The numbers: SWE-Bench Pro 51.2% vs. Claude Haiku 4.5 at 35.2%; up to 60% fewer tokens on SWE-Bench Verified; +28.9 points on instruction-following.[15] These are vendor numbers — read with the usual caution. More practically: the model is now in the VS Code Copilot picker, across all tiers (Free/Pro/Pro+/Max), plus OpenRouter, Fireworks, and Baseten.[16] Try Worth a spin as a cheap, fast default for routine edits — especially since it claims token efficiency.

Copilot Shifts to Usage-Based Billing

Since June 1, GitHub Copilot has run on usage-based billing with monthly AI Credits across all plans.[17] Concretely: Free $0, Pro $10/month (includes $15 credits), Pro+ $39/month ($70), Max $100/month ($200).[18] Plus a quiet but pricey shift: Copilot Code Review now counts against included Actions minutes.[19]

For agentic sessions this is real leverage: long autonomous runs (exactly what GPT-5.6 and Fable 5 pitch) are no longer "flat" — they consume credits. If you're heavy on background agents, watch your credit burn — the priciest models in the tightest loop can blow through a plan fast.

SWE-Bench Verified — and Why to Be Skeptical

SWE-Bench Verified — Top 6, as of 06/30/2026 (llm-stats)

The current snapshot per llm-stats (updated 06/30): Fable 5 95.0%, Claude Mythos Preview 93.9%, Opus 4.8 88.6%, Opus 4.7 87.6%.[20] Key to keep in mind — this is a ranking across 500 verified real GitHub issues, not a real-world guarantee.

Skepticism The aggregators disagree: BenchLM.ai lists a "Claude Mythos 5" at 95.5%; a third source puts the top contender at only ~78%.[21] These spreads come from different scaffolding (the agent harness around the model) and different data splits — you're not measuring the model alone, but model + harness. So read absolute percentages as magnitude, not gospel.

LiveCodeBench tells a different story: DeepSeek-V4-Pro-Max (93.5%) and -Flash-Max (91.6%) lead per llm-stats,[22] while ArtificialAnalysis sees Gemini 3 Pro Preview at 91.7% (as of 06/24).[23] Again: two boards, two peaks — a hint that "the one best coder" barely exists.

Tooling: Cursor, Claude Code, WebMCP

Cursor 3.8 (06/18) adds an /automate skill to describe automations in plain language directly in an agent session, plus five Slack-emoji triggers, five GitHub triggers, and Computer Use.[24] The 3.7 wave (06/04–06/17) already shipped Design Mode (annotate UI elements right in the canvas), a Context Explorer (token breakdown), and Bugbot with /review.[25] Try The /automate flow is the lowest-friction Cursor automation onramp yet.

Claude Code v2.1.195 (06/27): new /rewind command, around 37% less CPU load during streaming (via text-update coalescing), fixed background agents, MCP-retry logic, and a sandbox.credentials setting.[26] The CPU piece is audible in local fan noise.

WebMCP — proposed at Google I/O on 05/19, co-developed with Microsoft — is an open web standard that annotates JavaScript functions and HTML forms as browser-agent-callable tools.[27] Roadmap: Chrome/Edge H2 2026 (Chrome-149 Origin Trial live), Firefox Q3, Safari Q4.[28] Secondary sources cite 67% fewer errors vs. visual scraping.[29] Conceptually it's "MCP for the open web" — agents call declared functions instead of clicking pixels.
