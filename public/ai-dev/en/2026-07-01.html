Claude Sonnet 5 — the new default in your terminal

Anthropic released Claude Sonnet 5 yesterday (06.30), and today Claude Code v2.1.197 makes it the standard model. This is the most practically relevant news of the day: If you have Claude Code open, you'll be working with a new model after the update without lifting a finger.[1][2]

The core numbers: 85.2% on SWE-bench Verified and 63.2% on the harder SWE-bench Pro (up from Sonnet 4.6's 58.1%). Plus a native 1M-token context window (up to 128k output) — the first time Sonnet in Claude Code runs with the long context length previously reserved for Opus.[1][3][2]

SWE-bench Verified vs. Pro: Both measure whether a model can fix real GitHub bugs independently. "Verified" is the human-reviewed, cleanly solvable selection (~500 cases); "Pro" is deliberately harder and contamination-resistant. Think TÜV vs. rally: the same test, but Pro is where the tires really burn — that's why Pro scores run 20+ points lower across all models.

Try this The introductory price of $2 per million input / $10 per million output tokens (through 08.31, then $3/$15) makes Sonnet 5 even cheaper than Sonnet 4.6 — with better bench numbers. For daily loops in Claude Code it's worth testing because you're getting switched over anyway; I'd still run a deliberate comparison against your current default before building 1M-context prompts (long context costs real money).[3][1]

Claude Code: this week's harness sprint

Alongside the model switch, Claude Code got several tangible features in a single week. In order, with dates:

v2.1.196 (06.30) brings two things worth taking seriously. First, an MCP security gate: Servers defined in .mcp.json no longer auto-start but require explicit approval. Second, subagents can now nest 5 levels deep — plus org-wide default model controls.[2]

MCP (Model Context Protocol): An open standard for agents to call external tools and data sources — like a USB port for AI tools: once defined, every client speaks to the same interface. The new gate is why it matters: A .mcp.json that a repo ships could otherwise launch a server unasked when you open it — now Claude Code asks first.

v2.1.191 (06.25) delivered the /rewind command (rewind conversation to an earlier point — handy when an agent run goes sideways), MCP retry logic, and ~37% CPU reduction via batched text updates (100-ms window). You'll notice this mainly on laptop battery life.[2]

Context: The 5-level subagent nesting is the statement in the race for "agents coordinating agents" — Cursor is going the same route with cloud VMs (below). The MCP gate is the more mature answer: control first, then autonomy.

GPT-5.6 Sol/Terra/Luna — preview, but without coding proof

OpenAI launched a limited preview of the GPT-5.6 lineup on 06.26: Sol (flagship), Terra (balanced for everyday work), and Luna (fast/cheap). Access is currently limited to ~20 partner organizations; the broader release should follow "in the coming weeks".[5][6]

What hard numbers exist: Terminal-Bench 2.1 score: Sol (max) 88.76% (Sol ultra 91.91%). Pricing: Sol at $5/$30, Terra at $2.50/$15, Luna at $1/$6 per million tokens.[5]

Important — what's missing For GPT-5.6 there is no public SWE-bench score (neither Verified nor Pro) — as of today the model doesn't appear on any coding leaderboards. Terminal-Bench measures agent shell behavior, not pure bug-fixing like SWE-bench; the two numbers aren't comparable. If you want to pit GPT-5.6 against Sonnet 5, you'll need to wait for the broader release and an independent bench entry. I'm deliberately not writing a comparison number here that doesn't exist.

Context The three-tier split (Sol/Terra/Luna) mirrors Anthropic's Opus/Sonnet/Haiku logic and Google's Gemini tiers — the market is converging on "one name, three price/performance points." Practically: As a solo developer, this isn't a toolbox decision yet as long as you don't have access. Keep it on the watchlist for now.

GLM-5.2 — the open field has a new leader

From the open-weights camp comes the strongest contribution: Z.ai released GLM-5.2 first to coding-plan subscribers (06.13) and then as open weights under MIT license (06.16/17). MIT means: you can run it locally, use it commercially, fine-tune it — no usage restrictions.[7][8]

The numbers that matter: 62.1% SWE-bench Pro (up from 58.4%) — making GLM-5.2 the leader among open models — and 81.0% Terminal-Bench 2.1 (up from 63.5%). On the Intelligence Index v4.1 it ranks first at 51 (MiniMax-M3 44, DeepSeek V4 Pro 44). Architecture: 753B parameters total, ~40B active as Mixture-of-Experts, 1M context.[8][7][9]

Mixture-of-Experts (MoE): Instead of running the entire 753-billion network for every token, the model activates only a matching sub-network (~40B). Like a hospital that doesn't call all doctors for every visit, just the specialty needed — full expertise in-house, but per case only a fraction works. Makes an otherwise massive model affordable to run.

Try this For local/sovereign deployment, this is the week's most interesting candidate: open weights, leading open SWE-bench-Pro number, 2.9× fewer FLOPs per token at 1M context (IndexShare attention). Via OpenRouter, pricing lands around ~$1.40/$4.40 — well below the proprietary flagships.[7]

One catch GLM-5.2 is chatty: 43k output tokens per task (Intelligence Index) versus 26k for GLM-5.1. With output-based billing, that eats into the cheap token price — noticeable in agent loops with many steps. Factor that into your comparison.[7]

SWE-bench Verified: the board with a skeptical eye [10][11]

The public SWE-bench-Verified leaderboard was updated today (07.01). The top of the table at a glance — with a caveat:

SWE-bench Verified — top 4, as of Jul 01, 2026

Fable 5 leads at 95.0%, Opus 4.8 at 88.6%, the new Sonnet 5 at 85.2%, GLM-5 (5.1) at 77.8%. On the harder SWE-bench Pro the field shifts: Opus 4.8 actively leads at 69.2%, GLM-5.2 at 62.1% as the best open model, Sonnet 5 at 63.2%.[10][11]

Skepticism required These are predominantly vendor-reported numbers. Scale/SEAL measures on a standardized, contamination-controlled set much lower: there GPT-5.4 (xHigh) leads at 59.1%, Opus 4.6 thinking at 51.9%. The rule of thumb from practice: Vendor scores run 15–30 points above the standardized ones. Take the 95% not as "nearly solved" but as a ranking hint, not absolute value.[11][12]

Why it matters to you: A model that shines on Verified but stumbles on your repo with unusual build setup isn't surprising — benchmark gaming and training-data proximity are real. The only reliable test is still your own code.
