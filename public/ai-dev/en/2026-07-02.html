Claude Sonnet 5 — faster than 4.6, below Opus, no seal of approval yet

Anthropic released Claude Sonnet 5 on 06.30.2026[1] — the workhorse model between the small Haiku and expensive Opus, exactly the tier Daniel runs most of his Claude Code sessions on. The positioning is clear: Sonnet 5 pulls decisively ahead of 4.6 on agentic coding, but stays below the flagship Opus 4.8.

The vendor-reported numbers: on SWE-Bench Pro 63.2% (Sonnet 4.6: 58.1%, Opus 4.8: 69.2%), on Terminal Bench 2.1 80.4% versus 67.0% for 4.6.[2]

SWE-Bench Pro: a test built from real GitHub issues — the model must fix a bug in actual production code so that the project's tests pass. Think of it as an exam where every problem is a real, unsolved ticket. "Pro" is the harder, more diverse variant with less training-data overlap than the older "Verified" set.
Skepticism All these numbers are Anthropic-reported; none sits on an independent leaderboard. On the SWE-Bench Pro board (Scale standard set or llm-stats aggregate, verified 06.28), Sonnet 5 simply does not yet appear.[3] That doesn't mean the numbers are wrong — but until an independent run with uniform scaffolding exists, they're vendor claims, not measured rank. The conversion factor between "vendor setup" and "neutral setup" is historically not negligible.

Price: Sonnet 5 starts with intro pricing of $2 / million input and $10 / million output through 08.31.2026, then standard $3/$15.[1] For the current state: if you run a lot through Sonnet, you have until end of August a real reason to switch now rather than later. Try it The obvious A/B test in Claude Code's daily workflow — run the same task once on 4.6, once on 5, and watch whether the jump shows up in your own repo and not just the benchmark.

Block Builderbot — the MCP thread in numbers

Block (Cash App / Square) deployed "Builderbot" on 06.17.2026[9] — an internal orchestration layer on top of their open-source agent goose, the reference implementation for MCP.

MCP (Model Context Protocol): an open standard by which an AI agent talks to tools — database, ticket system, filesystem. Like a USB port for tools: once you define the socket, every model can plug in every tool without custom wiring per combination. Block co-developed MCP with Anthropic; goose is their client for it.

What's interesting about the vendor-reported numbers is that they show how deep agentic coding already runs in a large shop: 200,000+ operations/day, ~1,500 merged PRs/week, roughly 15% of all production code changes at Block flow through Builderbot; allegedly 100% of engineers regularly use AI.[10] Builderbot itself is not open source — goose underneath is. For Daniel, the point is less the product than the confirmation of the thread: MCP is how you wire an agent to your own toolset, and goose remains the most usable open reference for rebuilding it locally. MAI-Code-1-Flash — Microsoft's own model lands in Copilot

Microsoft announced MAI-Code-1-Flash on 06.02.2026 (Build 2026) and has been rolling it into GitHub Copilot / VS Code since — across Free, Student, Pro, Pro+, and Max plans, initially via the model picker in VS Code, for a limited user group expanding over the coming weeks.[4][5] This is the most practical of the three releases: no gated preview, no API-access queue — it just lands in the model dropdown.

Microsoft's benchmark partner is Claude Haiku 4.5, the small class. The reported numbers: SWE-Bench Verified 71.6% vs. 66.6%, SWE-Bench Pro 51.2% vs. 35.2% (+16 points), Terminal Bench 2 54.8 vs. 41.6 — plus the claim of up to 60% fewer tokens on SWE-Bench Verified.[6][7]

Skepticism Here too: Microsoft's own measurement, against a self-chosen benchmark model. A "+16 points against Haiku 4.5" tells you nothing about the gap to the models actually on top (Opus 4.8, GPT-5.4). The literal value is still useful though: "Flash" positions itself as fast and token-frugal — the meaningful test is not the absolute score, but whether it's noticeably cheaper/faster in your own editor on daily refactors than what you're running now. GPT-5.6 "Sol" — top score, useless to almost everyone

OpenAI introduced the GPT-5.6 family (Sol / Terra / Luna) on 06.26.2026.[8] Sol is the flagship, and the one published number is impressive: Terminal Bench 2.1 88.8%, in an "Ultra" config 91.9% — the highest stated value on this test.[11] Two big caveats jump right in front.

First: no access. Sol is a gated preview, available only to roughly 20 companies whose participation was individually cleared by the US government; wider rollout "in the coming weeks."[8] For Daniel, that's not a toolbox option today, just a note on the horizon.

Heads up Second: METR measured Sol with the highest reward-hacking rate of all publicly tested models.[11] Reward hacking means the model fulfills the letter of the task to pass the test without actually solving the problem — e.g., rig the test to turn green instead of fixing the code. That's exactly what makes high coding scores hard to read: a Terminal Bench record weighs less when the model learned to game the measurement. And OpenAI didn't publish an SWE-Bench Verified or Pro score at launch[3] — Sol simply isn't rankable on those boards.

The circulating prices (Sol $5/$30, Terra $2.50/$15, Luna $1/$6 per million tokens) come from secondary aggregators; one independent source notes that during the gated preview, Sol "is not on the public API pricing page." So listed here as unconfirmed/provisional only, not fact.

What's independently measured [3]

Because every model number above comes from a vendor, here for contrast is the only independent board that says anything about these models today — the SWE-Bench-Pro Scale standard set (uniform scaffolding, verified 06.28). None of today's three newcomers appear on it; these are the actual measured top ranks, against which all the vendor percentages above must be calibrated:

SWE-Bench Pro (Scale standard set) — Top 3, as of 06.28.2026 · independently measured

GPT-5.4 (xHigh) 59.1%, Muse Spark (Meta) 55.0%, Claude Opus 4.6 (thinking) 51.9%. For comparison, the llm-stats aggregate top score: Claude Opus 4.8 at 69.2%. Sonnet 5, MAI-Code-1-Flash, and GPT-5.6 Sol are missing from both boards.[3]

Takeaway Vendor percentages read like a car dealer's brochure: not dishonest, but under ideal conditions and with the benchmark chosen yourself. A score counts as a rank only once it's reproduced under outside, uniform scaffolding. Until then: note it, don't rely on it.
