Sonnet 5 is now your default Claude

This is the news that directly affects your workday. Anthropic released Claude Sonnet 5 on 06-30 and immediately made it the default model for Free and Pro plans — and, more importantly for you: it's now the default in Claude Code (as of version 2.1.197, with native 1M-token context) and available in GitHub Copilot.[1][2][3]

The positioning is "near Opus, but much cheaper": introductory pricing $2 / $10 per million tokens (input/output) through August 31, 2026, then $3 / $15. That's Opus-class performance at a fraction of the cost — for the vast majority of coding turns that don't need the top model, the obvious choice. TRY Update to 2.1.197 and watch for a day whether your agent loop runs cheaper before escalating.

Heads up tokenizer Sonnet 5 has a new tokenizer: the same input produces roughly 1.0–1.35× more tokens than Sonnet 4.6, according to Anthropic — depending on content type. The price per token drops, but the token count rises slightly; when comparing costs, use the real token count, not just the sticker price.

On Anthropic's supplied benchmarks: 46.8% on Humanity's Last Exam (with tools), 78.5% on OSWorld-Verified — solid, but no coding score you should directly pit against Fable/Opus.[1]

Fable 5 is back globally — Mythos 5 for partners only Claude

Quick recap of the episode: On 06-12, the US government placed export controls on Claude Fable 5 and Mythos 5, restricting access for foreign nationals.[4] The trigger was a report from Amazon researchers who could bypass a Fable-5 safety guardrail — the model produced code in one case that demonstrated how to exploit a software vulnerability.[4]

On 06-30, the controls were lifted; since 07-01 Fable 5 is globally available again, now with an additional security classifier.[4][5] The sharper sibling model Mythos 5 (same weights, fewer guardrails) returned only for select US organizations under Project Glasswing — for defensive cybersecurity, not the general public. For you outside the US, concretely: the access that was blocked for non-US users for three weeks is back.

Export control on a model: Normally you regulate physical goods this way (chips, weapons). Here a software model was treated as a controlled technology — the provider then has to manage who from which country gets access. Imagine a tool so powerful that customs decides who can use it. That's exactly what happened here for three weeks.

Performance-wise, Fable 5 leads the coding benchmarks (see next door). But the real news value is the precedent: a frontier coding model was regulated for the first time because of an exploit capability. This is a pattern that could repeat.

Benchmarks: what really counts — and what's marketing Bench

On the coding leaderboards, skepticism is a must. Here's the state as of 07-03-2026, with honest fine print.

SWE-Bench Verified — Top Models, as of 07-03-2026

SWE-Bench Verified: Mythos 5 leads with 95.5%, Fable 5 follows at 95.0%, Opus 4.8 sits at 88.6%.[6][7] The Fable-5 number was independently confirmed by vals.ai at 95.00% — that's the most reliable number in the field.

Read skeptically Three pitfalls today: (1) SWE-Bench Pro — Fable 5 is listed at ~80.0–80.3%, but that's a vendor number with Anthropic's own scaffolding, not independently verified. (2) Aider leaderboard is stale — last updated 11-20-2025, over seven months old; the GPT-5 scores shown there are not a current comparison. (3) The Artificial Analysis Intelligence Index shows Fable 5 at the top of coding-relevant models, but exact point values contradicted themselves between two calls — so here only the ranking claim, no hard number.[7][8]
SWE-Bench Verified: A test from real GitHub issues where the model has to write a working patch that passes the repo's tests. Like a hands-on trade test instead of a theory exam: not "does it know the answer" but "does it fix the broken thing". "Verified" means humans reviewed the tasks — that's the most credible coding board. Still: high scores can reflect training contamination or scaffolding optimized to the test, not pure capability.

LiveCodeBench: Here not Claude leads, but Gemini 3 Pro Preview at 91.7, ahead of Gemini 3 Flash Preview (90.8) and DeepSeek V3.2 Speciale (89.6).[9] Different board, different tasks, different ranking — exactly why you should never take a single leaderboard as gospel.

OSS: China ships LongCat-2.0 OSS

According to a VentureBeat report (the site returned HTTP 429 on direct fetch, so here reconstructed from search synthesis), Meituan around 06-30 open-sourced LongCat-2.0 under permissive MIT license: a 1.6-trillion-parameter mixture-of-experts coding model that activates only ~33–56B parameters per token and carries a 1M-token context window.[13]

Mixture-of-Experts (MoE): Instead of firing the whole giant model for each request, a router activates only a few matching "expert" sub-networks. Like a big law firm where not every lawyer works every case — only the two or three with the right specialty. Result: 1.6 trillion parameters of total knowledge, but only a fraction of the compute cost per token.

Benchmark figures (vendor-reported): SWE-bench Pro 59.5, Terminal-Bench 70.8 — solid for open-weights, but below Fable/Opus.[13] The real headline is the claim that the model trained entirely on a domestic 50,000-card cluster with Chinese hardware (Meituan uses Huawei's HCCL library — the chipmaker isn't explicitly named, but that's a reasonable inference, not a confirmed fact). TRY If you're hunting a strong, truly open MoE for local/on-prem experiments: the MIT license makes LongCat-2.0 a candidate — provided you have the hardware for a 1.6-trillion-parameter model (most will need to wait for quantized/hosted variants).
