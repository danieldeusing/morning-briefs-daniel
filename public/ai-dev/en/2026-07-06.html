Claude Sonnet 5 is now the default in Claude Code

Anthropic released Claude Sonnet 5 on June 30, calling it its "most agentic Sonnet model yet" — one that can "make plans, use tools like browsers and terminals, and run autonomously" at a level that previously required larger, pricier models.[1] For Daniel's daily workflow, the real news is the second part: Sonnet 5 is now the default model in Claude Code (version 2.1.197), with a native 1-million-token context window.[2]

"Agentic": A model doesn't just produce a single answer — it works through a task in multiple steps of its own choosing: reading files, calling tools, checking results, and correcting itself, similar to an intern given a goal instead of step-by-step instructions. The more "agentic," the longer it can run reliably without needing to check in.

Anthropic's own framing: Sonnet 5 delivers performance "close to that of Opus 4.8, but at lower prices," and can match Opus-4.8-level results on agentic tasks even at medium compute effort.[1] For security/cyber work requiring reduced safeguards, Anthropic still recommends Opus 4.8. In practice, that means the expensive Opus run is no longer needed for most coding sessions — Sonnet 5 covers the bulk of it, and the price gap is real (see box).

Anthropic cites improvements over Sonnet 4.6 "especially in reasoning, tool use, coding, and knowledge work," listing eval categories including BrowseComp (agentic search) and OSWorld-Verified (computer use).[1] An independent data point comes from the Artificial Analysis Intelligence Index: there, Sonnet 5 scores 53 in adaptive-reasoning/maximum-effort mode, against a category average of roughly 28.[3] That's an aggregated overall intelligence score, not a pure coding score — for the coding question, the benchmark caveat in the next section is what matters.

Pricing Launch pricing through Aug 31, 2026: $2/million input tokens, $10/million output tokens.[1] After that, standard pricing kicks in: $3 input, $15 output per million tokens. Anyone running a lot of agentic experiments should use this cheap window. try Update to 2.1.197 and point Sonnet 5 at a real repo — the 1M context makes whole-repo context possible without RAG workarounds.

One caveat: the details on Claude Code version 2.1.197 and the 1M context come from secondary aggregator sources (releasebot.io, gradually.ai), not the primary changelog — plausible, but not primarily confirmed. The model and pricing facts, on the other hand, come straight from Anthropic's announcement.

Benchmarks: the only Sonnet 5 numbers are Anthropic's own

Skepticism is required here. The only coding numbers circulating for Sonnet 5 come from Anthropic's own, internally measured agentic coding eval — no independent leaderboard confirmation exists. On that eval (SWE-Bench Pro, Anthropic's own vendor measurement), Anthropic reports: Sonnet 5 63.2%, Opus 4.8 69.2%, Sonnet 4.6 58.1%.[14]

Anthropic's own agentic coding eval — vendor-reported numbers, as of 2026-07-06

Why this needs a careful read: vendor-reported numbers aren't a neutral measure. Benchmark gaming (tuning a model specifically to the test tasks), training contamination (test data leaking into the training set), and weak real-world generalization are all known pitfalls — and an eval that the vendor itself defines and measures can't rule any of them out. Interestingly, Sonnet 5 scores below Opus 4.8 here: the "near Opus 4.8" claim refers to the overall picture across many tasks, not to this specific coding number.

The independent cross-check for Sonnet 5 itself is missing: as of 2026-07-06, SWE-Bench Verified still doesn't list "Claude Sonnet 5" as its own entry.[4] The 95% scores currently circulating belong to other models: on June 9, 2026, Anthropic announced the Mythos class — Claude Fable 5 (the generally available version) and Claude Mythos 5 (the same model with certain safeguards lifted in specific domains, initially available only to cyber defenders and infrastructure providers via "Project Glasswing").[12] For Fable 5, third-party trackers (vals.ai, llm-stats.com) report roughly 95% on SWE-Bench Verified — vals.ai lists Fable 5 at the top with 95.00%, ahead of Opus 4.8 at 88.60% (as of Jul 1, 2026).[13] That figure sits on an independent leaderboard, but is not officially confirmed by Anthropic as a benchmark number on its own announcement page (there, Anthropic points to Cognition's "FrontierCode" eval, not a SWE-Bench score). In short: Fable/Mythos 5 are real and strong, but the 95% SWE-Bench figure is tracker-reported, not primarily confirmed — and it doesn't concern Sonnet 5 anyway. For Sonnet 5, the bottom line stands: until it shows up on an independent leaderboard, the coding number is a vendor promise, not a measured fact.

SpaceX buys Cursor for $60 billion

On June 16, it was announced: SpaceX is acquiring Anysphere, Cursor's parent company, for $60 billion in an all-stock transaction — according to Forbes, "the largest acquisition of a venture-backed startup in history."[5] Closing is expected in Q3 2026, pending regulatory approval. An exclusive option had already been secured on April 21, backed by a break-up fee of roughly $10 billion.[6]

All-stock deal: Instead of paying $60 billion in cash, SpaceX gives Cursor's owners its own shares — swapping company equity for company equity. For the buyer, this preserves cash; for the sellers, the real value depends on what SpaceX shares are worth down the road.

Why SpaceX is buying a coding tool: the strategic core is xAI/Grok. Cursor's coding data is set to feed Grok's training pipeline, Cursor in turn gets access to xAI's "Colossus" supercluster, and both are meant to jointly train a model that lands in Cursor and in "Grok Build" — also serving as a talent backfill after xAI lost all eleven co-founders by the end of March.[7] These rationale points come from secondary sources; the hard deal facts ($60B, all-stock, June 16, Q3 closing) are consistent across Forbes and several independent aggregators.

What this means for users Nothing changes about Cursor itself today for Daniel. But a coding tool whose usage data will eventually train a competing model (Grok) is a governance signal: anyone working on sensitive codebases should keep an eye on privacy settings and enterprise commitments once the deal closes.

For context on how hot Cursor is right now: per multiple sources, its annualized revenue curve ran from roughly $100 million (Jan 2025) to over $4 billion (early summer 2026), with roughly 75% enterprise share — the exact week of the $4B milestone varies by source, but the monthly-level figure is consistent.[8]

Tool roll call: Copilot CLI and Codex catch up

Equal weight for every tool — here are this round's dated moves:

GitHub Copilot CLI shipped version 1.0.69-1 on Jul 4 (23:38 UTC): a new /mcp list command, an auto-allow-all mode for LLM-evaluated requests, and a stayInAutopilot setting.[9] This pushes Copilot's terminal agent further toward what Claude Code and Codex already do — MCP management directly from the CLI is the interesting bit. try Anyone already holding Copilot licenses can test the /mcp list flow against their existing MCP server setup.

MCP: Model Context Protocol — a unified "plug standard" that lets a coding agent talk to external tools and data sources (file system, database, API). Like USB for models: register a server once, and multiple tools can use it instead of building a custom integration for every app.

OpenAI Codex Remote reached general availability across all ChatGPT plans, along with a new DigitalOcean droplet workspace plugin and a security fix against trace-log leaks (which had let complete Responses WebSocket payloads get written into logs).[10] "Remote" means the agent runs in a hosted sandbox instead of locally — handy for long-running tasks, with the usual trade-off that the code leaves your own machine.
