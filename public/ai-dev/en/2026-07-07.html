Fable 5: still included today, metered from tomorrow

The only story with a real deadline today: Anthropic has redeployed Claude Fable 5 — and changed the billing at the same time. Through today, July 7 inclusive, Fable 5 is included in Pro, Max, Team, and select Enterprise plans, "for up to 50% of weekly usage limits."[1] That ends tomorrow: anyone who wants to keep using Fable 5 must enable usage credits. Without credits enabled, users simply lose access — no silent fallback to a cheaper model.[2]

The backstory explains why this is landing so abruptly: Fable 5 had been suspended in the meantime. The US government imposed export controls on June 12 requiring real-time nationality verification — something Anthropic says it couldn't reliably implement.[3] The current redeploy comes with additional safeguards, but also with the new metered logic.

Price — officially confirmed The credit price is $10/M input, $50/M output — twice as expensive as Opus 4.8 at standard rates ($5/$25). This figure is stated in Anthropic's own official API pricing documentation (platform.claude.com) — for Fable 5, exactly $10/$50 per million tokens, $5/$25 in batch mode.[4] So it's solid for team budgeting, not just "reported."

My take: This is less a price increase than a shift into visibility. Fable 5 was never "free" — it just counted against your weekly limit. Now the usage gets an explicit price tag, and the bigger effect is psychological: a model you used in flow because it was "already in the plan" suddenly has a meter running. For Daniel's setup, concretely: check whether your daily coding actually needs Fable 5, or whether Opus 4.8 (half the price, marginally below on most bench metrics) is enough. For a lot of sessions, it is.

OpenAI GPT-5.6 Sol — a security variant, but invite-only for now

OpenAI has announced GPT-5.6 in three variants (Sol, Terra, Luna). For Daniel, Sol is the interesting one: built for long-horizon security work — vuln research, secure code review, patch development, debugging, defensive testing.[5] That's exactly the category where an agent stays on a task for hours instead of just answering a question.

Stay skeptical OpenAI's selling point — that Sol is "competitive with Anthropic Mythos Preview" on cyber benchmarks, at only about 1/3 the output tokens — is a vendor-supplied, independently unconfirmed figure.[6] No independent leaderboard lists GPT-5.6 (consistent with its preview status). It's absent from the Artificial Analysis Coding Index accordingly — there, Fable 5 (76), GPT-5.5 xhigh (75), and Opus 4.8 (74) lead, as of 07/07/2026.[7]

And the catch: GPT-5.6 is only available in a limited-partner preview via API and Codex, with GA "in the coming weeks." So you can't try it yet — worth noting the GA date, but the token-efficiency claim can only really be judged once third parties can measure it.

SWE-Together: the benchmark that's honest about multi-turn

The most interesting bench move of the week isn't a new record — it's a new yardstick. Meta-affiliated researchers have released SWE-Together: 109 repository-level tasks distilled from 11,260 real user-agent sessions (~0.97% of which made it into the dataset). Public on GitHub, Apache-2.0 licensed.[8]

Multi-turn benchmark: The usual coding benchmarks give the model a fixed task and measure whether it solves it in one shot. SWE-Together instead replays real conversation flows — the (simulated) user refines the goal, adds conditions, corrects mistakes across multiple rounds. Like real pair programming, not like an exam question.

The punchline is in the contrast. On SWE-Together, Claude Opus 4.8 leads with 63% pass@1.[9] The same model reaches 88.6% on SWE-Bench Verified (the single-turn standard everyone quotes), as of 07/06/2026 — third place behind Mythos 5 (95.5%) and Fable 5 (95%).[10] That's a gap of about 26 points for the same model, purely because the tasks are interactive instead of one-shot.

My take: These 26 points are the most honest number of the week. They explain the feeling every agent user knows: the model that scores "88.6%" on a benchmark stalls on your real session by the third follow-up. SWE-Bench Verified measures the exam; SWE-Together measures the pair-programming. If you're comparing tools, the multi-turn number is the one closer to your actual workday. Caveat still stands: 109 tasks is a small, "high-precision" curated set, and the user is simulated, not real.

GitLab measures the "AI paradox" — across six countries, n=1,528

GitLab's 2026 AI Accountability Report (Harris Poll, fielded February 2026, published June 23; 1,528 developers/buyers from the US, Germany, UK, France, Japan, Australia) puts numbers behind a gut feeling many already had.[11]

78% say developers have been writing and committing code faster since adopting AI. But 79% also say individual productivity has risen while overall software delivery has not kept pace.[12] That's the "AI paradox": more code, not more shipped product.

Where the bottleneck moved 85% say AI has shifted the bottleneck to review and validation. 82% see risk of a new kind of technical debt, 84% name "governance after creation" the biggest challenge, and 43% can't reliably tell AI-written code from human-written code in their own repo.[13]

My take: These four supplementary figures come from an aggregator (InfoQ) summarizing the GitLab report — consistent in tone with the primary-sourced 78%/79%, but not separately verified against GitLab's own PDF. The core holds: the bottleneck is no longer typing, it's checking. That lines up exactly with SWE-Together above — producing code is solved, making code correct and accountable across multiple rounds is not. If you want to make your team faster in 2026, invest in review tooling, not more generation speed.
