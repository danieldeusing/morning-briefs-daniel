OpenAI GPT-5.6 — Sol, Terra, Luna

OpenAI rolled out the GPT-5.6 family on Thursday (07/09) as an "expanding preview": three sizes — Luna ($1/$6 per Mtok), Terra ($2.50/$15), and the flagship Sol ($5/$30). The headline: Sol sets a new record on Artificial Analysis's Coding Agent Index at 80.0 — 2.8 points above Claude Fable 5 — and, according to OpenAI, does so with less than half the output tokens and roughly a third of the cost. Terra (max) and Luna (max) land at 77 and 75 respectively.

The catch comes one line later: on SWE-bench Pro, the picture flips — Fable 5 hits ~80%, Sol only 64.6%. OpenAI's response is notably combative: it considers SWE-bench Pro itself unreliable, estimating that ~30% of the tasks are "broken." On the general Intelligence Index, Sol scores 59, one point behind Fable 5 (60) — but at about a third of the cost. GitHub Copilot rolled out Sol/Terra/Luna across all surfaces the very same day.

Coding Agent Index vs SWE-bench Pro: Two exams, two different questions. The Coding Agent Index measures how well a model performs as an autonomous coding agent across many tasks — weighted by cost, time, and token usage. Like grading a contractor not just on whether the house stands, but on how fast and cheap the build was. SWE-bench Pro, by contrast, is a fix-it exam: real GitHub bugs the model has to repair until the tests turn green — a pure hit rate, no cost credit. That's why the same model can lead one board and sit mid-pack on the other.

xAI/SpaceXAI Grok 4.5 — built for efficiency

A day earlier (07/08), Grok 4.5 arrived — according to xAI, the first model built specifically for coding and agentic work (Elon Musk calls it "opus-class"). Price: $2/$6 per Mtok. On the hard coding benchmarks it's solid but not a frontrunner: SWE-bench Pro 64.7% (Opus 4.8 at 69.2%, Fable 5 at 80.4%, GPT-5.5 at 58.6%), Terminal-Bench 2.1 at 83.3%. On the Intelligence Index it lands in fourth place, behind Fable 5, GPT-5.5, and Opus 4.8.

The real selling point is efficiency: for a SWE-bench Pro task, Grok 4.5 uses about ~15,954 output tokens on average, versus ~67,020 for Opus 4.8 (max) — a 4.2x difference. Tokens are the word-building-blocks a model outputs; fewer tokens for the same result simply means cheaper and faster. A side note on corporate structure: "SpaceXAI" is xAI, which moved under SpaceX after the IPO — X (Twitter) is now a SpaceXAI subsidiary.

Z.ai GLM-5.2 + ZCode — open weights that keep pace

For context upfront: not a new release this week — but the strongest openly licensed coder for self-hosters, which is why it's here. Z.ai actually released GLM-5.2 back in mid-June (model announcement 06/13, open weights + API on 06/16); the ZCode IDE built around the model followed on 07/02, and the wave of articles from 07/08 were retrospective/analysis pieces, not the release announcement. So it's roughly four weeks old — but the numbers still stand: 753 billion parameters, MIT license, $1.40/$4.40 per Mtok. (The 753-billion and MIT figures rest on a single source — VentureBeat; the second source only generically confirms "open-source.") On SWE-bench Pro, GLM-5.2 hits 62.1%, beating GPT-5.5 (58.6%) and its own predecessor GLM-5.1 (58.4%); on the long-horizon FrontierSWE, 74.4% vs GPT-5.5's 72.6%. On agentic tool use (MCP-Atlas 77.0) and Terminal-Bench (81.0), it stays just behind Opus 4.8 (77.8 / 85.0).

Alongside it comes ZCode, a free "Agentic Development Environment" built around the model — 1 million-token context, plus the quirk of remote-controlling running coding agents via WeChat/Feishu/Telegram. Try-worthy Worth trying if you want an openly-licensed coder near the frontier and value self-hosting or low cost.

Benchmarks: who "leads" depends on the board

This week is a lesson in benchmark skepticism. Take SWE-bench Pro — the resolve rate on real GitHub issues, vendor-reported (chart below). Claude Fable 5 dominates at ~80%, clearly ahead of Opus 4.8 (69.2). This week's three newcomers, plus the openly licensed GLM-5.2 (a mid-June release), all crowd into a tight 61–65% band: Grok 4.5 (64.7), GPT-5.6 Sol (64.6), GLM-5.2 (62.1), Muse Spark 1.1 (61.5), with GPT-5.5 (58.6) below that.

Two caveats. First, nearly all of these are vendor-reported figures — take with caution. The cross-check is at least somewhat reassuring: the "rival" numbers (GPT-5.5 at 58.6, Opus 4.8 at 69.2) show up identically across OpenAI, xAI, and Meta, so the board is at least roughly comparable. Second, the winner's rank flips depending on the metric: on the cost-/token-weighted Coding Agent Index, it's GPT-5.6 Sol that leads with 80.0 — the very same model that's only fourth on SWE-bench Pro. And OpenAI disputes SWE-bench Pro outright (~30% broken tasks). Translation: a single leaderboard spot doesn't say much these days — what matters is which task (autonomous agent vs. precise bug fix) and which cost bracket is relevant to you.

Google Gemini Managed Agents — built for production

On 07/07, Google expanded the Gemini API's Managed Agents — four building blocks toward production readiness: background tasks (background: true → the API immediately returns a poll ID while the task keeps running server-side), connections to remote MCP servers, custom function calling, and mid-session credential refresh (renewing short-lived keys via environment_id).

MCP (Model Context Protocol): an open standard through which an AI agent talks to external tools and data sources — file system, database, API. Like a universal USB plug for agent peripherals: standardize once, every tool fits. "Remote MCP" then means the agent is allowed to connect to MCP servers running elsewhere, not just locally on the same machine.

PyTorch 2.13 — FlexAttention on Apple Silicon

For anyone running local models on a Mac, this is the week's most practical release (07/08): FlexAttention now runs on Apple Silicon (MPS). On sparse attention patterns, PyTorch measures ~35 ms versus ~431 ms for the standard SDPA path — roughly a 12.3x speedup on a 32k-long sequence with a 256-element sliding window; a smaller 8k case brings ~4.15x. On dense patterns, SDPA still wins — no universal gain. There's also a deterministic CUDA backward path with <1% overhead (e.g., +0.2% at S=32768), making reproducible training essentially free.

Sparse attention / FlexAttention: "Attention" is how a model decides which parts of the text matter right now — normally every token gets compared against every other, which gets expensive for long contexts. "Sparse" means comparing only the truly relevant pairs (say, a sliding window of neighboring words). FlexAttention is PyTorch's way of computing such sparse patterns efficiently on the hardware — now on Apple's chip too.

Try-worthy Worth trying if you run long contexts locally on Apple Silicon — the speedup kicks in exactly for long-context inference.

It's Monday, July thirteenth. Last week was a genuine release avalanche for anyone coding with AI: three labs — OpenAI, xAI, and Meta — each shipped a new model tuned for coding within a few days of each other. And the real story isn't who won — it's that the vendors are now openly fighting over the benchmarks themselves.

On Thursday, OpenAI's GPT five point six arrived in three sizes: Luna, Terra, and the flagship Sol. Sol tops one of the major coding leaderboards with a score of eighty, while working noticeably more efficiently than the competition. But on a tougher test called SWE-bench Pro, the picture flips: Claude Fable five leads clearly there at around eighty percent, with Sol at only about sixty-five. OpenAI counters that this test is broken, claiming roughly thirty percent of its tasks are faulty.

A day earlier, xAI — now under the SpaceX umbrella — released its Grok four point five model, the company's first built specifically for coding. On raw scores it's solid but not a leader. Its real selling point is efficiency: for the same task, it uses roughly four times fewer output tokens than Anthropic's top model, and fewer tokens simply means cheaper and faster.

For self-hosters, the most interesting openly licensed coder remains GLM five point two from Z.ai — though it's not a new release this week, it actually came out back in mid-June; it has open weights under a free license, it's affordable, and on SWE-bench Pro it's actually ahead of GPT five point five. It also comes with a free development environment called ZCode. And Meta launched a paid model interface for the very first time: Muse Spark one point one trails Anthropic's Opus on pure coding, but shines at orchestrating tools for agents.

The week's key takeaway: almost all of these numbers are self-reported by the vendors. The cross-check is somewhat reassuring, since the rivals' figures look consistent across different providers. But which model leads flips depending on the leaderboard. A single leaderboard rank doesn't say much these days — it comes down to whether you need an autonomous agent or a precise bug fix, and what you're willing to pay for it.

Beyond the models: PyTorch two point thirteen brings a faster attention variant to Apple's chip — up to twelve times faster on certain patterns, useful for long contexts on the Mac. Google made its Gemini agents more production-ready, with background tasks and remote tools. And at Claude Code, this week's real news is a built-in browser, a diagnostics command, and a security boost that catches an accidental delete command — not the old computer-use story that's wrongly making the rounds again.

In the toolbox: Cursor brings side chats, Copilot already has GPT five point six integrated, and JetBrains is bringing in two agents at once. And one warning worth remembering: Continue.dev has been discontinued, and its cloud data gets deleted the day after tomorrow — if you still have anything stored there, export it today.

Tomorrow's news day will likely be quieter, as the model wave gets digested. What's worth watching: whether independent leaderboards confirm the vendors' numbers — and whether OpenAI's attack on SWE-bench Pro sparks a real debate about broken benchmarks.
