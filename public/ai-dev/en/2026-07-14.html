AgentPrizm — Audited Memory and a Skill Marketplace for Agents

This week's freshest move isn't a model, it's infrastructure. AgentPrizm publicly launched two products on 07/09/2026: AgentMemory and AgentSkills. Both target something coding agents structurally lack — memory across sessions and shared, versioned capabilities.

Agent memory: A coding agent starts every session from zero — what you agreed on last week, which convention applies in the repo, which bug already came up three times: all gone the moment the context window fills up or you close the chat. Picture an intern whose notebook gets wiped every night after work. AgentMemory is an external notepad that multiple agents share — except this one assigns a confidence score to every note and logs who filed what, when.

"Governed, confidence-scored" means concretely: every memory carries a confidence score and an audit entry, instead of the agent blindly replaying some old scrap of a note. AgentSkills is a versioned SKILL.md marketplace — the same kind of skill files Daniel already writes for Claude Code, but shareable, versioned, and pullable from a catalog. Both connect to Claude Code, Cursor, and Claude Desktop via REST and MCP — Anthropic's open standard through which an agent reaches tools and data sources, a kind of universal plug for agent tooling.

Why this matters for Daniel: the SKILL.md workflow is already lived practice for him, but so far local and per-project. Claude Code can already remember a bit and load skills project-locally, but only per project and per machine; AgentPrizm pulls memory and the skill catalog out of the repo and makes them shareable across tools. That's exactly the missing piece for reusing skills across repos without copying them. Try it — the free tier is enough to test it: 10,000 memories + 4,500 retrievals/month, AgentSkills is free on every plan.

Skepticism The launch came via PR Newswire, the vendor is young, and "audited/confidence-scored" stay marketing words until retrieval quality holds up in a real repo. The real question isn't the feature list, it's whether the memory pulls back relevant notes instead of noise — and the lock-in: the team's memory then lives in a startup's cloud. That it connects via MCP instead of proprietary at least takes the edge off that.

GPT-5.6 in the Benchmark Reality Check — One Week Later

The launch of GPT-5.6 (Sol/Terra/Luna) on 07/09 has been covered extensively in the last few days' briefs — including pricing and the 07/13 "which board leads" dispute. Here's just where the numbers stand now, and why the "No. 1" headline is misleading.

SWE-Bench Pro: a collection of real, open GitHub issues from large repos. The model has to deliver a patch that makes the project's tests pass — not answer a quiz. Closer to day-to-day job reality than HumanEval & co., which is why it's the harder test — and why it's where marketing scores part ways from real capability.

On the Artificial Analysis Coding Agent Index (which blends DeepSWE, Terminal-Bench v2, and SWE-Atlas-QnA), Sol (max) leads with 80 and, per the vendor, costs ~40% less per task than Claude Fable 5 (max) and ~10% less than Opus 4.8 (max) in Claude Code.[1] This very index is behind the "No. 1" headline. The catch: on SWE-Bench Pro, Sol lands at 64.6% — 15.7 points behind Claude Fable 5 (80.3%).[2][3] Terra (63.4) and Luna (62.7) sort in just below, barely ahead of predecessor GPT-5.5 (59.4).

SWE-Bench Pro — Coding Models, as of 07/10/2026

The punchline is the self-relativization: OpenAI itself calls ~30% of SWE-Bench Pro tasks "broken" and has published that openly.[2] Convenient when you're losing on that board right now — but it's also a genuinely known problem with the benchmark. And the rank keeps flipping depending on the board: on the independent Terminal-Bench 2.1, Sol Ultra leads at 91.9%, while Claude comes out on top elsewhere.[4] Even within one benchmark, the trackers disagree: on LiveCodeBench, Artificial Analysis has Gemini 3 Pro Preview ahead at 91.7%, while VALS AI has Claude Fable 5 ahead at 89.78% — same benchmark, different cutoff dates and methods, two trackers, two different winners.[8] Almost all of these scores are vendor-reported; an official SWE-Bench Verified figure for GPT-5.6 couldn't be confirmed (the board loads via JavaScript, and secondary sources say OpenAI hasn't published one). The takeaway stays the same as on 07/13: GPT-5.6 is a solid, affordable coder — it only wears the crown on the board its vendor shows first.

Google Antigravity 2.0 — the Multi-Agent Command Center (Follow-Up)

An honestly dated follow-up: Antigravity 2.0 isn't from today — Google showed it at I/O on 05/19/2026. It never came up in recent briefs, though, and it's the one big agent-IDE move that deserves a proper look.

Multi-agent command center: A "main agent" works like a technical project lead. Give it a complex task, and it breaks the task down itself and clones itself into several subagents that work on subtasks in parallel. Instead of chatting with one bot, you're directing a small team that distributes itself.

Concretely, Antigravity 2.0 supports full multi-repo workspaces and cron-like scheduled tasks that run in the background.[1] Pricing-wise, the expanded feature set hangs on a new $100 "AI Ultra" tier, which gives five times AI Pro's Antigravity usage limits.[2]

Relevant for tool users Part of the overhaul: the open Gemini CLI is being discontinued (06/18/2026) and folded into a closed-source Antigravity CLI — independently confirmed alongside Google's own "CLI consolidation." Anyone scripting against the open Gemini CLI should plan for the switch; an open toolchain shutting down is exactly the kind of move this brief flags.

Take: a command-center bet — heavy, Google-locked, and the step into closed source runs against the open-tool ethos. Watch, don't necessarily switch.

Short & Linked — What's Still Running

ARD vs. MCP. The enterprise standard push Agentic Resource Discovery (ARD) — backed by Google, Microsoft, Salesforce plus Cisco, Databricks, GitHub, GoDaddy, Hugging Face, NVIDIA, ServiceNow, and Snowflake, announced 06/17–18 — is being sold as "complementary" to MCP/A2A, but neither OpenAI nor Anthropic is on the backer list. That reads as a competing enterprise discovery layer to Anthropic's MCP (Nov 2024, donated to the Linux Foundation, 97M+ SDK downloads/month as of Dec 2025). Google and Microsoft confirm the backer list and MCP relationship in their own developer blogs; the full story ran in the 07/10 brief.[5]

Ona/Gitpod. OpenAI acquires Ona (ex-Gitpod, Kiel) — announced 06/11 — and folds persistent cloud sandbox execution into Codex, so agents keep working when the laptop is closed. Codex now hits over 5M weekly users, +400% since the start of the year; terms undisclosed, deal still under antitrust review. Already covered on 07/10/07/13.[6]
