Continue Merges Into Cursor — the Cloud Dies Today [1][2]

Continue.dev, the open-source coding agent for VS Code, CLI, and JetBrains, has been acquired by Cursor (Anysphere) — announced around 06/16/2026, with no press release, just a line on the homepage ("Continue has been acquired by Cursor"). The last release, v2.0.0-vscode, shipped on GitHub on 06/19.[1]

The part that matters for Daniel is dated and today: the hosted platform is being wound down, and the cloud data stored there — chat history, saved configs, team settings — will be deleted after 07/15/2026.[2] Anyone who used Continue only locally in the IDE isn't affected; anyone who ran the hosted platform with team settings should export today.

Act today Using Continue's hosted platform? Export configs and team settings now — after today they're gone. The local, open-source Continue remains on GitHub (last release v2.0.0-vscode, 06/19).

Context: this is the second major consolidation in agent tooling within a few weeks — the buyer pool around Cursor is absorbing open projects rather than letting them run as competition. For the tool landscape that means less diversity among hosted agents; the local, self-hosted camp (Cline, Aider, OpenHands) gains relative weight as a result.

GPT-5.6 vs. Muse Spark vs. Fable 5 — who can actually code? [3][4][5]

Two model families arrived on the same day, 07/09 — but at different levels of maturity. OpenAI's GPT-5.6 became generally available (three tiers — Sol, Terra, Luna — after a limited preview since 06/26). Meta's Muse Spark 1.1, an agentic model with a 1-million-token context that speaks MCP and runs its own skills, is by contrast only in public preview: the Meta Model API currently runs only in the US (not yet in the EU), and the only thing generally live so far is the consumer "Thinking Mode" in the Meta AI app.[3][4][5]

MCP / Tool Use: Model Context Protocol is a unified "plug" that lets an AI agent talk to external tools — file system, database, API. Like USB-C for tools: implement it once, use it everywhere. "Tool use" means the model calls these tools itself instead of just producing text.

The marketing numbers: GPT-5.6 Sol leads the Artificial Analysis Coding Agent Index with 80 points and reports 88.8% on Terminal-Bench 2.1 and 72.7% on DeepSWE v1.1.[3] Muse Spark 1.1, in turn, leads on pure tool use (MCP Atlas 88.1), ahead of Opus 4.8 and GPT-5.5.[5]

The skepticism: On SWE-Bench Pro — the tougher test that sits closer to real work — the picture flips. There, Anthropic's Claude Fable 5 beats GPT-5.6 Sol clearly, at roughly 80% vs. 64.6%, and Muse Spark 1.1 (61.5) trails Opus 4.8 (69.2). On DeepSWE 1.1 and Terminal-Bench 2.1, Muse Spark likewise lands behind GPT-5.5 and Opus 4.8.[4][5] Notably: OpenAI has published no SWE-Bench Verified score for GPT-5.6 at all — and responded to the Pro loss with a public critique that "~30% of the SWE-Bench Pro tasks" are broken.[4]

SWE-Bench Pro vs. Verified: SWE-Bench measures how many real GitHub bug fixes a model can resolve on its own — every task has a test that proves whether the fix holds. "Verified" is the human-checked, clean 500-task selection (hence the high scores); "Pro" is a larger, tougher set — which is why scores are lower there and models spread further apart.

What Daniel takes away: Vendor numbers aren't a ranking, they're a marketing cherry-pick — everyone leads exactly the board they choose to show. For daily coding, the index win matters less than the Pro/Verified reality, and this week that favors the Claude tier. Important caveat on sourcing: OpenAI's own GPT-5.6 pages were unreachable (403); the GA/pricing/bench numbers come from two matching secondary sources (marktechpost, simonwillison.net), not from OpenAI directly.

Prices (per 1M tokens): GPT-5.6 Sol $5/$30, Terra $2.50/$15, Luna $1/$6.[3] Muse Spark 1.1, per secondary reports, $1.25/$4.25 (plus $20 starting credit) — not primarily confirmed.

Claude Code: Built-In Browser + a Faster CLI [7][8]

Claude Code picked up a built-in, tabbed sandbox browser in the 06.–10.07. release window (v2.1.202–206), opened with Cmd+Shift+B (macOS) or Ctrl+Shift+B (Windows). Safety classifiers check every write action on third-party sites; purchases or account creation require explicit approval.[7] Try For "read this doc / check this web app" loops, this saves the detour through an external browser.

Sandbox browser: A sandbox is a fenced-off playground. The browser runs in an isolated area where the agent can click and type without touching your real system or your real accounts.

The CLI itself is at v2.1.210 (07/14) — three releases (v2.1.208–210) shipped that day as one big bugfix/performance bundle: a screen-reader mode (plain-text rendering), up to 7x faster tool rounds with many MCP tools, and up to 79x smaller session transcript size in edit-heavy sessions.[8] There's no pure coding-feature leap here — it's maintenance, but noticeable maintenance.

GitHub Copilot — Three Small Steps [9][10]

Copilot is rolling out in small steps. 07/01: agentic browser tools in VS Code reach GA — on by default, the agent can navigate pages, inspect content, take screenshots, and check web apps directly in the IDE. The model provider picker also now launches straight from VS Code (VS Code June releases, 07/08).[9] 07/14: the Copilot app's /security-review command is in public preview, and the JetBrains integration expands its BYOK capabilities.[10]

BYOK: "Bring Your Own Key" — you plug your own API key from a model provider into the tool instead of using the tool maker's quota. Like bringing your own streaming account instead of borrowing someone else's.

The convergence is notable: Copilot's browser tools (07/01) and Claude Code's sandbox browser (week 28) land in the same window — "the agent drives a browser itself" is becoming a standard capability rather than a differentiator.

But: there's no confirmed release date this week; the entry looks older, so explicitly no freshness claim here.

MoE (Mixture-of-Experts): The model has 80B parameters but activates only 3B of them per token. Like a big law firm where, for each question, only the two relevant lawyers answer instead of the whole firm — the performance of a large model at the compute cost of a small one.
