Kimi K3: an open model at the top of the arena

Moonshot AI released Kimi K3 via app and API on 07/16 — a model with 2.8 trillion parameters and a 1-million-token context, described by its makers as their "first open 3-trillion model." The eye-catcher is the ranking: in the Arena.ai Frontend Code Arena, K3 sits at #1 with 1,679 Elo, ahead of Claude Fable 5 (1,631) and GPT-5.6 Sol (1,618). Across the arena's seven frontend domains it leads in six; only in "Gaming" does Fable 5 come out ahead. For scale: the predecessor Kimi K2.6 sat at rank 18 (1,515) on the same board.

winzheng.com · tomshardware.com · simonwillison.net · as of 07/22/2026
Arena Elo & MoE: An "arena" doesn't measure with test questions but with crowd voting: two models tackle the same task, a human picks the better output — like a blind-tasting tournament that produces a chess-style Elo score. That tells you which output people like better, not whether the code actually runs in a real repo. "MoE" (Mixture of Experts) is the trick behind the 2.8 trillion parameters: instead of one giant brain, there are many specialists, and a router only wakes up a handful per token — huge on disk, but affordable on the bill. The catch: all the parameters still have to fit in memory. This stays a server model, not a laptop model.

Frontend winner, agentic middle-of-the-pack

Two caveats before you crown K3 the new king. First: the weights aren't out yet. Moonshot promises the open-weights drop for 07/27 — until then, "open" is an announcement, not something you can host yourself; right now it's only usable via the Kimi API and the Kimi Code CLI. Second, and more important: the arena measures frontend by taste. On the Coding Agent Index from Artificial Analysis — which measures real multi-step agent work (reading files, editing, running tests) — K3 lands at rank 5 with 57 points, right in the middle of the closed top tier: GPT-5.6 Sol 61, Fable 5 59, Grok 4.5 58, K3 57, Opus 4.8 55. On Terminal-Bench v2 it scores 84%.

That's the honest read: ahead in the frontend-heavy taste arena, level with — not ahead of — the closed models in hard agentic work. try If you're keeping an eye on an open model for coding, K3 becomes the candidate to test yourself starting 07/27; until then, an API trial run against your own frontend repo is worth more than trusting the arena rank.

Why no chart The three arena scores sit close together at 1,679/1,631/1,618 — a zero-based bar chart would flatten the roughly 60-Elo-point gap into three nearly identical bars, hiding more than it shows. That's why the numbers stay in the text, where the gap remains readable.

deepseekv4pro.com · api-docs.deepseek.com · as of 07/22/2026
Peak/off-peak pricing: Like off-peak electricity rates, for the API. Instead of one flat price per million tokens around the clock, requests cost more during peak hours and less outside them — the provider smooths out its server load that way. For you, that means: batch jobs that aren't in a hurry can be scheduled into the cheap windows, roughly halving the output price.

The reported prices

A time-window pricing model like this is expected to arrive with the release. Reported figures for V4-Pro are around $0.87 per million output tokens off-peak and $1.74 at peak; for V4-Flash, $0.28 and $0.56. The report names 09:00–12:00 and 14:00–18:00 as the peak window. Two caveats: these numbers come from a prep source, not a DeepSeek price list, and they only apply once GA is actually live. Until then, they're ballpark figures, not something to budget against.

MCP goes stateless — the biggest overhaul since launch

The final MCP specification ships on 07/28 — the release candidate has already been publicly available since 05/21, with beta SDKs for Python/TS/Go/C#. According to the project, it's "the biggest revision of the protocol since launch." The core overhaul: the protocol becomes stateless — the previous initialization handshake and the Mcp-Session-Id go away. New additions are MCP Apps (servers can ship their own UI) and a Tasks extension for longer-running jobs. The Roots, Sampling, and Logging capabilities are being dropped. So the RC and beta SDKs are already out — 07/28 is the ship date for the final version (the spec's dated version stamp), not the RC date.

blog.modelcontextprotocol.io · as of 07/22/2026
Stateless & handshake: A "handshake" is the introduction round at the start of a connection — client and server agree on who can do what, and keep that in mind for the session. "Stateless" means there's no such memorized session anymore. Think of the difference between a phone call, where both sides stay on the line the whole time, and postcards, each of which is complete on its own. The upside: stateless servers scale more easily and survive restarts, because there's no session memory to lose. The cost: anyone running MCP servers today will need to make adjustments when they switch over.

Why this matters to you: MCP now sits underneath nearly every agent tool you use — it's the standard plug that Claude Code and friends use to reach external tools and data sources. An overhaul this large means self-built or hosted servers will sooner or later need to catch up, and that the capabilities being dropped (like Sampling) can break existing integrations. No urgency today — but it's a date worth noting for the end of the week.
