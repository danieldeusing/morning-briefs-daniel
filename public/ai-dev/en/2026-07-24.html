GPT-5.6 Multi-agent: the model delegates to itself

The actually interesting part of the GPT-5.6 family (Luna/Terra/Sol, July 9) isn't the pricing for us — that was already set last week. It's two new dials. First, an additional reasoning tier called max — the ladder now runs none · low · medium · high · xhigh · max, with max giving Sol the most time to think. Second, a beta mode called Multi-agent (unlocked via the beta flag responses_multi_agent=v1) that goes beyond what a single agent can do: it launches multiple subagents concurrently and merges their work into one request. OpenAI explicitly markets this for multi-step research and long coding projects — examples cited include scanning separate parts of a large codebase and writing independent test suites in parallel.

developers.openai.com · simonwillison.net · As of 2026-07-24
Subagents / multi-agent: Instead of one model chewing through an entire task solo from start to finish, it splits the task up and sends out several copies of itself in parallel — one reads the tests, one searches the repo, one writes the patch — then stitches the results back together at the end. Think of a foreman sending three people out at once instead of working through everything himself in sequence. The payoff is speed on broad tasks; the cost is that the pieces can't see each other, so the foreman has to catch any contradictions. With "Multi-agent," this all happens inside a single request — you don't orchestrate the subagents yourself.

The hard numbers for context: a 1.05M-token context window, a maximum of 128,000 output tokens, knowledge cutoff February 16, 2026 — both confirmed by the developer docs and an independent source that match exactly. Dev pricing is unchanged from last week: Sol $5/$30, Terra $2.50/$15, Luna $1/$6 per million tokens. For agent work, context is the real lever — 1M tokens means a mid-sized repo fits in whole, with no chunked reloading.

developers.openai.com · simonwillison.net · As of 2026-07-24
Stay skeptical OpenAI is touting a record score of 53.6 for Sol on "Agents' Last Exam," reportedly 13.1 points above Claude Fable 5. First, that's OpenAI's own benchmark; second, it's not a coding-specific measurement — it says little about how Sol handles your ticket. Treat it as a marketing number, not a basis for comparison. The Multi-agent mode is also still beta (and only reachable behind the responses_multi_agent=v1 flag): it's worth a try on a real multi-hour refactor, but not yet a default setting for everyday work.

Muse Spark 1.1: open API, silent on pricing and benchmarks

Meta's Meta Model API has been in public preview since July 9, and Muse Spark 1.1 is the first model behind it: 1M context. Meta doesn't publish a price list — no token price appears in the announcement blog or in trade coverage. The only thing circulating is a ratio: according to a Vals AI post cited by TestingCatalog, Muse Spark is "10x cheaper and twice as fast" as Claude Fable 5 — a third-party claim, not a confirmed pricing figure. What's interesting for us is the early partners — Replit, Cline, and Box are on the list, meaning two coding environments right away. Anyone working in Cline gets Muse Spark as a selectable model with no detour.

ai.meta.com · testingcatalog.com · As of 2026-07-24

Meta itself is cagey about the coding numbers, and that's exactly where it pays to look closely. On SWE-Bench Pro, Muse Spark 1.1 scores 61.5 — behind Claude Opus 4.8 (69.2, leading) and just above GPT-5.5 (58.6). On Terminal-Bench 2.1 it's 80.0, where GPT-5.5 leads with 83.4 ahead of Opus 4.8 (82.7). Both figures come from Meta's own evaluation report, passed along via an aggregator — no independent re-measurement. Read as self-reported numbers, that says: solid mid-tier, not a frontrunner.

kingy.ai · As of 2026-07-24
A number you shouldn't cite Aggregator coverage of Muse Spark 1.1 has a "77.4% on SWE-Bench Verified" figure circulating. That's mislabeled: the 77.44 is Muse Spark's overall composite score on BenchLM's own weighted leaderboard (rank 6 of 200), not a SWE-Bench Verified percentage. An independently published SWE-Bench Verified score for Muse Spark simply doesn't exist at launch. If you run into this number anywhere — it's a textbook case of a figure that looks clean but is fabricated.

Benchmark check: where the coding frontier really stands

With two new coding models going live this week, it's worth checking the independent board — and it's soberingly clear-cut. SWE-Bench Verified (as of July 23) is completely dominated by Anthropic: Claude Mythos 5 95.5%, Claude Fable 5 95%, Claude Opus 4.8 88.6%. Neither GPT-5.6 nor Muse Spark 1.1 is even listed. That's the real story of the board: this week's launches haven't been independently re-measured yet.

benchlm.ai · As of 2026-07-23
SWE-Bench Verified: A human-filtered set of 500 real GitHub issues from open-source Python projects. The model gets the repo and the bug report, writes a patch — and it only counts if the project's tests pass afterward. Not multiple choice, but "does the code run or not." That makes it the most honest of the common coding benchmarks. The catch: models can be trained specifically toward these exact 500 cases, and a vendor reporting its own run is grading its own homework.

Fable 5 for coding — the context behind second place

Why is Anthropic so far ahead on coding? Fable 5 and Mythos 5 have been generally available since June 9 (after a brief suspension June 12–July 1), Fable 5 at $10/$50 per million tokens. Anthropic claims for Fable 5, on the FrontierCode evaluation from Cognition (the company behind Devin and Windsurf), that it "score highest among frontier models" — so the assessment comes from a third party, but it's circulated as vendor phrasing with no published percentage and no independently viewable FrontierCode board. The often-cited evidence is a Stripe customer quote: a 50-million-line Ruby migration in one day instead of two months by hand. Impressive — but a single customer anecdote relayed by Anthropic, not reproduced.

Tool wave: routers, cloud agents, MCP

Fitting the model theme, the tooling layer also delivered on orchestration this week. Cursor rolled out the Cursor Router (Auto-Mode) on July 22: it "analyzes every request and sends it to the right model," with three tiers — Intelligence (maximum quality), Balance (recommended for everyday use), and Cost (token-frugal). For Team plans, this is the new default.

cursor.com · As of 2026-07-24
Model router: A front gate that glances at every request and decides on its own which model answers it — the expensive, powerful one for the tricky refactor, the cheap, fast one for "rename this variable." Like ER triage: not every patient needs the attending physician. The payoff is that you stop paying for the most expensive model on trivial stuff across the board; the risk is that the triage misjudges and hands a hard task to the weak model. That's why there are three tiers — you tell the router how aggressively it's allowed to save.

Copilot wires agents into the tracker

GitHub Copilot made two agent building blocks generally available on July 23: the cloud agent for Linear is GA, and the GitHub MCP server now supports the next MCP specification. On top of that, agent automation for GitHub Issues is in public preview. In practice this means a ticket in Linear or a GitHub issue can trigger an agent that handles the work in the background — the model capability of "working alone for a long stretch" meets the wiring of "gets triggered by the tracker."

github.blog · As of 2026-07-24
MCP (Model Context Protocol): A unified plug standard that an AI agent uses to talk to external tools and data sources — filesystem, database, issue tracker, browser. Instead of writing a bespoke integration for every tool, everything speaks the same language, like USB-C for peripherals. GitHub's MCP server now supporting the "next" spec mainly means: the connectors you build for your agents stay compatible for longer.

Across the rest of the field: JetBrains bundles AI Assistant and its coding agent Junie under one subscription — with a free tier (unlimited code completion, access to local models); Pro/Ultimate raise the quotas. Windsurf has fully rebranded to "Devin Desktop"; the old Cascade engine was sunset on July 1 and replaced by Devin Local (claimed to be roughly ~30% more token-efficient, with parallel subagents — a vendor claim, not independently measured). And Copilot's new model picker has also carried Gemini 3.6 Flash since July 21; Kimi K2.7 has been selectable there since July 7 — its own launch belongs in the ai-usecases brief.

Claude Code — what changed under the hood

Same standard for everyone: Anthropic's own agent also got some dev-relevant small changes. Auto-Mode arrived on Bedrock, Vertex AI, and Foundry with no opt-in flag needed (July 11); turn it off via disableAutoMode. New are session-wide caps against runaway loops — 200 WebSearch calls and 200 subagent launches per session each, adjustable via environment variables. And a security fix from July 15: permission previews mirrored into chat channels now neutralize character tricks (bidirectional control characters, zero-width characters, look-alike quotation marks) that could visually forge an approval message.
