Claude Code 2.1.149 · desglose de uso y tres correcciones de seguridad [1]

En un día en que la mayoría de los proveedores está de fin de semana, Claude Code 2.1.149 (22/05, 22:09 UTC) es el único release estable sustancial — y tiene dos caras: una función simpática y un bloque inusualmente denso de correcciones de seguridad.[1]

La función visible: /usage con desglose

/usage ahora muestra un desglose por categoría — cuánto del presupuesto de tokens va a skills, subagents, plugins y cada servidor MCP individual. Para quien corre varios servidores MCP conectados, es la primera forma de ver qué servidor cuesta cuánto, en vez de leer solo una suma total. Además: la vista de detalle de /diff ahora tiene scroll por teclado (flechas, j/k, PgUp/PgDn), Markdown renderiza checkboxes de task-list GFM, y enterprise obtiene un ajuste gestionado allowAllClaudeAiMcps.

La sustancia de verdad: las correcciones de seguridad

Más pesadas que la función son tres correcciones que tocan directo el setup de Daniel. Dos bypasses de permisos en PowerShell se cerraron: comandos integrados como cd.. o cd\ podían escapar del directorio de trabajo autorizado, y las reglas de allow con prefijo/comodín podían rodearse. Más un fix de sandbox de worktree (la allowlist de escritura no se aplicaba bien a los worktrees de git) y — el más desagradable — un bug de find que agotó la tabla de vnodes de macOS y tiró la máquina entera.[1]

Tabla de vnodes: macOS lleva internamente una lista de todos los archivos y carpetas que están "tocados" en el momento — como un guardarropa con un número limitado de ganchos. Un find fuera de control colgó tantos abrigos que el sistema se quedó sin ningún gancho libre y se congeló. No solo se cayó la herramienta — se cayó el host.

Contexto: los bypasses de permisos en un agente que ejecuta comandos de shell no son cosmética — son justo la clase de bug en la que una promesa de sandbox empieza a tener fugas en silencio. En lo práctico, para Daniel: actualizar directo al 2.1.149 (o 150), sobre todo quien corre Claude Code con servidores MCP conectados o en Windows/PowerShell.
