Objetivo de lanzamiento del candidato de versión de especificación MCP (PR #2750 fusionada)

Antigravity 2.0 · una semana después del bang [1][2][3]

Hace nueve días, Google presentó su herramienta de escritorio agent-first en la versión 2.0 en I/O — aplicación independiente para macOS/Linux/Windows, sub-agentes paralelos, tareas de fondo programadas, una CLI agy, un SDK, integración con AI Studio/Android/Firebase. La arquitectura, ese es el consenso de las reseñas independientes de los últimos días, es real: no obtienes nada como esto hoy de Claude Code o Cursor.[1]

La entrega fue un problema. Google automáticamente aplicó parches a instalaciones existentes de v1 en el día del lanzamiento — removiendo el editor incorporado, moviendo configuraciones a una nueva ruta (= perdido), eliminando el control de fuente y las conexiones remotas de la imagen predeterminada. Cualquiera que ejecutara Antigravity en CI/CD encontró una interfaz de chat en lugar de un IDE el miércoles. El ecosistema se fragmentó ese día en tres descargas separadas: Antigravity 2.0 (shell de agentes), Antigravity IDE (codificación tradicional), y Antigravity CLI. Quien quisiera el editor de vuelta tenía que instalar la IDE manualmente.[2]

Nivel gratuito y brecha en la CLI

Dos números que contribuyeron al ambiente: el nivel gratuito bajó de 250 a 20 solicitudes/día — muy poco para evaluar seriamente la herramienta sin una cuenta paga. Y la CLI agy, que reemplaza la Gemini CLI el 18.06., no estaba disponible a través de un gestor de paquetes público hasta el 21.05. Eso creó presión de migración pura en tuberías CI que dependían de npm i -g @google/gemini-cli, sin sistema objetivo a la vista.[2]

Actualización desde ayer: El instalador oficial ahora está disponible. curl -fsSL https://antigravity.google/cli/install.sh | bash coloca el binario nativo agy en ~/.local/bin (Unix) o %LOCALAPPDATA%\Antigravity\ (Windows). Sin paquete npm — Google explícitamente se apartó del flujo de trabajo Node.[3]

Para Daniel: no instales hoy

Antigravity 2.0 no es un reemplazo directo para Claude Code; "revisa en 60–90 días" (el consejo honesto de la reseña) es el veredicto. Lo que vale tu tiempo hoy del mundo de Antigravity es el servidor MCP integrado en él — Chrome DevTools for agents, utilizable sin Antigravity (siguiente sección).

Chrome DevTools for agents · estándar silencioso [4][5]

Mientras Antigravity acapara titulares, algo más sustancial se ejecuta en segundo plano: Chrome DevTools for agents 1.0 se estabilizó el 19.05., el paquete npm chrome-devtools-mcp estaba en 1.1.1 ayer (etiqueta de lanzamiento 27.05.). Esto le da a tu agente de codificación verdaderos ojos de navegador: registros de consola, solicitudes de red, trazas de rendimiento, snapshots de memoria, auditorías lighthouse, emulación de dispositivos — todo a través de MCP en lugar de un sandbox de captura de pantalla.[4]

Chrome DevTools for agents (vía MCP): Imagina a tu agente de IA sentado ciego frente a la pantalla, escribiendo código de frontend — solía tener que adivinar si la página ni siquiera cargaba. Con este servidor MCP, obtiene la misma herramienta de inspección que abres en Chrome vía F12: ve qué solicitudes fallan, qué imagen tarda 3 segundos, qué campos de entrada carecen de etiqueta de lector de pantalla. En lugar de "creo que el error está aquí" → "Lighthouse dice Z, arregla Y."

La lista de clientes es la sorpresa real: preinstalado en Antigravity, plugin de marketplace en Claude Code, extensión MCP en Gemini CLI, además oficialmente listado para Cursor, Copilot (VS Code + CLI), Codex, Cline, Command Code, Factory CLI, JetBrains AI Assistant, Junie, Katalon, Mistral Vibe, OpenCode, Qoder, Visual Studio, Warp, Windsurf.[5] Esto ya no es solo una herramienta de Google en cualquier sentido significativo excepto marca — es infraestructura compartida.

Lo que puedes agregar hoy

En Claude Code: el plugin oficial del Marketplace de Anthropic; Codex/Cursor: vía configuración de servidor MCP (npx chrome-devtools-mcp@latest). El estudio de caso de LY Corporation que Google cita: un sistema de auditoría automatizado reduciendo análisis manual de rendimiento hasta en un 98% — la cifra viene del blog de Google, reportes secundarios la confirman sin objeciones, pero permanece citada por el proveedor. Prueba Entrada con baja fricción, alto apalancamiento — especialmente para sesiones de front-end donde el error es visual y el modelo lo adivina ciego.

Managed Agents + AI-Studio Android [8]

Managed Agents en la API Gemini (Preview)

Una llamada a API inicia una sandbox Linux aislada donde un agente razona, usa herramientas y ejecuta código — el mismo arnés detrás de Antigravity. Defines tus propios agentes como archivos versionables AGENTS.md y SKILL.md. Disponible vía API de Interacciones + AI Studio, "Public Preview." Esa es la respuesta de Google al Codex-as-a-Service de OpenAI y Agent SDK de Anthropic — el mismo bloque conceptual, proveedor diferente.[8]

Managed Agents: En lugar de construir un bucle de agente tú mismo (llamada de modelo → llamada de herramienta → resultado → siguiente paso), obtienes un contenedor de agente listo del proveedor donde solo dices "haz X." Ventaja: menos boilerplate, aislamiento automático de sandbox. Desventaja: menos control, lock-in del proveedor, límite de seguridad es el proveedor.

Apps Android nativos en AI Studio

Un prompt → proyecto Kotlin/Jetpack Compose, vista previa en vivo en un emulador de navegador o directamente al dispositivo vía ADB, un clic en Google Play Internal Test Track. Esta es la pieza de I/O que voló más bajo el radar, y la más práctica: nadie tuvo que instalar Android Studio, nadie tuvo que firmar, nadie tuvo que publicar. Interesante para proyectos de hobby y prototipado; tangencial al trabajo principal de Daniel (stack TypeScript/Python).

Es jueves, veintiocho de mayo. Nueve días después de Google I/O, el polvo está assentándose y el cuadro es mixto. Una gran historia se divide en dos mitades: una áspera, una sustancial.

La mitad áspera es Antigravity dos punto cero, la nueva app de escritorio agent-first de Google. La arquitectura es real: orquestrador de sub-agente paralelo, tareas de fondo programadas, un agente de navegador. No obtienes nada así hoy de Claude Code o Cursor. Pero Google empujó una actualización automática a instalaciones existentes el día del lanzamiento — y removió el editor integrado, perdió configuraciones, rompió tuberías de integración continua. Cualquiera que ejecutara Antigravity en producción encontró una interfaz de chat en lugar de un ambiente de desarrollo el miércoles. Las reseñas están en tres de cinco. El consejo del reseñador: mira de nuevo en sesenta a noventa días. Para Daniel hoy: no instales.

La mitad sustancial es Chrome DevTools for agents en versión uno punto cero. Ese es el servidor Model Context Protocol dándole a tu agente verdaderos ojos de navegador — registros de consola, solicitudes de red, trazas de rendimiento, auditorías lighthouse. Imagina tu agente sentado ciego frente a la pantalla de otra forma adivinando si la página ni siquiera carga; este servidor es la inspección F-doce que abres tú mismo, solo legible por máquina. Preinstalado en Antigravity — pero también como plugin en Claude Code, Cursor, Copilot, JetBrains, Codex y una docena de otros. Se convirtió silenciosamente en infraestructura compartida. Recomendación de Daniel: engánchalo en Claude Code hoy, baja fricción, alto apalancamiento en trabajo de front-end.

Los números del proveedor son fuertes: Terminal Bench en setenta y seis punto dos, M-C-P-Atlas en ochenta y tres punto seis. Pero en razonamiento abstracto puro, queda detrás de Gemini tres punto uno Pro.
