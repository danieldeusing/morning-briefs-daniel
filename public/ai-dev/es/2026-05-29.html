Opus 4.8 · números de vendors y la brecha Verified [1][2][3]

Ayer a las 18:00 UTC, Claude Opus 4.8 estuvo disponible — simultáneamente en todos lados (claude.ai, API, Claude Code, Bedrock, Vertex, Foundry). Anthropic lo posiciona como "un colaborador más efectivo": juicio más agudo, más honestidad sobre su propio progreso, menos trampa en código. La frase de marketing que pega: Opus 4.8 es "cuatro veces menos probable que su predecesor" de pasar por alto sus propios errores de código sin comentario.[1]

"Pasar por alto errores de código": Cuando un agente escribe una función que rompe una prueba, y el modelo aún dice "hecho" aunque vio la falla de la prueba — eso es "pasar por alto". Según Anthropic, en Opus 4.7 esto pasaba en X% de casos, en 4.8 en X/4%. La base de números que Anthropic no publica; esto es un comportamiento que solo puedes verificar en tus propios logs.

La tabla de benchmarks que Anthropic envía con su post se ve respetable: Agentic Coding 64,3 → 69,2%, Multidisciplinary Reasoning with Tools 54,7 → 57,9%, Agentic Computer Use 82,8 → 83,4%, trabajo de conocimiento (Elo) 1.753 → 1.890, Agentic Financial Analysis 51,5 → 53,9%.[2] Lo que no está en esa lista y tampoco en el verificable SWE-Bench-Verified: un score para Opus 4.8. Extrajimos el JSON Verified directamente del repo — sin entrada para Opus 4.8, sin entrada para Mythos, sin entrada para Sonnet 4.6 o 4.7. La partición más reciente visible (bash-only) data del 17/02/2026.[3]

Dynamic Workflows · Claude escribe su propio script [5][6]

La mitad menos llamativa, pero estructuralmente más importante del lanzamiento Anthropic de ayer: Dynamic Workflows, Research Preview, desbloqueado a partir de Claude Code v2.1.154. En lugar de que Claude genere subagentes uno por uno y cada resultado intermedio llene la ventana de contexto, cada solicitud "workflow" ahora recibe su propio script JavaScript independiente, que Claude escribe él mismo — y un runtime dedicado ejecuta el script en background, mientras la conversación permanece responsiva.[5]

Dynamic Workflow: Imagina pedirle a un chef un menú de tres platos. Hasta ahora, pensaba en cada plato en paralelo y mantenía todo el plan en su cabeza — eventualmente el primer plato se olvidaba porque su memoria se llenaba. Ahora escribe una nota: "entrada dos chefs fría, plato fuerte cuatro chefs caliente, postre secuencialmente" — la nota es el script, la cocina es el runtime. El chef (Claude) puede volver a la conversación en cualquier momento sin interrumpir el trabajo; el script corre separado. Y la nota es código — la puedes leer, guardar, ejecutar de nuevo con un comando la próxima vez.

Los límites duros están en la documentación oficial: Hasta 16 agentes en ejecución simultánea por ejecución (menos en hardware débil), 1.000 agentes total por ejecución (previene loops descontrolados), sin entrada de usuario a mitad de ejecución (solo prompts de permiso pueden pausar). Disponible en Pro, Max, Team, Enterprise, más Anthropic API, Bedrock, Vertex y Microsoft Foundry. En Pro cambias la línea Dynamic-Workflows en /config.[6]

Tres maneras de iniciar un workflow

  • Deja la palabra "workflow" en tu prompt — Claude Code la destaca y escribe el script en lugar de trabajar uno por uno. (alt+w desactiva el gatillo por prompt.)
  • Workflow incluido: /deep-research <pregunta> es el integrado — distribuye búsquedas web en tu pregunta, las fuentes se verifican entre sí, el resultado es un reporte citado donde claims que no sobrevivieron la verificación cruzada ya están filtrados.
  • /effort ultracode: acopla razonamiento xhigh con orquestación automática de workflow — Claude decide solo cuándo una tarea necesita un workflow. Mayor quema de tokens, latencia más alta por solicitud; el gatillo manual permanece como predeterminado.

Workflow como un comando versionable + demostración Bun

Cuando una ejecución hace lo que quieres, presiona s en la vista /workflows — el script se convierte en un slash-command en tu repo (.claude/workflows/ o ~/.claude/workflows/) y es ejecutable en la siguiente rama como /<nombre>. Los docs son limpios: subagent = worker, skill = instructions, workflow = el script orquestando ambos. La propia demostración Anthropic: un puerto Zig→Rust del runtime Bun impulsado por workflow con "aproximadamente 750.000 líneas de Rust", 99,8% pruebas verdes, en 11 días.[5] Caso de uso de vendor, no benchmark independiente — Bun tampoco es representativo de codebases de mercado medio. Muestra el techo, no la expectativa de lunes. Prueba Entrada baja: /deep-research en una pregunta de investigación; workflows personalizados: lee el script antes de hacer commit con s.

Es viernes, veintinueve de mayo. Ayer fue el día más ruidoso de AI-coding desde Google I-O — Anthropic entregó dos cosas grandes a la vez, y Cognition recaudó mil millones el día anterior. Tres historias, una línea estratégica.

Primera historia. Anthropic lanzó Claude Opus cuatro-punto-ocho — la nueva versión flagship, disponible en todos lados inmediatamente. Los números de vendor se ven sólidos: agentic coding salta de sesenta y cuatro coma tres a sesenta y nueve coma dos por ciento, trabajo de conocimiento en la escala Elo de mil setecientos cincuenta y tres a mil ochocientos noventa, computer-use sube ligeramente. La frase de marketing que pega: Opus cuatro-punto-ocho es cuatro veces menos probable que su predecesor de pasar por alto sus propios errores de código sin comentario. Más honesto consigo mismo. Ese es el pitch.

El calificador. Extrajimos el archivo SWE-Bench-Verified directo del repo, el dataset en que este brief normalmente se basa para benchmarks. Sin entrada para Opus cuatro-punto-ocho. Sin entrada para Mythos Preview. Sin entrada para Sonnet cuatro-punto-seis o cuatro-punto-siete. La partición visible más reciente es de diecisiete de febrero. Mientras eso sea cierto, los números de Anthropic son solo eso — números de Anthropic. Pueden ser correctos, pero no están corroborados independientemente.

Segunda historia, la mitad estructuralmente más importante del lanzamiento de ayer: Dynamic Workflows en Claude Code. En lugar de que Claude genere un subagente tras otro y cada resultado intermedio se coma la ventana de contexto, Claude ahora escribe su propio script JavaScript para el trabajo, y un runtime separado ejecuta el script en background. Hasta dieciséis agentes en paralelo, hasta mil total por ejecución. Disponible como research preview comenzando Claude Code versión dos-punto-uno-punto-ciento-cincuenta-y-cuatro, en Pro, Max, Team y Enterprise. Tres gatillos: escribe la palabra "workflow" en tu prompt, o usa el comando slash-deep-research incluido, o establece slash-effort-ultracode — entonces Claude decide solo cuándo una tarea necesita un workflow. Si la ejecución se ve bien, presionas S en la vista de workflow y el script se guarda como un slash-comando versionado en tu repo. Para tu stack solo, esa es la palanca más directa del lanzamiento de ayer — barrera de entrada baja, sustancia alta.

Anthropic también confirmó que el siguiente modelo — Mythos — viene "en las próximas semanas" para disponibilidad general, actualmente en preview con un pequeño conjunto de organizaciones haciendo trabajo de cybersecurity. Eso cambia la pregunta si Opus cuatro-punto-ocho permanece como el mejor modelo públicamente disponible de una cuestión de meses a una cuestión de semanas.

Para tus elecciones de herramientas solo, Devin mismo no es relevante (precios enterprise, por-ticket), pero el posicionamiento contra Anthropic es la cuestión estratégica central detrás de ambas historias.
