El campo ahora es cuádruple

La historia de hoy no es un anuncio único, sino un patrón que se completó esta semana: Hasta hace poco, "IA para código" era esencialmente una lucha de dos vías entre Anthropic (Claude Code/Opus) y OpenAI (Codex/GPT). Con los modelos MAI de Microsoft y la Copilot App en Build (02/06) y el Gemini 3.5 Flash más Antigravity de Google (I/O, 19/05), todos los cuatro labs principales ahora tienen su propio stack de código, verticalmente integrado, en el campo.

CNBC resume la situación el 1 de junio así: Microsoft y Google están "retrasados, pero es absolutamente esencial que compitan por este crecimiento".[1] La razón está en los datos de adopción esta semana: Claude Code es el agente de código que más crece — por la encuesta de desarrolladores 2026 de JetBrains, subiendo de ~3% (abr 2025) a 18% (ene 2026), aproximadamente 6× en nueve meses — y entre desarrolladores experimentados (10+ años) el claro favorito: 46% elige Claude Code para trabajo diario versus 9% Copilot.[2] GitHub Copilot permanece líder en números brutos (encuesta Stack Overflow fin-2025: ~51% de desarrolladores profesionales, aunque ha caído de 67%; Cursor nuevo en ~18%, Claude Code en ~10% en primera aparición). El ingreso en tasa de corrida de Claude Code fue superior a 2,5 mil millones de dólares en feb 2026. Exactamente ese territorio — desarrolladores que pagan diariamente por un agente — Microsoft y Google no quieren ceder a Anthropic y OpenAI.

Stack de código verticalmente integrado: Cuando un proveedor suministra todo de una mano — el modelo, la herramienta alrededor (IDE o agente harness) y la nube en que se ejecuta. Como un fabricante de automóviles que no solo construye el motor, sino también el automóvil y las gasolineras. Ventaja: el modelo y la herramienta se entrenan uno al otro (MAI-Code-1 fue entrenado directamente en el harness de producción de Copilot). Desventaja para ti: bloqueo — el stack te empuja hacia un ecosistema.

Para la caja de herramientas de Daniel, la lección es sobria: la competencia es buena para precios y ritmo, pero cuatro stacks no significa cuatro opciones reales. Los novatos tienen que demostrar que sus modelos funcionan fuera de sus propias diapositivas de marketing — y ahí es donde entra el filtro de escepticismo (próxima sección).

Microsoft MAI: su primer modelo propio de código y razonamiento Bench

En Build, Microsoft presentó su familia MAI — el punto notable: estos son los primeros modelos entrenados por Microsoft para este rol, un paso deliberado lejos de la pura dependencia de OpenAI.[3] Dos son relevantes para nosotros:

MAI-Code-1-Flash — el pequeño, rápido modelo de código que aterriza directamente en GitHub Copilot y VS Code (selector de modelo, lanzamiento en Free/Student/Pro/Pro+/Max, gradualmente "durante las próximas semanas").[4] El truco real: Fue entrenado directamente en el harness de producción de Copilot y resuelve tareas con "hasta 60% menos tokens" gracias a la longitud de respuesta adaptativa — corto para consultas simples, más presupuesto de razonamiento para las difíciles.[5]

MAI-Thinking-1 — el primer modelo de razonamiento de Microsoft, entrenado desde cero sin destilación, en datos "limpios, licenciados comercialmente". Un MoE con ~35 mil millones de parámetros activos, contexto de 256K.[6]

Destilación (destilación de modelos): Un modelo pequeño aprende imitando las respuestas de un modelo "profesor" grande — como un aprendiz mirando por encima del hombro del maestro. Más rápido y barato, pero el modelo pequeño hereda los trucos del profesor. Microsoft enfatiza "sin destilación, desde cero" — mensaje de marketing: "no copiamos un modelo de OpenAI". Relevante para ti principalmente por los orígenes de licencia limpia de los datos de entrenamiento.
Escepticismo de Bench Los números son consistentemente proveedores o fuentes secundarias, no clasificación independiente. Reportado: MAI-Code-1-Flash ~51% SWE-Bench Pro (+16 puntos vs. Claude Haiku 4.5 en 35,2%, aproximadamente tier GPT-5.3); MAI-Thinking-1 ~53% SWE-Bench Pro y así "emparejado con Claude Opus 4.6", 97% AIME 2025.[6][7] Para contexto: Opus 4.8 está en 69,2% Pro — así que los modelos MAI juegan una clase abajo, pero aciertan su segmento de precio/tamaño. Y cuidado: el recuento de parámetros de MAI-Code-1-Flash se reporta de manera contradictoria (sitio Microsoft/múltiples fuentes secundarias: 5B; una fuente: "137B MoE") — no trataría ninguno de los dos como resuelto hasta que aparezca un detalle de model card.

GitHub Copilot App: de plugin de editor a centro de comando de agentes try

La noticia de Build más prácticamente relevante para mí no es un modelo, sino una herramienta: La GitHub Copilot App es una aplicación de escritorio independiente (Win 11, macOS, Linux), separada de la extensión VS Code. Convierte Copilot en "un centro de comando para sesiones de agentes IA paralelas — cada una en su propio, aislado Git worktree".[8]

Git worktree: Múltiples copias de trabajo del mismo repositorio lado a lado, cada una en su propia rama — sin cambiar constantemente de un lado a otro. Como múltiples bancos de trabajo en el mismo taller: cada uno tiene una reparación diferente en marcha, sin meterse en el camino del otro. Es exactamente lo que usa la app para que tres agentes trabajen simultáneamente en tres tareas sin sobrescribir los archivos uno del otro.

Desde una vista "My Work" ves sesiones en ejecución, issues, PRs y automaciones en segundo plano en múltiples repositorios. Es exactamente el patrón orquestrador que Daniel ya conoce de Claude Code — GitHub lo coloca en una app nativa. Además, el Copilot SDK ahora es GA en seis idiomas (Node/TS, Python, Go, .NET, Rust, Java); el mismo runtime de agente en que se ejecuta la app es programable.[9]

Por qué probar Si ya estás en el ecosistema GitHub y quieres ejecutar múltiples agentes en paralelo: la app es un competidor serio de Claude Code/Cursor para workflows de múltiples agentes — y el SDK hace posible la orquestación personalizada. Vista Técnica para Pro/Pro+/Business/Enterprise. Sáltalo para trabajo puro de un único agente individual.

Google Gemini 3.5 Flash + Antigravity 2.0

La respuesta de Google se ha implementado ampliamente desde el I/O (19/05) y pertenece a este cuadro de panorama — rozada ayer bajo "Pro wait", hoy con sustancia: Gemini 3.5 Flash es GA en Antigravity, la Gemini API en AI Studio y Android Studio, Gemini Enterprise y la app Gemini.[10] El argumento de venta es velocidad: ~289 tokens/segundo, aproximadamente 4× más rápido que Opus o GPT-5.5, en Antigravity hasta 12×.[11]

Escepticismo de Bench Los valores de código reportados son números propios de Google en suites parcialmente curadas por Google: Terminal Bench 2.1 76,2%, MCP Atlas 83,6%, GDPval-AA 1656 Elo.[10] Independiente es aproximado: Artificial Analysis ubica Gemini 3.5 Flash en el Índice de Inteligencia en 55 — campo medio sólido, no en la cima de código (allá GPT-5.5 xhigh con Índice de Código 59, Opus 4.8 adelante en Índice de Inteligencia en 61,4).[12] Así que Flash no compite contra Opus 4.8 en calidad, sino en calidad por segundo por dólar — ese es su territorio.

Antigravity 2.0 es la verdadera palanca dev: una app de escritorio independiente más CLI, construida agent-first, con Agentes Administrados en la Gemini API — una única llamada de API entrega un agente más sandbox Linux alojado (Bash/Python/Node, E/S de archivos, navegación, skills definidas en Markdown, montajes repo/GCS).[13] Ese es el ataque directo de Google a la capa IDE-agente donde viven Claude Code y Cursor.

Qué significa esto para la caja de herramientas de Daniel

Mi lectura: el mercado de cuatro labs es bueno para Daniel — más competencia reduce precios e impulsa ritmo. Pero "cuatro stacks" ≠ "cuatro opciones reales". Hoy Opus 4.8 sigue siendo la referencia de código, y Claude Code no sin razón es el agente que más crece y favorito de desarrolladores experimentados. Los novatos abordan diferentes ejes: MAI el segmento barato en-Copilot, Gemini 3.5 Flash velocidad.

Concretamente digno de intentar: la GitHub Copilot App si ejecutas workflows de múltiples agentes en el ecosistema GitHub, y una mirada a Antigravity 2.0 si los agentes administrados vía API te atraen. Con los modelos MAI y los puntajes de código de Gemini, esperaría benchmarks independientes antes de cualquier cambio de stack — hasta ahora casi solo números de vendedores están adentro. Y el próximo deadline real sigue siendo Gemini 3.5 Pro en la ventana de junio.

Es domingo, siete de junio, y el día de noticias en sí está tranquilo. Este es un buen momento para dar un paso atrás, porque esta semana el mercado de modelos de inteligencia artificial para código se ha completado.

Hasta hace poco, esto era esencialmente una disputa de dos vías entre Anthropic con Claude Code y OpenAI con Codex. Esta semana los otros dos grandes jugadores se han unido. Microsoft presentó sus propios modelos en su conferencia Build el dos de junio, y Google ha lanzado ampliamente su Gemini tres punto cinco Flash junto al ambiente de desarrollo Antigravity. Eso significa que los cuatro labs principales ahora tienen su propio stack de código. CNBC llama a Microsoft y Google rezagados, pero dice que es esencial que compitan. La razón es simple: Claude Code es el agente de código que más crece ocho meses después del lanzamiento y el favorito de desarrolladores experimentados. Según la encuesta de JetBrains, cuarenta y seis por ciento lo elige para trabajo diario versus nueve por ciento para Copilot. GitHub Copilot aún lidera en números brutos, con cerca de cincuenta y uno por ciento en la encuesta del Stack Overflow.

Microsoft tiene dos modelos que nos importan. MAI-Code-uno-Flash es un pequeño, rápido modelo de código que va directamente a GitHub Copilot y Visual Studio Code y fue entrenado en el verdadero harness de producción de Copilot. MAI-Thinking-uno es el primer modelo de razonamiento de Microsoft, entrenado desde cero sin destilación. El mensaje detrás es menos dependencia de OpenAI. Lo que es importante es mi filtro de escepticismo: todos los números reportados son números de vendedores, aún no verificados independientemente. En la prueba más dura, SWE-Bench Pro, los modelos MAI anotan alrededor de cincuenta y uno a cincuenta y tres por ciento, mientras Opus cuatro punto ocho de Anthropic anota sesenta y nueve por ciento. Así que juegan una clase abajo, pero aciertan su objetivo de precio y tamaño.

Lo más prácticamente interesante no es un modelo, sino una herramienta: la nueva aplicación GitHub Copilot. Esa es una aplicación de escritorio independiente, separada de la extensión del editor, que orquesta múltiples agentes de inteligencia artificial en paralelo. Cada agente trabaja en su propio Git worktree aislado — su propia copia del proyecto — para que no se estorben mutuamente. Ese es exactamente el patrón orquestrador que Daniel conoce de Claude Code, ahora como una app nativa. Además, el kit de desarrollador Copilot ahora está disponible en seis idiomas.

Google apuesta por velocidad con Gemini tres punto cinco Flash: alrededor de cuatro veces más rápido que Opus o GPT, y hasta doce veces más rápido en Antigravity. Aquí también se aplica el filtro de escepticismo, porque los puntajes de código provienen del propio Google y parcialmente de sus propias suites de prueba. Clasificado independientemente, Flash es un campo medio sólido, no en la cima de código. Su territorio es calidad por segundo y por dólar, no una comparación directa de calidad con Opus. Antigravity dos punto cero en sí es el ataque directo de Google contra la capa de agentes de Claude Code y Cursor, con agentes alojados directamente a través de la interfaz de programación.

Hoy había solo cosas menores: Anthropic expandió su red de socios, OpenAI habilitó la compartición de plugins Codex por defecto en la versión empresarial de ChatGPT. Entre modelos abiertos, nada nuevo ha sucedido desde MiniMax M-tres del uno de junio, y herramientas como Aider, Cline y OpenHands permanecen tranquilas.

Mi conclusión: cuatro stacks no significan cuatro opciones reales. Opus cuatro punto ocho sigue siendo la referencia de código, los novatos abordan ángulos diferentes. Vale la pena intentar la aplicación GitHub Copilot para trabajo de múltiples agentes y una mirada a Antigravity. Pero antes de cualquier cambio de stack, esperaría los benchmarks independientes. Próxima semana estoy principalmente mirando si Google finalmente entrega Gemini tres punto cinco Pro.
