Fable 5 es ahora un modelo pagado [1][2][3]

Esta es la noticia que golpea directamente tu flujo de trabajo. Anthropic eliminó Fable 5 (lanzado el 09/06/2026) de los límites del plan el 23 de junio — el uso continuado ahora funciona con créditos de API pagados.[2] Los precios: $10/M tokens de entrada, $50/M salida.[1] Según ayautomate, este es "el modelo frontier más caro generalmente disponible que Anthropic envía, el doble de la tarifa de Opus 4.8".

$/M tokens: Pagas por millón de bloques de construcción de palabras que entran y salen — como un taxímetro que avanza conforme lees y conforme respondes. La salida es más cara porque el modelo tiene que "pensar" para producirla. A $50/M salida, una ejecución de agente con mucha salida de herramienta se suma rápido.

Consecuencia práctica: quien ejecute Fable 5 por encima de la cuota del plan lo quema según future-stack-reviews en "roughly double the rate of Opus 4.8".[3] Para la configuración de Daniel, eso significa: Fable 5 es ahora el modelo para el 10% difícil — la decisión de arquitectura, el refactor complicado — no el conductor predeterminado. Opus 4.8 sigue siendo mejor relación calidad-precio para el trabajo diario, y para código rutinario vale la pena enrutar a modelos más baratos (o localmente: Qwen/Gemma). Un reflejo "usa Fable 5 para todo" acaba de ponerse caro.

SpaceX quiere comprar Cursor — reportado, no sellado [4][5]

Gran titular, pero lee con cautela: Según TechCrunch (16/06/2026), SpaceX acordó adquirir Cursor (Anysphere), valuando la startup de codificación en $60 mil millones — completamente en acciones.[4] Punto clave: esta es una transacción reportada, aún no cerrada; el cierre se espera solo para Q3 2026, así que podría fracasar o cambiar.

Los números detrás: Cursor logró "roughly $4 billion in annualized revenue in under four years", según CNBC ~$2.6 mil millones de B2B empresarial — el acuerdo la valúa en aproximadamente 15 veces los ingresos.[5] Por qué esto importa para tu caja de herramientas: si un fabricante de cohetes compra el editor de código de IA más utilizado, es una señal de que las herramientas de codificación se están convirtiendo en infraestructura estratégica — y un recordatorio de no apostar demasiado en un único editor propietario cuyo propietario (y roadmap) podría cambiar mañana. Cursor continuó entregando en paralelo: v3.9 (22/06) agrupa plugins, skills y MCPs en una Customize Page; v3.8 (18/06) trajo automaciones con triggers GitHub/Slack.[5]

Google Managed Agents: agente mediante una única llamada de API prueba [6]

Google está lanzando Managed Agents como preview en la API Gemini (API Gemini + AI Studio Playground).[6] Idea: un agente que "razona, usa herramientas y ejecuta código en un entorno Linux aislado y efímero con una única llamada de API". Impulsado por Gemini 3.5 Flash ($1.50/$9.00 por M tokens). Los agentes se definen en Markdown — AGENTS.md + SKILL.md, así que son versionados y declarativos.

Por qué probarlo: No tienes que ejecutar tu propia infraestructura de sandbox — el contenedor Linux viene con la llamada de API. Para un rápido "el agente construye y prueba algo aislado" sin configuración de harness personalizado, es un punto de entrada barato. Nota de límite: la Gemini Enterprise Agent Platform (el envoltorio de industria/negocio alrededor de esto) pertenece al brief de ai-usecases — aquí solo lo que es relevante para manos de dev: el bloque de construcción de API en sí.

  1. blog.google

Benchmarks: números fuertes, evidencia débil [7][8][9]

El campo de rankings se está moviendo — pero el número más importante no es una puntuación, es una advertencia de métodos. En el board SWE-Bench-Verified en llm-stats.com, Fable 5 lidera en 95.0 % (entre 102 modelos evaluados), seguido por Claude Mythos Preview (93.9 %) y Opus 4.8 (88.6 %); Gemini 3.1 Pro y DeepSeek-V4-Pro-Max empatan en 80.6 %.[7] A partir del scrape del leaderboard del 24/06/2026.

Lee escépticamente Según digitalapplied, de 100 modelos en ese board (al 16/06), solo 1 resultado fue verificado independientemente — los otros 99 proveedores los enviaron a sí mismos.[8] Eso significa: este 95 % es una cifra reportada por Anthropic, no medida neutralmente. Para puntuaciones autorreportadas en este rango (95–99 %), se requiere cautela — game de benchmark, contaminación de entrenamiento y scaffolding (el framework agéntico alrededor del modelo) a menudo importan más que la capacidad bruta del modelo.

Para poner "con scaffolding" en perspectiva: en SWE-Bench Pro (benchmark agéntico de Anthropic), Fable 5 publica 80.3 % vs Opus 4.8 (69.2 %)[9] — la misma brecha, diferente métrica. Así que lee el 95 % no como "resuelve 95 de 100 PRs reales" sino como "lidera en un board donde los proveedores ingresan sus propios números".

Terminal-Bench: Codex al frente, figura Fable con salvedades [10]

En el benchmark terminal agéntico, según los snippets disponibles, Codex CLI con GPT-5.5 lidera el board 2.1 en 83.4 %, Claude Code con Fable 5 sigue cerca en 83.1 %, Claude Code con Opus 4.8 en 78.9 %.[10]

Estado no está claro La cifra Fable (83.1 %) viene solo de un snippet de búsqueda, no de un fetch fresco del leaderboard. Un fetch directo devolvió una tabla Terminal-Bench-2.0 con valores diferentes (GPT-5.5 82.7 %, Opus 4.8 69.4 %). Trata el orden 2.1 como tendencia aproximada, no como posición confirmada. Terminal-Bench: mide si un agente resuelve tareas reales de shell en una terminal end-to-end — más cerca de "¿esta cosa realmente puede construir y probar mi repo?" que snippets de código aislados.

El leaderboard Aider se queda fuera hoy: los datos de versión/fecha no pudieron verificarse de manera confiable para junio de 2026 (los metadatos de GitHub se veían anticuados), así que ningún número estimado — una brecha es mejor que un valor incorrecto.

Claude Code & Codex: semana densa de herramientas [11][12]

Claude Code funcionó en alta cadencia del 17/06–23/06. v2.1.187 (23/06) trae login/logout MCP vía CLI, una configuración sandbox.credentials (bloquea comandos en sandbox de leer archivos de credenciales) y restricciones de modelo de org.[11] Mientras tanto: bloqueó comandos git destructivos en modo auto, un comando /cd para cambiar el directorio de trabajo en la sesión, cadenas de sub-agente hasta 5 niveles de profundidad, y artefactos como páginas web vivas compartibles (historial de versión en beta para team/enterprise).

MCP: Un enchufe estándar, a través del cual el agente de codificación habla con herramientas y fuentes de datos externas — como un puerto USB donde conectas una base de datos, sistema de tickets o navegador sin necesitar un cable separado para cada uno. El nuevo login/logout CLI administra estas conexiones de puerto de forma más limpia.

Codex CLI 0.142.0 (22/06) trae canje de crédito de uso, organización de plugin por categoría y presupuestos de token de rollout configurables.[12] Ya el 15/06 llegó /import para migrar desde Claude Code; el 18/06, "Record & Replay" para macOS (convierte flujos de trabajo demostrados en skills reutilizables). La ruta /import es notable: OpenAI está activamente reduciendo el costo de cambio de Claude Code a Codex.

Codex vs. Claude Code: dos filosofías [10]

La comparación de harness es oportuna. Codex funciona en 8 sub-agentes paralelos y una ventana de contexto de 200K; Claude Code en una ventana de 1M-token — pero quema "3–4× more tokens" para tareas idénticas (en el ejemplo del plugin Figma, 6.2M vs 1.5M tokens).[10] Este es el trade-off en forma pura: Claude Code ve más contexto a la vez, cuesta más por tarea — que afecta dos veces después del aumento de precio de Fable 5 de hoy. Codex es parco en tokens pero tiene que administrar activamente su contexto más ajustado. En adopción: Claude Code afirma "326K+ daily commits (~10% of all public commits)" según morphllm; Codex está en 91 mil estrellas de GitHub.[10]

Concreto para las matemáticas de costo: si Claude Code extrae 3–4× más tokens y ejecutas Fable 5 más caro en él, los dos factores se multiplican. Para ejecuciones rutinarias intensivas de tokens, el camino más barato ahora es más claro que nunca — un modelo más barato en el harness, o donde cabe, uno local (Qwen-Coder, Gemma) para tareas que no necesitan razonamiento frontier. Gasta tokens frontier solo donde el problema realmente los demanda.
