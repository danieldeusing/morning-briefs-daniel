GPT-5.6: Anuncio Grande, Distribución Limitada

El 26 de junio, OpenAI reveló la familia GPT-5.6 — tres modelos con nombres celestiales: Sol (flagship), Terra (equilibrado) y Luna (rápido/económico).[1] El pitch es codificación agentiva: tareas largas y autónomas en lugar de preguntas aisladas. Sol cuesta $5 entrada / $30 salida por 1M de tokens; Terra $2,50/$15; Luna $1/$6.[1] Sol también recibe dos nuevos niveles de razonamiento.

Modo de Razonamiento ("max"/"ultra"): Imagina darle a alguien más tiempo para pensar antes de responder. Exactamente eso aquí: en "ultra", el modelo razona internamente más largo y cuidadosamente antes de generar código — más costoso y lento, pero más preciso en problemas complejos.

La parte verdaderamente notable es cómo se distribuye: una vista previa restringida para "un pequeño grupo de socios confiables", coordinado con el gobierno de EE.UU.; acceso más amplio solo "en las próximas semanas".[1] OpenAI se distancia públicamente del requisito — "No creemos que este tipo de proceso de acceso controlado por el gobierno deba convertirse en el estándar a largo plazo."[2] Prácticamente para Daniel: aún no comprobable, quizás solo a través de Codex y socios seleccionados.

Y ahí fue donde se puso concreto rápidamente: según TechTimes, OpenAI silenciosamente activó GPT-5.6 en Codex para algunos usuarios — descubierto porque un system prompt filtrado reveló el cambio de modelo.[3] Si construyes sobre Codex, es mejor verificar qué modelo está realmente respondiendo.

Advertencia de Benchmark En la vista previa, solo Terminal Bench 2.1 está disponible: Sol alcanza 88,8% (configuración ultra 91,9%), por delante de Fable 5 con 83,4%.[4] OpenAI intencionalmente no publicó un score SWE-Bench — irónicamente, el benchmark donde Claude lidera "simplemente no existe".[5] Hasta que esa comparación exista, GPT-5.6 vs. la línea Claude no puede clasificarse justamente en codificación. Un solo benchmark no prueba generalización.

Fable 5: Tres Días en Copilot, Luego Desapareció

Una lección sobre lo frágil que se ha vuelto la disponibilidad de modelos en contexto empresarial. Fable 5 de Anthropic — según GitHub, el "primer modelo clase Mythos, diseñado para trabajo de codificación y conocimiento largo autónomo" — entró en GA en GitHub Copilot el 9 de junio.[6] Para el 12 de junio fue removido, "suspendido en todas las superficies de GitHub Copilot".[7]

La razón es una directiva de control de exportación de EE.UU.: El 12 de junio, el Secretario de Comercio Howard Lutnick — invocando autoridad de control de exportación y seguridad nacional — ordenó a Anthropic bloquear Fable 5 y Mythos 5 para todos los usuarios extranjeros mundialmente.[7] El disparo fue un "jailbreak" reportado que podría desbloquear las capacidades cibernéticas de los modelos. Anthropic disputa públicamente que el jailbreak sea tan grave.[7] Las versiones más antiguas de Claude no son afectadas y permanecen disponibles en Copilot.

Control de Exportación: El gobierno de EE.UU. puede prohibir algunas tecnologías de llegar a usuarios extranjeros si ve un riesgo de seguridad — como un embargo de armas, pero para modelos de IA. Aquí el gobierno temió que un "jailbreak" pudiera desbloquear las capacidades cibernéticas de Fable 5 y Mythos 5. Resultado: Anthropic tuvo que desligar los dos modelos mundialmente para todos los usuarios extranjeros — no por mala calidad, sino por orden del gobierno.

Separadamente, como contexto operacional: Fable 5 también exigió almacenar prompts y salidas por hasta 30 días para clasificadores de seguridad[8] — un requisito único entre modelos Copilot que choca con la postura de retención cero de datos de Microsoft. Pero esa es una quirk separada, no la razón de la suspensión.

Cualitativamente, eso es doblemente lamentable: Fable 5 actualmente lidera la clasificación SWE-Bench Verified (95,0% según llm-stats, al 30/06) — pero no es utilizable ahora.[9] El mejor modelo activamente disponible en la lista es Claude Opus 4.8 con 88,6%.[10] Si eres productivo en Copilot hoy, sigues funcionando bien en Opus — y no deberías sentirlo como un paso atrás.

MCP-RC: La Base se Reconstruye [11][12][13]

La entrada más consecuente del día — aunque no suene dramática. El Candidato de Lanzamiento del Protocolo de Contexto de Modelo para la versión 28/07/2026 está disponible desde el 21 de mayo; la especificación final se lanza el 28 de julio, con una ventana RC de 10 semanas para que SDKs tier-1 se actualicen.

MCP (Model Context Protocol): El socket unificado a través del cual agentes de IA hablan con herramientas — sistemas de archivos, bases de datos, APIs. En lugar de construir un adaptador personalizado para cada herramienta, todo habla el mismo idioma. La configuración de Claude Code de Daniel depende enteramente de este socket; cambia el enchufe, y los servidores tienen que seguir.

Tres cambios que importan:

  • Núcleo sin estado: El handshake initialize/initialized y el header Mcp-Session-Id desaparecen — cada solicitud ahora lleva su propio contexto. Esto desacopla un cliente de una única instancia de servidor y hace que el escalado horizontal (balanceadores de carga, múltiples réplicas) sea finalmente trivial.
  • JSON Schema 2020-12 completo: inputSchema y outputSchema de herramientas ahora pueden usar el estándar completo — contratos de herramientas más estrictos y expresivos.
  • Endurecimiento OAuth 2.0 / OIDC más un Framework de Extensiones (Tasks + MCP Apps). A cambio, Roots, Sampling y Logging se deprecan — con ventanas de 12 meses.
Lo que Esto Significa para Daniel Si ejecutas tus propios servidores MCP, prueba el RC ahora contra tu implementación de servidor — especialmente si usas Sampling o Roots. La ventana de deprecación de 12 meses brinda tiempo, pero sin estado es un cambio arquitectónico real, no solo una bandera.

Microsoft MAI-Code-1-Flash: Pequeño, Nativo, en el Selector

Microsoft reveló MAI-Code-1-Flash el 2 de junio — su propio modelo de codificación con apenas 5 mil millones de parámetros y contexto de 256k, enfatizando que fue "entrenado desde cero en datos limpios y rastreables, sin destilación de modelos de terceros".[14]

Destilación: Un modelo pequeño aprende mirando por encima del hombro de un modelo grande e imitando sus respuestas — como un aprendiz copiando al maestro. Microsoft es explícito en que MAI-Code no sucedió así — relevante porque "copiado de OpenAI/Anthropic" tiene complicaciones de licencia y legales.

Los números: SWE-Bench Pro 51,2% vs. Claude Haiku 4.5 con 35,2%; hasta 60% menos tokens en SWE-Bench Verified; +28,9 puntos en instruction-following.[15] Estos son números de proveedor — lee con la cautela usual. Más prácticamente relevante: el modelo ahora está en el selector Copilot de VS Code, en todos los planes (Free/Pro/Pro+/Max), además en OpenRouter, Fireworks y Baseten.[16] Try Vale la pena probar como modelo predeterminado barato y rápido para ediciones rutinarias — especialmente si realmente ahorra tokens.

Copilot Cambia a Facturación por Uso

Desde el 1º de junio, GitHub Copilot funciona con facturación por uso en todos los planes con AI-Credits mensuales.[17] Concretamente: Free $0, Pro $10/mes (incl. $15 de créditos), Pro+ $39/mes ($70), Max $100/mes ($200).[18] Además un cambio silencioso pero costoso: la revisión de código Copilot ahora cuenta contra los minutos de Actions incluidos.[19]

Para sesiones agentivas, esa es una palanca de comportamiento real: ejecuciones largas y autónomas (exactamente lo que GPT-5.6 y Fable 5 anuncian) ya no son "planas" sino queman créditos. Si trabajas mucho con agentes de fondo, observa tu consumo — los modelos más caros en el bucle más apretado pueden agotar un plan rápidamente.

SWE-Bench Verified — y Por Qué Ser Escéptico

SWE-Bench Verified — Top 6, al 30/06/2026 (llm-stats)

Los puntajes según llm-stats (actualizado 30/06): Fable 5 95,0%, Claude Mythos Preview 93,9%, Opus 4.8 88,6%, Opus 4.7 87,6%.[20] Ten en mente — esta es una clasificación sobre 500 issues reales de GitHub verificados, no una garantía del mundo real.

Escepticismo Los agregadores se contradicen: BenchLM.ai lista un "Claude Mythos 5" en 95,5%; una tercera fuente coloca al líder en solo ~78%.[21] Tales diferencias vienen de scaffolding diferente (el envoltorio del agente alrededor del modelo) y splits de datos diferentes — no es solo el modelo siendo medido, sino modelo + harness. Lee porcentajes absolutos como orden de magnitud, no verdad absoluta.

LiveCodeBench cuenta una historia diferente: aquí DeepSeek-V4-Pro-Max (93,5%) y -Flash-Max (91,6%) lideran según llm-stats,[22] mientras ArtificialAnalysis ve Gemini 3 Pro Preview en 91,7% en la cima (al 24/06).[23] Nuevamente: dos boards, dos líderes — una pista de que "el único mejor programador" apenas existe.

Herramientas: Cursor, Claude Code, WebMCP

Cursor 3.8 (18/06) trae una skill /automate que te permite describir automaciones en lenguaje natural directamente en una sesión de agente, más disparadores Slack-emoji, cinco disparadores GitHub y Computer Use.[24] La onda 3.7 (04–17/06) ya había entregado Design Mode (anotar elementos de UI directamente en canvas), un Context Explorer (desglose de tokens) y Bugbot con /review.[25] Try El flujo /automate es el punto de entrada de menor fricción para automaciones Cursor hasta ahora.

Claude Code v2.1.195 (27/06): nuevo comando /rewind, aproximadamente 37% menos carga de CPU durante streaming (vía text-update coalescing), agentes de fondo reparados, lógica de retry MCP y un setting sandbox.credentials.[26] La parte del CPU es notablemente mejor para el ruido del ventilador local.

WebMCP — propuesto en Google I/O el 19/05, co-desarrollado con Microsoft — es un estándar web abierto que anota funciones JavaScript y formularios HTML como herramientas invocables por agentes-navegador.[27] Timeline: Chrome/Edge H2 2026 (Chrome-149 Origin Trial en vivo), Firefox Q3, Safari Q4.[28] Fuentes secundarias citan 67% menos errores vs. scraping visual.[29] Conceptualmente, es "MCP para la web abierta" — agentes invocan funciones declaradas en lugar de hacer clic en píxeles.
