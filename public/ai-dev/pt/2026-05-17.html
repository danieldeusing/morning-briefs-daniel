OpenAI · Brockman fica com tudo que toca código

Enquanto a Anthropic continua apagando o incêndio do credit pool desde terça e o Google deliberadamente não diz nada no fim de semana, a OpenAI repintou na sexta às 17h PT um organograma que importa diretamente para devs. TechCrunch e Benzinga confirmam: ChatGPT, Codex e a API de desenvolvedor estão agora sob um único product lead — Greg Brockman em caráter permanente (em vez de interim, durante o afastamento médico de Fidji Simo). Thibault Sottiaux, antes head de Codex, assume core-product e plataforma nos três pilares. Nick Turley vira para enterprise.

Por que isso importa para você

Até agora a API rodava com cadência de roadmap própria, o Codex com a sua, o ChatGPT com a sua — o que explica por que a mesma lógica do GPT-5.5 chegou nas três superfícies em três momentos diferentes. Um lead compartilhado significa uma sequência de release compartilhada: features que saem primeiro no ChatGPT consumer devem pingar mais rápido na API. Em troca, sobe a pressão para empurrar features de API cedo para o fluxo do Codex.

Essa é a leitura que o próprio Brockman prefere — a leitura dos repórteres é mais incisiva: "três produtos com um stack rank do que afina primeiro." Ambas podem ser verdade ao mesmo tempo. Na prática: SLAs de estabilidade da API ficarão mais exigentes nos próximos meses porque features de consumer podem ganhar prioridade no backlog. Quem tem pipelines de produção na API responses, deve ler o changelog mais de perto.

Correção · ACP da JetBrains é Client, não Control

Uma correção importante a ontem. O brief de sábado falou em "Agent Control Protocol" da JetBrains como "conexão de agente vendor-neutral, concorrente do MCP." Ambos errados.

O que ACP é de fato

ACP significa Agent Client Protocol. É co-desenvolvido com Zed, não exclusivo da JetBrains e não posicionado como rival do MCP. O ponto: MCP governa a comunicação entre agente e tools/dados ("me busca o schema dessa base"). ACP governa a comunicação entre editor e agente ("usuário apertou cmd-K na linha 42, aqui está a posição do cursor e o código selecionado"). Os dois protocolos vivem em camadas diferentes; não competem.

A JetBrains é participante no ACP registry desde janeiro de 2026; Cursor entrou em março; o ponto genuinamente fresco desta semana é o anúncio do ReSharper 2026.2 EAP de 11/05: ACP agora roda dentro do Visual Studio via Junie — primeiro host fora do Zed e das IDEs da JetBrains.

agentclientprotocol.com · blog.jetbrains.com/dotnet
Usuário no editor → ACP → Agente → MCP → Tools / dados
Lição aprendida Com nomes de protocolos de três letras prefixados por "Agent": double-check mais uma vez do que parece necessário. ACP, A2A, MCP, ANP, AGNTCY — a cada três semanas alguém inventa outro. Na dúvida, vá à spec site, não a um recap.

Realidade dos benchmarks · Verified vs Pro [1][2][3]

Os fornecedores citam o SWE-Bench Verified porque ali os modelos chegam a 80–93%, o que fica bem no palco. A própria OpenAI abandonou Verified em fevereiro, com a justificativa: "todos os modelos frontier estão contaminados" e "59,4% das hard tasks têm testes falhos." Quem quer uma comparação realista olha o SWE-Bench Pro — a suíte sucessora, mais difícil de memorizar.

Contaminação de benchmark: como quando um candidato traz por engano as questões da prova do ano passado para estudar. Ele resolve as questões "com sistema", mas o sistema é decoreba, não habilidade. Em LLMs significa: as questões do benchmark vazaram para o dataset de treino. Scores altos em Verified, scores medíocres em Pro = provável contaminação.

METR Time Horizon — saturado, não extrapolável

O Time Horizon 1.1 da METR (29/01/2026) expandiu a task suite de 170 para 228, o slice de 8h+ de 14 para 31 tasks. Taxa de duplicação pós-2023 ~131 dias / 4,3 meses vs. 210 dias antes. Estimativas de GPT-5/Opus 4.5 subiram +55% / +11%, GPT-4 mais antigos caíram -35 a -57%. A própria METR nota no post que o valor de 50% é "ruidoso porque a task suite atual está quase saturada." O horizonte de 718 minutos do Opus 4.6 depende de 2–3 tasks de 8h resolvidas. Um sigma a menos e o número cai pela metade.
