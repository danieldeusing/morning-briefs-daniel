Hangzhou responde · Qwen 3.7 Max — flagship sim, pesos não [1][2][3]

Enquanto o Google teve o palco ontem à noite, a Alibaba entregou a resposta hoje: no Cloud Summit em Hangzhou (20–21/05), o Qwen 3.7 Max foi oficialmente revelado — construído para cargas agentic, "operações sustentadas, multi-etapa", entregue via Bailian (fora da China: Model Studio). Isso transforma o "preview sighting" dos últimos dias em um lançamento de verdade — com uma ressalva importante.

A ressalva: sem lançamento aberto

Até hoje não há pesos abertos para o Qwen 3.7: Hugging Face e ModelScope ainda têm só a geração 3.6 (Qwen3.6-27B, Qwen3.6-35B-A3B sob Apache 2.0), sem repo QwenLM/Qwen3.7.[2] O Qwen 3.7 Max é API-only hoje. Isso bate com o padrão: a Alibaba mantém — como já fez com o 3.6 Max/Plus — seus modelos mais fortes proprietários, e as variantes abertas de mid-tier chegam (se chegarem) depois e menores. Leitura cética: revelar um flagship ainda não é um presente para a comunidade de self-hosting. Quem espera por pesos abertos do Qwen 3.7 para o stack local segue esperando.

alibabacloud.com · scmp.com

A parte de hardware: Zhenwu M890

Mais substancial que o próprio modelo talvez seja o chip: a subsidiária T-Head da Alibaba mostrou o Zhenwu M890 — 3× o desempenho do antecessor Zhenwu 810E, 144 GB de memória on-chip, precisão de FP32 até FP4. Mais o supernode Panjiu AL128 (128 aceleradores) e um chip de rede ICN Switch 1.0 (25,6 Tbit/s). O que significa para o Daniel: esta é a resposta da China aos controles de exportação — uma estratégia de stack vertical (chip próprio + modelo próprio + nuvem própria) que, no médio prazo, define quão barata fica a inferência agentic fora do mundo NVIDIA.

business-standard.com
FP4 (precisão de 4 bits): o quanto um modelo "arredonda" ao calcular. FP32 = precisão total, muita memória; FP4 = só 4 bits por número, bem menor e mais rápido, mas mais grosseiro. Um chip que desce até FP4 consegue rodar modelos grandes mais barato — desde que a qualidade não sofra demais com o arredondamento grosseiro.

Google I/O · Gemini 3.5 Flash vira o padrão [4][5][6]

A verdadeira história do I/O é uma decisão de padrão. O Google torna o Gemini 3.5 Flash o modelo padrão imediatamente no app Gemini, no AI Mode da Busca, na API, no AI Studio e no Antigravity. O modelo Pro de fronteira (3.5 Pro) só chega "mês que vem". A aposta: um modelo rápido e barato em nível de fronteira vence o último ponto de benchmark.

Números & preço — a leitura cética

Segundo o material de I/O do Google, o 3.5 Flash vence o antecessor 3.1 Pro na suíte agentic: Terminal-Bench 2.1 76,2% (3.1 Pro 70,3%), MCP Atlas 83,6% (78,2%), Finance Agent v2 57,9% (43,0%), GDPval-AA 1656 Elo (1314). O número mais independente: SWE-Bench Pro Public 55,1% (54,2%) — só +0,9 ponto.[5] Preço: US$ 1,50/1M de input, US$ 9,00/1M de output, cacheado US$ 0,15/1M; contexto 1.048.576 in / 65.536 out; quatro níveis de thinking. Nota cética: Terminal-Bench, MCP Atlas e GDPval são os benchmarks de palco escolhidos a dedo pelo Google, e a comparação é contra o próprio antecessor antigo — não contra o Opus 4.7 ou o GPT-5.5. O salto agentic é real; "vence a fronteira" é uma manchete.

llm-stats.com · latent.space

Antigravity 2.0 · a plataforma fica séria [4][7]

No I/O, o Antigravity chegou na versão 2.0 — o passo de IDE para plataforma: um app desktop com interface agent-first, uma CLI com subagentes especializados, um SDK para controle programático do harness, e Managed Agents na API Gemini com sandbox Linux hospedado.

O número de palco do SO em 12 horas

A demo de palco: o Antigravity + Gemini 3.5 Flash construiu um sistema operacional funcional do zero em 12 horas — com 93 subagentes paralelos, 15.000+ requests ao modelo, 2,6 bi de tokens, menos de US$ 1.000 em créditos. Leia como o que é: uma rodada de força bruta maximamente paralelizada para o palco. A mensagem aos desenvolvedores é "paralelismo massivo fica acessível", não "seu próximo SO se escreve sozinho".

developers.googleblog.com
Managed Agents (sandbox hospedado): em vez de o seu agente rodar código na sua máquina, ele roda numa caixa Linux isolada na nuvem do Google — como um Linux descartável só para esta tarefa. Se algo der errado, a caixa some e a sua máquina fica limpa. Prático para agentes paralelos, arriscado para confiança de dados (seu código fica brevemente com o Google).

Ecossistema MCP · WebMCP como proposta de padrão [6]

Agentes de navegador ganham um padrão aberto de ferramentas

Um dos anúncios de dev mais substanciais do I/O: o WebMCP, um padrão web aberto proposto que deixa sites exporem ferramentas estruturadas — funções JS e formulários HTML — para que agentes baseados em navegador executem tarefas complexas neles. O origin trial experimental começa no Chrome 149. O MCP venceu no lado do servidor; o WebMCP leva a mesma ideia para o navegador. Ainda cedo, mas a direção é clara.

developers.googleblog.com
MCP (Model Context Protocol): um plugue único pelo qual um modelo de IA conversa com ferramentas externas — como o USB para ferramentas de IA. Em vez de fazer uma conexão sob medida para cada app, todo servidor fala "MCP" e todo agente o entende. O WebMCP traz esse plugue para dentro do próprio site.
