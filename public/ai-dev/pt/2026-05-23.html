Claude Code 2.1.149 · detalhamento de uso e três correções de segurança [1]

Num dia em que a maioria dos fornecedores está de fim de semana, o Claude Code 2.1.149 (22/05, 22:09 UTC) é o único release estável substancial — e tem dois rostos: um recurso simpático e um bloco incomumente denso de correções de segurança.[1]

O recurso visível: /usage com detalhamento

O /usage agora mostra um detalhamento por categoria — quanto do orçamento de tokens vai para skills, subagents, plugins e cada servidor MCP individual. Para quem roda vários servidores MCP conectados, é a primeira forma de ver qual servidor custa quanto, em vez de ler só uma soma total. Além disso: a visão de detalhe do /diff agora é rolável pelo teclado (setas, j/k, PgUp/PgDn), o Markdown renderiza checkboxes de task-list GFM, e o enterprise ganha uma configuração gerenciada allowAllClaudeAiMcps.

A substância de verdade: as correções de segurança

Mais pesadas que o recurso são três correções que tocam diretamente no setup do Daniel. Dois bypasses de permissão no PowerShell foram fechados: comandos embutidos como cd.. ou cd\ conseguiam escapar do diretório de trabalho liberado, e regras de allow com prefixo/curinga podiam ser contornadas. Mais um fix de sandbox de worktree (a allowlist de escrita não se aplicava direito a worktrees do git) e — o mais desagradável — um bug do find que esgotou a tabela de vnodes do macOS e travou a máquina inteira.[1]

Tabela de vnodes: o macOS mantém internamente uma lista de todos os arquivos e pastas atualmente "tocados" — como um guarda-volumes com um número limitado de ganchos. Um find fora de controle pendurou tantos casacos que o sistema ficou sem nenhum gancho livre e travou. Não foi só a ferramenta que caiu — foi o host.

Contexto: bypasses de permissão num agente que executa comandos de shell não são cosmética — são exatamente a classe de bug em que uma promessa de sandbox vaza em silêncio. Na prática, para o Daniel: atualizar direto para o 2.1.149 (ou 150), sobretudo quem roda o Claude Code com servidores MCP conectados ou no Windows/PowerShell.
