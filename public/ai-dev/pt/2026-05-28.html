Antigravity 2.0 · uma semana após o bang [1][2][3]

Nove dias atrás, Google apresentou sua ferramenta de desktop agent-first na versão 2.0 no I/O — aplicação independente para macOS/Linux/Windows, sub-agentes paralelos, tarefas de background agendadas, uma CLI agy, um SDK, integração com AI Studio/Android/Firebase. A arquitetura, esse é o consenso das resenhas independentes dos últimos dias, é real: você não consegue nada disso hoje em Claude Code ou Cursor.[1]

A entrega foi um problema. Google automaticamente aplicou patches em instalações existentes da v1 no dia do lançamento — removendo o editor integrado, movendo configs para um novo caminho (= perdido), derrubando controle de código-fonte e conexões remotas da imagem padrão. Qualquer um rodando Antigravity em CI/CD encontrou uma interface de chat em vez de uma IDE na quarta-feira. O ecossistema se fragmentou naquele dia em três downloads separados: Antigravity 2.0 (shell de agentes), Antigravity IDE (codificação tradicional), e Antigravity CLI. Quem queria o editor de volta tinha que instalar a IDE manualmente.[2]

Camada grátis e lacuna na CLI

Dois números que contribuíram para o clima: a camada grátis caiu de 250 para 20 requisições/dia — muito pouco para avaliar seriamente a ferramenta sem uma conta paga. E a CLI agy, que substitui a Gemini CLI em 18.06., não estava disponível via um package manager público até 21.05. Isso criou pressão de migração pura em pipelines CI que dependiam de npm i -g @google/gemini-cli, sem sistema-alvo à vista.[2]

Atualização desde ontem: O instalador oficial agora está disponível. curl -fsSL https://antigravity.google/cli/install.sh | bash coloca o binário nativo agy em ~/.local/bin (Unix) ou %LOCALAPPDATA%\Antigravity\ (Windows). Sem pacote npm — Google explicitamente se afastou do fluxo de trabalho Node.[3]

chatforest.com (Resenha, 3/5) · antigravity.google/docs/cli-features

Para Daniel: não instale hoje

Antigravity 2.0 não é um substituto direto para Claude Code; "revisitar em 60–90 dias" (o conselho honesto da resenha) é a chamada. O que vale a pena seu tempo hoje do mundo Antigravity é o servidor MCP construído nele — Chrome DevTools for agents, utilizável sem Antigravity (próxima seção).

Chrome DevTools for agents · padrão silencioso [4][5]

Enquanto Antigravity marca os destaques, algo mais substancial roda em segundo plano: Chrome DevTools for agents 1.0 ficou estável em 19.05., o pacote npm chrome-devtools-mcp estava em 1.1.1 ontem (tag de lançamento 27.05.). Isso dá ao seu agente de codificação verdadeiros olhos de navegador: console logs, requisições de rede, traces de performance, snapshots de memória, audits lighthouse, emulação de dispositivos — tudo via MCP em vez de um sandbox de captura de tela.[4]

Chrome DevTools for agents (via MCP): Imagine seu agente de IA sentado cego diante da tela, escrevendo código de frontend — costumava ter que adivinhar se a página até carregava. Com este servidor MCP, ele consegue a mesma ferramenta de inspeção que você abre no Chrome via F12: vê quais requisições falham, qual imagem leva 3 segundos, quais campos de entrada carecem de rótulo para leitor de tela. Em vez de "acho que o bug está aqui" → "Lighthouse diz Z, conserte Y."

A lista de clientes é a surpresa real: pré-instalado em Antigravity, plugin de marketplace em Claude Code, extensão MCP em Gemini CLI, além de oficialmente listado para Cursor, Copilot (VS Code + CLI), Codex, Cline, Command Code, Factory CLI, JetBrains AI Assistant, Junie, Katalon, Mistral Vibe, OpenCode, Qoder, Visual Studio, Warp, Windsurf.[5] Isso não é mais só uma ferramenta Google em qualquer sentido significativo exceto marca — é infraestrutura compartilhada.

Managed Agents + AI-Studio Android [8]

Managed Agents na API Gemini (Preview)

Uma chamada de API dispara uma sandbox Linux isolada onde um agente raciocina, usa ferramentas e executa código — o mesmo arnês atrás de Antigravity. Você define seus próprios agentes como arquivos versionáveis AGENTS.md e SKILL.md. Disponível via Interactions API + AI Studio, "Public Preview." Essa é a resposta do Google ao Codex-as-a-Service do OpenAI e Agent SDK do Anthropic — mesmo bloco conceitual, fornecedor diferente.[8]

Managed Agents: Em vez de construir um loop de agente você mesmo (chamada de modelo → chamada de ferramenta → resultado → próximo passo), você consegue um container de agente pronto do fornecedor onde você só diz "faça X." Vantagem: menos boilerplate, isolamento automático de sandbox. Desvantagem: menos controle, lock-in de fornecedor, limite de segurança é o fornecedor.

Apps Android nativos no AI Studio

Um prompt → projeto Kotlin/Jetpack Compose, visualização ao vivo em um emulador de navegador ou direto para dispositivo via ADB, um clique no Google Play Internal Test Track. Esta é a peça do I/O que voou mais sob o radar, e a mais prática: ninguém teve que instalar Android Studio, ninguém teve que assinar, ninguém teve que publicar. Interessante para projetos de hobby e prototipagem; tangencial ao trabalho principal de Daniel (stack TypeScript/Python).

A metade áspera é Antigravity dois-ponto-zero, o novo app de desktop agent-first do Google. A arquitetura é real: orquestrador de sub-agente paralelo, tarefas de background agendadas, um agente de navegador. Você não consegue nada assim hoje de Claude Code ou Cursor. Mas Google empurrou uma atualização automática para instalações existentes no dia do lançamento — e removeu o editor integrado, perdeu configs, quebrou pipelines de integração contínua. Qualquer um rodando Antigravity em produção encontrou uma interface de chat em vez de um ambiente de desenvolvimento na quarta-feira. Resenhas estão em três de cinco. A metade substancial é Chrome DevTools for agents na versão um-ponto-zero. Esse é o servidor Model Context Protocol dando ao seu agente verdadeiros olhos de navegador — console logs, requisições de rede, traces de performance, audits lighthouse. Imagine seu agente sentado cego diante da tela de outra forma e adivinhando se a página até carrega; este servidor é a inspeção F-doze que você abre você mesmo, apenas legível por máquina. Pré-instalado em Antigravity — mas também como plugin em Claude Code, Cursor, Copilot, JetBrains, Codex e uma dúzia de outros. Tornou-se silenciosamente infraestrutura compartilhada. Recomendação de Daniel: conecte em Claude Code hoje, baixa fricção, alto alavancagem em trabalho de front-end.
