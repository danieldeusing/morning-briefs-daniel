Opus 4.8 · números de vendors e a lacuna Verified [1][2][3]

Ontem às 18h00 UTC, Claude Opus 4.8 foi ao ar — simultaneamente em todos os lugares (claude.ai, API, Claude Code, Bedrock, Vertex, Foundry). Anthropic o posiciona como "um colaborador mais eficaz": julgamento mais aguçado, mais honestidade sobre seu próprio progresso, menos manipulação em código. A frase de marketing que gruda: Opus 4.8 é "quatro vezes menos provável que seu predecessor" de deixar passar seus próprios erros de código sem comentário.[1]

"Deixar passar erros de código": Quando um agente escreve uma função que quebra um teste, e o modelo ainda diz "pronto" mesmo tendo visto a falha do teste — isso é "deixar passar". Segundo Anthropic, no Opus 4.7 isso acontecia em X% dos casos, no 4.8 em X/4%. A base de números que Anthropic não publica; isso é um comportamento que você só pode verificar em seus próprios logs.

A tabela de benchmarks que Anthropic envia com seu post fica respeitável: Agentic Coding 64,3 → 69,2%, Multidisciplinary Reasoning with Tools 54,7 → 57,9%, Agentic Computer Use 82,8 → 83,4%, trabalho de conhecimento (Elo) 1.753 → 1.890, Agentic Financial Analysis 51,5 → 53,9%.[2] O que não está nessa lista e também não está no verificável SWE-Bench-Verified: um score para Opus 4.8. Dynamic Workflows · Claude escreve seu próprio script [5][6]

A metade menos chamativa, mas estruturalmente mais importante do lançamento Anthropic de ontem: Dynamic Workflows, Research Preview, desbloqueado a partir de Claude Code v2.1.154. Em vez de Claude spawnar subagentes turn-by-turn e cada resultado intermediário inchar a janela de contexto, cada requisição "workflow" agora recebe seu próprio script JavaScript independente, que Claude escreve ele mesmo — e um runtime dedicado executa o script em background, enquanto a conversa permanece responsiva.[5]

Dynamic Workflow: Imagine pedir a um chefe um menu de três pratos. Até agora, ele pensava em cada prato em paralelo e mantinha o plano todo na cabeça — eventualmente o primeiro prato escapava porque sua memória ficava cheia. Agora ele escreve uma anotação: "entrada dois chefs frio, prato principal quatro chefs quente, sobremesa sequencialmente" — a anotação é o script, a cozinha é o runtime. O chefe (Claude) pode pular de volta na conversa a qualquer momento sem interrupção do trabalho; o script roda separado. E a anotação é código — você pode ler, salvar, executar novamente com um comando na próxima vez.

Os limites duros estão na documentação oficial: Até 16 agentes em execução simultânea por rodada (menos em hardware fraco), 1.000 agentes total por rodada (previne loops descontrolados), sem entrada de usuário no meio da rodada (apenas prompts de permissão podem pausar). Disponível em Pro, Max, Team, Enterprise, plus Anthropic API, Bedrock, Vertex e Microsoft Foundry. Em Pro você alterna a linha Dynamic-Workflows em /config.[6]

Três maneiras de iniciar um workflow

  • Deixe a palavra "workflow" no seu prompt — Claude Code a destaca e escreve o script em vez de trabalhar turn-by-turn. (alt+w desativa o gatilho por prompt.)
  • Workflow incluído: /deep-research <pergunta> é o built-in — dispara buscas web em sua pergunta, fontes se checam mutuamente, resultado é um relatório citado onde claims que não sobreviveram ao cross-check já estão filtrados.
  • /effort ultracode: acopla raciocínio xhigh com orquestração automática de workflow — Claude decide sozinho quando um task precisa de um workflow. Maior consumo de tokens, latência maior por request; gatilho manual permanece padrão.

Workflow como um comando versionável + showcase Bun

Quando uma rodada faz o que você quer, pressione s na visualização /workflows — o script vira um slash-command no seu repo (.claude/workflows/ ou ~/.claude/workflows/) e é executável na próxima branch como /<nome>. Docs são limpas: subagent = worker, skill = instructions, workflow = o script orquestrando ambos. O próprio showcase Anthropic: uma porta Zig→Rust do runtime Bun orientada por workflow com "aproximadamente 750.000 linhas de Rust", 99,8% testes verdes, em 11 dias.[5] Caso de uso de vendor, não benchmark independente — Bun também não é representativo de codebases de mercado médio. Mostra o teto, não a expectativa de segunda-feira. Tente Entrada baixa: /deep-research em uma pergunta de pesquisa; workflows customizados: leia o script antes de commitar com s.

Se a rodada fica boa, você pressiona S na visualização de workflow e o script fica salvo como um slash-comando versionado no seu repo. Para seu stack solo, esse é o alavanca mais direto do lançamento de ontem — barreira de entrada baixa, substância alta.
