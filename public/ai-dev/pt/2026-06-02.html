Claude Opus 4.8 — o único grande lançamento que você pode tocar hoje

Em 28/05, Anthropic entregou Opus 4.8 — sem preview, sem fila de espera, API desde o primeiro dia. Para um dev, essa é a história do dia porque é o único dos quatro modelos "grandes" que realmente existe agora (os outros três abaixo são anúncios). O salto atinge exatamente onde Daniel trabalha: o score agentic-coding (SWE-Bench Pro) salta de 64,3 % para 69,2 % — medido por Anthropic, então leia com o ceticismo usual.[1][2]

Programação agentic: Não "escreva essa função para mim", mas "trabalhe na tarefa inteira por conta própria" — ler arquivos, rodar testes, ver erros, iterar, em muitas etapas. Como um estagiário que não pede orientação após cada linha. Isso é exatamente o que o score agentic mede: sucesso em toda uma cadeia de tarefas, não em um único prompt.

A parte realmente agradável: o preço fica igual ao Opus 4.7 — $5 por milhão de tokens de entrada, $25 por milhão de saída.[1] Mais poder pela mesma taxa é raro; normalmente você paga pelo modelo melhor. Mais duas adições relevantes para Claude Code: Dynamic Workflows (Research Preview) deixam Claude planejar uma tarefa e depois orquestrar centenas de subagentos paralelos em uma sessão — e um controle Effort Control ao lado do seletor de modelo para ajustar quanto "pensamento" o modelo coloca em uma resposta.[1]

Fast Mode agora roda a ~2,5x de velocidade e é três vezes mais barato que o Fast Mode anterior ($10/$50 versus anterior mais caro).[1] tente Por que tentar: Para execuções agentic longas em Claude Code, 4.8 é um upgrade quase gratuito — mesma taxa, e Fast Mode reduz custos dos passos intermediários rápidos notavelmente. Apenas aumente o modelo padrão e execute uma tarefa real contra ele.

SWE-Bench Verified: Empatado no topo — e uma entrada que você não pode usar

Com o lançamento de 4.8, vale a pena olhar para o leaderboard — mas com o ceticismo que o spec exige. A partir de 02/06, conforme llm-stats.com, Opus 4.8 está em 88,6 % e GPT-5.5 (reportado por OpenAI) em 88,7 % — dentro da tolerância de medição, então empatado, sem vantagem.[3][4] Opus 4.7 mantém 87,6 %, Gemini 3.1 Pro 80,6 %. A página oficial do SWE-Bench renderiza sua tabela via JavaScript e não forneceu números ao buscar — daí a fonte independente de cross-check, verificada contra andrew.ooo (a partir de 11/05).

SWE-Bench Verified: 500 bugs reais do GitHub de projetos open-source reais, verificados como solúveis por humanos. O modelo recebe o repositório e o ticket, escreve um patch — passa apenas se os testes reais ficarem verdes depois. "88 %" significa: quase nove em dez desses bugs, o modelo realmente consertou.
Ceticismo No topo, llm-stats lista uma entrada "Claude Mythos Preview" com 93,9 %.[3] Essa é real: a Anthropic a anunciou em 07/04 com system card completo — mas deliberadamente não disponível ao público, restrita a um pequeno grupo de parceiros do "Project Glasswing" para trabalho de segurança defensiva. O score é genuíno, mas você não consegue nem chamar o modelo pela API pública nem reproduzir o número. Trate os modelos GA abaixo como o sinal acionável — Mythos é um preview de capacidade, não algo em que dá para produzir.

O que o número não diz: Gemini 3.5 Flash ainda não aparece no board Verified independente — Google cita ~78 % (auto-medido), mas só o 3.1 Pro mais antigo está listado.[4] Número de fornecedor ≠ medido independentemente; até que Flash esteja no board, esse 78 % é uma promessa.

Gemini 3.5 Pro: anunciado, não entregue

Trechos de busca gostam de reclamar que Gemini 3.5 Pro está "agendado para junho" — isso é verdadeiro apenas como uma promessa. No changelog oficial da API Gemini, ainda não há entrada para 3.5 Pro até hoje (02/06), nem GA nem preview; só Flash está listado, GA desde 19/05.[5] No palco da I/O, Pichai disse aproximadamente "nos dê até o próximo mês" — esse mês está correndo agora.[6] Isso significa para Daniel: nada para tentar ainda, independentemente do que os agregadores de blogs sugiram com tabelas de spec (contexto de 2M, Deep Think — tudo não confirmado).

O que é realmente utilizável hoje é Gemini 3.5 Flash: Google reporta 76,2 % no Terminal Bench 2.1 — sólido para um tier Flash, mas pouco atrás de GPT-5.5 (78,2 %) e também medido pelo fornecedor.[7] Para programação agentic, sólido e barato; não alcança Opus 4.8 / GPT-5.5 nas tarefas difíceis.

MiniMax M3 — o fio de pesos abertos se move, mas na metade

Ontem (01/06) lançado: MiniMax M3, um modelo de pesos abertos chinês com contexto de 1 milhão de tokens, construído para execuções longas de agent coding.[8] Os números do fornecedor são impressionantes — SWE-Bench Pro 59,0 %, Terminal Bench 2.1 66,0 % — mas duas ressalvas que o spec exige: (1) todos os benchmarks são medidos pelo fornecedor, e (2) os pesos ainda não estão fora — API está ao vivo, pesos abertos e relatório técnico seguem em "~10 dias".[9]

Pesos abertos: Os pesos do modelo acabado são publicados — você pode baixar o modelo e rodar em seu próprio hardware sem uma API em nuvem. Como uma receita que você leva para casa em vez de comer apenas no restaurante. Mas enquanto os pesos não estão saindo, "peso aberto" é apenas uma promessa — você não pode auto-hospedar ou medir independentemente nada.

Para configurações locais (mundo Ollama/Qwen de Daniel), M3 é ainda não uma opção — espere até que os pesos apareçam no Hugging Face e os primeiros re-testes independentes venham. No board Verified independente, os líderes de pesos abertos permanecem DeepSeek-V4-Pro-Max (~80,6 %) e Qwen 3.7 Max (~80,4 %), seguidos de perto pelo M2.5 mais antigo do MiniMax.[3]

Ferramentas: Cursor se afasta, Claude Code fica cauteloso

Cursor 3.6 — agente roda mais sem um clique

Em 29/05 veio o "modo auto-review run": as chamadas de shell, MCP e fetch do agente não são mais aprovadas individualmente — chamadas de allowlist rodam instantaneamente, as sandboxáveis rodam na sandbox, o resto vai para um subagentoClassificador que decide: permitir, tentar diferente, ou perguntar.[11] Essa é a mesma tendência dos Dynamic Workflows do Opus — menos fricção de aprovação para execuções longas. Antes disso (20/05) veio /loop para execuções de agent local repetidas e Composer 2.5 ($3/$15 Fast).[11] tente Qualquer um rodando refactors longos em Cursor se poupa dos cliques constantes com auto-review.

Claude Code v2.1.160 — prompts de segurança

As compilações recentes estão apertando a segurança: um prompt antes de escrever em arquivos de inicialização do shell (.zshenv, .zlogin, .bash_login) e antes de configurações de build-tool que desbloqueiam execução de código — até em modo acceptEdits.[12] Uma resposta sensata à superfície de ataque "agente silenciosamente escreve em seu init do shell". Além disso, plugins de .claude/skills agora carregam automaticamente, sem o marketplace.[13]

Atenção · 15/06 Cobrança de Claude Code se divide: uso interativo de terminal/IDE mantém assinatura, mas uso programático (Agent SDK, claude -p, integração GitHub Actions) passa a usar um novo pool mensal, cobrado a taxas de API completas.[14] Se usa Claude Code em scripts/CI, calcule primeiro.

Na margem

Anthropic supostamente apresentou um pedido de IPO confidencialmente — essa é uma história de mercado/finanças que pertence ao brief de stocks-crypto, não aqui. Para devs, a única questão relevante é se uma listagem pública muda o ritmo de lançamento (recentemente 4.7 → 4.8 em menos de dois meses) e preços — observe.

É terça-feira, dois de junho. Hoje é um dia para olhar com cuidado o que foi realmente entregue versus o que foi apenas anunciado — porque os anúncios se acumulam, mas apenas um deles você pode realmente usar agora.

Esse lançamento real é Claude Opus quatro vírgula oito. Anthropic entregou na quinta passada, vinte e oito de maio — sem fila de espera, direto pela interface de programação. O ponto-chave para Daniel: o score para programação agentic salta de sessenta e quatro para sessenta e nove por cento, medido pela Anthropic, então leia com ceticismo. A melhor parte é o preço fica igual — cinco dólares por milhão de tokens de entrada, vinte e cinco para saída. Mais poder pela tarifa antiga é raro.

Olhando para o leaderboard, Opus quatro vírgula oito e GPT cinco vírgula cinco estão praticamente empatados neste teste para bugs reais do GitHub, ambos em torno de oitenta e oito por cento. No topo absoluto existe uma entrada chamada Claude Mythos Preview com quase noventa e quatro por cento. Essa é real e anunciada pela Anthropic, mas deliberadamente restrita a poucos parceiros de segurança — então você não consegue usá-la nem reproduzir o número. Trato os modelos disponíveis ao público como o sinal real.

Com Google, a situação é mais clara do que os títulos alegam. Gemini três vírgula cinco Pro não foi entregue. O changelog oficial não tem nenhuma entrada para ele. Apenas a versão mais rápida Flash está disponível, desde dezenove de maio. Pichai disse no palco, nos dê até o próximo mês — e esse mês está correndo agora.

Do mundo dos modelos abertos veio MiniMax M3 ontem, um modelo chinês com janela de contexto de um milhão de tokens. Mas também aqui duas ressalvas: os benchmarks são todos medidos pelo fornecedor, e os pesos nem foram publicados ainda. Quem quer rodar algo localmente espera aproximadamente dez dias até você poder baixar e re-testar independentemente.

Grok cinco de xAI permanece um ponto no mapa. O modelo ainda está em treinamento, e mercados de previsão dão apenas um terço de probabilidade de que saia até fim de junho. Não planeje com isso.

Em ferramentas, coisas mais concretas estão acontecendo. Cursor na versão três vírgula seis agora deixa seu agente rodar mais sem cliques constantes de aprovação. Claude Code na versão dois vírgula um vírgula cento e sessenta fica cauteloso e pergunta antes de escrever em seus arquivos de inicialização do shell. E Aider está quieto desde doze de fevereiro — mais de três meses sem lançamento.

Importante marcar: em quinze de junho, cobrança de Claude Code muda. Se usa em scripts ou pipelines automatizados, você pagará de um pool separado com taxas de API completas.

Amanhã veremos se os primeiros benchmarks MiniMax independentes chegam assim que pesos são testáveis — e se Google cumpre sua promessa Pro. Até lá: tente o que realmente existe, o resto ainda é roadmap.
