Microsoft treina seu próprio modelo Copilot

Esta é a história do dia, e é mais estratégica do que parece. No keynote Build 2026 em 2 de junho, Microsoft revelou sete modelos proprietários MAI — imagem, fala, transcrição, um modelo de raciocínio (MAI-Thinking-1) — e no meio, o único que importa para nós desenvolvedores: MAI-Code-1-Flash, o primeiro modelo de codificação proprietário da Microsoft, lançado no GitHub Copilot desde 2 de junho.[1] Aparece no seletor de modelos do VS Code e é distribuído em todos os planos — Gratuito, Pro, Pro+, Max.

O ponto real: até agora, Copilot funcionava em modelos de terceiros — OpenAI, Anthropic, Google. MAI-Code-1-Flash é o primeiro que Microsoft possui, de ponta a ponta. De acordo com Microsoft, foi treinado em "dados limpos e devidamente licenciados", explicitamente sem destilação de modelos OpenAI[2] — uma crítica ao parceiro do qual Microsoft agora está ganhando independência.

Destilação: Um modelo pequeno aprende imitando as respostas de um modelo grande e caro — como um aprendiz observando o mestre pelo ombro. Microsoft diz: não deixamos nosso aprendiz observar OpenAI. Importante, porque modelos destilados estão legal e contratualmente ligados ao seu "professor".

Arquitetura: uma Mixture-of-Experts esparsa com 137 bilhões de parâmetros no total, mas apenas 5 bilhões ativos por token, contexto de 256K.[3] (O cartão do modelo oficial diz 137B/5B; o post da família Microsoft encurta chamando de um "modelo 5B" — significando a contagem de parâmetros ativos. Isso causou confusão.[4])

Mixture-of-Experts (esparsa): Em vez de executar todo o modelo gigante para cada consulta, o modelo tem muitos "especialistas" e encaminha cada token apenas para os poucos que combinam — como um hospital que não examina você por todos os 137 médicos de uma vez, mas apenas pelos 5 relevantes. Resultado: o poder de um modelo grande ao custo computacional de um pequeno.

Os benchmarks — leia com ceticismo

O cartão do modelo de Microsoft coloca MAI-Code-1-Flash diretamente contra Claude Haiku 4.5, a classe de comparação óbvia (modelo pequeno, rápido). Os números são, e isto é crucial, medições próprias do fabricante — MAI-Code-1-Flash ainda não está (a partir de 05.06) no índice de codificação independente de Artificial Analysis.[5] Até que a replicação chegue, é: interessante, mas não confirmado.

SWE-Bench Verified — Alegações do fabricante, fontes mistas, a partir de 02.06.2026

No SWE-Bench Verified, o cartão relata 71,6% para MAI-Code-1-Flash versus 66,6% para Haiku 4.5 — e, quase mais interessante que o score: em aproximadamente 10,8K em vez de 27,3K tokens por tarefa.[3] No SWE-Bench Pro mais difícil, a diferença é maior (51,2% vs. 35,2%), no SWE-Bench Multilingual mais apertada (65,5% vs. 62,7%). Nas tarefas difíceis, Microsoft alega até 60% menos tokens.[4]

SWE-Bench Verified: Uma coleção de bugs reais do GitHub com suites de teste reais. O modelo deve encontrar o bug no repositório e escrever um patch que torne os testes verdes — como um mecânico que tem que consertar o carro de um estranho e no final passar em um test drive real. "Verified" significa: um humano verificou que as tarefas são solucionáveis e os testes são justos.
Filtro de ceticismo Duas ressalvas. Primeiro: fabricante mede a si mesmo — Microsoft treinou MAI-Code-1-Flash especificamente no harness Copilot contra o qual é benchmarked aqui; vantagem de casa é real e pode ser distribuída diferentemente na natureza. Segundo: eficiência de token é a parte mais credível da história. Menos tokens = custos mais baixos e latência para os milhões de usuários do Copilot — e é isto (não o último ponto percentual SWE-Bench) que é o motivo real de Microsoft.

Copilot se torna agêntico: Agent HQ tente

O segundo grande tema Build para codificação: Agent HQ. GitHub agora permite que você execute múltiplos agentes de codificação em paralelo — direto no GitHub, GitHub Mobile e VS Code — e não apenas o próprio Copilot, mas também Claude da Anthropic e OpenAI Codex (ambos em Preview Pública) lado a lado.[6] Para usuários Pro+ e Enterprise.

Isto é mais que marketing. Um agente orquestrador pode atribuir subagentes a workstreams separados — linting, geração de testes, documentação, revisão de segurança rodam simultaneamente em vez de sequencialmente. Se você trabalha com Claude Code ou seu próprio harness, conhece o padrão; o que é novo é GitHub construir como um recurso first-party e cross-vendor, com contexto, histórico e revisão em um lugar.

Por que tentar Se já tem Copilot Pro+: Agent HQ é o primeiro lugar onde você pode colocar Claude e Codex um contra o outro no mesmo workflow sem três ferramentas abertas. Segundo relatos (techtimes.com), Enterprise receberá um Modo de Agente Autônomo que escreve, testa e faz commit em branches inteiros de features — merge ainda com aprovação humana; GitHub ainda não nomeou ou agendou oficialmente este recurso.[7]

Codex agora disponível no Amazon Bedrock

Em paralelo, um dia antes de Build: o agente de codificação Codex da OpenAI, juntamente com GPT-5.5 e GPT-5.4, tornou-se disponível no Amazon Bedrock em 1º de junho.[8] Acesso via App Codex, CLI e integrações IDE (VS Code, JetBrains, Xcode). OpenAI relata mais de 5 milhões de desenvolvedores por semana usando Codex.

O apelo para lojas AWS: toda a inferência funciona através de Bedrock e assim sob os controles de governança AWS que você já tem — IAM, isolamento VPC/PrivateLink, criptografia KMS, auditoria CloudTrail. Prompts não são usados para treinamento. Preço "corresponde às taxas first-party da OpenAI", pay-per-token, sem licenças de assento.

Contexto Três dias, três labs, um padrão: Microsoft agora possui seu modelo Copilot, OpenAI distribui Codex através do braço de cloud de seu concorrente, GitHub torna todos os três intercambiáveis no Agent HQ. O modelo de codificação em 2026 não é mais um produto, mas um componente que você conecta dependendo de plataforma, preço e conformidade.

Gemini 3.5 Flash como um modelo de codificação

Um pouco mais antigo (Google I/O, 19.05), mas pertence aqui no contexto de codificação: Gemini 3.5 Flash é o modelo mais forte do Google para programação e agentes autônomos — não para chat. Google o vende abertamente como um modelo agêntico: planejar, construir, iterar com intervenção humana mínima, aproximadamente 4× mais rápido que modelos frontier comparáveis.[9]

Os números (novamente, do fabricante): Terminal-Bench 2.1 76,2%, MCP Atlas 83,6%, GDPval-AA 1656 Elo — cada um sobre Gemini 3.1 Pro. Disponível via Antigravity, Gemini API e Enterprise. Gemini 3.5 Pro deve chegar "próximo mês" (Junho), já in-house; um contexto de 2M-token circula apenas em relatos secundários e ainda não foi confirmado oficialmente pelo Google.[10] Para nós, Flash é principalmente interessante como um backend agêntico barato e rápido em harnesses que tornam modelos livremente intercambiáveis — veja Agent HQ acima.

Open-Weights: MiniMax M3 à frente, Qwen eficiente Local

Para o stack local de Daniel com Ollama/Qwen, algo também está se movimentando. Dois lançamentos notáveis:

MiniMax M3 (Junho) lidera o ranking open-weight no SWE-Bench Pro com 59,0% e é o primeiro modelo open-weight combinando codificação frontier, contexto de 1M e multimodalidade nativa, segundo kilo.ai.[11]

Qwen3-Coder-Next permanece o rei da eficiência: 80B MoE / apenas 3B ativos, contexto de 256K, funciona em um MacBook de 64GB ou uma RTX 5090. Números do fabricante: SWE-Bench Verified 70,6 / Pro 44,3.[12] Notável: isso é desempenho de codificação de nível Sonnet-4.5 localmente — sem conta de token.

DeepSeek V4 (24.04) completa o campo — V4-Pro (1,6T/49B) e V4-Flash (284B/13B), ambos contexto de 1M, licença MIT. Muito grande para a maioria localmente, mas valioso como modelo frontier auto-hospedável.

Chamada de ferramentas

Cursor 3.6 (29.05) traz Revisão automática: um modo de execução que deixa Cursor trabalhar mais sem perguntar. Chamadas Shell, MCP e fetch ou passam imediatamente (allowlist), executam em sandbox — ou um subagente classificador decide permitir, resolver diferentemente ou perguntar mesmo assim.[13] Este é o mesmo trend "menos fricção de aprovação" que o Modo de Agente Autônomo do Copilot — agentes conseguem rodar mais tempo sozinhos, humanos controlam apenas os limites.

Cline v3.85 (25.05) alcançou GPT-5.5 e DeepSeek V4 Flash/Pro; 61k+ estrelas.[14]

Quieto nas sombras Aider ainda não tem nenhum MCP nativo — apenas bridges da comunidade, a partir de maio de 2026; isto por si é notícia (estagnação). Continue e OpenHands sem movimento notável esta semana. JetBrains AI e Plandex quietos.

O que isto significa para a toolbox de Daniel

Minha leitura: o dia confirma um trend que é bom para nós — lock-in de modelo está se dissolvendo. Agent HQ torna Claude, Codex e Copilot intercambiáveis no mesmo editor; MAI-Code-1-Flash reduz custos na extremidade baixa; Codex no Bedrock desacopla OpenAI de uma única nuvem. Se você já está trabalhando com harnesses intercambiáveis e LLMs locais, como Daniel, você ganha: mais escolha, menos dependência de fornecedor.

Especificamente digno de tentar esta semana: MAI-Code-1-Flash no seletor VS Code para edições de rotina baratas e rápidas (se o lançamento o alcançar), e Cursor 3.6 Auto-review se prompts de aprovação em execuções de agente longos o incomodam. Eu esperaria pela replicação SWE-Bench independente dos números MAI antes de confiar nos 71,6% de Microsoft.

É sexta-feira, cinco de junho. Três dias após a conferência Build de Microsoft, ferramentas de codificação estão girando em torno de uma única ideia: o modelo de IA com o qual você escreve código se torna uma peça intercambiável.

A história principal vem de Microsoft. Pela primeira vez, Microsoft colocou seu próprio modelo de codificação no GitHub Copilot, chamado MAI-Code-1-Flash. Até agora Copilot funcionava em modelos de fora de OpenAI, Anthropic e Google. Este Microsoft treinou a si mesmo, em dados licenciados limpos segundo suas contas, e pontualmente não observando OpenAI. É pequeno e rápido, um modelo chamado Mixture-of-Experts que calcula apenas uma fração de seu tamanho por solicitação.

Estou lendo os números de benchmark com cuidado. O cartão do modelo próprio de Microsoft relata no teste SWE-Bench Verified uma pontuação de setenta e um ponto seis por cento contra sessenta e seis ponto seis para Claude Haiku, e isso com significativamente menos tokens gastos. Mas isso é uma medição de fabricante. Um lugar independente como Artificial Analysis ainda não verificou o modelo novamente. Mais credível para mim que o último ponto percentual é a eficiência de token, porque menos tokens significa custos mais baixos, e esse é exatamente o motivo real de Microsoft.

O segundo tema de Build é Agent HQ. GitHub agora permite que você execute múltiplos agentes de codificação simultaneamente, e não apenas Copilot, mas Claude e Codex lado a lado, no mesmo editor. Um orquestrador atribui trabalho como testes, documentação e revisão de segurança em paralelo a subagentes. Segundo relatos de terceiros, para clientes empresariais deve vir um modo que escreve branches inteiros de features e faz commit neles — mas merge apenas com aprovação humana; o GitHub ainda não confirmou nem datou isso oficialmente.

OpenAI se encaixa bem. Um dia antes de Build, o agente de codificação Codex se tornou regularmente disponível no Amazon Bedrock, juntamente com os novos modelos GPT. Mais de cinco milhões de desenvolvedores usam Codex cada semana, e no Bedrock tudo funciona sob controles de segurança AWS familiares. Três dias, três labs, um padrão: o modelo se torna infraestrutura que você conecta dependendo de plataforma e preço.

No Google, Gemini 3.5 Flash é agora o modelo mais forte do Google para programação e agentes autônomos, aproximadamente quatro vezes mais rápido que a concorrência. A versão maior, Gemini 3.5 Pro, deve chegar depois em junho.

Para o stack local de Daniel, duas notas: o modelo open-weight MiniMax M3 agora lidera o ranking aberto, e Qwen3-Coder-Next permanece o rei da eficiência, chegando perto da qualidade de modelos caros enquanto ainda funciona em um MacBook. Entre as ferramentas, Cursor na versão três ponto seis adicionou um modo que pergunta menos vezes, e Cline alcançou os novos modelos. Aider permanece sem MCP nativo, o que por si é uma declaração.

Meu resumo: lock-in de modelo está se dissolvendo, e é bom para qualquer um trabalhando com ferramentas intercambiáveis. Próxima semana estou assistindo principalmente duas coisas: se uma fonte independente confirma os números SWE-Bench de Microsoft, e se Google realmente entrega seu Gemini 3.5 Pro ainda em junho.
