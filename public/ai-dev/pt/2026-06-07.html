O campo agora é quádruplo

A história de hoje não é um anúncio único, mas um padrão que se completou esta semana: Até recentemente, "IA para código" era essencialmente uma disputa de duas vias entre Anthropic (Claude Code/Opus) e OpenAI (Codex/GPT). Com os modelos MAI da Microsoft e o Copilot App no Build (02/06) e o Gemini 3.5 Flash mais Antigravity do Google (I/O, 19/05), todos os quatro grandes labs agora têm seu próprio stack de código, verticalmente integrado, no campo.

A CNBC resume a situação em 1º de junho assim: Microsoft e Google estão "atrasados, mas é absolutamente essencial que compitam por esse crescimento".[1] A razão está nos dados de adoção desta semana: Claude Code é o agente de código que mais cresce — pela pesquisa de desenvolvedores 2026 do JetBrains, subindo de ~3% (abr 2025) para 18% (jan 2026), aproximadamente 6× em nove meses — e entre desenvolvedores experientes (10+ anos) o claro favorito: 46% escolhem Claude Code para trabalho diário versus 9% Copilot.[2] GitHub Copilot permanece líder em números brutos (pesquisa Stack Overflow fim-2025: ~51% dos desenvolvedores profissionais, embora tenha caído de 67%; Cursor novo em ~18%, Claude Code em ~10% na primeira aparição). A receita em taxa de corrida do Claude Code foi superior a 2,5 bilhões de dólares em fev 2026. Exatamente esse território — desenvolvedores que pagam diariamente por um agente — Microsoft e Google não querem ceder à Anthropic e OpenAI.

Stack de código verticalmente integrado: Quando um fornecedor fornece tudo de uma mão — o modelo, a ferramenta ao seu redor (IDE ou agente harness) e a nuvem em que ele roda. Como um fabricante de carros que não apenas constrói o motor, mas também o carro e os postos de gasolina. Vantagem: modelo e ferramenta são treinados um no outro (MAI-Code-1 foi treinado diretamente no harness de produção do Copilot). Desvantagem para você: lock-in — o stack o puxa para um ecossistema.

Para a caixa de ferramentas de Daniel, a lição é sóbria: concorrência é boa para preços e tempo, mas quatro stacks não significa quatro opções reais. Os novatos têm que provar que seus modelos funcionam fora de seus próprios slides de marketing — e é aí que entra o filtro de ceticismo (próxima seção).

Microsoft MAI: seu primeiro modelo próprio de código e raciocínio Bench

No Build, a Microsoft apresentou sua família MAI — o ponto notável: estes são os primeiros modelos treinados pela própria Microsoft para esse papel, um passo deliberado longe da pura dependência do OpenAI.[3] Dois são relevantes para nós:

MAI-Code-1-Flash — o pequeno, rápido modelo de código que chega diretamente em GitHub Copilot e VS Code (selecionador de modelo, lançamento em Free/Student/Pro/Pro+/Max, gradualmente "ao longo das próximas semanas").[4] O truque real: Foi treinado diretamente no harness de produção do Copilot e resolve tarefas com "até 60% menos tokens" graças ao comprimento de resposta adaptativo — curto para consultas simples, mais orçamento de raciocínio para as difíceis.[5]

MAI-Thinking-1 — o primeiro modelo de raciocínio da Microsoft, treinado do zero sem destilação, em dados "limpos, licenciados comercialmente". Um MoE com ~35 bilhões de parâmetros ativos, contexto de 256K.[6]

Destilação (destilação de modelo): Um modelo pequeno aprende imitando as respostas de um modelo grande "professor" — como um aprendiz olhando por sobre o ombro do mestre. Mais rápido e barato, mas o modelo pequeno herda as peculiaridades do professor. A Microsoft enfatiza "sem destilação, do zero" — mensagem de marketing: "não copiamos um modelo do OpenAI". Relevante para você principalmente pelas origens de licença limpa dos dados de treinamento.
Ceticismo de Bench Os números são consistentemente fornecedores ou fontes secundárias, não placar independente. Relatado: MAI-Code-1-Flash ~51% SWE-Bench Pro (+16 pontos vs. Claude Haiku 4.5 em 35,2%, aproximadamente tier GPT-5.3); MAI-Thinking-1 ~53% SWE-Bench Pro e assim "equiparado com Claude Opus 4.6", 97% AIME 2025.[6][7] Para contexto: Opus 4.8 está em 69,2% Pro — então os modelos MAI jogam uma classe abaixo, mas acertam seu segmento de preço/tamanho. E cuidado: a contagem de parâmetros do MAI-Code-1-Flash é relatada contraditoriamente (site Microsoft/múltiplas fontes secundárias: 5B; uma fonte: "137B MoE") — não trataria nenhum dos dois como resolvido até que um detalhe do model card apareça.

GitHub Copilot App: de plugin de editor para centro de comando de agentes try

A notícia do Build mais praticamente relevante para mim não é um modelo, mas uma ferramenta: O GitHub Copilot App é um aplicativo desktop independente (Win 11, macOS, Linux), separado da extensão VS Code. Transforma Copilot em "um centro de comando para sessões de agentes IA paralelas — cada uma em seu próprio, isolado Git worktree".[8]

Git worktree: Múltiplas cópias de trabalho do mesmo repo lado a lado, cada uma em seu próprio branch — sem constantemente ir e vir. Como múltiplas bancadas na mesma oficina: cada uma tem uma revisão diferente acontecendo, sem se atrapalharem. É exatamente isso que o app usa para que três agentes trabalhem simultaneamente em três tarefas sem sobrescrever os arquivos um do outro.

De uma visualização "My Work" você vê sessões em execução, issues, PRs e automações em background em múltiplos repos. É exatamente o padrão orquestrador que Daniel já conhece de Claude Code — GitHub a coloca em um app nativo. Além disso, o Copilot SDK agora é GA em seis linguagens (Node/TS, Python, Go, .NET, Rust, Java); o mesmo runtime de agente em que o app roda é programável.[9]

Por que experimentar Se você já está no ecossistema GitHub e quer rodar múltiplos agentes em paralelo: o app é um competitor sério de Claude Code/Cursor para workflows de múltiplos agentes — e o SDK torna orquestração customizada possível. Visualização Técnica para Pro/Pro+/Business/Enterprise. Pule se for trabalho puro de um único agente solo.

Google Gemini 3.5 Flash + Antigravity 2.0

A resposta do Google foi amplamente lançada desde o I/O (19/05) e pertence a este quadro de paisagem — acariciada ontem sob "Pro wait", hoje com substância: Gemini 3.5 Flash é GA em Antigravity, a Gemini API em AI Studio e Android Studio, Gemini Enterprise e o app Gemini.[10] O argumento de vendas é velocidade: ~289 tokens/segundo, cerca de 4× mais rápido que Opus ou GPT-5.5, em Antigravity até 12×.[11]

Ceticismo de Bench Os valores de código relatados são números próprios do Google em suites parcialmente curadas pelo Google: Terminal Bench 2.1 76,2%, MCP Atlas 83,6%, GDPval-AA 1656 Elo.[10] Independente é aproximado: Artificial Analysis localiza Gemini 3.5 Flash no Índice de Inteligência em 55 — meio-campo sólido, não no topo de código (lá GPT-5.5 xhigh com Índice de Código 59, Opus 4.8 à frente no Índice de Inteligência com 61,4).[12] Então Flash não compete contra Opus 4.8 em qualidade, mas em qualidade por segundo por dólar — esse é seu território.

Antigravity 2.0 é a real alavanca dev: um app desktop independente mais CLI, construído agent-first, com Agentes Gerenciados na Gemini API — uma única chamada de API entrega um agente mais sandbox Linux hospedado (Bash/Python/Node, E/S de arquivo, navegação, skills definidas em Markdown, montes repo/GCS).[13] Esse é o ataque direto do Google à camada IDE-agente onde Claude Code e Cursor vivem.

O que isso significa para a caixa de ferramentas de Daniel

Minha leitura: o mercado de quatro labs é bom para Daniel — mais concorrência reduz preços e impulsiona tempo. Mas "quatro stacks" ≠ "quatro opções reais". Hoje Opus 4.8 permanece a referência de código, e Claude Code não é sem razão o agente que mais cresce e favorito dos desenvolvedores experientes. Os novatos abordam diferentes eixos: MAI o segmento barato em-Copilot, Gemini 3.5 Flash velocidade.

Concretamente digno de experimentação: o GitHub Copilot App se você roda workflows de múltiplos agentes no ecossistema GitHub, e um olhar em Antigravity 2.0 se agentes gerenciados via API o atraem. Com os modelos MAI e os scores de código do Gemini, eu aguardaria benchmarks independentes antes de qualquer troca de stack — até agora quase apenas números de fornecedores estão em. E o próximo deadline real é ainda Gemini 3.5 Pro na janela de junho.

É domingo, sete de junho, e o dia de notícias em si está calmo. Esse é um bom momento para dar um passo atrás, porque esta semana o mercado de modelos de inteligência artificial para código se completou.

Até pouco tempo, isso era essencialmente uma disputa de duas vias entre Anthropic com Claude Code e OpenAI com Codex. Esta semana os outros dois grandes players entraram no jogo. Microsoft apresentou seus próprios modelos em sua conferência Build em dois de junho, e Google lançou amplamente seu Gemini três ponto cinco Flash ao lado do ambiente de desenvolvimento Antigravity. Isso significa que todos os quatro grandes labs agora têm seu próprio stack de código. A CNBC chama Microsoft e Google de atrasados, mas diz que é essencial que compitam. A razão é simples: Claude Code é o agente de código que mais cresce oito meses após o lançamento e o favorito dos desenvolvedores experientes. Segundo a pesquisa do JetBrains, quarenta e seis por cento o escolhem para trabalho diário versus nove por cento para Copilot. GitHub Copilot ainda lidera nos números brutos, com cerca de cinquenta e um por cento na pesquisa do Stack Overflow.

Microsoft tem dois modelos que importam para nós. MAI-Code-um-Flash é um modelo de código pequeno e rápido que vai direto para GitHub Copilot e Visual Studio Code e foi treinado no verdadeiro harness de produção do Copilot. MAI-Thinking-um é o primeiro modelo de raciocínio da Microsoft, treinado do zero sem destilação. A mensagem por trás disso é menos dependência do OpenAI. O que é importante é meu filtro de ceticismo: todos os números relatados são números de fornecedores, ainda não verificados independentemente. No teste mais duro, SWE-Bench Pro, os modelos MAI marcam por volta de cinquenta e um a cinquenta e três por cento, enquanto Opus quatro ponto oito da Anthropic marca sessenta e nove por cento. Eles estão, portanto, um nível abaixo, mas acertam seu alvo de preço e tamanho.

A coisa mais praticamente interessante não é um modelo, mas uma ferramenta: o novo aplicativo GitHub Copilot. Esse é um aplicativo desktop independente, separado da extensão do editor, que orquestra múltiplos agentes de inteligência artificial em paralelo. Cada agente trabalha em seu próprio Git worktree isolado — sua própria cópia do projeto — para que não se atrapalhem. Esse é exatamente o padrão orquestrador que Daniel conhece do Claude Code, agora como um app nativo. Além disso, o kit de desenvolvedor Copilot agora está disponível em seis idiomas.

Google está apostando em velocidade com Gemini três ponto cinco Flash: cerca de quatro vezes mais rápido que Opus ou GPT, e até doze vezes mais rápido em Antigravity. Aqui também o filtro de ceticismo se aplica, porque os scores de código vêm do próprio Google e parcialmente de suas próprias suites de teste. Classificado independentemente, Flash é meio-campo sólido, não no topo de código. Seu território é qualidade por segundo e por dólar, não uma comparação direta de qualidade com Opus. Antigravity dois ponto zero em si é o ataque direto do Google contra a camada de agentes do Claude Code e Cursor, com agentes hospedados direto através da interface de programação.

Hoje havia apenas coisas menores: Anthropic expandiu sua rede de parceiros, OpenAI habilitou compartilhamento de plugin Codex por padrão na versão empresarial do ChatGPT. Entre modelos abertos, nada de novo aconteceu desde MiniMax M-três de primeiro de junho, e ferramentas como Aider, Cline e OpenHands permanecem quietas.

Minha conclusão: quatro stacks não significam quatro opções reais. Opus quatro ponto oito permanece a referência de código, os novatos abordam ângulos diferentes. Vale a pena experimentar o aplicativo GitHub Copilot para trabalho de múltiplos agentes e um olhar em Antigravity. Mas antes de qualquer mudança de stack, eu esperaria pelos benchmarks independentes. Semana que vem estou principalmente olhando se Google finalmente entrega Gemini três ponto cinco Pro.
