Claude Fable 5: a camada Mythos fica pública

A Anthropic anunciou dois modelos simultaneamente ontem, 9 de junho: Claude Fable 5 e Claude Mythos 5.[1] Eles compartilham os mesmos pesos — a mesma base de modelo treinado. A diferença é a coleira: Fable 5 é a variante protegida, disponível publicamente; Mythos 5 é o mesmo modelo com proteções relaxadas, disponível apenas para clientes autorizados no "Project Glasswing".

"Camada Mythos": a Anthropic classifica seus modelos em camadas — Haiku (pequeno/rápido), Sonnet (intermediário), Opus (principal). "Mythos" é uma nova camada acima de Opus. Pense na linha de modelos como motores: Mythos é o motor de corrida que vivia apenas no laboratório até agora. Fable 5 é exatamente esse motor, mas com um limitador de rotação para a estrada.[1]

As especificações básicas dos documentos de modelo oficiais:[2] ID da API claude-fable-5, janela de contexto de 1M tokens, até 128k de saída em tokens, preço de $10/$50 por milhão de tokens (entrada/saída). Extended Thinking não é um comutador — em vez disso, Adaptive Thinking está sempre ativo: o modelo decide por quanto tempo raciocinar. GA desde 9 de junho em Claude API, AWS Bedrock, Vertex e Microsoft Foundry.

O preço é a história real para a caixa de ferramentas: Fable 5 custa o dobro de Opus 4.8 ($5/$25), mas a Anthropic diz que é "menos da metade" do preview anterior do Mythos.[1] Uma camada acima de Opus, pelo dobro do preço de Opus — se isso compensa depende inteiramente de você confiar nos benchmarks. E é aí que fica interessante.

Os números — e o asterisco [3][4]

A Anthropic relata um salto para Fable 5 que quase maximiza a escala SWE-Bench-Verified: 95,0%, versus 88,6% para Opus 4.8. No SWE-Bench Pro mais difícil, 80,0% (Opus 4.8: 69,2%; GPT-5.5: 58,6%; Gemini 3.1 Pro: 54,2%).[3]

SWE-Bench Verified: um teste de bugs reais do GitHub — o modelo tem que corrigir um bug em um repositório real para que o conjunto de testes do projeto passe. "Verified" significa que um humano confirmou que cada tarefa é solucionável. "Pro" é o irmão mais difícil com tarefas complexas multi-arquivo.

Por que você precisa olhar duas vezes aqui — três coisas:

(1) Todos os números são relatados pelo fornecedor. Vêm do próprio anúncio da Anthropic e cartão do sistema, não de medição independente.[3] O quadro independente SWE-Bench-Verified (última atualização maio de 2026) não lista nem Fable 5 nem Mythos 5 — e nem Opus 4.8. GPT-5.5 lidera lá com 88,7%, seguido por Opus 4.7 com 87,6%.[4] Até que Fable 5 apareça verificado no quadro, os 95% é uma afirmação de fábrica, não um resultado medido.

(2) As proteções puxam o score real para baixo. Esta é a ressalva crítica: solicitações que toquem cibersegurança, biologia, química ou destilação de modelo caem automaticamente para Opus 4.8.[1] Então os números de Mythos se aplicam apenas à variante desprotegida. No Terminal-Bench 2.1, 20,9% das tentativas acionaram uma recusa e caíram — o score efetivo de Fable 5 caiu de 88,0% (Mythos 5) para 84,3%.[3] Se você escreve código adjacente à segurança — autenticação, criptografia, ferramentas de pen-test — você não obtém o modelo Mythos para partes disso; você obtém Opus.

(3) O novo tokenizador. Fable 5 usa o tokenizador do Opus 4.7 em diante — o mesmo texto produz ~30% mais tokens do que modelos mais antigos.[2] Com a taxa de $10/$50, isso significa que o custo efetivo por linha de código é maior do que o preço por token sozinho sugere. Contra Opus 4.8 (mesmo tokenizador) se anula; contra Sonnet ou modelos mais antigos, não.

Linha de fundo: mesmo com ceticismo, o salto é real — de 88,6% para algo acima de 90% mesmo conservadoramente. Mas o "95%" de manchete é melhor caso sem fricção de proteção. Para o dia a dia de Daniel (aplicações web, ferramentas, sem domínio bio/cyber), Fable 5 deve chegar perto do número relatado; para trabalho de segurança, provavelmente mais perto da paridade com Opus.

Fable 5 em toda parte desde o primeiro dia [5][6][7]

O que é notável é menos o modelo do que o lançamento. Ao contrário de lançamentos de fronteira anteriores, não havia gargalo de preview: Fable 5 estava disponível no primeiro dia em todos os ambientes que Daniel realmente usa — imediato via API e Enterprise baseado em consumo, enquanto os planos por assinatura (Pro/Max/Team e Enterprise baseado em assento) se desenrolam até 22 de junho.[1]

Claude Code v2.1.170 (09.06.) entrega suporte a Fable 5 — o changelog coloca claramente: "As capacidades de Fable superam as de qualquer modelo que já tornamos disponível publicamente."[5] Um dia antes, em v2.1.169 (08.06.), chegaram duas pequenas mas úteis melhorias na CLI: --safe-mode inicia Claude Code sem nenhuma personalização (útil quando um hook ou skill se comporta mal), e /cd muda o diretório de trabalho no meio da sessão sem invalidar o cache de prompt.[5] teste Só /cd economiza a reinicialização tediosa ao pular entre repositórios.

GitHub Copilot lançou Fable 5 como GA para Pro+, Max, Business e Enterprise — em todos os modos (Chat, Ask, Edit, Agent).[6] Um detalhe que os times precisam saber: Fable 5 força retenção de dados — prompts são mantidos por até 30 dias para os sistemas de segurança da Anthropic. Outros modelos Claude no Copilot não têm esse requisito. Para código de cliente sob NDA, esse é um critério real.

Atenção retenção de 30 dias em Fable 5 no Copilot se aplica independentemente do plano Enterprise. Se você trabalha com código de cliente altamente confidencial, verifique isso antes de mudar — ou fique com Opus 4.8 para repositórios sensíveis.

Cursor relata que Fable 5 lidera "todos os outros modelos" no CursorBench (próprio benchmark de codificação do mundo real do Cursor — número não divulgado, então trate com cautela).[7] Existe uma variante Thinking para tarefas que recompensam raciocínio extra; custa aproximadamente 2× Opus 4.8. O preço em lote ($5/$25 por milhão de tokens) é o desconto em lote padrão de 50% da Anthropic repassado — sem precificação específica do Cursor. AWS Bedrock e Databricks (via Unity AI Gateway) adicionaram dois caminhos a mais seguros para governança no mesmo dia.

Fable 5 vs. Opus 4.8 — quando usar qual

Escolha Fable 5 quando …
  • … você enfrenta uma tarefa longa e assíncrona de agente onde Opus pede esclarecimento com muita frequência — esse é o argumento da Anthropic.
  • … o domínio não é segurança/bio/química (senão retrocede para Opus).
  • … você realmente precisa de contexto grande (1M tokens).
  • … o dobro do custo é compensado por menos iterações.
Fique com Opus 4.8 quando …
  • … o custo importa (metade do preço; 88,6% de SWE-Bench geralmente é suficiente).
  • … código adjacente à segurança retrocede para Opus mesmo assim.
  • … retenção de 30 dias (Copilot) é um obstáculo.
  • … você quer confirmação do quadro independente primeiro.

Meu parecer: para o stack típico de Daniel (web, ferramentas, experimentos locais), uma execução de teste de Fable 5 em uma tarefa real vale a pena — mas Opus 4.8 continua sendo o operário padrão até que os 95% sejam verificados independentemente.

Atualização desde a semana passada

Atualização desde seg Gemini 3.5 Pro permanece preso em preview limitado no Vertex — sem GA. Anunciado no Google I/O (19.05.), apenas Flash entrou em produção; Pichai adiou para "próximo mês". Pro tem como alvo contexto de 2M e um modo "Deep Think".[8] Com o lançamento de Fable 5, a pressão sobre o Google é palpável: a Anthropic lança uma camada inteira nova enquanto o modelo de codificação frontier do Google não sai do acesso antecipado.

"Preview limitado": o modelo funciona, mas apenas para um grupo selecionado em uma plataforma em nuvem (aqui Google Vertex). "GA" (General Availability) significa que qualquer pessoa com uma conta pode chamá-lo via API. Frequentemente um ou dois meses entre.

inalterado Claude Sonnet 4.8 permanece não anunciado — sem anúncio, sem cartão de modelo, sem ID de API. Camada intermediária oficialmente fica em Sonnet 4.6 ($3/$15, melhor razão velocidade/inteligência).[9] Que a Anthropic lance a camada superior antes de atualizar o mercado intermediário se encaixa no padrão: manchete primeiro, modelos de volume depois.

É quarta-feira, dez de junho. Ontem a Anthropic revelou uma camada de modelo inteiramente nova acima de Opus — e estava disponível em quase todos os lugares onde Daniel trabalha no mesmo dia.

O modelo se chama Claude Fable cinco. A Anthropic o coloca em uma nova camada chamada Mythos, que eles dizem ficar acima de Opus. Há um modelo irmão, Mythos cinco, que é tecnicamente idêntico mas funciona sem barreiras de segurança e só é acessível para clientes autorizados. Fable cinco é a variante pública, protegida. Tem uma janela de contexto de um milhão de tokens e custa dez dólares por milhão de tokens de entrada e cinquenta dólares por milhão de tokens de saída — aproximadamente o dobro de Opus quatro ponto oito.

Os destaques são os benchmarks. A Anthropic relata noventa e cinco vírgula zero por cento em SWE-Bench Verified, comparado com oitenta e oito vírgula seis para Opus quatro ponto oito. Mas há um asterisco importante nesse número. Primeiro, é auto-relatado — no quadro independente atualizado pela última vez em maio, Fable cinco nem aparece; G-P-T cinco ponto cinco ainda lidera lá com oitenta e oito vírgula sete.

Segundo, e este é o pega mais importante: em tópicos como cibersegurança, biologia ou química, Fable cinco automaticamente retrocede para Opus quatro ponto oito. Em um dos testes, quase vinte e um por cento das tentativas acionaram uma recusa, que derrubou o score efetivo de oitenta e oito para oitenta e quatro vírgula três por cento. Se você escreve código adjacente à segurança, você não obtém o novo modelo de nível superior para partes disso — você obtém Opus.

O que foi verdadeiramente impressionante foi o lançamento. Fable cinco estava disponível no primeiro dia em toda a Claude A-P-I, GitHub Copilot, Cursor, AWS Bedrock, Vertex, Microsoft Foundry e Databricks. Claude Code versão dois ponto um ponto cento e setenta entregou suporte a Fable-cinco imediatamente. Um dia antes, versão dois ponto um ponto cento e sessenta e nove adicionou dois comandos pequenos mas práticos: um modo de inicialização segura e um comando de mudança de diretório no meio da sessão.

Um detalhe para trabalho em equipe: no GitHub Copilot, Fable cinco força armazenamento de dados de até trinta dias — ao contrário dos outros modelos Claude lá. Para código de cliente confidencial, essa é uma consideração real.

Na Google, as coisas estão quietas: Gemini três ponto cinco Pro permanece em preview limitado, sem data de lançamento firme. Com a nova camada da Anthropic, a pressão sobre o Google está aumentando. E Claude Sonnet quatro ponto oito ainda não foi anunciado — camada intermediária oficialmente fica em Sonnet quatro ponto seis.

Meu conselho para a caixa de ferramentas de Daniel: uma execução de teste de Fable cinco em uma tarefa real vale a pena. Mas até que os noventa e cinco por cento sejam confirmados independentemente e para repositórios sensíveis, Opus quatro ponto oito continua sendo o padrão sensato — metade do preço, sem retenção de dados forçada.
