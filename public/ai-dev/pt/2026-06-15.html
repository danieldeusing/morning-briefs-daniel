Divisão de faturamento Anthropic: SDK de agentes recebe seu próprio fundo [1]

Hoje, 15 de junho, entra em vigor a mudança mais importante do trimestre para a própria cadeia de ferramentas de Daniel: o uso programático de Claude é desacoplado do limite de assinatura e se alimenta de um pool de créditos mensal separado. Anunciado em 14 de maio, a data de vigência é agora.[1]

Fundo de créditos vs. limite de assinatura: Até agora, uma assinatura Max tinha um único "copo" por 5 horas, do qual tudo bebia—o humano na TUI assim como um script de fundo via claude -p. Se um agente de longa duração esvaziasse o copo durante a noite, de manhã o trabalho interativo também era limitado por taxa. A partir de hoje há dois copos: o copo de assinatura antigo para trabalho interativo (TUI, Cowork, claude.ai) e um copo separado, medido em dólares, para tudo o mais programático. Usuários de scripts não se bloquearão mutuamente—mas o fundo de scripts é finito, não se renova para o mês seguinte, e não é compartilhável em times.

O que se move para o novo fundo: Claude Agent SDK, claude -p (CLI headless), Claude Code GitHub Actions, apps de terceiros via Agent Client Protocol.[1] O que permanece no limite de assinatura: Claude Code interativo (TUI do Terminal, plugin IDE), Claude Cowork, claude.ai (Web/Desktop/Mobile).[1]

Operacionalmente para Daniel (Max 20×) $200 parece generoso, mas é por usuário (sem pool de time), expira mensalmente (sem rollover), e sai pelo preço de tabela. Uma execução longa de sub-agente com Opus 4.8 consome tudo mais rapidamente que o esperado. A partir de hoje, rastreie volume de fundo (scripts, orquestrador de briefing, revisões GitHub Action)—uma semana de dados é suficiente para saber se $200 é bastante ou se descer para Sonnet faz sentido.

Lacuna-chave: os números vêm de digitalapplied.com, não diretamente do Help Center da Anthropic (tabela de visão geral não foi exposta no momento da pesquisa). Compradores de Team/Enterprise devem abrir a documentação oficial por conta própria antes de ajustar o provisionamento.

Fable 5 + Mythos 5 desativados globalmente—controle de exportação [2][3][4]

A nova classe Mythos da Anthropic—Fable 5 (GA desde 09/06) e o preview Mythos 5—não estão disponíveis globalmente desde 12 de junho de 2026, 17h21 ET.[3] O secretário de comércio dos EUA Howard Lutnick emitiu uma diretiva de controle de exportação para Dario Amodei que bloqueia o acesso para todo nacional estrangeiro—dentro e fora dos EUA, incluindo a equipe internacional da própria Anthropic.[3] A Anthropic desativou o acesso para todos os usuários porque o status de nacional estrangeiro não pode ser verificado de forma confiável em uma API pública.

No GitHub Copilot, Fable 5 foi desativado na sexta-feira 12/06, no mesmo dia: "effective today, access to Claude Fable 5 has been suspended across all GitHub Copilot experiences"[2]—três dias após o lançamento GA em 09/06.[2]

Controle de exportação em um modelo de API: Classicamente, o Departamento de Comércio dos EUA regula a exportação de chips, algoritmos criptográficos ou software de uso duplo. Aqui, pela primeira vez, um modelo generativo de IA está sendo tratado como tecnologia de uso duplo—acusação: a capacidade de "encontrar fraquezas em código" é reutilizável para ataques cibernéticos. A contraproposta da Anthropic: a mesma capacidade existe no GPT-5.5; "este padrão interromperia relançamentos em toda a indústria."[3]

Gatilho, conforme Marktechpost: um jailbreak "narrow and non-universal" que um "parceiro de confiança altamente credível" da Anthropic e do governo dos EUA demonstrou—a técnica pede ao modelo que verifique uma base de código procurando buracos de segurança.[3][4]

Operacionalmente O fallback de modo automático em Claude Code v2.1.176 (12/06) reverte corretamente para o melhor Opus disponível.[8] Pins manuais (model: "claude-fable-5") devem trocar para Opus 4.8 hoje ou falham completamente. Sem prazo oficial de levantamento—até lá, Opus 4.8 é o melhor modelo Anthropic disponível (88,6 % em SWE-Bench Verified, veja próxima seção).

Gemini CLI → Antigravity CLI: quinta-feira é a data limite [6]

Google desliga Gemini CLI em 18 de junho de 2026, 23h59 PT (Sex 19/06, 08h59 CEST / 03h59 BRT).[6] O substituto é Antigravity CLI, disponível em geral desde 19 de maio—baseado em Go (não Python), fluxos de trabalho multi-agente assíncronos sem bloqueio de terminal, unificado com o harness de desktop Antigravity 2.0.[6]

Mude esta semana, não em quatro semanas. Quem tem Gemini CLI fixado em CI ou scripts locais verá "comando não encontrado" a partir de sexta-feira. Exceção: clientes com licença Gemini Code Assist Standard/Enterprise mantêm seu acesso—EOL afeta apenas usuários gratuitos/login.[6]

Fluxo de trabalho multi-agente (CLI): Um agente único faz loop: plano → chamada de ferramenta → resultado → próximo plano. Em tarefas longas (indexação, testes em execução, escrita de patches) isso bloqueia o terminal. Multi-agente: a ferramenta principal delega a sub-agentes que rodam em paralelo em segundo plano, e entrega o resultado. Nativo em Antigravity CLI; no antigo Gemini CLI, você tinha que construir isso você mesmo.

GitHub Copilot: faturamento por uso ativo, novo nível Max [7]

Desde 1º de junho de 2026, solicitações GitHub Copilot Premium são cobradas em AI Credits: 1 AI Credit = $0,01.[7] Completações de código e sugestões next-edit permanecem gratuitas (não cobradas em créditos); o limite se aplica apenas a chat, modo agente, revisões de pull request e espaços.[7]

O nível Max é novo—para "fluxos de trabalho de agente de alto volume sustentado": taxa fixa de $100, orçamento de $200 em créditos, efetivamente o dobro do volume por dólar. A justificativa do GitHub: "uma pergunta rápida de chat e uma sessão de codificação autônoma de múltiplas horas podem custar ao usuário o mesmo valor … o modelo de solicitação premium atual não é mais sustentável."[7] Visual Studio Magazine entitulou em 27/04: "You Will Get Less, but Pay the Same Price"—power users não veem aumento de volume, apenas custo efetivo mais alto por execução.[7]

Em paralelo com Anthropic Marcante: GitHub e Anthropic se movem em sincronismo—ambos separando consumo de fundo do limite interativo e mudando para créditos em $. As cargas de trabalho de agentes estão explodindo em queima de tokens; um limite de assinatura unificado não é mais sustentável do lado do provedor. Para Daniel: monitoramento de consumo é agora higiene de ferramentas.

OpenCode ultrapassa Claude Code em estrelas [9]

O ponto de dados OSS mais notável da semana: OpenCode—agente de código de terminal com licença MIT—está em 172.198 estrelas GitHub e é agora o agente de código open-source mais estrelado.[9] Para comparação: Claude Code ~124.000, Gemini CLI ~105.000, openai/codex ~90.000.[9] OpenCode suporta 75+ provedores de IA conforme byteiota (Claude, GPT, Gemini, Ollama)—a agnóstica de modelo é seu diferencial central.

Verificação de realidade Estrelas ≠ qualidade. Um revisor de medium.com comparou OpenCode em tarefas idênticas: ~78 % mais lento que Claude Code.[9] Voz única, sem benchmark oficial—mas acompanha o padrão de que "75+ provedores" frequentemente significa uma camada de adaptador genérico. tente-isto execute-o em um projeto pessoal—o trade-off de velocidade aparece rapidinho; se for negligenciável, a escolha de modelo é uma vantagem real.

Cursor Bugbot fica rápido e barato [10]

Cursor entregou uma atualização de Bugbot em 10/06: tempo de revisão por PR de ~5 min para ~90 s, 10 % mais bugs, 22 % mais barato.[10] Novo: comando /review. Menos de dois minutos significa para bots de PR: revisão executa enquanto o autor ainda está na mesma aba—o fluxo de trabalho fica assíncrono. Cursor está posicionando Bugbot claramente contra revisões do Claude Code via GitHub Action e CodeRabbit.
