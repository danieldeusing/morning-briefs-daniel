Fable 5 desativada—via diretiva de exportação [1][2][3]

A notícia mais importante desta semana é uma que você não encontrará em um changelog. Anthropic desativou ambos os seus mais novos modelos de topo—Claude Fable 5 e Claude Mythos 5—para todos os clientes em 12/06/2026, apenas três dias após o lançamento em 09/06. Motivo: uma diretiva de controle de exportação do governo americano que Anthropic diz ter recebido "hoje às 5:21pm (ET)".

A parte crítica para você está na linguagem. A diretiva exige que "todo acesso a Fable 5 e Mythos 5 por qualquer estrangeiro, dentro ou fora dos Estados Unidos" seja bloqueado.[1] Você é um cidadão alemão no Brasil—você se enquadra explicitamente naquela proibição. Mesmo se Anthropic reabrisse os modelos amanhã para cidadãos americanos, você não conseguiria acessá-los. Na prática, Anthropic virou a chave para todos: "devemos desativar abruptamente Fable 5 e Mythos 5 para todos os nossos clientes para garantir conformidade."[1]

Controle de exportação em um modelo na nuvem: Imagine que o governo proíbe uma oficina de emprestar uma ferramenta especializada a estrangeiros. Para bens físicos (chips, armas) é rotina. O que é novo é um modelo de software entregue pela nuvem ser apanhado por isso—e "estrangeiro" se aplica até mesmo a um alemão usando a ferramenta remotamente via API.

O motivo alegado é desconfortável para nossa categoria: o governo supostamente encontrou uma maneira de "quebrar" o modelo—e segundo Anthropic, esse método se resume a "pedir ao modelo que leia uma base de código específica e corrija falhas de software."[1] Esse é exatamente o fluxo de trabalho de codificação agentic que usamos esses modelos para. Anthropic o chama de "quebra estreita e não-universal" cujas capacidades são "amplamente disponíveis em outros modelos," e o contesta publicamente: acham que está errado "desabilitar um modelo comercial implantado para centenas de milhões de pessoas" sobre uma possível quebra estreita.[1]

Atenção Trate isso como reportado e contestado, não resolvido. A declaração do Anthropic é a única fonte detalhada; Fortune e National Law Review confirmam data, diretiva e escopo "estrangeiro", mas reportam a versão do Anthropic dos detalhes técnicos, não uma conta verificada independentemente. O que importa para você: Fable 5 e Mythos 5 estão indisponíveis como ferramenta, com um fim aberto.

GitHub Copilot: baseado em uso com "AI Credits" [4][5]

Desde 01/06/2026, GitHub Copilot cobra por uso em todos os planos: "a cobrança por uso do GitHub Copilot agora está disponível para todos os usuários."[4] A unidade de cobrança é o AI Credit, e a taxa de câmbio é agradavelmente simples: segundo a documentação do GitHub "1 AI credit = $0,01 USD"—um centavo americano por credit.[5]

Cobrança por uso / AI Credits: Antes, Copilot era como um plano de internet de taxa fixa—preço mensal fixo, não importa quanto você usa. Agora é mais como eletricidade: uma provisão base está na assinatura, depois o medidor funciona. Cada "credit" é um centavo virtual; modelos caros (modelos de raciocínio grande) custam mais credits por requisição do que os baratos.

Chave para o trabalho diário: conclusões de código e sugestões de próxima edição não são cobradas em AI credits—"Conclusões de código e sugestões de próxima edição não são cobradas em AI credits."[5] O autocomplete clássico com tab permanece ilimitado; o medidor só funciona para chamadas de agente/chat mais caras para modelos maiores. Preços de modelos são públicos na documentação: Claude Opus 4.x custa $5 entrada / $25 saída por 1M tokens, Claude Sonnet 4.x $3 / $15.[5]

O que o GitHub não lista limpo oficialmente são as provisões de credit por plano. Fontes secundárias reportam Pro ≈ 1.500, Pro+ ≈ 7.000, e Max ≈ 20.000 credits/mês; Pro+ supostamente custa $39/mês e inclui $39 credits.[6] Trate esses números como reportados—o changelog oficial do GitHub não os nomeia, apenas a taxa de centavo e preços de modelos são confirmados de primeira mão.

Terminal-Bench 2.1: líder lidera—mas com um fantasma [7]

Movimento fresco no leaderboard Terminal-Bench-2.1, o placar para trabalho agentic em um shell real. No topo: Codex CLI + GPT-5.5 em 83,4 % ± 2,2 (enviado 01/05). No #2 segue Claude Code + Claude 5 Fable em 83,1 % ± 2,0—enviado em 17/06.[7]

Terminal-Bench & o "± 2,0": O benchmark dá a um agente tarefas reais em uma linha de comando (construir pacotes, corrigir arquivos, fazer scripts funcionarem) e mede qual fração ele resolve. O "± 2,0" é incerteza de medição: em 83,4 % vs 83,1 %, as barras de erro completamente se sobrepõem—classificação 1 e 2 são estatisticamente indistinguíveis. Uma diferença de meio ponto percentual é ruído aqui, não um resultado.

E é exatamente ali que o leaderboard colide com a realidade: o modelo #2 Fable 5 foi globalmente desativado desde 12/06 (veja à esquerda)—a data de submissão 17/06 é cinco dias após a desativação. Se essa é uma computação anterior à desativação enviada tarde ou um artefato de dados não pode ser determinado de fora. Na prática: a entrada #2 é não-reproduzível hoje. Classificação 3 (Terminus 2 + Fable 5, 80,4 %) também depende do modelo morto. A melhor entrada utilizável do Anthropic é Claude Code + Opus 4.8 em 78,9 % ± 2,5 (29/05).

CLI Gemini está morta, longa vida a CLI Antigravity [10][11]

Google desativou a CLI Gemini para os níveis gratuito, Pro e Ultra em 18/06/2026: "Em 18 de junho de 2026, Gemini CLI e extensões Gemini Code Assist IDE deixarão de servir requisições para Google AI Pro e Ultra, bem como aqueles que a usam gratuitamente."[10] O sucessor é a CLI Antigravity—um binário Go de código fechado substituindo a CLI Python Gemini anteriormente de código aberto (Apache 2.0); "Construído em Go para melhor desempenho."[10]

O porém está na quota grátis. A antiga CLI Gemini oferecia "60 requisições de modelo por minuto e 1.000 requisições por dia sem custo" no lançamento.[11] Para Antigravity, essa quota grátis supostamente está caindo de 1.000 para cerca de 20 requisições por dia—reportado por The Register; Google não publicou um número concreto de Antigravity grátis em seu post oficial, então reportado, não confirmado oficialmente.[12] Se você usa a CLI Gemini via uma licença Code Assist Standard/Enterprise, Google diz que você não é afetado.

Aberto → Fechado A perda real não é a quota, é a licença: uma CLI de código aberto e hackeável é substituída por um binário fechado. Se você contou com capacidade de script e auto-correção, isso é um retrocesso—e um argumento para preferir harnesses agnósticos de agente que não dependem de um binário do fornecedor.

Windsurf → Devin Desktop, mais ACP [13][14]

Windsurf se tornou oficialmente Devin Desktop com a atualização OTA em 02/06/2026.[13] O agente local anterior Cascade recebe um fim de vida em 01/07/2026 ("Cascade permanece disponível até 1º de julho") e é substituído por Devin Local—reescrito em Rust e segundo o fornecedor "até 30% mais eficiente em tokens."[13]

ACP (Agent Client Protocol): Um padrão de conector comum entre editor e agente de codificação. Antes, cada agente era conectado em sua IDE; com ACP você pode conectar agentes de terceiros na mesma interface—Devin Desktop "suporta Codex, Claude Agent, OpenCode e qualquer outro agente compatível com ACP."[14] Pense em USB-C para agentes: uma porta, muitos dispositivos.

Essa é a parte genuinamente interessante: em vez de trancá-lo em um agente da casa, Devin Desktop se abre para a concorrência. Junto com o movimento CLI Gemini acima, um padrão emerge—shell de ferramenta e modelo estão se desacoplando. Bom para você, porque seu fluxo de trabalho sobrevive então à falha de qualquer modelo único (veja Fable 5).

Experimente Cursor v3.8: Automations & Bugbot mais rápido [15][16]

Cursor v3.8 chegou em 18/06/2026.[15] Novo é Automations: via uma skill /automate você pode pendurar execuções de agente em gatilhos Slack e GitHub, incluindo uso de computador na nuvem. Mais prático dia a dia: Bugbot agora revisa em ~90 s em vez de ~5 min, encontra ~10 % mais bugs, e custa ~22 % menos por execução.[16]

Por que experimentar: Se você já passa algumas revisões de PR para um bot, um Bugbot 3× mais rápido é o ganho rápido mais tangível da semana—loops mais curtos, mais acertos, menor custo. (Números do Bugbot apenas via fonte secundária — o changelog oficial do Cursor não cita métricas do Bugbot.)
