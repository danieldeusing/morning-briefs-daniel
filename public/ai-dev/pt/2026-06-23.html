Cursor anuncia "Origin" — e é comprada pela SpaceX [1][2][3]

Na sua primeira conferência própria Compile (16.06., Fort Mason, San Francisco), Cursor soltou duas coisas no mesmo dia. Primeiro, um novo produto: Origin, uma plataforma de hospedagem Git não construída como uma interface web envolvida em torno do Git, mas do zero para agentes de codificação. Segundo — no mesmo dia — o próprio Cursor foi adquirido: "SpaceX announced a formal agreement to buy the artificial intelligence startup Cursor for $60 billion worth of stock."[1] 100% em ações, sem dinheiro — um acordo anunciado (16.06.) com fechamento esperado em Q3 2026, ainda não concluído.

Os números da Origin, que Cursor mostrou em um único repositório de demonstração, são incomumente concretos: cerca de 296.000 clones por hora, ~81.000 pushes e 22,6 commits por segundo.[2] O ponto de venda real é a latência: "Global synchronization operates 'under 400 milliseconds', with automatic failover within 10 milliseconds."[2] Tecnicamente, Origin se assenta na fundação re-arquitetada da aquisição Graphite de dezembro de 2025 — "Built on re-architected technology from its December 2025 acquisition of Graphite."[3]

Hospedagem Git "centrada em agentes": GitHub é uma interface web para humanos operarem — você clica para fazer branches, PRs, reviews. Hospedagem centrada em agentes inverte isso: é otimizada para agentes de software que clonam, modificam e fazem push em centenas de repositórios em paralelo sem um humano clicando junto. Os 296.000 clones por hora não são propaganda — são a carga que você obtém quando seu enxame de agentes, não você, é o usuário.

Para você como desenvolvedor solo, Origin ainda não é alcançável: "Origin is expected to officially launch in fall 2026, with a waitlist now open."[2] O que importa é a direção. Se o host Git dominante de repente enfrenta concorrência de um player que trata a carga de agentes como caso de uso primário, isso muda, a longo prazo, onde seu código vive e como CI e review se parecem. Eu classificaria a mudança de propriedade de $60 bilhões separadamente: diz mais sobre a situação de avaliação (~$4 bilhões em ARR, dos quais ~$2,6 bilhões B2B[4]) do que sobre o roteiro do seu editor.

Contexto Trate os números da Origin como demonstração de fornecedor, não teste de carga independente — vêm do próprio show de palco do Cursor. E um lançamento outono 2026 com lista de espera significa: nada para o qual você está migrando esta semana. A conclusão estratégica é mais sobre não trancar seu workflow muito fortemente a um host Git.

Gemini para codificação: Flash está aqui, Pro ainda está esperando [5][6]

Com o ângulo de codificação do Google você precisa manter dois modelos separados. Gemini 3.5 Flash está ao vivo desde 19.05.2026 e é ajustado para trabalho agentic: Google afirma que supera o 3.1 Pro mais antigo em "challenging coding and agentic benchmarks like Terminal-Bench 2.1 (76.2%)" e MCP Atlas (83,6%), em "4 times faster than other frontier models" em tokens de saída/seg.[5] Para loops de agentes rápidos e baratos, esse é o mais interessante dos dois.

O grande Gemini 3.5 Pro — o modelo que você quer para raciocínio pesado durante codificação — ainda não é geralmente disponível: "Gemini 3.5 Pro remains in limited preview for select Vertex AI enterprise customers only" (a partir de 19.06.).[6] Uma GA é esperada "em junho" e poderia chegar entre 19 e 30 — mas isso é anunciado, não entregue. Espere uma janela de contexto de 2 milhões de tokens e um modo de raciocínio "Deep Think".

"Limited Preview" vs. "GA": Preview significa que o modelo está rodando, mas apenas para clientes enterprise selecionados via plataforma cloud do Google (Vertex AI) — você como desenvolvedor solo não pode alcançá-lo com uma chave de API regular. GA (General Availability) é o ponto onde abre para todos, com preços publicados e garantias de estabilidade. Até então: especificações legais, mas nenhuma ferramenta que você possa planejar hoje.

A estimativa de preço circulante — ~$15 por milhão de entrada, $60 por milhão de saída — é uma estimativa secundária, não uma taxa oficial: "roughly ten times the cost of Flash".[6] Sem um post GA oficial com preços reais, isso é uma estimativa, não um número de orçamento.

Experimente GitHub Copilot SDK é GA [7][8]

A partir de 02.06.2026, o Copilot SDK está geralmente disponível: "GitHub Copilot SDK is now generally available."[7] O núcleo: você pode incorporar o mecanismo Copilot agentic em seus próprios aplicativos em vez de apenas usá-lo no editor. GA vem em seis linguagens — "Node.js/TypeScript, Python, Go, .NET, Rust (new at GA), and Java (new at GA)"[8] — Rust e Java são novos em GA.

A parte mais importante para você está na letra miúda: o acesso é "to all existing GitHub Copilot subscribers … and to non-Copilot users via BYOK."[8] BYOK significa que você pode trazer suas próprias credenciais de modelo em vez de estar preso ao backend do Copilot — e o SDK fala com servidores MCP.

SDK + BYOK (Traga Sua Própria Chave): Um SDK é um kit de ferramentas que permite puxar a funcionalidade de outro para seu próprio código. Aqui: a lógica do agente Copilot se torna uma biblioteca que você pode incorporar à sua ferramenta. "Traga Sua Própria Chave" significa que você usa suas próprias credenciais de API — digamos, para Claude ou outro modelo — em vez do modelo integrado do Copilot. Você está usando a maquinaria de agente do Copilot, mas a alimentando com o modelo de sua escolha.

Por que experimentar: Se você vinha construindo suas próprias automações de codificação e costumava ter que costurar manualmente o loop de agentes, esse SDK lhe dá um mecanismo testado — com MCP e BYOK em cima. Esse é o bloco de construção mais concreto "pegue hoje" da semana, nenhuma promessa outono-2026.

Claude aterrissa como agente no JetBrains [9][10]

Nas IDEs um padrão está se solidificando: o shell do editor e o agente estão se desacoplando. O JetBrains AI Assistant (v262.8117.29, 19.06.) agora agrupa dois agentes estrangeiros nativamente no chat — "Claude Agent, powered by Anthropic's Agent SDK, is now seamlessly integrated"[9] ao lado do agente Codex do OpenAI. Mais "Next Edit Suggestions" em Preview.

Em paralelo e na mesma trilha: GitHub lançou "Claude as agent provider preview in JetBrains IDEs"[10] em 22.06. via rota Copilot. Você está recebendo Claude no JetBrains agora em duas caminhos — diretamente via JetBrains AI Assistant próprio e via Copilot como provedor.

Padrão Junto com o SDK BYOK do Copilot (acima), isso mostra o mesmo movimento que em CLIs: ferramenta ≠ modelo. Quando você pode escolher o agente livremente no IDE, você sobrevive à interrupção ou bloqueio de qualquer modelo único com mais suavidade — um argumento para favorecer shells agnósticos a modelos.

SWE-Bench Verified: todos os scores auto-relatados [11]

Verificação rápida de sanidade nos placar porque números altos estão circulando por toda parte esta semana. Em SWE-Bench Verified, 0 de 102 modelos listados são verificados independentemente de acordo com llm-stats — todo score superior é auto-relatório de fornecedor.[11] O melhor modelo utilizável de codificação do Google na lista, Gemini 3.1 Pro, está em 80,6% (auto-relatado).

"Auto-relatado" em benchmarks: O fornecedor executa seu próprio modelo através do teste e relata o resultado por si próprio — ninguém independente o reproduz. Isso não é automaticamente falso, mas convida contaminação de treinamento (o modelo viu as tarefas de teste antes) e seleção de cereja. Um badge "verificado" significaria que um terceiro reproduziu a mesma execução. Isso não aconteceu para nenhuma das 102 entradas aqui.

Claude Code endurece modo de agente [12]

Na linha Claude Code v2.1.179–v2.1.185 há um detalhe que importa no trabalho agentic: modo automático agora bloqueia comandos Git e Terraform destrutivos a menos que você solicite explicitamente — "blocking of destructive git commands and terraform operations unless explicitly requested" (v2.1.183, 19.06.).[12] Também: /config key=value como nova sintaxe, "Artifacts" (páginas web compartilháveis de sessões, beta Team/Enterprise), e sub-agentes podem gerar sub-agentes — limitado a cinco níveis de profundidade.

O guardrail do modo automático é a mudança praticamente relevante: um agente rodando desacompanhado não deveria acidentalmente lhe entregar um git reset --hard ou terraform destroy. Freio padrão inteligente para exatamente os workflows onde você executa a ferramenta desacompanhada.
