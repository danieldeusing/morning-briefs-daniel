Fable 5 agora é um modelo pago [1][2][3]

Esta é a notícia que bate direto no seu fluxo de trabalho. A Anthropic removeu Fable 5 (lançado em 09/06/2026) dos limites do plano em 23 de junho — o uso continuado agora roda com créditos de API pagos.[2] Os preços: $10/M tokens de entrada, $50/M saída.[1] Segundo ayautomate, este é "o modelo frontier geralmente disponível mais caro que a Anthropic fornece, o dobro da taxa do Opus 4.8".

$/M tokens: Você paga por milhão de blocos de construção de palavras que entram e saem — como um taxímetro que marca conforme você lê e conforme responde. A saída é mais cara porque o modelo tem que "pensar" para produzi-la. Em $50/M saída, uma execução agêntica com muita saída de ferramenta se soma rápido.

Desfecho prático: quem rodar Fable 5 acima da cota do plano queima segundo future-stack-reviews em "roughly double the rate of Opus 4.8".[3] Para a configuração de Daniel, isso significa: Fable 5 é agora o modelo para os 10% difíceis — a decisão de arquitetura, o refator complicado — não o motorista padrão. Opus 4.8 continua melhor custo-benefício para o trabalho do dia a dia, e para código rotineiro ainda vale a pena rotear para modelos mais baratos (ou localmente: Qwen/Gemma). Um reflexo "use Fable 5 para tudo" ficou caro agora.

SpaceX quer comprar Cursor — anunciado, não selado [4][5]

Grande manchete, mas leia com cuidado: Segundo TechCrunch (16/06/2026), a SpaceX concordou em adquirir Cursor (Anysphere), valorizando a startup de código em $60 bilhões — inteiramente em ações.[4] Ponto-chave: esta é uma transação anunciada, ainda não fechada; o fechamento é esperado apenas em Q3 2026, então poderia não se concretizar ou se deslocar.

Os números por trás disso: Cursor atingiu "roughly $4 billion in annualized revenue in under four years", e segundo CNBC ~$2,6 bilhões de B2B empresarial — o acordo o valida em aproximadamente 15 vezes a receita.[5] Por que isso importa para sua caixa de ferramentas: se um construtor de foguetes compra o editor de código de IA mais usado, é um sinal de que ferramentas de codificação estão se tornando infraestrutura estratégica — e um lembrete para não apostar demais em um único editor proprietário cujo proprietário (e roadmap) pode mudar amanhã. Cursor continuou entregando em paralelo: v3.9 (22/06) agrega plugins, skills e MCPs em uma Customize Page; v3.8 (18/06) trouxe automações com triggers GitHub/Slack.[5]

Google Managed Agents: agente via uma única chamada de API experimente [6]

O Google está lançando Managed Agents como preview na API Gemini (API Gemini + AI Studio Playground).[6] Ideia: um agente que "razoa, usa ferramentas e executa código em um ambiente Linux isolado e efêmero com uma única chamada de API". Alimentado por Gemini 3.5 Flash ($1,50/$9,00 por M tokens). Agentes são definidos em Markdown — AGENTS.md + SKILL.md, então são versionados e declarativos.

Por que experimentar: Você não precisa rodar sua própria infraestrutura de sandbox — o contêiner Linux vem com a chamada de API. Para um rápido "agente constrói e testa algo isolado" sem setup de harness customizado, é um ponto de entrada barato. Nota de limite: a Gemini Enterprise Agent Platform (o wrapper de indústria/negócios em volta disso) pertence ao brief de ai-usecases — aqui apenas o que é relevante para mãos de dev: o bloco de construção de API em si.

Benchmarks: números fortes, evidência fraca [7][8][9]

O campo de rankings está se movendo — mas o número mais importante não é um score, é um aviso de métodos. No board SWE-Bench-Verified em llm-stats.com, Fable 5 lidera em 95,0 % (em 102 modelos avaliados), seguido por Claude Mythos Preview (93,9 %) e Opus 4.8 (88,6 %); Gemini 3.1 Pro e DeepSeek-V4-Pro-Max empatam em 80,6 %.[7] Em 24/06/2026 pelo scrape do leaderboard.

Leia com ceticismo Segundo digitalapplied, de 100 modelos naquele board (em 16/06), apenas 1 resultado foi verificado independentemente — os outros 99 fornecedores submeteram a si mesmos.[8] Isso significa: estes 95 % é uma figura relatada pela Anthropic, não medida neutralmente. Para scores autorrelatados nesta faixa (95–99 %), cautela é necessária — gaming de benchmark, contaminação de treinamento e scaffolding (o framework agêntico em volta do modelo) frequentemente importam mais que a capacidade bruta do modelo.

Para colocar "com scaffolding" em perspectiva: em SWE-Bench Pro (benchmark agêntico da Anthropic), Fable 5 registra 80,3 % vs Opus 4.8 (69,2 %)[9] — mesmo gap, métrica diferente. Então leia os 95 % não como "resolve 95 de 100 PRs reais" mas como "lidera em um board onde fornecedores entram seus próprios números".

Terminal-Bench: Codex na frente, figura Fable com ressalvas [10]

No benchmark terminal agêntico, indo pelos snippets disponíveis, Codex CLI com GPT-5.5 lidera o board 2.1 em 83,4 %, Claude Code com Fable 5 segue perto em 83,1 %, Claude Code com Opus 4.8 em 78,9 %.[10]

Status não está claro A figura Fable (83,1 %) vem apenas de um snippet de busca, não de um fetch fresco do leaderboard. Um fetch direto retornou uma tabela Terminal-Bench-2.0 com valores diferentes (GPT-5.5 82,7 %, Opus 4.8 69,4 %). Trate a ordenação 2.1 como tendência aproximada, não como posição confirmada. Terminal-Bench: mede se um agente resolve tarefas reais de shell em um terminal end-to-end — mais perto de "esta coisa realmente pode construir e testar meu repo" do que snippets de código isolados.

O leaderboard Aider fica de fora hoje: dados de versão/data não puderam ser verificados de forma confiável para junho de 2026 (metadados GitHub pareciam desatualizados), então nenhum número estimado — uma lacuna é melhor que um valor errado.

Claude Code & Codex: semana densa de ferramentas [11][12]

Claude Code rodou em alta cadência de 17/06–23/06. v2.1.187 (23/06) traz login/logout MCP via CLI, uma configuração sandbox.credentials (bloqueia comandos em sandbox de ler arquivos de credenciais) e restrições de modelo de org.[11] No meio disso: bloqueou comandos git destrutivos no modo auto, um comando /cd para trocar diretório de trabalho na sessão, cadeias de sub-agentes até 5 níveis de profundidade, e artefatos como páginas web vivas compartilháveis (histórico de versão em beta para team/enterprise).

MCP: Um plug padrão, através do qual o agente de codificação fala com ferramentas e fontes de dados externas — como uma porta USB em que você conecta um banco de dados, sistema de tickets ou navegador sem precisar de um cabo separado para cada um. O novo login/logout CLI gerencia essas conexões de porta mais limpo.

Codex CLI 0.142.0 (22/06) traz resgate de crédito de uso, organização de plugin por categoria e orçamentos de token de rollout configuráveis.[12] Já em 15/06 veio /import para migrar do Claude Code; em 18/06, "Record & Replay" para macOS (transforma fluxos de trabalho demonstrados em skills reutilizáveis). O caminho /import é notável: OpenAI está ativamente abaixando o custo de troca do Claude Code para Codex.

Codex vs. Claude Code: duas filosofias [10]

A comparação de harness é oportuna. Codex roda em 8 sub-agentes paralelos e uma janela de contexto de 200K; Claude Code em uma janela de 1M-token — mas queima "3–4× more tokens" para tarefas idênticas (no exemplo do plugin Figma, 6,2M vs 1,5M tokens).[10] Este é o trade-off em forma pura: Claude Code vê mais contexto de uma vez, custa mais por tarefa — que corta duas vezes após o aumento de preço do Fable 5 de hoje. Codex é token-escasso mas tem que gerenciar ativamente seu contexto mais apertado. Em adoção: Claude Code afirma "326K+ daily commits (~10% of all public commits)" segundo morphllm; Codex está em 91 mil stars do GitHub.[10]

Concreto para a matemática de custo: se Claude Code puxa 3–4× mais tokens e você roda Fable 5 mais caro nele, os dois fatores se multiplicam. Para execuções rotineiras pesadas de token, o caminho mais barato é agora mais claro que nunca — um modelo mais barato no harness, ou onde se encaixa, um local (Qwen-Coder, Gemma) para tarefas que não precisam de raciocínio frontier. Gaste tokens frontier apenas onde o problema realmente os exige.
