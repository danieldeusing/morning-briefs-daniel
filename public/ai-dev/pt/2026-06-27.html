Anthropic desativa Fable 5 + Mythos 5 globalmente [1][2]

O maior movimento este mês não é um lançamento — é uma retirada. Em 12.06. às 17:21 ET, a Anthropic recebeu uma diretiva de exportação dos EUA e reagiu imediatamente: ambos os modelos principais, Fable 5 e Mythos 5, foram desativados para todos os clientes — "para garantir conformidade", nas próprias palavras da Anthropic.[1] Este é o próprio produto da Anthropic; estou relatando isso da forma mais neutra possível porque o alcance é independente da propriedade.

A diretiva visa o uso "por qualquer cidadão estrangeiro, dentro ou fora dos EUA, incluindo funcionários estrangeiros da Anthropic", de acordo com a Anthropic.[2] Como o acesso a modelos não pode ser filtrado de forma limpa por nacionalidade, a única resposta em conformidade foi: desligamento global. A Anthropic cita um "jailbreak estreito e não universal" — fazer o modelo ler uma base de código e corrigir vulnerabilidades — como o gatilho e disputa publicamente se isso justifica um recall de um modelo comercial.[1]

Diretiva de controle de exportação: Uma ordem governamental que proíbe a exportação de certa tecnologia para certos destinatários — como um embargo no envio de tecnologia avançada. Aqui atingiu software que não é enviado fisicamente mas acessado via API — então todo acesso teve que ficar offline.

O que isso significa para você: Todos os outros modelos Claude (Opus 4.8, Sonnet, Haiku, Claude Code) continuam funcionando sem mudanças — a Anthropic enfatiza isso explicitamente.[3] Mas é a lição mais aguda possível de aprisionamento a fornecedor: os dois modelos no topo absoluto da lista SWE-Bench Verified independente (Mythos 5 95,5 %, Fable 5 95,0 %) ficaram inacessíveis da noite para o dia — não por um bug ou preço, mas por uma ordem governamental.[4] Se você fixou seu fluxo de trabalho de agente a um modelo frontier, você deveria ter um caminho alternativo por agora.

GLM-5.2: a ponte auto-hospedável [5][6][7]

Quando um modelo frontier pode desaparecer da noite para o dia, independência de fornecedor muda de uma ideia legal para um seguro. É exatamente onde GLM-5.2 da Zhipu/Z.ai chega no momento certo: API desde 13.06., pesos desde ~17.06. no HuggingFace — sob licença MIT sem restrições regionais.[5] Você pode auto-hospedá-lo sem que ninguém puxe a tomada.

Arquitetura: 753B parâmetros totais, ~40B ativos por token (MoE), com janela de contexto de 1M-token.[5]

MoE (Mixture of Experts): Em vez de executar todo o modelo gigante para cada palavra, ele é dividido em muitos "especialistas" e apenas uma pequena fração é ativada por token — como um grande escritório onde cada tarefa traz apenas os dois colegas relevantes aos seus pés em vez de todos os cem. Resultado: o poder de um modelo de 753B com custo de computação mais próximo a ~40B.

Nos benchmarks, GLM-5.2 está surpreendentemente próximo do Opus 4.8, de acordo com o blog de Z.ai: SWE-Bench Pro 62,1, FrontierSWE 74,4 (Opus 4.8: 75,1), Terminal-Bench 2.1 81,0 (Opus 4.8: 85,0).[5] Ressalva importante: os números de comparação com Opus vêm do próprio blog de Z.ai. Confirmado independentemente é a classificação #1 entre pesos abertos no Índice de Inteligência da Artificial-Analysis (score 51) — essa é uma medição neutra de fornecedor.[6] Integração no Dia 1 em Claude Code, Cline, OpenCode, Roo Code, Goose, Crush e Kilo Code.[7] Teste isto Quer um fallback frontier com zero risco de fornecedor: GLM-5.2 é hoje a escolha auto-hospedável mais plausível — vale a pena ler.

MAI-Code-1-Flash para Copilot está em GA [8][9]

A partir de 26.06., MAI-Code-1-Flash — o modelo de codificação interno da Microsoft — está disponível em geral para GitHub Copilot Business + Enterprise.[8] Não é posicionado como um novo grande frontier, mas como uma ferramenta de trabalho barata e rápida para fluxos de trabalho agentic do Copilot, medido por consumo.

A Microsoft o posiciona explicitamente contra Claude Haiku 4.5: números de fornecedor citam +16 pontos em SWE-Bench Pro (51,2 % vs. 35,2 %), +28,9 em IF Bench, e "até 60 % menos tokens" em SWE-Bench Verified — com apenas 5B parâmetros por fontes secundárias.[9] Todos esses números são reportados pela Microsoft; uma entrada SWE-Bench Verified independente está faltando. A eficiência de tokens é o verdadeiro pitch: se você executa grandes varreduras de agente, você paga por token, não por score.

Contexto: A Microsoft agora controla outro modelo padrão dentro de sua própria pilha — como com a mudança do Copilot de Premium Request Units para "GitHub AI Credits" em 01.06., o aprisionamento de preços da plataforma cresce, não diminui.

  1. github.blog — MAI-Code-1-Flash GA, 26.06.2026.
  2. microsoft.ai — Benchmarks de fornecedor (SWE-Bench Pro, IF Bench, eficiência de tokens), 02.06.2026; 5B params via chatforest.com (secundário).

Cursor engole Continue — exporte dados até 15.07. [10][11]

Cursor adquiriu Continue (anunciado ~22.06.) — e desativou o produto. É uma aquisição de contratação: o time se move para Cursor, o repositório open-source (`continuedev/continue`, 34K stars) fica somente leitura com lançamento final v2.0.0.[10] Usuários do Continue têm até 15.07. para exportar seus dados — depois é fim.[11]

Detalhe picante: Cursor deve ser adquirida pela SpaceX em um acordo anunciado de ~60B$ (conclusão prevista para o 3º tri de 2026) — então Continue também passaria para a SpaceX.[10] Para desenvolvedores: se Continue estiver integrado a um fluxo de trabalho, exporte agora e migre para um caminho mantido ativamente — Cline e Kilo Code são os sucessores open-source óbvios.

Padrão da semana Três das quatro grandes histórias são consolidação ou retirada — Anthropic puxa modelos, Microsoft constrói sua própria pilha, Cursor/SpaceX compra um concorrente open-source. O único movimento em direção a mais opções vem da China (GLM-5.2, Kimi). Essa é a verdadeira história do mês.

Kimi K2.7-Code: reclamações fortes, independentemente não confirmadas [12][13]

Moonshot AI lançou Kimi K2.7-Code em 12.06.: 1T parâmetros totais, 32B ativos por token (384 especialistas, contexto de 256K), sob MIT Modificada no HuggingFace (~595 GB de pesos).[12] Em benchmarks de propriedade de fornecedor, ele salta significativamente sobre K2.6: +21,8 % em Kimi Code Bench v2, +31,5 % em MLS Bench Lite; SWE-Bench Pro reportado por fornecedor 58,6.[13]

Cuidado com benchmarks Para Kimi K2.7-Code, não existe (a partir de ~14.06.) nenhuma entrada independente de terceiros em suites padrão — SWE-Bench Verified, SWE-Bench Pro, Terminal-Bench, LiveCodeBench. Os saltos citados são todos auto-medidos. Em benchmarks de fornecedor (Kimi Code Bench, MLS Bench), contaminação de treinamento é o risco padrão. Trate os números como hipótese, não medição — até que uma entrada independente apareça.

  2. vm0.ai — nenhuma entrada independente de terceiros (a partir de ~14.06., secundário); SWE-Bench Pro 58,6 fornecedor via totalum.app.

É sábado, vinte e sete de junho. O lado das ferramentas está tranquilo, mas a pilha de modelos está no meio de uma turbulência — e a maior história do mês não é um modelo novo, é um que desaparece.

Em doze de junho, a Anthropic desativou seus dois modelos de codificação mais fortes, Fable cinco e Mythos cinco, globalmente para todos os clientes. O gatilho não era uma falha, mas uma diretiva de exportação dos EUA que não podia ser filtrada por nacionalidade — então a única resposta era tirar completamente ambos os modelos do ar. A Anthropic disputa a ordem publicamente, mas desativou por segurança mesmo assim. Todos os outros modelos Claude continuam funcionando. Ainda assim, é a lição mais aguda de aprisionamento de fornecedor que você pode imaginar: os dois modelos que se classificavam independentemente bem no topo da lista SWE-Bench se tornaram inutilizáveis da noite para o dia.

O mundo dos pesos abertos fornece a ponte. G-L-M cinco ponto dois da Zhipu está disponível com pesos sob licença MIT no HuggingFace desde meados de junho — auto-hospedável, com ninguém capaz de puxar a tomada. É um modelo Mixture-of-Experts com mais de setecentos bilhões de parâmetros, mas apenas cerca de quarenta bilhões deles trabalham por token. No Índice de Análise Artificial independente, é número um entre modelos abertos. Se você está procurando um fallback com zero risco de fornecedor, você deveria olhar para ele.

Na Microsoft, o modelo de codificação interno M-A-I-Code-um-Flash entrou em disponibilidade geral para Copilot Business e Enterprise em vinte e seis de junho. Não é um grande frontier, mas uma ferramenta de trabalho barata e rápida com supostamente apenas cinco bilhões de parâmetros. Os números de benchmark da Microsoft parecem bons, mas são todos auto-medidos — uma entrada independente está faltando.

Cursor adquiriu o concorrente open-source Continue e o desativou. O repositório fica somente leitura, e os usuários têm até quinze de julho para exportar seus dados. Cursor deve ser adquirida pela SpaceX (acordo anunciado, conclusão prevista para o 3º tri de 2026) — então Continue também passaria para a SpaceX.

Da China também vem Kimi K-dois-ponto-sete-Code da Moonshot, um modelo de trilhões de parâmetros sob licença aberta. Os saltos de benchmark parecem impressionantes, mas não existe uma única entrada independente de terceiros — então trate como hipótese, não medição.

Muito permanece tranquilo nas sombras: Aider está preso na versão zero ponto oitenta e seis ponto dois desde fevereiro, Roo Code é arquivado e vive apenas nos forks Zoo Code e Kilo Code. E marque seu calendário para oito de julho: a partir de então, a Anthropic exige verificação de identidade para contas de consumidor sinalizadas.

Próxima semana o brief observará principalmente se uma entrada de benchmark independente aparece para Kimi ou se novos números da Microsoft surgem — e se a situação em torno dos modelos desativados da Anthropic se move.
