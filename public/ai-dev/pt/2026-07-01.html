Claude Sonnet 5 — o novo padrão no seu terminal

Anthropic lançou Claude Sonnet 5 ontem (30.06), e hoje Claude Code v2.1.197 torna-o o modelo padrão. Esta é a notícia mais praticamente relevante do dia: se você tem o Claude Code aberto, estará trabalhando com um novo modelo após a atualização sem fazer nada.[1][2]

Os números principais: 85,2% no SWE-bench Verified e 63,2% no SWE-bench Pro mais difícil (acima do 58,1% do Sonnet 4.6). Além de uma janela de contexto nativa de 1M tokens (até 128k de saída) — a primeira vez que o Sonnet no Claude Code funciona com o tamanho de contexto longo anteriormente reservado para o Opus.[1][3][2]

SWE-bench Verified vs. Pro: Ambos medem se um modelo pode corrigir bugs reais do GitHub de forma independente. "Verified" é a seleção revisada por humanos e facilmente solucionável (~500 casos); "Pro" é deliberadamente mais difícil e resistente à contaminação. Pense em TÜV vs. rali: o mesmo teste, mas Pro é onde os pneus realmente queimam — é por isso que os scores Pro são 20+ pontos mais baixos em todos os modelos.

Tente isto O preço introdutório de $2 por milhão de tokens de entrada / $10 por milhão de saída (até 31.08, depois $3/$15) torna o Sonnet 5 ainda mais barato que o Sonnet 4.6 — com melhores números de benchmark. Para loops diários no Claude Code vale a pena testar porque você está sendo mudado de qualquer forma; eu ainda executaria uma comparação deliberada contra seu padrão atual antes de construir prompts de 1M contexto (contexto longo custa dinheiro real).[3][1]

Claude Code: o sprint de harness desta semana

Além da mudança de modelo, o Claude Code obteve vários recursos tangíveis em uma única semana. Em ordem, com datas:

v2.1.196 (30.06) traz duas coisas que valem a pena levar a sério. Primeiro, um portão de segurança MCP: os servidores definidos em .mcp.json não iniciam mais automaticamente, mas requerem aprovação explícita. Segundo, subagentes agora podem se aninhar 5 níveis de profundidade — além de controles de modelo padrão em nível organizacional.[2]

MCP (Model Context Protocol): Um padrão aberto para agentes chamarem ferramentas e fontes de dados externas — como uma porta USB para ferramentas de IA: uma vez definido, cada cliente fala com a mesma interface. O novo portão é por que isso é importante: um .mcp.json que um repositório fornece poderia iniciar um servidor sem ser solicitado quando você o abre — agora o Claude Code pergunta primeiro.

v2.1.191 (25.06) entregou o comando /rewind (retroceder a conversa para um ponto anterior — útil quando uma execução de agente dera errado), lógica de nova tentativa de MCP e redução de ~37% de CPU através de atualizações de texto em lote (janela de 100 ms). Você notará isso principalmente na vida útil da bateria do laptop.[2]

Contexto: o aninhamento de subagentes de 5 níveis é a declaração na corrida por "agentes coordenando agentes" — o Cursor está indo pelo mesmo caminho com VMs de nuvem (abaixo). O portão MCP é a resposta mais madura: controle primeiro, depois autonomia.

GPT-5.6 Sol/Terra/Luna — preview, mas sem comprovação de programação

A OpenAI lançou uma preview limitada da linha GPT-5.6 em 26.06: Sol (navío-almirante), Terra (equilibrado para trabalho do dia a dia) e Luna (rápido/barato). O acesso está atualmente limitado a ~20 organizações parceiras; o lançamento mais amplo deve seguir "nas próximas semanas".[5][6]

Quais números concretos existem: pontuação Terminal-Bench 2.1: Sol (máx) 88,76% (Sol ultra 91,91%). Preço: Sol a $5/$30, Terra a $2,50/$15, Luna a $1/$6 por milhão de tokens.[5]

Importante — o que falta Para GPT-5.6 não há nenhuma pontuação pública de SWE-bench (nem Verified nem Pro) — atualmente o modelo não aparece em nenhum quadro de programação. Terminal-Bench mede o comportamento do agente na shell, não a correção de bugs pura como SWE-bench; os dois números não são comparáveis. Se você quiser comparar GPT-5.6 com Sonnet 5, precisará esperar pelo lançamento mais amplo e uma entrada de benchmark independente. Estou deliberadamente não escrevendo um número de comparação aqui que não existe.

Contexto A divisão de três níveis (Sol/Terra/Luna) espelha a lógica Opus/Sonnet/Haiku da Anthropic e os níveis Gemini do Google — o mercado está convergindo para "um nome, três pontos de preço/desempenho". Na prática: como desenvolvedor solo, isso ainda não é uma decisão de caixa de ferramentas enquanto você não tiver acesso. Mantenha-o na lista de observação por enquanto.

GLM-5.2 — o campo aberto tem um novo líder

Do acervo de pesos abertos vem a contribuição mais forte: Z.ai lançou GLM-5.2 primeiro para assinantes do plano de programação (13.06) e depois como pesos abertos sob licença MIT (16./17.06). MIT significa: você pode executá-lo localmente, usá-lo comercialmente, ajustá-lo — sem restrições de uso.[7][8]

Os números que importam: 62,1% SWE-bench Pro (acima de 58,4%) — fazendo GLM-5.2 o líder entre modelos abertos — e 81,0% Terminal-Bench 2.1 (acima de 63,5%). No Índice de Inteligência v4.1 fica em primeiro com 51 (MiniMax-M3 44, DeepSeek V4 Pro 44). Arquitetura: 753B de parâmetros no total, ~40B ativos como Mixture-of-Experts, contexto de 1M.[8][7][9]

Mixture-of-Experts (MoE): Em vez de executar a rede inteira de 753 bilhões para cada token, o modelo ativa apenas uma sub-rede correspondente (~40B). Como um hospital que não chama todos os médicos para cada visita, apenas a especialidade necessária — competência completa no consultório, mas por caso apenas uma fração trabalha. Torna um modelo de outro modo massivo acessível para executar.

Tente isto Para implantação local/soberana, este é o candidato mais interessante da semana: pesos abertos, número líder de SWE-bench-Pro aberto, 2,9× menos FLOPs por token em contexto de 1M (atenção IndexShare). Via OpenRouter, o preço chega a ~$1,40/$4,40 — bem abaixo dos navíos-almirantes proprietários.[7]

Uma desvantagem GLM-5.2 é tagarela: 43k tokens de saída por tarefa (Índice de Inteligência) versus 26k para GLM-5.1. Com faturamento baseado em saída, isso consome parte do preço de token barato — notável em loops de agentes com muitas etapas. Inclua isso em sua comparação.[7]

SWE-bench Verified: o quadro com olhar cético [10][11]

O quadro público de SWE-bench-Verified foi atualizado hoje (01.07). O topo da tabela num relance — com uma ressalva:

SWE-bench Verified — top 4, a partir de 01.07.2026

Fable 5 lidera com 95,0%, Opus 4.8 em 88,6%, o novo Sonnet 5 em 85,2%, GLM-5 (5.1) em 77,8%. No SWE-bench Pro mais difícil o campo muda: Opus 4.8 lidera ativamente em 69,2%, GLM-5.2 em 62,1% como melhor modelo aberto, Sonnet 5 em 63,2%.[10][11]

Ceticismo obrigatório Estes são predominantemente números reportados por fornecedores. Scale/SEAL mede em um conjunto padronizado e controlado por contaminação muito mais baixo: lá GPT-5.4 (xHigh) lidera em 59,1%, Opus 4.6 thinking em 51,9%. A regra de ouro da prática: pontuações de fornecedor ficam 15–30 pontos acima das padronizadas. Tome os 95% não como "quase resolvido", mas como dica de classificação, não valor absoluto.[11][12]

Por que isso importa para você: um modelo que brilha no Verified mas tropeça no seu repositório com configuração de compilação incomum não é surpreendente — gamificação de benchmark e proximidade de dados de treinamento são reais. O único teste confiável ainda é seu próprio código.

Nas margens

Fable 5 e Mythos 5 da Anthropic aparecem no quadro Verified em 95,0%, mas estiveram brevemente indisponíveis a partir de 12.06 devido a uma diretiva de controle de exportação dos EUA (esperado retornar por volta de 01.07). A história de disponibilidade de modelos e exportação pertence ao campo vizinho — detalhes no brief ai-usecases (incluindo GeneBench Pro lá). Para nós aqui: quando Fable 5 aparece como uma referência do quadro, não é uma ferramenta do dia a dia para você, mas o medidor na extremidade superior.

É quarta-feira, primeiro de julho, e é um dia repleto de notícias no cenário de codificação com IA. Três classes de modelos se movem simultaneamente — um novo modelo padrão em seu terminal, uma visualização do grande concorrente e um forte challenger aberto.

A notícia mais praticamente importante: Anthropic lançou Claude Sonnet cinco ontem, e a partir de hoje, Claude Code versão dois ponto um ponto cento e noventa e sete o torna o modelo padrão. Se você tiver o Claude Code aberto, trabalhará automaticamente com ele após a atualização. Sonnet cinco atinge oitenta e cinco vírgula dois por cento no SWE-bench Verified e sessenta e três vírgula dois por cento na variante Pro mais difícil, agora tem uma janela de contexto de um milhão de tokens — e custa no preço introdutório apenas dois dólares de entrada e dez de saída por milhão de tokens até o final de agosto. Isso o torna ainda mais barato que seu antecessor.

Na OpenAI, uma visualização da linha GPT-cinco-ponto-seis está em execução desde sexta-feira passada: Sol, Terra e Luna, três níveis como os outros provedores também. Mas o acesso está limitado a cerca de vinte organizações parceiras, e — isto é crucial — ainda não há um único benchmark de programação pública para ele. Apenas um valor de Terminal-Bench em torno de oitenta e oito por cento, que mede algo diferente de SWE-bench. Estou deliberadamente não nomeando um número de comparação contra Sonnet que simplesmente não existe. Até o lançamento mais amplo, GPT cinco-ponto-seis é apenas uma entrada de lista de observação.

Do acervo de modelos abertos vem a contribuição mais forte: GLM cinco-ponto-dois da Z.ai, lançado sob a licença MIT gratuita. Lidera com sessenta e dois vírgula um por cento no SWE-bench Pro em todo o campo aberto, tem uma janela de contexto de um milhão de tokens e é executado com acessibilidade apesar de seu tamanho graças a uma arquitetura Mixture-of-Experts. Para implantação local ou soberana, este é o candidato mais interessante da semana. Uma desvantagem: é tagarela e notavelmente queima mais tokens de saída por tarefa — você vai querer levar isso em conta em seu cálculo de custo.

O Claude Code em si passou por um sprint: subagentes agora podem se aninhar cinco níveis de profundidade, um novo portão de segurança impede que servidores MCP iniciem sem ser solicitados, há um comando de retrocesso para desfazer execuções descarrilhadas, e a carga de CPU caiu cerca de trinta e sete por cento. O Cursor está acompanhando com subagentes de nuvem e um novo beta iOS que permite orientar agentes do seu telefone.

Uma palavra de cautela sobre benchmarks: o quadro Verified atualizado mostra os melhores scores acima de noventa por cento, mas esses são principalmente números de fornecedores. Em conjuntos padronizados e controlados por contaminação, os mesmos modelos são executados quinze a trinta pontos mais baixos. Pegue a classificação, não os valores absolutos — e teste no seu próprio código.

Amanhã provavelmente veremos feedback de uso real do Sonnet cinco em uso real, e em algum momento nas próximas semanas vem o lançamento mais amplo do GPT-cinco-ponto-seis com os primeiros números de programação reais. Até lá.
