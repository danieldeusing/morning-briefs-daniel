O lançamento triplo do Google: dois para usar, um só para observar

Na terça-feira, dia 21/07, o Google publicou três modelos de uma só vez. O carro-chefe é o Gemini 3.6 Flash: 1,50 $ por milhão de tokens de entrada, 7,50 $ por milhão de tokens de saída, disponível desde o primeiro dia na API do Gemini, no AI Studio, no Android Studio e no app do Gemini. O Google não vende o modelo só pelo preço, mas pela economia: o modelo precisaria de 17 % menos tokens de saída que o 3.5 Flash, chegando a até 65 % menos em benchmarks pontuais. Esse é um dado do próprio fornecedor, tirado do post de lançamento, não uma medição independente — mas a lógica por trás faz sentido.

blog.google · Dados de 23/07/2026
Preço por token × consumo de tokens: Um modelo custa para você duas vezes — uma para cada pedaço de palavra que você coloca, outra para cada pedaço que sai. No trabalho com agentes, o segundo item domina, porque um agente produz páginas de raciocínio intermediário antes de mudar uma única linha de código. Pense num táxi cuja tarifa por quilômetro cai e que ao mesmo tempo faz trajetos mais curtos: os dois efeitos se multiplicam, não se somam. É exatamente esse o argumento do Google — preço menor por token, com menos tokens consumidos.
Cuidado Circula também na imprensa especializada que o corte de conhecimento do 3.6 Flash teria avançado de janeiro de 2025 para março de 2026 (techtimes.com, 22/07). Na própria página de anúncio do Google não há nada sobre isso — conferimos diretamente. Trate isso como afirmação da imprensa, não como especificação confirmada; se o seu caso de uso depender do corte de conhecimento, pergunte diretamente ao modelo antes de confiar nisso.

Flash-Lite: o mais barato que pode clicar

O Gemini 3.5 Flash-Lite custa 0,30 $/2,50 $ por milhão de tokens — e tem o "Computer Use" não como produto à parte, mas como ferramenta embutida. Essa é a notícia de verdade: a capacidade de operar sozinho uma interface está migrando para a faixa de preço mais baixa. Para tarefas em que um agente precisa simplesmente clicar num formulário web ou numa interface administrativa, isso muda bastante a conta.

blog.google · Dados de 23/07/2026
Computer Use: Em vez de só devolver texto, o modelo pode operar um computador — recebe capturas de tela e responde com "clique aqui", "digite isso", "role para baixo". É como um estagiário para quem você vira o monitor, em vez de ditar comandos: ele enxerga a mesma interface que você e vai trabalhando nela. A vantagem é que funciona em qualquer lugar sem API. O preço é lentidão e muitos tokens por clique — o que, justamente por isso, faz sentido estar na faixa mais barata.

Flash Cyber: o modelo atrás da cerca

O terceiro modelo é o caso interessante. O Gemini 3.5 Flash Cyber é um modelo de segurança construído sobre o 3.5 Flash, "ajustado para encontrar, validar e corrigir vulnerabilidades" — e, segundo o próprio blog da DeepMind, vai ficar disponível exclusivamente para governos e parceiros de confiança via CodeMender. Ou seja, ele existe, é oficial, e você não vai poder usá-lo. O número que o Google divulga: na engine JavaScript V8, o modelo teria encontrado 55 problemas únicos confirmados, contra 47 do 3.5 Flash e 36 do Claude Opus 4.6. Também essa é uma medição do próprio fornecedor, sem contraprova independente.

deepmind.google · Dados de 23/07/2026

Minha leitura: a restrição de acesso é a parte mais honesta do anúncio. Um modelo que encontra, valida e corrige vulnerabilidades é, no mesmo movimento, um modelo que encontra vulnerabilidades — e não só para defesa. A resposta do Google a essa simetria é um porteiro, não uma etiqueta de preço. Para você, isso não significa nada concreto hoje, exceto uma indicação de rumo: o valor está migrando de "modelo escreve código" para "modelo fecha o ciclo de encontrar, verificar e corrigir". Quem quiser construir um equivalente aberto desse ciclo acaba de ganhar uma descrição de objetivo bem clara.

A primeira conta da Meta: um quarto do preço mais alto

No dia 09/07, a Meta cobrou dinheiro pela primeira vez por um modelo próprio. A Meta Model API, de autoatendimento, precifica o Muse Spark 1.1 em 1,25 $ por milhão de tokens de entrada e 4,25 $ por milhão de tokens de saída. Mark Zuckerberg deixou o posicionamento claro em entrevista à Bloomberg: a nova API custaria "cerca de 25 % dos preços mais altos de OpenAI e Anthropic". Isso não é um comentário de passagem, é a estratégia — a Meta está comprando participação de mercado com a etiqueta de preço, não com um recorde de benchmark.

A escada de preços, medida

No topo continua, sem mudança, o GPT-5.6 Sol, o carro-chefe da OpenAI para programação: 5 $/30 $ por milhão de tokens, janela de contexto de 1,05 milhão de tokens, no máximo 128.000 tokens de saída. Com isso, um milhão de tokens de saída no Sol custa quatro vezes mais que no Gemini 3.6 Flash, sete vezes mais que no Muse Spark e doze vezes mais que no Flash-Lite. Uma ressalva sobre disponibilidade: o próprio post da OpenAI se chama "Previewing GPT-5.6 Sol" e cita 26/06 como início de uma prévia restrita, enquanto rastreadores registram 09/07 como disponibilidade ampla. Antes de você migrar por causa disso: preço por token é só metade da conta. Um modelo mais barato que precisa de três tentativas para a mesma mudança sai mais caro no fim do mês do que o modelo caro que acerta de primeira — e é exatamente essa taxa de acerto que nenhuma tabela de preço mede. A medição que vale é custo por tarefa concluída no seu próprio repositório: as mesmas cinco tarefas, dois modelos, anotando consumo de tokens e retrabalho. testar Flash-Lite para tarefas repetitivas de clique e formulário, Sol para as tarefas em que uma segunda tentativa sairia mais cara que a diferença de preço — essa é a divisão que os números sugerem.

Duas brechas na cadeia do IDE

No dia 14/07, foram divulgadas duas vulnerabilidades que atingem exatamente as ferramentas usadas aqui todos os dias. A CVE-2026-50510 afeta o plugin do GitHub Copilot para JetBrains, com escore base CVSS 3.1 de 7,8, e todas as versões do plugin anteriores à 1.13.0-251. A recomendação no aviso é inequívoca: usuários de IntelliJ IDEA, PyCharm, WebStorm, Rider e Android Studio devem atualizar a extensão imediatamente. É a tarefa de dois minutos do dia.

windowsnews.ai · Dados de 23/07/2026

A CVE-2026-50520 afeta o Visual Studio Code: uma injeção de comando (CWE-77) com CVSS 8,4. Um detalhe que se perde nas manchetes: o vetor de ataque é local. O "Remote" em "Remote Code Execution" aqui se refere a entradas controladas pelo atacante, não a um ataque pela rede — a cadeia realista é, portanto, um repositório de terceiros aberto ou uma configuração maliciosa injetada, não uma varredura de portas. Isso não torna a falha inofensiva, só deixa claro no que prestar atenção: código de terceiros no editor continua sendo código de terceiros.

PyTorch 2.13: FlexAttention no Mac

O PyTorch 2.13 saiu em 08/07 — 3.328 commits de 526 colaboradores desde a versão 2.12. O ponto relevante para experimentos locais: o FlexAttention agora roda no Apple Silicon (backend MPS). Em padrões de atenção esparsos, os saltos são grandes: numa forma 1×8×32768×64 com janela deslizante de 256 elementos (0,8 % de densidade), o FlexAttention leva cerca de 35 ms contra cerca de 431 ms do SDPA — fator 12,3. Um caso menor, com comprimento 8192 e janela de 64, chega a fator 4,15.

pytorch.org · Dados de 23/07/2026
Attention e "esparso": "Attention" é a etapa em que cada palavra do contexto olha para todas as outras para decidir o que importa agora — com 32 mil palavras, isso já passa de um bilhão de pares. "Esparso" quer dizer: a maioria desses pares nem interessa, cada palavra olha só para os 256 vizinhos mais próximos. É como uma sala de aula em que nem todo mundo conversa com todo mundo, só com quem senta ao lado. O FlexAttention nem chega a calcular os pares sem interesse, enquanto o SDPA mais antigo carrega todos eles — daí o fator de diferença. testar Se você roda modelos locais no Mac, esse é o motivo para instalar o 2.13.

O resto da nota de lançamento é utilidade do dia a dia: nn.LinearCrossEntropyLoss reduz o pico de uso de memória em até quatro vezes, um novo backend torchcomms e comunicação sobreposta do FSDP2 miram no treinamento distribuído, e há wheels para o Python 3.15.

pytorch.org · Dados de 23/07/2026
