Licitações Públicas (TED EU)

Contratos públicos com critérios obrigatórios documentados — por isso uma análise genuína de lacunas de parceiro é possível aqui. A twiceD é uma operação solo/2 pessoas; qualquer limite acima dessa capacidade exige um parceiro.

Licitação TED: Quando um órgão do governo alemão quer contratar software por mais de cerca de €220k, ele precisa publicar o processo em toda a UE — pelo "Tenders Electronic Daily". Imagine um leilão público onde não vence quem cobra menos, mas quem demonstra cumprir todos os requisitos: o órgão define mínimos obrigatórios antecipadamente (receita anual, tamanho da equipe, referências, certificações). Quem não cumprir um único critério nem pode participar.

Serviços Python — Administração Federal ⚠️ Parceiro

O Escritório de Compras Federais do BMI está adjudicando um contrato-quadro de serviços de TI em Python para o restante da administração federal (Lote 2). Valor máximo do quadro €22.159.550, prazo qui 02.07.2026, 11:30 CET. O stack (Python) é compatível com o perfil de backend da twiceD — mas não menos do que cinco critérios obrigatórios estão muito além da capacidade solo: certificação ISO-9001, pelo menos €6 M de receita anual em desenvolvimento de software, 42 especialistas em Python por ano e três projetos de referência com valor mínimo de €100k cada. Não é um caso de "uma lacuna" — é um caso de consórcio.

Parceiro fornece Comprovação de receita anual mínima de €6 M + certificado ISO-9001 + capacidade de pessoal para 42 FTE Python/ano + três referências federais/governamentais ≥ €100k. A twiceD poderia participar realisticamente apenas como subcontratada de um grande licitante principal — não como licitante líder.

AWplus — Web UI + PostgreSQL/PostGIS, BAF Langen ⚠️ Parceiro

O Escritório Federal de Aviação (BAF) em Langen está licitando o desenvolvimento contínuo do "AWplus" — proteção de instalações conforme § 18a LuftVG: uma interface web mais banco de dados PostgreSQL/PostGIS, hospedado no ITZBund. Valor €670.000, prazo ter 21.07.2026, 11:00 CET (CPV 72261000/72240000). Stack adjacente: desenvolvimento web + PostgreSQL são correspondência direta no CV, PostGIS (geodados) é aprendível — Java/Angular não são exigidos explicitamente. A lacuna é menor do que no contrato do BMI, mas dois critérios obrigatórios estão faltando.

Parceiro fornece Comprovação de €800k de receita em três exercícios fiscais (twiceD solo abaixo disso) + 2 referências no setor público com ≥ 100 dias-pessoa cada (twiceD não tem referências amplas em órgãos públicos). Mínimo de 4 FTE e seguro de responsabilidade de €2 M/€1 M são adicionalmente exigidos — viável como modelo de parceria com um prestador de serviços público estabelecido.

O que o board está dizendo hoje

O mercado de contratos está sóbrio e estável: 78 vagas verificadas, das quais 51 são freelance, e de longe o maior cluster é o pão com manteiga de Daniel — Java + Spring Boot + Angular, 100% remoto. As 7 primeiras (fit 85–90) são praticamente intercambiáveis: a mesma combinação de stack, os mesmos clientes finais de Frankfurt/Munique, a mesma faixa de 90–120 €/h. Quem quiser apresentar proposta hoje tem escolha, não escassez.

Dois padrões se destacam. Primeiro, durações incomummente longas: iSAX (41 meses), BridgingIT (45), 1st solution (46) e Kinteo (47 meses de Angular puro) não são preenchedores de lacunas de três meses, mas posições quase permanentes via contrato freelance — valiosas para segurança de planejamento, mas com risco de concentração porque travam a capacidade por muito tempo.

Segundo, o prêmio OAuth2/Segurança. A vaga principal (iSAX, fit 90) gira explicitamente em torno de autenticação — OAuth2, FIDO2, AWS Cognito — e esse é o maior diferencial de Daniel. Ao lado: o SDK de criptografia SINC JOSE/JWT (fit 85) e a vaga de Identity da Camunda. Onde segurança é o núcleo, as taxas são negociáveis para cima, não para baixo.

Sinal do Mercado Skills recorrentes nas vagas da Camada 2 (10+ ocorrências cada): Java, Spring Boot, Angular, Kubernetes, Kafka, AWS, PostgreSQL. Nicho hoje: OCPP/E-Mobility (Computer Futures), MQTT/Smart Energy (A3Watt), Java/Groovy Automotivo em Ingolstadt (HENNING) — todas na faixa de 100–130 €/h.

Mercado freelance: primeira queda desde 2016

O Freelancer-Kompass 2026 da freelancermap (pesquisa de 17/11/2025 a 08/02/2026, cerca de 5.400 participantes) traz a manchete do mercado do dia: a taxa horária média de todos os freelancers DACH em 2026 é de €103/h — contra €104/h em 2025. Parece ruído, mas é a primeira queda desde o início do estudo em 2016; após anos de crescimento constante, estagnação aparece pela primeira vez.

Para Daniel, o contexto importa mais do que a manchete. Primeiro: os frequentemente citados €94/h são a média da categoria Desenvolvimento de TI (Web/Mobile/Software) do estudo de 2025 — não um valor específico de Java. O objetivo de Daniel (€100/h+ para Java/Angular, 110–140 para nichos) está claramente acima dessa média ampla, o que as faixas do board de hoje (90–130 €/h) confirmam.

Segundo: Java é a segunda skill mais requisitada na freelancermap, logo atrás do SAP. A demanda pelo stack exato de Daniel, portanto, continua alta — a queda de taxa é um efeito de mercado amplo, não um efeito Java. O volume de projetos se estabilizou em torno de 90.000 projetos (nível de 2020, após o pico de 2022 de ~132.000).

Taxa média vs. sub-índice: "€103/h" é como o preço médio de todos os carros na Alemanha — do popular ao sedan de luxo. "€94/h Desenvolvimento de TI" é a média de um segmento, e "Senior Java Remoto" é um modelo específico e bem remunerado dentro dele. Aplicar a média geral a uma taxa de Senior Java seria como precificar uma S-Klasse pela média de todos os carros do mercado.
