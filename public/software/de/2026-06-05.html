Miasma: ein npm-Wurm, der sich über binding.gyp fortpflanzt

Die Story der Woche ist kein Modell-Launch, sondern ein Lieferketten-Vorfall — und einer mit neuer Mechanik. Ein selbstverbreitender npm-Wurm namens Miasma hat in einer Welle am 3. Juni laut StepSecurity 57 npm-Pakete in über 286 bösartigen Versionen in unter zwei Stunden kompromittiert. Zuerst erwischte es @vapi-ai/server-sdk (über 408.000 Downloads/Monat) um 23:30 UTC, dann u. a. ai-sdk-ollama (über 120.000/Monat).[1]

binding.gyp: Eine Konfig-Datei, mit der npm-Pakete C/C++-Anteile bauen. Stell sie dir vor wie den Bauplan, den ein Möbel-Paket beim Auspacken automatisch ausführt — normalerweise harmlos. Miasma nutzt eine 157 Byte kleine Datei mit gyp-Kommandosubstitution, um beim npm install Code auszuführen — komplett ohne die üblichen package.json-Lifecycle-Scripts, auf die jeder Scanner schaut. Der blinde Fleck ist das Konzept.

Der Payload erntet Multi-Cloud-Credentials breit: AWS-Keys und IMDS-Tokens, GCP-Service-Accounts, Azure-Identitäten, GitHub-Actions-Secrets (inklusive Speicher-Scraping des Runners), npm-Tokens, Vault- und SSH-Credentials. Mit den gestohlenen Tokens veröffentlicht der Wurm dann weitere infizierte Pakete — daher „Wurm" und nicht nur „Diebstahl".[1]

Abstammung: Miasma ist laut Microsoft-Analyse aus der Mini-Shai-Hulud-Malware abgeleitet, die ein Akteur am 12. Mai quelloffen ins Netz stellte — quasi ein Baukasten, den jetzt mehrere nachbauen.[2] Wer betroffen sein könnte: Lockfile prüfen, Install-Scripts beim CI mit --ignore-scripts blocken, Tokens rotieren.

Red Hat bestätigt: 32 kompromittierte npm-Pakete

Die erste große Miasma-Welle traf am 1. Juni Red Hat selbst: Unter dem Namespace @redhat-cloud-services wurden 96 Versionen über 32 Pakete kompromittiert — zusammen rund 116.991 Downloads/Woche (RHSB-2026-006).[3] Es sind Frontend-Libs der Hybrid Cloud Console. Einfallstor war ein gekapertes GitHub-Konto, über das bösartiger Code an der Code-Review vorbei eingeschleust wurde.

Red Hats Entwarnung ist instruktiv: Es wurde während des Vorfalls keine Console-Release publiziert, und der Publish-Prozess strippt Install-Scripts vor dem Deployment — die Schutzschicht hat gehalten. Für Endkunden „keine Aktion nötig". Aber wer die Pakete direkt aus npm zog, ist nicht durch dieselbe Wand geschützt.[3]

Einordnung Parallel meldete Microsoft eine separate Kampagne: 33 npm-Pakete, die per Dependency-Confusion Entwickler-Umgebungen ausspähen.[4] Zwei unabhängige Angriffe in einer Woche — die Registry-Schicht ist gerade der Hotspot, nicht der Code darin.

Wenn die Signatur lügt: SLSA-Provenance als Tarnung

Der unangenehmste Teil von Miasma ist nicht der Diebstahl, sondern die Tarnung. Der Wurm tauscht gestohlene Tokens gegen echte Sigstore-Signaturzertifikate von Fulcio ein und legt Einträge im Rekor-Transparency-Log an — die reinfizierten Pakete sehen damit legitim gebaut aus, inklusive gültiger SLSA-Provenance.[1]

SLSA-Provenance: Eine signierte Quittung, die belegt „dieses Paket wurde in dieser Pipeline aus diesem Commit gebaut". Gedacht als Echtheits-Siegel — wie ein Notar-Stempel. Miasma zeigt: Wer den Notar bestiehlt, kann jeden Stempel selbst setzen. Das Siegel beweist Herkunft, nicht Gutartigkeit.

Praktische Konsequenz für uns: „Hat Provenance" ist kein Freifahrtschein. Die Kette ist nur so stark wie der schwächste Token-Inhaber darin. Sinnvoller bleibt Tiefe — gepinnte Hashes, Install-Script-Blocking, und Misstrauen gegenüber Paketen, die übergroße Nicht-Entry-Point-Dateien mitbringen.

Achtung Miasma kippt zusätzlich Backdoor-Dateien in Repos — .claude/setup.mjs, .cursor/rules/setup.mdc, .vscode/tasks.json. Es vergiftet also die Tooling-Schicht, mit der Code generiert wird, nicht nur den Code. Diese Verzeichnisse mit auditieren.[1]

Build 2026, die nicht-KI-Hälfte: Windows wird zur Linux-freundlichen Dev-Box

Die Keynote-Schlagzeilen gehörten den Modellen — die laufen im ai-dev-Brief. Für uns interessanter ist die Plattform-Hälfte, und die ist diesmal substanziell.

Coreutils für Windows (GA)

Microsoft macht über 75 Unix-Style-Kommandozeilen-Tools nativ in Windows generell verfügbar — ls, cat, cp und Co. ohne WSL, ohne Git-Bash-Krücke. Sie kommen aus dem Open-Source-Projekt uutils, einer Reimplementierung der GNU Coreutils in Rust.[5] Für Cross-Plattform-Scripts ein echter Reibungs-Abbau.

WSL-Container (Preview)

Linux-Container lassen sich künftig direkt aus WSL heraus erstellen und betreiben — mit eigener CLI/API und Policy-Management für Unternehmen. Das zielt offen darauf, die Abhängigkeit von Docker Desktop & Co. zu senken.[6] Seit dem Open-Sourcing von WSL bei Build 2025 kommen laut Microsoft über 200 Pull Requests pro Monat rein — das Projekt lebt.[5]

Intelligent Terminal (experimentell)

Ein Terminal, das fehlgeschlagene Befehle erkennt, Kontext zeigt und Fixes vorschlägt — über ein Agent-Protokoll (ACP). Spannend, aber experimentell; hier nur als Plattform-Notiz, die Agent-Substanz ist ai-dev-Revier.[6]

Azure Linux 4.0 & Cobalt 200: Microsofts Linux-Maschinenraum

Azure Linux 4.0 ist in der Public Preview — und es ist Microsofts erste als General-Purpose deklarierte Server-Distro, nicht mehr nur Container-Host. Sie ist jetzt Fedora-basiert, quelloffen auf GitHub (azurelinux), mit kleinerem Paket-Footprint und strafferer Default-Deny-SELinux-Policy.[7]

Auf der Hardware-Seite ging die Cobalt 200-VM-Generation in Early-Access-Preview: Microsofts zweite Arm-Server-Generation (SoC auf Arm Neoverse V3), die laut Microsoft bis zu 50 % mehr Generationsleistung als Cobalt 100 bringt.[8] Für alle, die ARM-Workloads in der Cloud fahren, ist das die relevante Zahl der Woche.

RTX Spark Dev Box & GitHub Enterprise Local

Zwei Hardware-/Infra-Ankündigungen, die für die Lokal-vs-Cloud-Debatte zählen. Die Surface RTX Spark Dev Box ist eine kompakte Dev-Workstation mit NVIDIA-RTX-Spark-Silizium, 1 Petaflop Rechenleistung und 128 GB Unified Memory — genug, um Modelle jenseits von 120 Mrd. Parametern lokal zu fahren, ohne API-Rechnung. Verfügbar später 2026, vorerst nur in den USA über Microsoft.com; Preis noch offen.[6]

Infrastrukturell relevanter ist GitHub Enterprise Local: eine GitHub-Enterprise-Variante, die auf Azure Local läuft und wahlweise connected oder air-gapped betrieben werden kann — mit On-Prem-Inferenz über Foundry Local. Für regulierte oder souveränitäts-getriebene Umgebungen (Stichwort EU-Behörden) ein Signal; Preis und Datum noch offen.[9]

CISA KEV: Zwei Jahre alte WebLogic-Lücke wird scharf

CISA hat am 1. Juni CVE-2024-21182 in Oracle WebLogic Server in den Known-Exploited-Vulnerabilities-Katalog aufgenommen — die Lücke wird aktiv ausgenutzt. CVSS 7,5 (HIGH), unauthentifizierter Netzzugriff über T3/IIOP, betroffen sind 12.2.1.4.0 und 14.1.1.0.0. Oracle patchte sie schon im Juli-2024-CPU; ausgenutzt wird sie also gegen Nachzügler.[10][11]

KEV-Katalog: CISAs Liste der Lücken, für die belegte aktive Ausnutzung existiert. Kein „theoretisch gefährlich", sondern „wird gerade benutzt". US-Bundesbehörden müssen sie binnen Frist schließen — hier war die Frist gestern, der 04.06. Für uns ein gutes Priorisierungs-Signal, auch ohne Behörden-Pflicht.

Exposure ist real: laut Security Affairs sind über 1.592 WebLogic-Server online erreichbar und verwundbar (961 auf 12.2.1.4.0, 631 auf 14.1.1.0.0).[12] Wer Fusion Middleware betreibt: CPU-Stand prüfen.

EU/DE: Der CRA-Apparat fährt hoch

Standing Thread Regulierung: Am 11. Juni greifen die Bestimmungen zur Benennung der Konformitätsbewertungsstellen unter dem Cyber Resilience Act — in Deutschland veröffentlicht das BSI als notifizierende Stelle die Verfahren dafür.[15] Klingt bürokratisch, ist aber die Schiene, auf der die CRA-Pflichten ab 2027 für Hersteller vernetzter Produkte konkret werden.

Der für Devs schärfere Termin kommt im Herbst: Ab 11. September gilt die Meldepflicht für aktiv ausgenutzte Schwachstellen — binnen 24 Stunden.[15] Bußgelder bei Verstoß: bis zu 15 Mio. € oder 2,5 % des globalen Jahresumsatzes; das BSI stockt dafür bis 2029 um 141 Stellen auf.[16]

Releases im Rückspiegel: Node 24 & PostgreSQL 18.4

Ruhig an der Sprach-/DB-Front, aber zwei Mai-Releases sind erwähnenswert. Node.js 24 (21.05.) bringt das TC39-Proposal Explicit Resource Management nativ über V8 13.6 — das using-Keyword für deterministisches Aufräumen von Ressourcen, ähnlich try-with-resources in Java.[17]

PostgreSQL 18.4 (14.05.) ist ein reines Security-/Maintenance-Update der 18er-Linie — 11 Sicherheits-Fixes und über 60 Bugfixes, kein neues Feature.[18] Die eigentlichen Highlights kamen schon mit PostgreSQL 18.0 (Sep. 2025): das asynchrone I/O-Subsystem (bis zu 3× in I/O-lastigen Lasten), virtuelle Generated Columns und natives uuidv7() — letzteres spart vielen den Custom-Generator für zeitlich sortierbare IDs. Wer auf 18 ist, sollte 18.4 wegen der Security-Fixes einspielen.

Es ist Freitag, der fünfte Juni. Die Software-Woche dreht sich um die Lieferkette — und um eine Microsoft-Build-Konferenz, deren spannendste Teile gar nichts mit künstlicher Intelligenz zu tun haben.

Im Mittelpunkt steht ein npm-Wurm namens Miasma. In einer einzigen Welle am dritten Juni hat er laut StepSecurity siebenundfünfzig Pakete in unter zwei Stunden kompromittiert. Das Neue daran: Er verbreitet sich über eine Build-Datei namens binding.gyp und führt seinen Code schon beim Installieren aus — komplett an den Stellen vorbei, auf die Sicherheits-Scanner normalerweise schauen. Er stiehlt Cloud-Zugangsdaten von Amazon, Google, Azure und GitHub und veröffentlicht damit gleich die nächsten verseuchten Pakete.

Die erste große Welle traf am ersten Juni Red Hat selbst: sechsundneunzig Versionen über zweiunddreißig Pakete, zusammen rund hundertsiebzehntausend Downloads pro Woche. Red Hats Schutzschicht hat zwar gehalten, aber der Vorfall zeigt, wie verwundbar die Registry-Schicht gerade ist.

Am unangenehmsten ist die Tarnung. Der Wurm fälscht mit gestohlenen Tokens echte Signaturen über Sigstore — und gibt seinen Paketen damit eine gültige sogenannte SLSA-Provenance. Die Lehre für uns: Eine Signatur beweist, woher ein Paket kommt, aber nicht, dass es gutartig ist.

Auf der Microsoft-Build-Konferenz war die Plattform-Hälfte handfest. Windows bekommt über fünfundsiebzig Unix-Kommandos nativ, geschrieben in Rust. WSL kann jetzt Linux-Container direkt betreiben, ohne Docker Desktop. Und Azure Linux vier Punkt null ist Microsofts erste echte General-Purpose-Server-Distribution, jetzt auf Fedora-Basis.

Bei der Sicherheit hat CISA eine zwei Jahre alte Oracle-WebLogic-Lücke auf die Liste der aktiv ausgenutzten Schwachstellen gesetzt, mit einem Schweregrad von sieben Komma fünf. Über tausendfünfhundert Server stehen verwundbar im Netz. Die Frist für Behörden war gestern, der vierte Juni — Nachzügler sollten jetzt patchen.

Und in Europa fährt der Cyber Resilience Act weiter hoch: Ab dem elften Juni benennt das deutsche BSI die Prüfstellen. Der schärfere Termin für Entwickler kommt im September, wenn die Vierundzwanzig-Stunden-Meldepflicht für ausgenutzte Lücken greift.

Morgen wird es ruhiger — die Build-Sessions laufen on-demand weiter, und wir behalten die npm-Advisories im Auge, ob Miasma noch eine Welle nachschiebt. Damit zurück zur Übersicht.
