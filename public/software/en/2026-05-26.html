Google leaks its own exploit — 42 months unpatched

The top story is a self-dismantling of the security discipline: Google accidentally published an entry complete with a working PoC exploit on the public Chromium bug tracker for a flaw that is tracked internally as serious and has been privately known since late 2022 — roughly 42 months — but is still not patched to this day. The post was deleted, but had already been archived and circulated. The flaw abuses the interplay of service workers and the Background Fetch API, so that JavaScript keeps running persistently — across browser restarts and in part even device restarts. Trigger: merely visiting a crafted page. Every Chromium derivative is affected: Chrome, Edge, Brave, Opera, Vivaldi, Arc [1][2].

Persistent service worker: A service worker is a small program the browser runs in the background so a web app can respond even offline — like a doorman who stays after the office closes. If it turns malicious and survives restarts, you effectively have a foreign bug in your browser: it can turn your machine into a bot for DDoS attacks or crypto mining and track you via IP, timestamp and user agent — entirely without installed malware.

The punchline is not the bug but who leaked it. Google is the loudest champion of Project Zero's "90-day disclosure" rule — and itself holds a serious browser bug open for 42 months, only to make it public through its own slip-up. No public CVE so far, no patch. Concretely for Daniel: keep Chromium browsers current as soon as the fix lands, and if in doubt, check registered service workers in DevTools (Application → Service Workers) and unregister them [3].

TrapDoor — one credential stealer, three registries at once

The next supply-chain attack takes no break: TrapDoor (named by Socket) distributes a credential stealer across 34+ packages and 384+ versions simultaneously over npm, PyPI and Crates.io. The disguise: generic dev tools and "security scanners", targeting crypto/DeFi/Solana and AI developers. The earliest package was eth-security-auditor@0.1.0 on PyPI on 22.05, 20:20 UTC. A different auto-run mechanism fires per ecosystem: npm via a postinstall hook (a 1,149-line harvester that validates AWS/GitHub tokens via a live API before exfiltration), PyPI on import (loads a payload from GitHub Pages via node -e — behavior changeable after the fact), Crates.io via build.rs during cargo build (XOR-encrypts keystores, exfiltrates via GitHub Gists) [1][2].

Install hook: Many package systems allow code that runs automatically on install or build — npm's postinstall, Rust's build.rs, Python import. It's meant for setup tasks (compiling a binary or similar). For an attacker it's the ideal door: you don't even have to use their package; just npm install or cargo build starts their code — with your privileges, your tokens, your SSH keys.

The remarkable twist: TrapDoor deliberately poisons AI assistants by manipulating .cursorrules and CLAUDE.md files with instructions hidden via zero-width Unicode — the assistant is supposed to carry out the exfiltration itself under the guise of an "automatic security scan". This is not an AI story hook but the next tooling layer as an attack surface. Good news: Socket detected the releases in about ~6 minutes on average. Daniel's takeaway stays the old one: fresh packages with a cooldown, --ignore-scripts where possible, take lockfiles and provenance seriously [3].

Ghost CMS — old fix, new mass campaign

A lesson in patch discipline: CVE-2026-26980 (Ghost CMS, NVD 7.5 / GitHub CNA 9.4) is an unauthenticated SQL injection in the Content API through which attackers can read arbitrary DB contents, including the Admin API keys. Affected: Ghost 3.24.0 to 6.19.0. The fix has existed since Feb 19 in 6.19.1 — yet, according to XLab/Qianxin, 700+ domains were hijacked, among them Harvard, Oxford, Auburn and DuckDuckGo. With the stolen keys, the attackers inject JavaScript into articles that fingerprints visitors and presents selected targets with a ClickFix trap [1][2].

ClickFix: The page pretends you have to prove you're human — a fake Cloudflare window, "copy this command and run it". Whoever pastes that into the Windows command prompt starts the malware themselves. The trick bypasses every download protection, because you run the command, not the page.

The real lesson is not the bug but the three months in between: a February patch that still leaves 700 sites open in May is a patch-uptake problem, not a 0-day. Anyone running Ghost: immediately move to ≥6.19.1 and rotate Admin API keys [3].

JDK 27 — the feature freeze forks on Thursday

For the JVM side, the most important distant date in view: JDK 27 reaches Rampdown Phase One (RDP1) on 04.06 — from then on the feature set is frozen and the code forks from the mainline. The schedule has been fixed since the review ended on 13.04: RDP2 on 16.07, RC1 on 06.08, GA on 14.09.2026. JDK 27 is the next LTS line, and two defaults are considered set: G1GC as the default garbage collector and Compact Object Headers as standard (noticeably lowering per-object heap overhead). The Vector API, meanwhile, enters its 12th incubator round (JEP 537) — it won't be finalized until Project Valhalla [1][2].

What forks on 04.06 lands in September — anyone on JDK 25 (the current LTS) sees the next migration target here. JDK 26 has been GA since 17.03, but remains a non-LTS intermediate step [3].

CADA — Brussels postpones for the fourth time

The clarification on yesterday's brief: the date expected for Wednesday 27.05 is off. The EU Commission postpones its entire tech-sovereignty package (CADA + Chips Act 2 + open-source strategy + AI-in-energy roadmap) to Wednesday 03.06 — the fourth attempt after 25.03, 15.04 and 27.05. EVP Henna Virkkunen wants to present it before the Telecom Council on 09.06. Reasons per reports: comments by the US ambassador to the EU about the trade consequences of digital regulation — and simply that the "400 pages weren't yet ready" for the commissioner's signature [1][2].

Digital sovereignty: The question of who can pull the plug in an emergency. If government data sits on a US cloud, the provider is subject to US law — a foreign government could demand access. "Sovereignty" means keeping critical data under your own legal jurisdiction, like a safe-deposit box in your own bank.

The explosive paragraph remains in the leaked draft (CNBC, 07.05): under consideration is restricting US cloud for sensitive government data (health, justice, finance) — not a ban, not for the private sector. Context: US providers hold ~70% of the EU public-cloud market (AWS ~31%, Azure ~22%, Google ~10%). Anyone serving public contracts in the EU reads the real text on 03.06 — if it arrives this time [3].
