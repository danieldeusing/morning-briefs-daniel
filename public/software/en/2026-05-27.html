KEV wakes up — a 10.0 on a three-day deadline

After four days of no movement, CISA KEV jumps to catalog version 2026.05.26 (released 2026-05-26, 17:02 UTC) and takes in a heavy one: CVE-2026-48172 in the LiteSpeed cPanel plugin. It's a privilege escalation in which any cPanel user runs arbitrary scripts as root via the improperly secured Redis enable/disable feature (lsws.redisAble). NVD rates it 9.8 (CVSS v3.1) and 10.0 (v4.0) — the maximum. The deadline stands out: CISA set it to Friday 2026-05-29, just three days instead of the usual two weeks — a signal of active mass exploitation. Affected: cPanel plugin <2.4.7 and WHM plugin <5.3.1.0; fixed in 2.4.7 and 5.3.1.0 respectively [1][2].

For Daniel this is news, not an alarm — he runs Caddy on his own infra, no cPanel/LiteSpeed. The lesson is in the contrast: a fresh 10.0 gets three days, today's expiring Drupal deadline (CVE-2026-9082) got five — but NGINX "Rift" (CVE-2026-42945, F5 CVSS 9.2, actively exploited since ~2026-05-16, patched since 2026-05-13) is still not in the catalog as of 2026.05.26. An actively exploited 9.2 web-server flaw missing ~13 days after a patch, while a cPanel plugin gets listed within hours — the KEV's prioritization is harder to read than its score promises [3].

Starlette — a Host header defeats path auth

Relevant to Daniel's Python side, but no panic factor: CVE-2026-48710 in Starlette (the ASGI substrate under FastAPI) is a missing Host-header validation. GitHub and NVD rate it moderate, CVSS 6.5 (vector C:L/I:L/A:N). A crafted Host header with special characters like / or ? poisons how Starlette computes request.url.path — which lets path-based security middleware be bypassed while the actual routing runs normally. Example: Host: example.com/health?x= can make middleware believe a request to /protected targets /health. Found by the X41 D-Sec team (OSTIF audit), patched in Starlette 1.0.1, affecting ≤1.0.0 [1].

Path-based auth middleware: a bouncer layer that checks, before the actual code runs: "Path /admin? Then require a login." Picture a bouncer who only glances at the sign above the door. If the attacker can forge the sign without changing the room behind it, the bouncer lets them through — even though they land in the restricted area.

Concretely: if you do auth in FastAPI via route dependencies or the security layer (the normal case), you're not affected — the flaw only bites with hand-rolled, path-prefix-based middleware. Still a cheap patch: pull starlette>=1.0.1 [1].

Webmin — 2FA bypassable, help function hands out root

From the 7-day recap, because it's dev-relevant: CVE-2026-42210 in Webmin (the widespread web admin panel for Linux servers) is a double problem rated CVSS 9.8. First, two-factor authentication can be bypassed if an attacker has valid credentials and basic auth is enabled; second, the help function escalates to a root shell. JDK 27 — the feature freeze forks on Thursday

For the JVM side, the week's key date: JDK 27 reaches Rampdown Phase One (RDP1) on 2026-06-04 — from then the feature set is frozen and the code forks off the mainline. The schedule has been fixed since the review closed (2026-04-13): RDP2 07-16, RC1 08-06, GA 2026-09-14. JDK 27 is the next LTS line, and two defaults are considered locked: G1GC as the default garbage collector and Compact Object Headers as default (cuts per-object heap overhead). The Vector API enters its 12th incubator round (JEP 537) — it won't go final until Project Valhalla. If you're on JDK 25, this is the next migration target [1][2].

Microsoft wants a safer C# — without turning it into Rust

A language note with substance: .NET PM Richard Lander has sketched out how the team wants to harden C#'s memory safety — via a new model and a redefinition of the existing unsafe keyword, without giving up automatic memory management (garbage collection). The goal is to lift C# into the set of languages known for their type- and memory-safety guarantees. Important for most readers: business-app developers won't notice, since they never write unsafe code — it affects the low-level corner. Lander calls it one of the "highest-leverage changes" they could make; reaction so far is largely positive. It's distinct from Microsoft's separate push to replace C/C++ in systems code with Rust [1][2].

Who controls the stack — Cisco opens SONiC, Alibaba boots RISC-V

Cisco: SONiC for all Nexus 9000 customers

Cisco is set to bring the open network operating system SONiC to its flagship Nexus 9000 switches — and for all customers, no longer just hyperscalers. SONiC ran on Cisco's routers before; soon N9000 buyers will be able to choose among NX-OS, ACI and SONiC on the same hardware, hardened and managed via the Nexus Dashboard. Reported by The Register (2026-05-27) [1].

SONiC / NOS: a "network operating system" is the OS of a switch — like Linux for a server, only for the box that forwards your data packets. SONiC is the open-source, Microsoft-initiated flavor of it. Cisco allowing it on its own hardware means: you're no longer chained to Cisco's in-house NX-OS, you can run the same free software the big cloud operators run.

Alibaba: Android 16 on its own RISC-V

Fitting the same thread: Alibaba's DAMO Academy has ported Android 16 to its own RISC-V silicon (XuanTie 9 series) and claims to be the first to get an RVA23 chip running Android 16. The work was shared with "strategic customers" to shorten the path from chip prototype to product. The subtext is sovereignty under US export controls — a homegrown, royalty-free instruction set as insurance against dependency [2].

Linux 7.1-rc5 — Torvalds tightens the reins

Mainline sits at 7.1-rc5 (released Sun 2026-05-24; stable 7.0.10 from 2026-05-23) — already in the 7.1 cycle, no longer 6.x. Torvalds called rc5 unusually large and got blunt: "yes, several of these series were triggered by AI code review," and he'll be pushing back on "pointless pull requests with fixes that just aren't that important" — because "large rc weeks are not conducive to long-term stability." That follows his 2026-05-18 remark that the kernel security list is "almost entirely unmanageable" from AI bug-hunters flooding duplicates. It's not about a model, it's about the load on the contribution process from machine-generated noise [1][2].
