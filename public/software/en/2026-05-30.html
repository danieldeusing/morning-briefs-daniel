Three supply-chain hits in one KEV update [1][2]

On May 27, 2026, CISA added three flaws to the Known Exploited Vulnerabilities (KEV) catalog — and all three are not classic code bugs but compromised supply chains. Affected: the React state-management ecosystem TanStack (CVE-2026-45321), the VS Code extension Nx Console (CVE-2026-48027), and the disk-image mounter DAEMON Tools Lite (CVE-2026-8398). Federal agencies must remediate by June 10.

Supply-chain attack: Instead of attacking your code directly, the attacker poisons something you install anyway — a library, a tool, an installer. Like someone not breaking into your house, but swapping out a package the delivery person brings to your door, which you carry inside yourself.

In TanStack, attackers abused GitHub Actions via cache poisoning, a misconfigured pull_request_target permission, and stolen OIDC tokens from runner memory to publish 84 malicious versions across 42 @tanstack npm packages. The fix is in 5.68.0. Nx Console v18.95.0 sat in the VS Marketplace and OpenVSX for around 36 minutes on May 19 — triggered because a developer machine had previously pulled a compromised TanStack package. DAEMON Tools, meanwhile, shipped trojaned, signed binaries directly from the vendor site between early April and early May; clean as of version 12.6.

The uncomfortable lesson: two of the three incidents took root through the developer's tooling layer — npm packages and an IDE extension — not the production system. Your laptop and your pipeline are the perimeter.

Mini-Shai-Hulud: the worm that steals from CI [3][4]

The TanStack compromise was just one wave of a larger campaign that Microsoft and Palo Alto Unit 42 bundle as "Mini-Shai-Hulud" — successor to the Shai-Hulud worm from September 2025. On May 19, the @antv npm organization was seized; through dependency chains, the package echarts-for-react — with over one million weekly downloads — was reached. In a single hour, Unit 42 counted 639 malicious versions across 323 packages — the largest single-hour wave to date.

The mechanics are interesting. The payload — roughly 499 KB of obfuscated JavaScript — runs silently on npm install and is purpose-built to pull secrets from GitHub Actions environments: GitHub tokens, AWS, HashiCorp Vault, npm, Kubernetes, even 1Password.

Runner-memory scraping: To prevent secrets from appearing in build logs, GitHub masks them (shows *** instead of the value). The malware bypasses this by not reading the log, but instead scraping plaintext secrets directly from the memory of the build process (Runner.Worker via /proc) — like someone who doesn't read redacted files but reads over the shoulder while the clerk types.

GitHub hit back hard: 640 malicious packages removed and 61,274 granular npm tokens with write permissions revoked. That even OpenAI published a statement about the TanStack incident shows how broad the impact was.

TrapDoor: three registries, one malware — and poisoned AI rules [5]

Fresh on the scene: TrapDoor, discovered by Socket: 34 malicious packages across 384 versions, spread across npm, PyPI and Crates.io simultaneously — each with ecosystem-specific execution path: build.rs in Rust, postinstall in npm, import-time in Python. First activity from May 22 (artifacts trace back to May 19). Target: crypto/DeFi/Solana and AI developers, after their wallet keys, SSH, and cloud credentials.

The remarkable twist: TrapDoor specifically poisons AI coding assistants by manipulating .cursorrules and CLAUDE.md files and injecting instructions hidden with zero-width Unicode — the assistant is supposed to exfiltrate credentials itself. This is exactly the AI-as-attack-surface secondary effect, not the model itself. Socket detected the releases at a median of 5 min 27 sec — the half-life of these attacks is now minutes, not days.

  1. socket.dev — TrapDoor campaign (May 2026)

What this means in practice

No "patch tonight" alarm, but three straightforward hygiene points that Unit 42 recommends and that apply directly to Daniel's setup (GitHub Actions, npm frontends, Caddy/Docker):

CI Hygiene (1) In CI, use npm ci instead of npm install + lockfile pinning. (2) Disable lifecycle scripts via ignore-scripts=true in .npmrc where possible. (3) Egress filtering on runners and provenance checks (slsa-verifier against the OIDC attestation). Rotate tokens that were in affected runs — not just GitHub, also npm and Vault tokens.

Cisco SD-WAN: CVSS 10.0 under active exploitation [6][7]

Cisco updated its advisory for CVE-2026-20182 again on May 27 — reason enough to call it out here. It's an authentication bypass (CWE-287) in the peer authentication of Cisco Catalyst SD-WAN Controller and Manager with the maximum CVSS score of 10.0. An unauthenticated remote attacker can gain admin rights.

SD-WAN: Software-controlled enterprise-wide area network — a central controller orchestrates how data flows between sites. Anyone who takes over the controller orchestrates the whole network. That's exactly what this flaw allows.

Cisco PSIRT speaks of "limited exploitation." Behind the scenes, a whole campaign has been running since February: the actor UAT-8616 has been exploiting Cisco SD-WAN since at least 2023, and after proof-of-concept code surfaced, roughly ten other clusters jumped in. CISA issued Emergency Directive 26-03 with a federal agency deadline of May 17. Patches are available for all supported releases.

EU Cyber Resilience Act: the 24-hour clock ticks starting September [9]

File this away for future reference, not this week's timeline: With the EU Cyber Resilience Act, starting Sept. 11, 2026, Article 14's notification requirement kicks in — an actively exploited vulnerability or serious incident must be reported initially within 24 hours, supplemented within 72 hours, and finalized within 14 days. Rules already apply to conformity assessment bodies from June 11; full applicability by Dec. 11, 2027.

For a one-person software shop like twiceD, this isn't a crisis, but it's relevant: "products with digital elements" sold in the EU fall under this — including B2B software. The incidents this week are exactly the scenario that would start this clock.

Google I/O — belongs elsewhere

The big headline of the day was Google's I/O AI launches (Gemini 3.5 Flash, Managed Agents, Antigravity, plus xAI Grok Build). That's an AI keynote and not a software lead — details in the ai-dev brief. What matters here is the secondary effect: the more autonomous agents move into our tooling and CI layer, the bigger becomes exactly the attack surface that TrapDoor and Mini-Shai-Hulud demonstrated this week.

It's Saturday, May thirtieth. While the tech world focused on Google's developer conference and its AI announcements, the more important week for developers played out one level deeper — at the package registries. The clear theme this week: the software supply chain has become the primary attack target.

On Tuesday, U-S security agency CISA added three compromised supply chains to its catalog of actively exploited vulnerabilities. Affected are the React tool TanStack, the Visual Studio Code extension Nx Console, and the disk tool DAEMON Tools. At TanStack, attackers abused GitHub Actions and injected eighty-four malicious versions across forty-two packages. Federal agencies must clean up by June tenth.

These three cases were just the tip. Microsoft and Palo Alto bundle an entire wave under the name Mini-Shai-Hulud. A roughly four-hundred-ninety-nine kilobyte malware runs silently during package installation and specifically targets secrets from GitHub Actions environments — by reading them directly from the memory of the build process, bypassing the usual masking. GitHub responded by removing six-hundred-forty packages and revoking over sixty-one thousand npm tokens.

Even fresher is a campaign called TrapDoor, discovered by the firm Socket. Thirty-four malicious packages across npm, PyPI, and Crates.io simultaneously — and particularly insidious: it even poisons the rule files of AI coding assistants like Cursor and Claude, with hidden characters so the assistant itself exfiltrates credentials. That's the real lesson from Google's announcements: the more agents move into our tools, the larger this attack surface becomes.

Alongside that, two classic threats. Cisco updated its advisory for a flaw in its SD-WAN network product on Tuesday — maximum score ten point zero, an authentication bypass under active exploitation. And PostgreSQL plugged eleven security holes and over sixty bugs in mid-May; that update belongs on the roadmap.

For releases: Angular twenty-two is in Release Candidate, final version comes in June — Angular nineteen reached end-of-life on May nineteenth. Node Point js twenty-six enables the new Temporal time interface by default. And Caddy now requires Go one point twenty-five.

One date to keep in mind: starting September eleventh, the European Cyber Resilience Act requires notification of actively exploited flaws within twenty-four hours. Exactly the incidents this week would trigger that clock.

The practical consequence is straightforward: npm ci instead of npm install, disable lifecycle scripts, rotate tokens. Supply-chain threats will likely continue next week — we're watching the Socket and Unit Forty-two trackers for new waves.
