Mini Shai-Hulud: an npm worm that signs itself validly [1][2][3]

The week's biggest software story is not a release, but a crack in the trust model. The group "TeamPCP" launched a self-propagating worm called Mini Shai-Hulud across npm and PyPI registries on May 11 — roughly 169 npm package names with over 370 malicious versions, including household names like @tanstack/* (TanStack Router serves ~12 million downloads per week), @mistralai/*, and over 70 @uipath packages.[1] Tracked under CVE-2026-45321, CVSS 9.6.[2]

SLSA Provenance: Imagine every package gets a sealed receipt proving "this artifact came from exactly this verified build pipeline." The worm issued that receipt to itself — it hijacked a real GitHub Actions pipeline using stolen OIDC tokens and generated valid Build Level 3 attestations with it. The receipt is genuine, the contents are poison. It's the first documented npm worm to produce validly attested malware packages.[3]

Mechanics: The worm hunts for an npm token with bypass_2fa = true (published without second factor), then queries all packages owned by that maintainer and poisons them further. TanStack traced the initial breach to a chain of pull_request_target, GitHub Actions cache poisoning, and memory extraction of an OIDC token from the runner process — three gaps, none of which alone would have sufficed.[1]

Heads up Practical consequence for Daniel's CI: Provenance signatures tell you which pipeline built something, not whether it behaved correctly. If you install npm lockfiles with --ignore-scripts, keep OIDC tokens short-lived, and enforce 2FA on publish tokens, you strip the worm of its three levers. Tokens that may have been exposed in the wave should be rotated.

Typosquatting targets CI/CD secrets directly [4]

Fitting the worm wave, a second supply-chain story that Microsoft analyzed on May 28: A single actor (alias vpmdhaj) published 14 malware packages within four hours, spoofing OpenSearch, ElasticSearch, and DevOps libraries (names like opensearch-setup, elastic-opensearch-helper).[4]

The twist is target selection: The ~195-KB credential harvester runs silently on install and targets AWS IAM/STS, HashiCorp Vault tokens, GitHub Actions secrets, and npm publish tokens — exactly the keys that compromise the supply chain further down. Microsoft's advice mirrors the above: npm install --ignore-scripts, and rotate all CI tokens that may have touched an affected runner or dev machine.[4]

PAN-OS GlobalProtect: CISA deadline expires today [6][7]

Palo Alto's GlobalProtect has an authentication bypass, CVE-2026-0257, CVSS 9.1. An unauthenticated attacker forges "Authentication-Override" cookies and builds an unauthorized VPN tunnel into the internal network.[6] Rapid7 spotted the first attacks starting May 17 (first wave May 18 from Vultr infrastructure, second starting May 21). CISA added the gap to the KEV catalog on May 29 with agency deadline June 1 — today.[7]

Affected: PAN-OS 10.2.x, 11.1.x, 11.2.x, and 12.1.x, plus Prisma Access deployments. Anyone running exposed GlobalProtect gateways without patches should upgrade to the fixed version for their branch (e.g., 10.2.13-h18).

KEV Catalog: CISA's list of "known exploited vulnerabilities." US agencies must patch entries by the stated deadline — for everyone else, a KEV listing is the strongest signal that "this is being actively exploited right now," not just theoretically possible.

Dirty Frag: 9-year-old kernel bug, PoC public [8][9]

In circulation since May 8, but remains an ongoing task: "Dirty Frag" (CVE-2026-43284 + CVE-2026-43500, plus CVE-2026-46300 in Red Hat's RHSB-2026-003) is a local privilege escalation in the in-place decrypt path of esp4/esp6 and rxrpc. The ESP portion was introduced in 2017, the RxRPC portion in 2023 — the gap is old.[8]

Critical: A working PoC is public, and any local unprivileged user reaches root with a single command. The exploit needs only standard syscalls and modules (esp4, esp6, rxrpc) that ship enabled in default kernel packages across all major enterprise distros — Ubuntu, RHEL, Alma, Fedora, openSUSE. Microsoft reported active post-compromise exploitation. Patches are linked from NVD and available from the distros; KernelCare delivered live patches within 24 hours.[9]

Assessment No remote vector — requires a foothold already. But for precisely that reason, it's a classic "privilege escalation after initial access" component that you patch on every Linux box (including ddmini/ddStudio) via regular distro updates.

BSI warns of 7-Zip gap [10]

From Germany: The BSI warned on May 26 about the 7-Zip vulnerability CVE-2026-48095 (CVSS 8.8, discovered by GitHub Security Lab). A crafted NTFS container lets the stream-buffer size shift to exponent 32 — the buffer is mis-allocated to 1 byte, the next read triggers a heap overflow, which enables code execution.[10] Treacherous: The signature fallback logic also routes files with endings .7z, .zip, or .rar through the NTFS parser. Fix is 7-Zip 26.01 (late April); everything through 26.00 is affected.

Angular 22 ships this week — the "signal-first" era [11][12]

For Daniel's frontend side, the release of the week: Angular 22 is scheduled for the week of June 1 (RC.1 has been in circulation since May 20).[11] It consolidates what's been building since Angular 17:

  • Zoneless as default for new projects, new components standardly OnPush.
  • Signal Forms stable — the signal-based form system experimental in 21 replaces the classic Reactive Forms pattern with a lighter API.
  • Selectorless Components: components can be imported directly in templates without a selector string.
  • Vitest is the new default test runner; experimental Jest/web-test-runner paths are removed.
Zoneless / Signals: Previously, Angular listened via Zone.js to every event, then checked the whole component tree for changes — like a janitor who walks the entire building after every noise. Signals are tiny containers that notify Angular when their value changes — Angular updates only the places that actually depend on that signal. Less work, faster UI, no Zone.js overhead.
Assessment If you're on Angular 21, the jump should be relatively smooth — the hard shift (signals, standalone, control flow) is behind us. Reactive Forms stay for now, but for new work, Signal Forms are worth exploring.

PostgreSQL collection release patches 11 gaps [13]

Database maintenance: PostgreSQL 18.4, 17.10, 16.14, 15.18, and 14.23 shipped on May 14 — 11 security fixes plus over 60 bugfixes.[13] Three are relevant for Daniel's Postgres setups: a missing authorization check in CREATE TYPE (an object creator can hijack foreign queries that resolve types via search_path); a covert timing leak in MD5 password comparison (does not affect scram-sha-256, the default); and uncontrolled recursion in SSL/GSS negotiation that permits sustained DoS via the AF_UNIX socket.

Chrome 148 patches two critical RCE gaps [14]

Browser hygiene: The Chrome update from May 19 (version 148.0.7778.178/179) patches multiple gaps, two critical — a use-after-free in WebRTC (CVE-2026-9111, RCE via a crafted page visit) and a faulty UI implementation on Windows. Neither was actively exploited before the patch. Anyone on managed or manually updated systems should not wait for the staggered rollout.[14]

CRA countdown: reporting obligation kicks in September [15]

Ongoing regulation thread: The EU Cyber Resilience Act activates reporting obligations on September 11, 2026 — actively exploited gaps and severe incidents in "products with digital elements" must be reported via Single-Reporting-Platform within 24 hours (early warning) or 72 hours, with a final report due within 14 days of available remediation.[15] It also applies retroactively to products already on the market. Practically, for anyone shipping software in the EU: set up SBOM and a vulnerability/VEX process now, not in September. Full product obligations kick in December 11, 2027.

It's Monday, June first. Today's news turns on software supply chains — and a line that was crossed this week.

An attacker group called TeamPCP launched a self-propagating worm named Mini Shai-Hulud across npm and PyPI registries. Around one hundred sixty-nine package names were poisoned, including household names like TanStack, Mistral, and UiPath. What's new: the worm shipped its malicious packages with a valid authenticity signature. This so-called Provenance is supposed to prove a package came from a verified build pipeline — and the worm issued that proof to itself by hijacking a real pipeline with stolen tokens. It's the first documented case of its kind. Anyone who might have been caught in the wave should rotate their CI tokens and install packages with the ignore-scripts option.

A second supply-chain story, analyzed by Microsoft in late May, fits right in: fourteen fake packages spoofing OpenSearch and Elastic libraries that specifically target AWS and Vault tokens — the exact keys used to compromise the chain further.

Meanwhile, a deadline expires today. The US Cyber Agency CISA added an actively exploited vulnerability in Palo Alto's GlobalProtect VPN to its list — an authentication bypass with a severity of nine point one, observed since May seventeenth. Agencies must be patched today. And in the Linux kernel, Dirty Frag remains current: an old privilege escalation for which a public proof of concept exists. Patches are available; roll them out.

On the brighter side: Angular twenty-two is expected to ship this week. It makes signal-based, zoneless architecture the standard, Signal Forms become stable, and Vitest replaces the old test runner. PostgreSQL closed eleven security gaps in mid-May, and Chrome is at version one hundred forty-eight with two critical fixes.

In the background, the clock is ticking for the European Cyber Resilience Act: from September eleventh onward, there's a reporting obligation for exploited gaps. Anyone shipping software in the EU should set up SBOM and a vulnerability process now.
