TrapDoor: a worm template across three registries at once

The biggest fresh supply-chain story is TrapDoor — a campaign that, according to Socket, rolled out 34 malicious packages in over 384 versions simultaneously across npm, PyPI and Crates.io (disclosed on 05/25).[1] The payload harvests everything valuable in a dev setup: crypto wallets (Sui, Solana, Aptos), SSH keys, browser profiles, AWS environment variables — then anchors itself persistently via systemd services, cron jobs, Git hooks, and shell hooks.

Supply-chain attack: Instead of hacking your machine directly, attackers poison a software ingredient you're going to download anyway — like someone putting poison in a flour bag at the supermarket instead of breaking into your home. On the next npm install, you bake it yourself.

What's genuinely new: TrapDoor explicitly targets AI coding workflows. It rewrites .cursorrules and CLAUDE.md in projects, so an AI agent willingly carries the malicious logic on the next run — the agent-tooling layer becomes a persistence vector, not the model. Good news: Socket caught the releases with a median detection time of 5 min. 27 sec. before they gained wide adoption. That's the point — detection speed beats prevention now, because publishing costs seconds.

CIFSwitch: 19-year-old Linux bug gives away local root

On 05/28, researcher Asim Manizada disclosed CIFSwitch — a local privilege escalation in the Linux kernel's CIFS client that has been dormant since roughly 2007.[2] When a CIFS share is mounted with Kerberos, the kernel calls the root-privileged helper cifs.upcall via the keyring system. By forging cifs.spnego key descriptions plus user and mount namespace tricks, an unprivileged local user forces the helper to load a malicious NSS library — and lands as root.

Red Hat lists this in RHSB-2026-005 as "Important" and flags RHEL 7, 8, 9, and 10 plus OpenShift, OpenStack, and Virtualization as affected.[3] A CVE number is still pending, the upstream kernel patch exists, but distro updates are rolling out — and a PoC is already public. For Daniel's Linux/Docker world, this means: hosts mounting CIFS/SMB shares are the problem children. Not a "patch tonight" emergency, but until the distro update lands, it's worth checking whether unprivileged users are allowed to mount CIFS at all.

Quarkus emergency releases: semicolon bypasses auth

Straight into Daniel's Java territory: Quarkus pushed emergency releases across all supported streams against CVE-2026-39852 (disclosed 05/05, Red Hat "Important").[4] The root is classic and instructive — a path normalization inconsistency: the security layer checks the raw URL path including matrix parameters (semicolons), while RESTEasy Reactive strips semicolons before routing. Result: /api/admin;anything bypasses the authorization policy for /api/admin, but still lands on the protected endpoint.

Fixed in 3.20.6.1, 3.27.3.1, 3.33.1.1, 3.35.1.1, 3.34.7, and 3.35.2. The lesson applies beyond Quarkus: if you build path-based authz, you must ensure security filters and router see the same normalized path. This class of bug surfaces in every framework that lets two layers match independently.

Project Lightwell: $5B to make open source "belong" to somebody

On 05/28, IBM and Red Hat announced Project Lightwell — a $5 billion commitment plus over 20,000 engineers to help enterprises secure open source.[5] The clever part: instead of forcing companies onto a new version, Lightwell aims to backport the fix to exactly the version already running in production. That's the real pain point in regulated shops, where a major upgrade costs a quarter.

Backporting: A repair built for the latest version, carefully adapted back to an older one — like reworking a replacement part meant for a new car model so it fits a ten-year-old model without swapping the whole engine.

First customers are eleven financial houses — Bank of America, Citi, Goldman Sachs, JPMorgan, Mastercard, Visa, and others; it will be sold as a subscription later. The AI angle: Anthropic's "Mythos Preview" found ~3,900 gaps with high or critical severity when scanning open-source software.

My take The model is genuinely useful — backport-over-upgrade is exactly what the middle market needs. But "IBM becomes security clearinghouse for open source" has an aftertaste: a single corporation as gatekeeper for which fixes count as trustworthy centralizes exactly what makes OSS strong in its decentralization. The Linux community is reacting accordingly mixed. For twiceD customers: adopt the idea (targeted backports), not the dependence on a gatekeeper.

Edge devices are burning: PAN-OS and WebLogic actively exploited

Two classics under live fire. With the PAN-OS GlobalProtect auth bypass (CVE-2026-0257, CVSS 7.8 per Palo Alto advisory), attackers forge override cookies to VPN in without logging in.[6] Rapid7 observed successful exploitation starting on 05/17 across multiple customers, in two waves on 05/18 and 05/21 — the CISA KEV deadline expired yesterday (06/01).[7]

Source note A security aggregator listed CVE-2026-0257 with CVSS 9.1; the official Palo Alto advisory states 7.8. I'm taking the vendor value.

In parallel, CISA added an old Oracle WebLogic gap (CVE-2024-21182) to KEV on 06/01: unauthenticated access via T3/IIOP protocols, agency deadline 06/04. Both stories tell the same lesson — the most dangerous attack surface is always the box at the network edge that everyone forgets about because it "just works." KEV deadlines formally apply only to US agencies, but they're the best free prioritization list available.

Chrome 149 is stable today

Google is rolling out Chrome 149 as stable today — solid web-platform substance without AI hooks.[8] Highlights for web devs: pages with open WebSocket connections (Notion, Figma, Slack Web) now qualify for BFCache, so the back button loads instantly; element.scrollTo() now returns a Promise (finally cleanly awaitable); and CSS Gap Decorations let you natively style gaps in grid/flex instead of manual separator divs.

BFCache (Back/Forward-Cache): The browser freezes a page when you leave instead of throwing it away — like a bookmark that doesn't just remember the page number, but holds the entire open book in exact position. Back = instant, no reload.

In the broader web-platform landscape, WebGPU is now Baseline across all major browsers, and Safari 26.5 brings the :open pseudo-class for <details>/<dialog> — small stones that together continue closing the platform gap between engines.

CRA clock: reporting obligation from September, plannable today

The EU Cyber Resilience Act casts its shadow: Starting 09/11/2026, the reporting obligation kicks in — makers of "products with digital elements" must disclose actively exploited vulnerabilities within 24 hours as early warning and 72 hours as full disclosure to ENISA/CSIRT.[9] The full load (secure-by-design, SBOM, CE marking, conformity assessment) arrives later on 12/11/2027, but the reporting machinery must be in place first.

SBOM (Software Bill of Materials): An ingredient list for software — which libraries in which versions are included. Like the label on a food package: if a poisoned ingredient shows up tomorrow, you instantly know whether it's in your product.

Penalties: up to €15 million or 2.5% of global annual revenue. For solo/mid-market devs like Daniel, this isn't a "later" topic: if you're selling software into the EU, you need a process by late summer that reports an exploited gap within 24 hours — that's organizational lead time, not a line of code.
