Miasma: an npm worm that spreads via binding.gyp

The story of the week is not a model launch, but a supply chain incident — and one with new mechanics. A self-spreading npm worm called Miasma went on a rampage on June 3, compromising according to StepSecurity 57 npm packages in over 286 malicious versions in under two hours. It hit @vapi-ai/server-sdk (over 408,000 downloads per month) first at 23:30 UTC, then among others ai-sdk-ollama (over 120,000 per month).[1]

binding.gyp: A config file that lets npm packages build C/C++ components. Think of it like an assembly blueprint that a furniture package auto-executes when you unbox it — normally harmless. Miasma uses a 157-byte file with gyp command substitution to run code during npm install — completely bypassing the usual package.json lifecycle scripts that every scanner watches. The blind spot is the concept itself.

The payload harvests multi-cloud credentials broadly: AWS keys and IMDS tokens, GCP service accounts, Azure identities, GitHub Actions secrets (including runner storage scraping), npm tokens, Vault, and SSH credentials. With the stolen tokens, the worm then publishes further infected packages — hence "worm" and not just "theft".[1]

Lineage: Miasma derives from Mini-Shai-Hulud malware, which an actor open-sourced to the net on May 12, according to Microsoft analysis — essentially a toolkit that multiple people are now rebuilding.[2] Who could be affected: check your lockfiles, block install scripts in CI with --ignore-scripts, rotate tokens.

Red Hat confirms: 32 compromised npm packages

The first major Miasma wave hit Red Hat itself on June 1: under the @redhat-cloud-services namespace, 96 versions across 32 packages were compromised — totaling around 116,991 downloads per week (RHSB-2026-006).[3] These are frontend libraries for the Hybrid Cloud Console. The entry point was a hijacked GitHub account used to sneak malicious code past code review.

Red Hat's all-clear is instructive: no console release was published during the incident, and the publish process strips install scripts before deployment — the protective layer held. For end customers "no action needed". But anyone who pulled the packages directly from npm is not behind that same wall.[3]

Context In parallel, Microsoft reported a separate campaign: 33 npm packages using dependency confusion to profile developer environments.[4] Two independent attacks in one week — the registry layer is the hotspot right now, not the code inside it.

When the signature lies: SLSA provenance as camouflage

The most uncomfortable part of Miasma is not the theft, but the disguise. The worm exchanges stolen tokens for real Sigstore signature certificates from Fulcio and puts entries in the Rekor transparency log — the re-infected packages thus look legitimately built, complete with valid SLSA provenance.[1]

SLSA provenance: A signed receipt proving "this package was built in this pipeline from this commit". Meant as an authenticity seal — like a notary stamp. Miasma shows: whoever robs the notary can set any stamp they want. The seal proves origin, not goodness.

Practical consequence for us: "has provenance" is no free pass. The chain is only as strong as the weakest token holder in it. Depth remains more valuable — pinned hashes, install-script blocking, and distrust of packages carrying disproportionately large non-entry-point files.

Heads up Miasma also drops backdoor files into repos — .claude/setup.mjs, .cursor/rules/setup.mdc, .vscode/tasks.json. It poisons the tooling layer through which code is generated, not just the code itself. Audit these directories too.[1]

Build 2026, the non-AI half: Windows becomes a Linux-friendly dev box

The keynote headlines belonged to the models — those run in the ai-dev brief. More interesting for us is the platform half, and it's substantial this time.

Coreutils for Windows (GA)

Microsoft is making over 75 Unix-style command-line tools natively available on Windows — ls, cat, cp and friends without WSL, without Git Bash crutches. They come from the open-source uutils project, a Rust reimplementation of GNU Coreutils.[5] For cross-platform scripts, real friction reduction.

WSL containers (Preview)

Linux containers can now be created and run directly from WSL — with its own CLI/API and policy management for enterprises. It's openly aimed at reducing dependency on Docker Desktop and friends.[6] Since WSL open-sourced at Build 2025, Microsoft says over 200 pull requests a month are coming in — the project is alive.[5]

Intelligent Terminal (experimental)

A terminal that detects failed commands, shows context, and suggests fixes — via an agent protocol (ACP). Interesting but experimental; noted here as a platform callout, the agent substance is ai-dev territory.[6]

Azure Linux 4.0 & Cobalt 200: Microsoft's Linux engine room

Azure Linux 4.0 is in public preview — and it's Microsoft's first server distro declared general-purpose, no longer just a container host. It's now Fedora-based, open source on GitHub (azurelinux), with a smaller package footprint and tighter default-deny SELinux policy.[7]

On the hardware side, the Cobalt 200 VM generation entered early-access preview: Microsoft's second Arm server generation (SoC on Arm Neoverse V3), delivering according to Microsoft up to 50% more generation-over-generation performance than Cobalt 100.[8] For anyone running Arm workloads in the cloud, that's the number of the week.

  1. infoq.com
  2. azure.microsoft.com

RTX Spark Dev Box & GitHub Enterprise Local

Two hardware/infra announcements that matter for the local-vs-cloud debate. The Surface RTX Spark Dev Box is a compact dev workstation with NVIDIA RTX Spark silicon, 1 petaflop compute and 128 GB unified memory — enough to run models beyond 120 billion parameters locally, no API bill. Shipping later in 2026, initially US-only via Microsoft.com; price TBD.[6]

More infrastructurally relevant is GitHub Enterprise Local: a GitHub Enterprise flavor that runs on Azure Local and can be operated connected or air-gapped — with on-prem inference via Foundry Local. For regulated or sovereignty-driven environments (think EU agencies), a signal; price and date TBD.[9]

CISA KEV: Two-year-old WebLogic hole goes hot

CISA added CVE-2024-21182 in Oracle WebLogic Server to the Known Exploited Vulnerabilities catalog on June 1 — the hole is being actively exploited. CVSS 7.5 (HIGH), unauthenticated network access via T3/IIOP, affects 12.2.1.4.0 and 14.1.1.0.0. Oracle patched it back in July 2024; it's being exploited now against stragglers.[10][11]

KEV catalog: CISA's list of holes with confirmed active exploitation in the wild. Not "theoretically dangerous", but "being used right now". US federal agencies must close them within a deadline — here it was yesterday, Jun 4. A good prioritization signal for the rest of us, agency mandate or not.

Exposure is real: over 1,592 WebLogic servers are reachable online and vulnerable, per Security Affairs (961 on 12.2.1.4.0, 631 on 14.1.1.0.0).[12] Anyone running Fusion Middleware: check your CPU level.

More KEV adds: Linux cgroups & Android framework

CISA added two more on June 2: CVE-2022-0492 (Linux kernel, cgroups privilege escalation, CVSS 7.8 per NVD) — a local attacker can potentially break out of a container — and CVE-2025-48595 (Android framework, integer overflow, CVSS 8.4, Android 14–16). On June 3 came CVE-2026-45247 (Mirasvit Full Page Cache Warmer, unsafe deserialization).[13][14]

The cgroups hole is the relevant one for us: if you're running containers on older kernels, align your patch level — container escape is no edge risk.

EU/DE: The CRA apparatus ramps up

Standing thread on regulation: on June 11 the rules for naming conformity assessment bodies under the Cyber Resilience Act take effect — in Germany the BSI as the notifying body publishes the procedures for that.[15] Sounds bureaucratic, but it's the track on which CRA mandates become concrete for manufacturers of networked products starting in 2027.

The sharper deadline for developers comes in the fall: starting September 11 the duty to report actively exploited vulnerabilities applies — within 24 hours.[15] Penalties for breach: up to EUR 15 million or 2.5% of global annual revenue; the BSI is hiring 141 positions by 2029 to staff it.[16]

  2. heise.de

Releases in review: Node 24 & PostgreSQL 18.4

Quiet on the language/DB front, but two May releases are noteworthy. Node.js 24 (May 21) brings the TC39 proposal Explicit Resource Management natively via V8 13.6 — the using keyword for deterministic resource cleanup, similar to try-with-resources in Java.[17]

PostgreSQL 18.4 (May 14) is a pure security/maintenance update of the 18 line — 11 security fixes and over 60 bugfixes, no new features.[18] The real highlights came with PostgreSQL 18.0 (Sep 2025): the async I/O subsystem (up to 3× on I/O-heavy workloads), virtual generated columns, and native uuidv7() — the latter saves many the custom generator for time-sortable IDs. If you're on 18, upgrade to 18.4 for the security fixes.

  2. postgresql.org

It's Friday, June fifth. The software week centers on supply chain and a Microsoft Build conference whose most interesting parts have nothing to do with artificial intelligence.

At the center is an npm worm called Miasma. In a single wave on June third, according to StepSecurity, it compromised fifty-seven packages in under two hours. The novelty: it spreads via a build file called binding.gyp and executes its code during installation, completely circumventing the places security scanners normally watch. It steals cloud credentials from Amazon, Google, Azure, and GitHub and then uses those to publish the next batch of infected packages.

The first major wave hit Red Hat itself on June first: ninety-six versions across thirty-two packages, totaling around one hundred seventeen thousand downloads a week. Red Hat's protective layer did hold, but the incident shows how exposed the registry layer is right now.

Most uncomfortable is the disguise. The worm forges real signatures using Sigstore with stolen tokens and gives its packages valid SLSA provenance. The lesson for us: a signature proves where a package comes from, not that it's safe.

At Microsoft Build, the platform half was concrete. Windows gets over seventy-five Unix commands natively, written in Rust. WSL can now run Linux containers directly without Docker Desktop. And Azure Linux four point zero is Microsoft's first true general-purpose server distribution, now on a Fedora base.

On security, CISA put a two-year-old Oracle WebLogic hole on the list of actively exploited vulnerabilities, with a severity of seven point five. Over fifteen hundred servers are exposed online. The deadline for agencies was yesterday, June fourth — stragglers should patch now.

And in Europe the Cyber Resilience Act keeps moving forward: starting June eleventh, Germany's BSI names the assessment bodies. The sharper deadline for developers comes in September, when the twenty-four-hour reporting requirement for exploited holes kicks in.
