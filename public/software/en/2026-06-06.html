Miasma: the autopsy—signature was camouflage, not protection [1][2][3]

The Red Hat npm incident from early this week is now fully analyzed, and the numbers are solid: 32 packages under the @redhat-cloud-services namespace were compromised, totaling around 117,000 downloads/week[3] (Wiz initially reported ~80,000), manipulated in two waves on 06/01.[1] Wiz, Microsoft, and Palo Alto's Unit 42 have independently reviewed it—and reach the same uncomfortable conclusion.

The root cause is clear: a compromised Red Hat employee GitHub account. It was used to push "orphan commits" (commits that don't hang from any branch and thus bypass code review) into two RedHatInsights repos. These then compromised the normal release mechanism: a GitHub Actions workflow requested an OIDC token and published the trojan package—with valid SLSA provenance.[1][2]

SLSA provenance / OIDC token: SLSA provenance is like a certified bill of lading that says "this package was built from exactly this commit, by exactly this pipeline". An OIDC token is the key the pipeline briefly presents to prove "it's really me"—like a day pass. If the attacker has compromised the pipeline, they get a real day pass and a real bill of lading. The certification then only proves "really from this pipeline"—not "really harmless".

That's exactly the lesson for us: provenance and attestation answer the question "where did this come from?", not "is this clean?" Anyone who reduces supply-chain security to "has a signature, so it's good" has answered the wrong question. The malware stole npm and GitHub tokens, SSH keys, cloud credentials (new: GCP and Azure identities), and CI/CD secrets—and republished further with them, so it spread like a worm.[1]

Unit 42: the "nuisance era" of npm is over [4]

Aptly timed with the Miasma incident, Unit 42 updated its npm threat landscape dossier on 06/02—and it reads like a taxonomy of what we're witnessing. Key takeaway: since the original Shai-Hulud worm, npm attacks have evolved from isolated typosquatting to systematic campaigns—with worm-like spread, infrastructure persistence, and multi-stage payloads.[4]

The practical takeaway for Daniel's own stacks (npm in every frontend, GitHub Actions as CI): priority is not "patch faster," but token hygiene. Keep personal access tokens and npm tokens short-lived, use OIDC instead of long-lived secrets in Actions, and regularly audit repos for unknown workflows, new tokens, and third-party actions. Attackers are targeting precisely the trust layer that makes our pipelines fast.

GitHub: two days of Dependabot lag, then an auth outage [5]

Not dramatic, but felt in daily workflows: between 06/01 (23:00 UTC) and 06/04 (04:11 UTC), scheduled Dependabot updates were delayed—pull requests arrived up to two days late, affecting approximately 1.5 million repos with active version updates according to GitHub. The cause was a change to an internal routing service.[5]

Right on its heels, on 06/05 (14:49–16:45 UTC), auth errors for legitimate requests—a newly enabled feature flag that was then disabled; Slack and Teams channel subscriptions also broke. No data leak, but two reminders in one package: security tooling that fires late is a silent risk factor (Miasma week!), and feature flags deserve the same rollout rigor as code.

Let's Encrypt goes post-quantum—via Merkle trees [6][7]

Arguably the most substantial web-standards news of the week: Let's Encrypt announced on 06/03 its path to post-quantum-safe authentication—and solves elegantly a problem most underestimate.

Post-quantum + Merkle-tree certificate (MTC): Post-quantum cryptography covers algorithms that a future quantum computer still can't break—but their signatures are fat. A Merkle tree is a family tree of hash values: you don't sign each leaf individually, but the root of the whole tree once, and for each leaf a short path upward serves as proof. Instead of "one fat signature per certificate," it's "one signature for a whole batch."

Today's pain point: a typical web PKI handshake carries five signatures and two public keys. If you naively swap them out for ML-DSA (the post-quantum standard), a single TLS handshake balloons to "well over 10 kilobytes"—and that's what breaks connections.[6] MTC packs the authentication load smaller than today. Timeline: staging environment by end of 2026, production-ready in 2027. The IETF working group PLANTS is standardizing, Cloudflare and Chrome are already running field trials, and Chrome has named MTC the preferred post-quantum path.[6][7]

Why it matters even though 2027 sounds distant: Let's Encrypt issues the majority of all public TLS certificates (industry reports cite ~54% in Q1 2026). What they choose as standard becomes the de-facto standard for encrypted web traffic. For us, that means: ACME, Caddy and others will need to catch up over the next one to two years—no action required today, but a date to keep in mind.

FROST: the browser reads your SSD [8][9]

Researchers at TU Graz (with Liebherr Transportation) demonstrated a side-channel attack called FROST that requires no click and no permission—entirely from the browser sandbox.[8]

Side-channel via OPFS: The "Origin Private File System" is browser storage where a website can place files without asking. FROST deliberately reads and writes there—and measures how long it takes. If another app simultaneously accesses the same SSD, tiny delays occur. From the pattern of these delays, the attacker infers which other apps or websites are open—like listening through a wall, but via SSD load.

Hit rate: ~89% F1 in the closed-world test, ~87% in the open scenario.[8] OPFS is present in Chrome, Firefox, and Safari, so it's relevant cross-engine. This is research, not an active exploit—but a clean example of how each new web platform API is also a new measurement surface. Browser vendors will need to tighten here (reduce timing precision, decouple OPFS I/O).

CVE cluster: actively exploited flaws this week [11][12][13][14]

Cisco Unified CM (CVE-2026-20230): An SSRF hole in the WebDialer service, rated critical by Cisco despite CVSS 8.6, because it leads to unauthenticated root write access and privilege escalation. Published 06/03 (advisory cisco-sa-cucm-ssrf-cXPnHcW), PoC code is already public. Some reassurance: WebDialer is off by default—but anyone using CTI or click-to-call has it enabled. Fix in 14SU6; version 15 not until September (15SU5), interim COP patches available.[11]

SSRF: "Server-side request forgery"—you trick the server into making a request on your behalf that you couldn't make yourself. Like convincing the security guard to unlock an interior door you're not authorized to access.

CISA KEV—newly added: On 06/02 two entries arrived: the Android framework zero-day CVE-2025-48595 (integer overflow, local privilege escalation; Google: "may be limited, targeted exploitation," typical spyware pattern; fixed in June Android update, federal agency deadline was 06/05) and the old Linux cgroups bug CVE-2022-0492—four years old, now actively abused, classic container-escape risk.[12][13] On 06/03 came CVE-2026-45247 (Mirasvit Full Page Cache deserialization)—Magento/Adobe Commerce ecosystem.[14]

Not a "patch tonight" alarm—but the mix is instructive: a fresh zero-day, a four-year zombie, and an e-commerce plugin hole. For Daniel's world, it mainly underscores: container hosts with old kernels are not a "done" problem, and KEV lists deserve a weekly glance.

It's Saturday, June sixth. The news day itself is quiet, but this week's biggest stories have substance now—especially the Red Hat npm incident, which three security teams have now thoroughly dissected.

The worm called Miasma infected thirty-two packages under the Red Hat Cloud Services namespace, totaling about eighty thousand downloads per week. Wiz, Microsoft, and Palo Alto's Unit forty-two reach the same uncomfortable conclusion: The malicious packages carried genuine supply-chain certification, so-called SLSA provenance. The attacker compromised an employee account, bypassed code review, and then used GitHub Actions' normal publishing mechanism to obtain a valid token. The lesson: a signature proves where something came from—not whether it's clean.

Unit forty-two updated its npm threat picture right on cue. The message: the harmless phase of npm attacks is over. Today it's worm-like, self-propagating campaigns that specifically steal tokens and CI secrets. For Daniel's own projects, that means above all: keep tokens short-lived and regularly audit repositories for unfamiliar workflows.

The best news of the week comes from Let's Encrypt. They announced a path to quantum-safe certificates that elegantly solves the biggest problem. Quantum-safe signatures are huge—a single handshake would otherwise balloon to over ten kilobytes. With so-called Merkle-tree certificates, you no longer sign each certificate individually, but instead batch-sign many at once. The test environment arrives by year-end, production in twenty twenty-seven. Since Let's Encrypt issues the majority of all web certificates, this will become the de-facto standard.

On the vulnerability front, there's fresh action. Cisco disclosed a critically-rated flaw in its telephony software for which a proof-of-concept is already circulating. CISA added several actively exploited flaws to the list—including an Android zero-day and, oddly, a four-year-old Linux bug that allows container escapes.

From Graz comes an elegant attack idea called FROST: a website can guess which other apps are currently open just from solid-state drive access times—no permission required, right from the browser. This is still research, but it shows that every new browser interface is also a new measurement surface.

GitHub suffered a double setback: two days of Dependabot update delays across one and a half million repositories, then a brief authentication outage. Nothing dramatic, but during a supply-chain week, an unwelcome reminder that delayed security tooling is itself a risk.

On Thursday, June eleventh, the EU's machinery ramps up on the Cyber Resilience Act: the BSI begins designating test facilities. Real manufacturer pressure comes later in September. Next week we'll watch the Microsoft patch day and the fallout from npm week. In the meantime: rotate those tokens and enjoy your Saturday.
