Miasma & IronWorm: the worm now crawls through coding tools

This is the story of the week, and it deserves the headline. Miasma is a self-replicating npm worm that has attacked the supply chain in multiple waves. Wave 1 (May 29) hit 32 packages from the Red Hat scope @redhat-cloud-services — 96 poisoned package versions in a 72-second window, nearly 10 million cumulative downloads affected, at least 210 repositories with stolen credentials.[1]

The real bombshell is Wave 3 (June 3–5): Miasma jumped into 73 Microsoft GitHub repositories (Azure, Azure-Samples, Microsoft, MicrosoftDocs) — GitHub disabled them in two automated waves over 105 seconds.[2] What's remarkable is the distribution vector: via stolen contributor credentials, manipulated .claude/config, AGENTS.md, and .cursor/rules files were injected into repos like azure/durabletask. The payload fires the moment a developer opens the repo in Claude Code, Gemini CLI, Cursor, or VS Code, then harvests their credentials.[3]

Supply-chain worm: Imagine a bakery's flour supplier gets poisoned flour — and that poisoned flour bakes itself into every loaf, ensuring the next bakery gets poisoned flour too. An npm package that manipulates another package on install and spreads itself does exactly that.

Why this belongs here and not in the AI brief: the angle isn't a new model, but that the agent/tooling layer has become the new attack surface and persistence vector. Config files for AI tools are practically never reviewed in CI/CD and code review the way code is. This is classic supply-chain negligence with a fresh coat of paint.

In parallel ran IronWorm: an npm worm written in Rust that distributed a 976 KB eBPF rootkit plus Tor C2 across around 36–37 packages from a compromised account — with no CVE.[4] Two separate, sophisticated worms in the same week: this is the shift away from single-package typosquatting toward industrial, self-spreading compromise.

Heads up Not a "patch tonight" alert, but a concrete action item: audit lockfiles for the affected version windows, and treat AI-tool config files (.claude/, .cursor/, AGENTS.md) with the same code-review rigor as executable code — they're now part of the attack surface.

Oracle PeopleSoft CVE-2026-35273: 9.8, actively exploited, KEV deadline yesterday

A no-auth remote code execution flaw in Oracle PeopleSoft Enterprise PeopleTools 8.61 and 8.62, CVSS 9.8 — no login, no user interaction required.[5] ShinyHunters and the group tracked by Mandiant as UNC6240 actively exploited it between May 27 and June 9.[6] ShinyHunters claims data from over 100 organizations, 68% universities, stolen — publishing leaks starting June 9 on their site.[7]

CISA KEV: The US cyber agency's "Known Exploited Vulnerabilities" catalog is essentially the wanted list of flaws that are actually being attacked right now. When one's listed, US agencies must patch by a deadline — and for everyone else, it's the strongest signal "this is real, not theoretical."

CISA added it to the KEV catalog on June 12, with a patch deadline of June 15.[8] That deadline has already passed — if PeopleSoft is running in-house, patch status needs checking today, not planning.

Patch Tuesday June: 206 CVEs yearly record, plus Ivanti Sentry at 10.0

Microsoft patched 206 CVEs in June (per CrowdStrike; 198–208 depending on count), the highest count of 2026, with 37 critical.[9] The sharpest entries:

CVE-2026-45657 (Windows kernel RCE, CVSS 9.8): remote, unauthenticated, code execution at SYSTEM level — and flagged as wormable.[9] This is the one to roll out first.

CVE-2026-47291 (HTTP.sys RCE, 9.8) and CVE-2026-44815 (DHCP client, 9.8, affects all Windows versions) are the next two 9.8s.[9] Plus three publicly pre-disclosed zero-days (Windows CTF EoP, BitLocker bypass, HTTP.sys DoS).

"Wormable": A flaw that malware can use to copy itself from an infected machine on its own to the next one — without anyone clicking anything. That's the property that turns a single bug into an epidemic (see WannaCry 2017).

Outside Microsoft, the most urgent: Ivanti Sentry CVE-2026-10520, CVSS 10.0 — unauthenticated OS command injection leading to root RCE.[10] watchTowr released a public PoC on June 10, active exploitation began within days.[11] Affected are 10.5.1, 10.6.1, 10.7.0 and earlier; patched in 10.5.2, 10.6.2, 10.7.1. CISA KEV deadline was June 14.

PostgreSQL 19 Beta 1 — and a serious May security release

Finally some good news in the stack: PostgreSQL 19 Beta 1 came out June 4, with over 200 changes from version 18.[12] The highlights that matter for database work:

  • SQL/PGQ graph queries — graph pattern matching natively in SQL, following the new ISO standard.
  • Parallel autovacuum — maintenance cleanup finally runs across multiple workers.
  • REPACK CONCURRENTLY — table bloat reduction without exclusive locks.
  • WAIT FOR LSN — clean waiting for replication progress.
  • ~2× faster foreign-key inserts, GROUP BY ALL, FOR PORTION OF (UPDATE/DELETE).

GA is expected around September/October 2026 — beta time is test time, not production.[12] This week is a good window to run a staging migration.

Less cheerful but more pressing: the May 14 security release (18.4, 17.10, …) fixed 11 CVEs, including four with CVSS 8.8 — among them CVE-2026-6473 (integer wraparound → undersized allocation), CVE-2026-6475 (pg_basebackup symlink overwrite), and CVE-2026-6477 (libpq lo_* stack overwrite).[13] If you're on an older minor: this is the mandatory upgrade, not beta 19.

EU CRA: notification level in effect, reporting mandate from September

The EU Cyber Resilience Act reached a milestone on June 11: rules for notifying conformity assessment bodies are now in effect — member states must designate their notifying authorities.[23]

CRA conformity assessment body: A state-recognized testing facility — like TÜV, but for cybersecurity. It certifies that a product with "digital elements" meets CRA requirements. Before testing can happen, there has to be a rule on who gets to be a testing facility — that's what just went live.

The next — and more tangible for developers — milestone is September 11, 2026: manufacturers' reporting obligations kick in (24-hour early warning, 72-hour full report for actively exploited flaws).[24] Full enforcement December 11, 2027, fines up to €15 million or 2.5% of global revenue. Still months away, but it's the date that changes actual behavior — anyone bringing OSS or products into the EU should have September marked on the calendar.

It's Tuesday, June sixteenth, and it was one of the toughest supply-chain weeks of the year in software. Let's start with the story that's really new.

An npm worm called Miasma spread through the supply chain in multiple waves. First it hit thirty-two packages from the Red Hat ecosystem — ninety-six poisoned versions in seventy-two seconds, nearly ten million downloads affected. It gets interesting with the third wave: the worm jumped into seventy-three Microsoft repositories on GitHub, and not through traditional code, but through config files for AI coding tools like Claude Code, Cursor, and the Gemini command line. The payload fires the moment a developer opens such a repo in their tool. In parallel ran a second, Rust-based worm called IronWorm with a rootkit in tow. The lesson: the tooling layer is the new attack surface.

Then the vulnerabilities. Oracle PeopleSoft has a critical flaw rated nine point eight — no login needed, remote code execution. The group ShinyHunters actively exploited it and stole data from over one hundred organizations, mostly universities. The US Cyber Agency set a patch deadline of June fifteenth — that's already passed.

Patch Tuesday in June was a record: two hundred six flaws from Microsoft (per CrowdStrike; ninety-eight to two hundred eight depending on how you count), thirty-seven of them critical, including a wormable Windows kernel bug rated nine point eight. Even more urgent because already under attack: a flaw in Ivanti Sentry with the maximum rating of ten point zero, which came under assault within days of a public proof of concept.

On the flip side, some good news: PostgreSQL nineteen is here as a first beta. It includes graph queries directly in SQL, parallel autovacuum, table deflation without locks, and roughly twice as fast foreign key inserts. The final version is expected in September or October — until then, beta time is testing time.

In the surrounding stack: Spring Framework six point two loses open-source support on June thirtieth, the current line is seven point zero point eight. Angular twenty-two brings stable signal forms, and Python three point fifteen is in its second beta, with lazy imports as standard and a faster JIT.

From Europe: the Cyber Resilience Act reached a milestone, but the really important stage with reporting obligations doesn't come until September. And the BSI is warning about an actively exploited flaw in Check Point's VPN solution.

Tomorrow the focus will probably be on patch follow-up and whether new KEV additions come in. Until then.
