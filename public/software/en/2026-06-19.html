Defender as entry point: RoguePlanet (CVE-2026-50656) [1]

Microsoft has acknowledged a critical zero-day in its own protection software: a race condition (TOCTOU) in Defender's malware protection module allows an unprivileged local attacker to spawn a cmd.exe with SYSTEM privileges — exactly the level the protection is supposed to prevent. Disclosed by the researcher collective "Nightmare Eclipse / Chaotic Eclipse" on 06/10; Microsoft published the MSRC entry on 06/16 and confirmed a patch is "in development." Notable: the vulnerability affects fully patched Win10/Win11 systems including June KB5094126.

TOCTOU (Time-Of-Check / Time-Of-Use): Imagine a bouncer checks your ID (check) and waves you through (use). If you swap your ID in the half-second between those steps, the wrong person gets in. That's exactly what happens here — Defender checks a file, and someone swaps it between the check and execution.

Assessment: a privilege-escalation flaw in the protection itself is the worst possible kind — Defender is the last active defense line on millions of machines. CVSS 7.8 ("Important") sounds moderate on paper, but the plus-factors are harsher: local code suffices (RDP world, phishing drops), and the jump to SYSTEM is atomic. Until the patch arrives, realistically all you can do is watch Defender service account locks and sharpen EDR telemetry for unusual cmd.exe births under SYSTEM — the discipline you should already have.

Double NGINX: HTTP/3 and HTTP/2, both CVSS 9.2 [2][3]

F5 published a security advisory on 06/17 closing two critical vulnerabilities in NGINX products. CVE-2026-42530 is a use-after-free in the HTTP/3 path: an unauthenticated attacker can reopen the QPACK encoder stream and trigger a worker crash (DoS) — and on systems without ASLR, theoretically RCE. Affected: NGINX Open Source 1.31.0–1.31.1, Gateway Fabric 1.3.0–2.6.3, Ingress Controller 3.x/4.x/5.x, Instance Manager 2.17.0–2.22.0. Fix in NGINX Open Source 1.31.2 and Gateway Fabric 2.6.4.

CVE-2026-42055 hits the HTTP/2 track: a heap buffer overflow in proxy_http_version 2 and gRPC setups once ignore_invalid_headers off is set and large_client_header_buffers > 2 MB. Hits NGINX Plus R33–R36, R37.0.0–37.0.1, Open Source 1.30.0–1.31.1, F5 WAF, App Protect WAF, DoS, Gateway Fabric, Ingress Controller. Fix in NGINX Open Source 1.30.3 and 1.31.2, NGINX Plus R37.0.2.1 and R36 P6.

Heads up Patches for NGINX Ingress Controller and Instance Manager are not yet shipped per the F5 advisory — that hits exactly the Kubernetes world rolling out over the weekend and through KubeCon aftermath. Anyone running Ingress NGINX in production clusters should at minimum disable HTTP/3 activation and reset HTTP/2 header buffers to defaults.

Context: NGINX already had an actively exploited CVSS 9.2 vulnerability in May ("NGINX Rift", CVE-2026-42945) in the rewrite module. This double is not actively exploited (as of 06/17), but two CVSS 9.2 hits in five weeks are no accident by NGINX standards — the HTTP/2 and HTTP/3 parsers are the new attack surface.

Oracle PeopleSoft on KEV: CVE-2026-35273 [4]

Assessment: This is a classic Oracle problem — an enterprise component that's been running in government for decades, with a developer-convenient update endpoint lacking authentication. For German universities the story was already critical on 06/11; now KEV inclusion forces every U.S. federal shop to patch. Waymo recalls 3,900 robotaxis — highway construction [8]

NHTSA case 26E035000: Waymo voluntarily recalls nearly 4,000 vehicles of the fifth generation (built 05/17/2022 through 05/19/2026). Reason: The autonomous software "failed to recognize and drove past entrance closure signs into pre-planned highway construction zones" because it prioritized other roadway hazards higher. 13 confirmed incidents in Phoenix (AZ) and the San Francisco Bay Area; Waymo suspended highway operation on 05/19, filed the recall on 06/08, fix is "in development." The fleet continues operating on city streets.

Software-engineering lesson, not cars story: This is one of the cleanest documented cases of an ML-policy collision in a safety system. The model had learned to maximize avoidance of roadway hazards on the highway — and that very prioritization then exceeded the read-signal "ramp closed." Anyone building similar multi-reward stacks in your own systems (routing, recommendations, risk scoring) should read this report: the failure isn't "the model missed the sign," it's "the model saw the sign and deliberately ignored it."

Docker Model Runner: SSRF in registry layer [10]

A small but conceptually interesting CVE — and an example of "AI as software-engineering side effect" that belongs in the software beat: CVE-2026-33990 in Docker Model Runner is a server-side request forgery in the OCI registry token swap. A malicious OCI registry sets the realm field to an internal URL (e.g. 127.0.0.1) — and Docker Model Runner happily sends GET requests to internal services. Fixed in Docker Desktop 4.67.0 (03/30) and Model Runner 1.1.25; Enhanced Container Isolation blocks the class additionally.

SSRF (Server-Side Request Forgery): Your server is supposed to fetch a page for you. If your server trusts the malicious page, it fetches internal pages instead — cloud metadata, admin endpoints — and returns them to the attacker, who couldn't have reached them from the outside.

Dossier note: No official CVSS for CVE-2026-33990 was found during research (NVD entry not fetched). The Docker advisory describes the hole only qualitatively as SSRF — but the patch data and fix path are confirmed.

PostgreSQL 19 Beta 1 — property graphs official [13]

PostgreSQL 19 Beta 1 shipped on 06/04. The headline feature is SQL/PGQ: ISO SQL property graph queries over existing relational tables — no separate graph engine needed. Also included: parallel autovacuum (autovacuum_max_parallel_workers), REPACK CONCURRENTLY for non-blocking table repack, ON CONFLICT DO SELECT RETURNING, async I/O autoscaling (io_min_workers/io_max_workers), GROUP BY ALL, temporal UPDATE/DELETE FOR PORTION OF, and a pg_stat_lock view. Plus MD5 auth deprecation warnings and SNI-based TLS certificate selection. GA expected Sep/Oct.

Property-Graph (SQL/PGQ): Instead of standing up a second database (Neo4j, Memgraph) for "who's connected to whom" questions, you can now declare your regular Postgres tables as nodes + edges and send graph queries against them. A typical example: "find all customers linked to customer X within two orders."

Assessment: three features stand out. SQL/PGQ makes graph analytics first-class without polyglot persistence; REPACK CONCURRENTLY kills one of the most annoying maintenance chores in production Postgres clusters (reordering large tables without a lock); and parallel autovacuum smooths I/O spikes every Postgres admin with large write tables knows. Beta testing is the task for the next 2–3 weeks.
