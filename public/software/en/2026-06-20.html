Arch User Repository: "Atomic Arch" poisons 1,500+ packages

This week's biggest software story is a supply-chain attack on the Arch User Repository (AUR). Over the weekend, the tally of compromised packages climbed from an initial 400+ to over 1,500, and Arch Linux locked new account registrations on 06-15 while cleaning house.[1]

AUR: a community shelf for Arch Linux. Picture a massive self-service rack next to the official, vetted supermarket — anyone can drop their own software build recipes there, over 107,000 of them. Fast and handy, but no one systematically checks what each person puts in. That's exactly what the attacker exploited.

The trick was devilishly simple: the attackers deliberately adopted orphaned AUR packages (unclaimed packages can be taken over) and slipped in a malicious npm dependency called atomic-lockfile. Anyone building an affected package would install a Rust-written infostealer plus eBPF rootkit — malicious code that harvests credentials and hides deep in the kernel layer instead of running as a mere process.[2]

Heads up: The core Arch distribution (official repos, Pacman) is unaffected — only the community-maintained AUR. If you built AUR packages in the past ten days (especially formerly orphaned ones), review the PKGBUILDs and search for atomic-lockfile. First confirmed on 06-12, a second, more sophisticated wave hit on 06-14.

The bigger lesson: the AUR has always been "use at your own risk" — but this campaign shows how cheap adoption takeover is as a persistence vector. The same mechanism applies to any ecosystem with claimable orphaned packages.

F5 patches two critical NGINX RCE flaws — out-of-band

On 06-17, F5 issued an out-of-cycle alert for two critical NGINX vulnerabilities, both rated CVSS v4.0 = 9.2. This is a genuine "don't leave your reverse proxy unpatched" case, because NGINX sits in front of everything at many shops.[3]

  • CVE-2026-42530 — Use-after-free in the HTTP/3 module (ngx_http_v3_module). Affects NGINX Open Source 1.31.0–1.31.1. Only relevant if HTTP/3/QUIC is enabled.
  • CVE-2026-42055 — Heap buffer overflow in HTTP/2 and gRPC proxy paths (ngx_http_proxy_v2_module / ngx_http_grpc_module). Affects the broad range 1.13.10–1.31.1.
Use-after-free / Heap overflow: both are memory errors. In use-after-free, the program accesses memory it just freed — like continuing to use an address where someone else has moved in. In heap overflow, the program writes past its reserved space into foreign memory. Both can often be leveraged into code execution.
Heads up: Fixed in NGINX Open Source 1.31.2 and 1.30.3 plus NGINX Plus 37.0.2.1 and R36 P6; Gateway Fabric and Ingress Controller also affected. Note on scoring: CVE-2026-42055 rates only 7.3 under the older CVSS v3.1 — the v4.0 score of 9.2 is higher because v4.0 weights the attack context more heavily.

Oracle June CPU: 243 CVEs, nearly half critical

Oracle's quarterly Critical Patch Update from 06-16 is another hefty batch: 243 individual CVEs in 245 patches across 11 product families, with roughly 49.8% rated critical.[4] Java SE is in the mix — if you live on the JVM, move to the current builds (JDK 21.0.11, 26.0.1, 11.0.31); older versions are end-of-life starting with the next CPU on 07-21. A specific top-CVSS value for Java SE alone wasn't confirmed from the primary source — Oracle's Security Alerts page was unreachable (403) at fetch time, so only the aggregate figures here.

Node.js: quarterly security release with 12 CVEs

Node.js shipped its coordinated security batch across all active lines (22.x, 24.x, 26.x) on 06-18: 12 CVEs (2 HIGH, 6 MEDIUM, 4 LOW). Fixed versions: 22.23.0, 24.17.0, 26.3.1.[5] The two HIGH flaws warrant a closer look:

  • CVE-2026-48618 (HIGH) — TLS hostname bypass: a Unicode dot separator causes a normalization gap that bypasses wildcard certificate validation. In other words, a cert for *.example.com will be accepted for a host it should not cover.
  • CVE-2026-48933 (HIGH) — WebCrypto integer overflow: calling subtle.encrypt() with input that is an exact multiple of 2 GiB crashes the process.

From the MEDIUM pile, two that sting in practice: an HTTP/2 DoS via unbounded memory growth from malformed ORIGIN frames (CVE-2026-48619) and a leak of proxy credentials in ERR_PROXY_TUNNEL error messages (CVE-2026-48615). Updated bundles also shipped: llhttp 9.4.2, nghttp2 1.69.0, OpenSSL 3.5.7.

Spring: statemachine RCE and a CVE-heavy June

The Spring ecosystem had a June to forget: 67 CVEs total, per HeroDevs.[6] The headliner for Java folks is CVE-2026-41862 in Spring Statemachine (HIGH): persisted state-machine contexts are deserialized via Kryo without enforcing a class allowlist — the classic deserialization door to remote code execution, here in the JPA/MongoDB/Redis/ZooKeeper backends. Fixed in 4.0.2 and later.

Unsafe deserialization: reassembling an object from bytes without checking what classes are allowed to be created. Like accepting and unwrapping every package with your name on it — including the one with a bomb inside. An allowlist says in advance: only these senders, only these contents.

Also relevant: CVE-2026-47825 (Spring Cloud Gateway, HIGH) forwards X-Forwarded-For / Forwarded headers from untrusted proxies — IP spoofing risk in both WebMVC and WebFlux. Plus three Spring GraphQL flaws (unsafe deserialization, cross-site WebSocket hijacking, authorization bypass on type hierarchies) and path traversal in Spring Integration (CVE-2026-40987). If you're still on an older Spring Boot line: the April-reported CVE-2026-40976 (security filter chain bypass, CVSS 9.1) remains the biggest issue — fixed in 4.0.6 / 3.5.14.

CRA: next compliance deadline in September

The EU Cyber Resilience Act hit a milestone on 06-11: member states must now have named their notifying authorities, and conformity assessment procedures are active.[7] More pressing for day-to-day operations is the next deadline, 09-11-2026: from then on, manufacturers must report actively exploited vulnerabilities and severe incidents to ENISA and national CSIRTs. Full compliance deadline remains 12-11-2027. For anyone shipping products with digital components into the EU, September is the cutoff for having a reporting process in place.

TypeScript 7.0 RC: the Go compiler is here

The good news of the week: TypeScript 7.0 entered Release Candidate on 06-18. This is the version with the native Go compiler — and the speed jump is real: TypeScript builds often run roughly ten times faster.[8] Microsoft's own example: the VS Code codebase (1.5 million lines) compiles in 8.74 seconds instead of 89 seconds.[9] Stable GA is expected "within the next month" — roughly July.

Breaking changes in RC When upgrading, watch out: target: es5 and AMD/UMD/SystemJS modules are now hard errors. baseUrl is gone (use paths instead), rootDir defaults to ./, and types defaults to []. Try it with npm install -D typescript@rc.

Beyond speed, the RC brings semantic highlighting, import sorting, and automatic unused-import removal over the Beta — and the language-server command failure rate has dropped over twenty-fold versus TS 6.0. TS 6.0 (03-23) was the last JavaScript-based compiler; 7.0 is the watershed.

PostgreSQL 19 Beta 1: SQL graphs and parallel autovacuum

On 06-04, PostgreSQL 19 Beta 1 arrived — over 200 changes versus PG 18, with GA expected September/October.[9] The highlights that matter for Daniel's stack:

  • SQL/PGQ — property graph queries directly in SQL, no separate graph database.
  • ON CONFLICT DO SELECT and a new REPACK command.
  • Parallel autovacuum and up to 2× faster foreign key checks.
  • MultiXactOffset widened from 32 to 64 bits — goodbye to an old wraparound cliff.

Security-wise, there's SNI support via pg_hosts.conf, password expiration warnings, and MD5 deprecation notices. Watch upgrade behavior: JIT is now off by default in PG 19, default_toast_compression is now lz4, and RADIUS authentication has been removed. Beta means: for testing, not production.

It's Saturday, June twentieth. The week was nothing but patch waves, and don't worry — you can cruise through most of it over the weekend, nothing's on fire right now.

The biggest story is playing out in the Arch User Repository, the community shelf for Arch Linux. A coordinated campaign poisoned over fifteen hundred packages with malware — a data stealer and a deeply hidden rootkit. The attackers deliberately took over orphaned packages and slipped in a malicious npm dependency called atomic-lockfile. Arch locked new account signups on June fifteenth to clean house. Important: the official Arch distribution itself is untouched — only the community-maintained AUR.

On the infrastructure side, F5 patched two critical NGINX vulnerabilities out-of-cycle, both with a severity of nine point two. One sits in the HTTP-three module, the other in the HTTP-two and gRPC paths. If you're running NGINX as a reverse proxy, you should upgrade to versions one thirty-one point two or one thirty point three.

Node.js released its quarterly security batch: twelve CVEs, two of them rated high. One lets you bypass wildcard certificate validation using a Unicode trick, the other crashes WebCrypto. The fixes are in versions twenty-two point twenty-three, twenty-four point seventeen, and twenty-six point three-one.

And Oracle dropped its big quarterly update: two hundred forty-three vulnerabilities across eleven product families, nearly half of them critical. Java SE is in there — if you live on the JVM, move to the latest builds.

On the bright side: TypeScript seven point zero is in release candidate. This is the version with the native Go compiler, and it's roughly ten times faster. Microsoft's own VS Code codebase compiles in under nine seconds instead of eighty-nine. The finished version is expected around July. Watch out when upgrading: older module formats and a few compiler options are gone.

PostgreSQL nineteen also came out as a first beta — with over two hundred changes, graph queries right in SQL, and parallel autovacuum. And the EU Cyber Resilience Act has its next deadline coming: starting September eleventh, manufacturers must report actively exploited vulnerabilities to regulators.

Next week we'll see if TypeScript seven makes the leap to final, and whether the Arch story gets a second wave. Until then: check your patches, and enjoy your weekend.
