PTC Windchill: a perfect 10.0, and the BSI picked up the phone [1]

This week's headline story doesn't directly affect Daniel — Windchill is PLM software (Product Lifecycle Management: the system where industrial firms manage CAD models, bills of materials, and product data) — but it shows how a critical vulnerability escalates today. CVE-2026-12569 is an unauthenticated remote code execution via unsafe deserialization, affecting Windchill PDMLink and FlexPLM, and carries a perfect 10.0 under CVSS 3.1 (still 9.3 under CVSS 4.0).

Unsafe deserialization: A program receives data from outside and "unpacks" it back into live objects — like an IKEA box that assembles itself. If no one checks what's in the box, it also assembles what the attacker put in: executable code here. No login needed, directly from the network — so easily automated.

That's exactly what's happening now: attackers are actively installing backdoors and web shells. The BSI flags the HTTP header X-windchill-req: ?x8Fmgow as an indicator. Notably, the response: the BSI called affected admins overnight — reports cite calls around 2:30 a.m. on 06/17 — to push on patch status. A government agency doesn't make calls like that over a 7-score vulnerability.

Status Patched as of 06/15 (Windchill 13.1.2.8, 13.1.3.4, 13.0.2.12, 12.1.2.27). Versions ≤ 11.0 M030 get no fix and must be isolated from the network.

NGINX: out-of-band patch for HTTP/3 and gRPC [2]

This is the story that touches Daniel. F5 patched NGINX out of cycle on 06/17 — two vulnerabilities in the newer mainline versions:

CVE-2026-42530: a use-after-free in HTTP/3/QUIC handling (ngx_http_v3_module), CVSS v3.1 = 8.1, even 9.2 under v4.0. CVE-2026-42055: a heap buffer overflow in HTTP/2 and gRPC proxying, CVSS v3.1 7.3.

Use-after-free: The program frees a memory region ("I don't need this anymore") and then accesses it anyway — like ending your lease, returning the key, and walking back in. Whoever moved in next controls what you find there. In a web server, this can lead to code execution.

Only the young mainline releases 1.31.0–1.31.1 are affected (for 42055 the broader range from 1.13.10 too); the fix ships in 1.31.2 and 1.30.3. If you run the stable branch, you're clear on 42530 — classic NGINX, new protocol features come with rough edges. Good news: no confirmed exploitation in the wild yet. HTTP/3 is still sparse; by the time it becomes a mass target, patches should be out.

Fortinet FortiSandbox: three critical flaws, all three under active attack [3][4]

FortiSandbox — Fortinet's malware analysis appliance — is seeing three critical CVEs exploited simultaneously. This is the worst category: unauthenticated, no user interaction, root.

  • CVE-2026-39808 (OS command injection): CVSS 9.8. An unauthenticated attacker executes code with root privileges. Affects 4.4.0–4.4.8, patched in 4.4.9+ (April 2026).
  • CVE-2026-39813 (path traversal in JRPC API): CVSS 9.1, auth bypass via crafted HTTP request. Also patched in April.
  • CVE-2026-25089 (command injection in web UI): CVSS 9.8; patched 06/09. Oddly: the circulating exploit shows signs of AI-generated development and contains functional bugs.

VulnCheck saw the first exploitation of 39808 on 06/09; Defused Cyber reported attacks on all three within 24 hours by 06/16. Fortinet hasn't officially confirmed this. Lesson for operations thinking: a security appliance that's unauthenticated-RCE-able is not protection, it's a door with a "protection" sign.

Microsoft Patch Tuesday: 198 CVEs — and a zero-day out of spite [5][6]

June's Patch Tuesday was Microsoft's largest ever on a single Tuesday, with 198 CVEs — 32 critical, 166 important. Three were known zero-days disclosed in advance, all rated "Exploitation More Likely": CVE-2026-50507 (BitLocker bypass), CVE-2026-49160 (HTTP.sys DoS), and CVE-2026-45586 (CTFMON privilege escalation).

But the oddest story came from outside: "RoguePlanet" (CVE-2026-50656), a race condition in Microsoft Defender that spawns a shell with SYSTEM privileges. A researcher named "Nightmare Eclipse" published the PoC on GitHub on 06/10 — hours after Patch Tuesday, by their own account out of frustration over a dispute with Microsoft. Affects fully patched Windows 10 and 11 systems. Microsoft says they're working on a "high-quality" fix; as of 06/17 there was no patch and no observed exploitation.

Source note Multiple secondary sources cite RoguePlanet's CVSS value (~7.8); the MSRC primary source was not directly checked — so treat as a secondary value.

Chrome V8 zero-day — and why your Electron apps should care [7]

CVE-2026-11645 is an out-of-bounds read/write in V8, Chrome's JavaScript engine — CVSS 8.8, actively exploited, patched in Chrome 149.0.7827.103 on 06/08. A crafted HTML page can execute code (within the sandbox). The anonymous reporter earned $55,000 bounty. CISA added it to the KEV list; US agencies' deadline is 06/23.

Why this sits here instead of just "browser news": V8 powers every Electron app — VS Code, Slack, Discord, Postman. A V8 flaw is de facto a desktop-app flaw. If you build or ship Electron apps, watch the Chromium version, not just the browser update.

  1. helpnetsecurity.com — Chrome zero-day, 06/09/2026

KEV roundup: Joomla, Cisco SD-WAN, Android [8][9][10]

Joomla JCE — CVE-2026-48907, a perfect 10.0: unauthenticated PHP code execution via creating new editor profiles. Affects JCE 1.0.0–2.9.99.4; patched in 2.9.99.5 (06/03). CISA added it to KEV on 06/16, automated attacks running. If you have any Joomla with the JCE editor anywhere: that's the patch entry you check today.

Cisco Catalyst SD-WAN — CVE-2026-20262 (path traversal, CVSS 6.5, arbitrary file write) entered the KEV list; agency deadline 06/29. Attackers drop a malicious .war file via vManage's WildFly server, which deploys as a Java web app. Alongside it sits CVE-2026-20245, a root escalation unpatched at time of warning.

Android June bulletin — 121 flaws, 13 critical. CVE-2025-48595 (CVSS 8.4, integer overflow → local privilege escalation in framework, no user interaction) is being exploited by Google "in a limited and targeted manner"; affects Android 14 through 16. In KEV early June.

Note KEV add data for Joomla come from secondary sources (THN, Security Affairs) — the CISA alert page itself returned 403 on fetch. Cross-checked multiple times, but not primary-confirmed.

PostgreSQL 19 Beta 1 is here [11]

On 06/04, the first beta of PostgreSQL 19 shipped — for Daniel's stack (Postgres is the house database), the most welcome entry today. Highlights:

  • Parallel autovacuum (autovacuum_max_parallel_workers) — vacuuming dead table rows now runs multi-threaded instead of blocking a single worker.
  • SQL/PGQ — property graph queries directly in SQL: graph operations without a separate graph database.
  • ON CONFLICT DO SELECT — on upsert, get the conflicting row back directly instead of fetching it again later.
  • Also: REPACK CONCURRENTLY, ~2× faster inserts with FK checks, online checksum toggling.

GA targeted for Sep/Oct 2026. Beta means: test, give feedback — don't run production on it. But ON CONFLICT DO SELECT alone saves a roundtrip query in many FastAPI endpoints.

  1. postgresql.org — Beta 1 announcement, 06/04/2026

EU CRA + NIS2: two deadlines drawing closer [12][13]

The EU Cyber Resilience Act hit a milestone on 06/11: notification rules for conformity assessment bodies now apply — member states need designated authorities. The developer-relevant deadline is 09/11/2026: as of then, vendors must actively report exploited vulnerabilities and serious incidents to ENISA and national CSIRTs. Full CE marking requirement kicks in December 2027.

CRA: An EU regulation that mandates basic cybersecurity for "products with digital elements" — from IoT gadgets to software libraries — with CE marks like a toaster. OSS maintenance has exceptions, but commercial distribution puts you in scope.

Parallel in Germany: the NIS2 registration deadline with the BSI was 03/06 — already passed. Late registration is possible but counts as its own violation; fines reach €10 million or 2% of annual turnover.

It's Sunday, June twenty-first. Last week was anything but quiet in the software world — four major security stories converged in just four days, plus a very welcome database beta.

The standout story: a flaw in P-T-C's Windchill industrial software earns a perfect ten. An attacker can execute code from the network without a login, and it's already happening — backdoors are being actively installed. Germany's B-S-I called affected system administrators in the middle of the night, around two thirty a.m., to push for patches. A government agency doesn't make calls like that unless it matters. The patch has been available since June fifteenth.

Closer to Daniel's day-to-day is NGINX. F-5 patched the web server outside the normal release cycle — a use-after-free in H-T-T-P-three handling with high severity, plus a heap overflow in gRPC proxying. Only the young mainline versions are affected; if you run the stable line, you're clear. Nothing being exploited yet.

At Fortinet, though, it's already burning. In the Sandbox appliance, three critical flaws are being attacked simultaneously — unauthenticated, with root privileges, without any user clicking anything. The most severe scores nine point eight. Bitter irony: a security appliance that becomes an open door itself.

Microsoft delivered a record on Patch Tuesday — one hundred ninety-eight flaws in a single Tuesday, more than ever before. Three were known zero-days in advance. And then a researcher, frustrated by a dispute with Microsoft, published another Defender zero-day just hours after Patch Tuesday, named RoguePlanet, still without a fix. It runs on fully patched systems too.

A Chrome zero-day in the JavaScript V-8 engine is also being exploited. That's more than browser news, because V-8 powers every Electron app — so VS Code, Slack, or Postman too. If you build those apps, you should watch the Chromium version, not just update the browser.

On a brighter note: PostgreSQL Nineteen is out as the first beta. Parallel autovacuum, graph queries right in S-Q-L, and a new ON CONFLICT DO SELECT that saves an extra query on upserts. The final version arrives in the fall.

On the regulatory side, the E-U Cyber Resilience Act is drawing near: as of June eleventh, the first rules take effect, and starting September eleventh, vendors must report exploited flaws to E-N-I-S-A. Germany is running N-I-S-Two in parallel, with fines up to ten million euros.

Next week should be quieter — the only hard deadline is Tuesday, when U-S agencies must have the Chrome zero-day closed. Talk tomorrow.
