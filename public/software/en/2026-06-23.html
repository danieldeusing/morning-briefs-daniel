Splunk: unauth RCE, actively exploited, KEV deadline already passed [1][2]

The vulnerability of the weekend: CVE-2026-20253 in Splunk Enterprise allows unauthenticated remote code execution via the bundled PostgreSQL Sidecar — CVSS 9.8. Affected are versions < 10.2.4 and < 10.0.7; fixed in 10.2.4 and 10.0.7 respectively. According to helpnetsecurity, it has been actively attacked since around June 15. CISA added it to the KEV catalog — the remediation deadline for US federal agencies was Sunday, June 21, so it has already passed.

Splunk often sits at the heart of a security stack as a SIEM; an unauthenticated RCE there is exactly the kind of vulnerability an attacker uses to pivot from the monitoring layer deeper into infrastructure. If you run an on-premises instance: the update is not optional, and "internally accessible" is no protection if the telemetry pipeline reaches far into the network.

libssh2: critical RCE — and still no release [3]

CVE-2026-55200 in libssh2 (CVSS 9.2) can be triggered via malformed SSH packets: memory errors leading to code execution, including DoS. Affected are all versions ≤ 1.11.1 — and the inconvenient part: there's no official release yet. The fix exists only as a GitHub commit 7acf3df; Debian has pushed 1.11.1-3 to Testing. A second vulnerability (CVE-2026-55199, High) comes along with it.

libssh2 is everywhere SSH gets embedded: routers, IoT devices, server management tools, Git clients. If you link libssh2 statically, you should apply the commit or wait for the distro package rather than hoping for an upstream tag that isn't there yet.

  1. heise.de

Oracle CPU: PeopleSoft RCE actively exploited [4][5]

Oracle's second monthly security update brought 243 CVEs in 245 patches, with 122 rated as critical. The sharp entry: CVE-2026-35273, an unauthenticated RCE in PeopleTools 8.61 and 8.62. According to Tenable, ShinyHunters (UNC6240) have compromised ~300 installations, with 68% in higher education. Oracle delivered an out-of-band fix on 06.10, and CISA added the vulnerability to the KEV catalog on 06.12.

PeopleSoft is campus ERP — student data, personnel records, finances. That a ransomware group is specifically targeting universities fits the pattern: older enterprise stacks, stretched IT teams, sensitive data.

WordPress supply chain hit twice [6][7]

Two supply-chain incidents in the WordPress ecosystem. First: at Awesome Motive (OptinMonster), attackers stole a CDN API key via an UpdraftPlus vulnerability and injected malicious JavaScript into OptinMonster, TrustPulse, and PushEngage — 1.2 million sites affected, according to Sansec. Second: at ShapedPlugin, attackers compromised the build and distribution chain via Easy Digital Downloads; the malware stole admin credentials plus 2FA secrets and set up remote access — vulnerable window April through June 2026.

In both cases the target was not the individual site but the update channel. That's the uncomfortable lesson for anyone who auto-updates plugins: auto-update also automatically distributes a poisoned version.

Heads up If you run affected plugins: check for CDN-injected scripts and unexpected admin accounts, rotate 2FA secrets.

AWS Lambda gets MicroVMs — up to 8 hours [8]

AWS announced Lambda MicroVMs on 06.22: isolated, stateful execution environments with runtime up to 8 hours (vs. the old 15-minute limit), up to 16 vCPU, 32 GB RAM, 32 GB disk, ARM64, built on Firecracker, with suspend/resume. Initially in five AWS regions (N. Virginia, Ohio, Oregon, Ireland, Tokyo). (Beta / Current Channel Preview, Version 2607, from ~July 21 2026).

The stated intent is clear: run code generated by users or AI in a sandbox. AWS is following the pattern others sell as "code interpreter sandbox" — only directly in the Lambda platform with longer runtime and state. For agent workloads that need more than a few minutes and execute untrusted code, this closes a real gap.

MicroVM: a minimal virtual machine that starts in milliseconds (Firecracker is the technology behind it). Heavier isolation than a container, lighter than a classical VM — ideal for safely locking down third-party code.

Spring Boot 4.1.0 is here — and 3.5 is ending [9][10]

Spring Boot 4.1.0 has been available since 06.10. Highlights: Spring gRPC auto-configuration, SSRF protection for the HTTP client (new InetAddressFilter), updated OpenTelemetry integration, Log4j file rotation, Jackson config properties, and Kotlin 2.3 support. Mark your calendar: Spring Boot 3.5 reaches EOL on 06.30 — after that there are no free fixes in the OSS branch.

For Java/Spring shops, that's the migration announcement of the month. The SSRF filter is more than cosmetic: it addresses exactly the class of server-side request forgery through which internal metadata endpoints are often scraped.

AMD removes memory encryption — quietly [11]

Odd and unappealing story: AMD removed Transparent Secure Memory Encryption (TSME) from Ryzen 9000 consumer CPUs (non-Pro) with firmware AGESA 1.2.7.0 — without any announcement. A Linux user discovered it in April via HSI audit; AMD confirmed the removal was intentional but remained quiet on further questions. A BIOS update in July is supposed to restore TSME.

TSME encrypts RAM transparently against physical attacks (cold boot, DMA sniffing). Turning it off quietly means devices lose a security feature without their owners knowing — the worst kind of security regression.

TSME: encrypts the entire memory automatically in hardware. Protects against attackers with physical access who want to read RAM contents directly — especially relevant for laptops and kiosk systems.

FastAPI 0.138.0 now serves frontends [12]

Small but practical for the Python crowd: FastAPI 0.138.0 (June 20) brings app.frontend() / router.frontend() to serve a built dist/ directory directly from the framework — plus Pydantic v2 deprecation fixes and a bump to Starlette 1.0.0. Saves a separate static server for small SPA-plus-API setups; especially for a FastAPI backend with Angular or React frontend that you want to ship as a single container, this is a welcome simplification. The Starlette 1.0 jump is the part that deserves a closer look — a 1.0 major usually means some behavioral changes under the hood.

Patch Tuesday: Record CVE count, six zero-days [13][14]

Microsoft's June patch day (June 9) was unusually large: depending on how you count, 198 to 209 CVEs, 32–33 of them critical, 28 critical RCEs. Plus six zero-days — but only one actively exploited: CVE-2026-42897 (Exchange Server spoofing). The other five were publicly known but not yet attacked, including CVE-2026-45585 (BitLocker bypass "YellowKey"), CVE-2020-17103 (Cloud Files EoP), CVE-2026-45586 (CTFMON EoP, 7.8 — SYSTEM privileges via speech recognition processes) and CVE-2026-49160 (HTTP.sys DoS, 7.5 — HTTP/2 "bomb", mitigated via new MaxHeadersCount registry value). Separately, CVE-2026-32193 stands out: a critical Azure Kubernetes Service RCE with CVSS 8.8.

The HTTP.sys DoS is the most interesting for web developers: an HTTP/2 header flood that can overwhelm the Windows HTTP stack — relevant for anything behind IIS or HTTP.sys. But the one already actively exploited is the Exchange Server spoofing (CVE-2026-42897) — exactly the kind of flaw an attacker reaches for first in a phishing or identity chain. Six zero-days in one patch day is unusually high; in sum, this is one of Microsoft's largest patch days ever.

EU CRA & Five Eyes: Regulation meets reality [17][18]

Standing thread on EU regulation: with the Cyber Resilience Act, the framework for designating notified bodies became effective on 06.11 — member states had to designate their notifying authorities. The next hard deadline is 09.11.2026: manufacturers must then report actively exploited vulnerabilities within 24 hours (early warning) or 72 hours (full report).

In parallel, a joint Five Eyes advisory warns that AI-driven cyberattacks are "months, not years" away, and boards should treat cyber risk as core to business continuity. Rhetoric with lead time — but the tone aligns with what the supply-chain incidents above practically demonstrate.

It's Monday, June 23. The weekend was anything but quiet in the software world — security dominates the day.

The top story: Splunk Enterprise has a critical vulnerability, unauthenticated code execution via the bundled PostgreSQL service, with the highest possible severity of nine point eight. It's been actively exploited since mid-June, and the deadline set by CISA, the US federal cybersecurity agency, already expired on Sunday. Anyone running a Splunk instance should update to versions ten point two point four or ten point zero point seven. That's according to helpnetsecurity and bleepingcomputer.

Nearly as serious: the widely deployed libssh2 library has a critical vulnerability with a severity of nine point two. It affects all versions up to and including one point eleven point one — and the inconvenient part is there's still no official release, just a single commit on GitHub. As heise reports, libssh2 is found in routers, IoT devices, and management tools.

Oracle had a vulnerability in the campus software PeopleSoft actively exploited; the group ShinyHunters compromised around three hundred installations, nearly seventy percent of them at universities. And in the WordPress ecosystem there were two supply-chain attacks: one via a stolen CDN key at Awesome Motive affecting one point two million sites, and one via compromised premium plugins from ShapedPlugin.

But there was also good news. AWS gave its Lambda service a new option called MicroVMs: isolated environments running up to eight hours with up to sixteen virtual cores and thirty-two gigabytes of memory. They're explicitly designed to safely lock down code generated by users or AI.

For the Java world, Spring Boot four point one arrived with gRPC support, new protection against server-side request forgery, and Kotlin two point three. Important: Spring Boot three point five reaches end of support on June thirtieth. And AMD quietly removed memory encryption from consumer Ryzen processors — a BIOS update in July should restore it.

Microsoft's patch day was record-breakingly large with around two hundred vulnerabilities and six zero-days (one actively exploited), plus a critical vulnerability in the Azure Kubernetes service. Android patched an actively exploited zero-day along with one hundred twenty-four vulnerabilities total.

On the regulation side, a milestone was reached with the European Cyber Resilience Act; starting September eleventh, there's a twenty-four-hour reporting requirement. And the Five Eyes nations warn that AI-powered attacks are only months away.
