PTC Windchill: a perfect 10.0, and the BSI reaches for the phone [1][2][3]

Today's story is engineering software most devs never touch, but whose outage paralyzes entire factories. PTC Windchill is a PLM system (Product Lifecycle Management) — the central database where industrial corporations manage their CAD models, BOMs, and design states. CVE-2026-12569 scored a perfect 10.0 in CVSS 3.1 (9.3 in CVSS 4.0): a remote, unauthenticated attacker can execute arbitrary code via unsafe deserialization.[1]

Deserialization: When a program "unpacks" a stored object (e.g., a Java object from the network), it trusts the instructions inside. Think of getting an assembled piece of furniture by mail and building it without checking — if a saw is packed inside, you build it in. When an attacker sends a rigged object, your server executes its code.

What's remarkable is the escalation: the BSI called affected companies at 2:30 a.m. — no agency makes that call for routine vulnerabilities. According to SecurityAffairs, backdoors are already being placed on vulnerable servers; an indicator is the HTTP header X-windchill-req: ?x8Fmgow in logs.[3] PTC patched on June 15, 2026 — fixed versions are Windchill 13.1.2.8, 13.1.3.4, 13.0.2.12, and 12.1.2.27.[2] Anyone running Windchill should check the named logs, not just deploy patches.

"Miasma": the npm worm crawling through Red Hat's supply chain [4][5][6]

The second major incident is exactly the kind of supply-chain attack this brief has been warning about for months — this time via a trusted brand. Microsoft, Wiz, and Snyk document a campaign called "Miasma": at least 32 npm packages from the official @redhat-cloud-services namespace were trojanned across over 90 versions; the malicious versions combined saw roughly 80,000 downloads per week.[4]

The entry point wasn't an npm bug, but a compromised GitHub account of a Red Hat employee — whose credentials had been in infostealer logs since April 13, 2026. Through the compromised CI/CD pipeline (RedHatInsights/javascript-clients), the tampered packages were cleanly signed and published.[5] The payload is a 4.29 MB obfuscated dropper in a preinstall hook — it fires the second someone types npm install and grabs ~/.npmrc tokens, SSH keys, GitHub tokens, GCP/Azure identities, and CI/CD secrets.[6]

preinstall hook: npm lets packages run a script before actual installation. Like a delivery driver who doesn't just drop off a package but runs into your house when you open the door — without you ever seeing the contents. This automation is exactly what makes supply-chain worms so effective.

The lesson is uncomfortable: a valid signature from a known vendor no longer means "safe". If you use @redhat-cloud-services, audit lockfiles, run builds without lifecycle scripts (--ignore-scripts), and rotate exposed tokens.

Mastra & Open VSX: the tooling layer remains the attack vector [7][8]

Miasma is not alone. In parallel, Phoenix Security reports 140 Mastra packages with malicious dependencies targeting crypto-wallet extensions.[7] And in the VS Code-compatible Open VSX marketplace, Dark Reading found over 72 malicious extensions since January 31, 2026.[8] Pattern: not the language, not the runtime — the layer of packages, plugins, and agent tooling is the attack surface. Dependency pinning and build isolation are no longer paranoia luxuries.

Squidbleed: a 29-year-old wound in the proxy [9]

Squid is the longest-serving open-source caching proxy — and has carried a heap over-read since 1997. CVE-2026-47729 ("Squidbleed", CVSS 7.5 per squidbleed.xyz; NVD confirmation was unavailable today) lives in the FTP directory listing parser: when reading past a buffer boundary, Squid leaks plaintext HTTP auth headers, cookies, and API keys of other users into foreign responses — Heartbleed mechanics, only in a proxy. All nine release families since 1997 are affected. The fix is a single line (a null check before strchr); anyone who doesn't need the FTP gateway can disable it as an immediate mitigation.[9]

  1. squidbleed.xyz

Splunk: CVSS 9.8, in the KEV catalog, PoC public [10][11]

Splunk Enterprise has CVE-2026-20253 (SVD-2026-0603, CVSS 9.8): an unauthenticated RCE via the PostgreSQL sidecar service — attackers can create or truncate arbitrary files.[10] This is more than theory — WatchTowr published a PoC on June 12, first exploit attempts were observed from June 17, and on June 18 CISA added it to their KEV catalog.[11] Affected are 10.0.0–10.0.6 and 10.2.0–10.2.3; fixed in 10.2.4 / 10.0.7.

CISA KEV catalog: The U.S. Cybersecurity and Infrastructure Security Agency maintains a list of vulnerabilities that are demonstrably being actively exploited. Like a wanted poster wall: if a CVE is on it, it's not "could be dangerous" but "is being abused right now" — that's the signal to expedite patches.

  2. socradar.io

Node.js: a batch release closes TLS bypass and WebCrypto DoS [12]

On June 18, a coordinated security release landed across all active lines — v22.23.0 / v24.17.0 / v26.3.1 — with two HIGHs and ten lower-severity CVEs. CVE-2026-48618 is a TLS hostname bypass: Unicode dot separators let you evade wildcard depth checking, an auth-bypass class. CVE-2026-48933 is an integer overflow in WebCrypto AES that crashes the process (DoS) when given input as a multiple of 2 GB. Also included: OpenSSL 3.5.7, undici, and nghttp2 1.69.0.[12] Mandatory update for anyone terminating TLS or using WebCrypto — so pretty much every Node server.

  1. nodejs.org

Docker: AuthZ bypass via 1-MB padding [16]

More context than breaking news, but relevant for anyone with Docker authorization plugins: CVE-2026-34040 (CVSS 8.8) bypasses AuthZ plugins by padding a request with over 1 MB — an incomplete fix for the older CVE-2024-41110. Patched in Docker Engine 29.3.1 and Docker Desktop 4.66.1.[16] Disclosure was in April, so this is just a reminder if your engine is still old.

Homebrew 6.0: Tap Trust and a built-in CVE scanner [17][18]

Fitting the supply-chain theme, Homebrew 6.0.0 (June 11) brings Tap Trust mechanics: a third-party tap can only execute code once you explicitly authorize it. Additions include brew vulns (checking installed formulas against the OSV database), Linux sandbox via Bubblewrap, an ask-mode as default, and parallel brew bundle.[17] Project lead quote on the state of package ecosystems: npm is "less vulnerable than it is today" compared to a decade ago.[18] Hard to disagree given the incidents above.

AWS Lambda MicroVMs: Serverless can now run for 8 hours [19]

The most exciting pure infra news of the week: AWS unveiled Lambda MicroVMs — built on Firecracker, with up to 8 hours runtime instead of the 15-minute Lambda ceiling, up to 16 vCPUs, 32 GB RAM, 32 GB disk, on ARM64.[19] This closes the gap between "quick function" and "persistent container": batch jobs, data pipelines, and longer renders that previously needed Fargate or an EC2 instance now fit the serverless model with VM-level isolation. Available in US-East (N. Virginia, Ohio), US-West (Oregon), Europe (Ireland), and Asia-Pacific (Tokyo).

Firecracker: The mini-virtualization AWS built for Lambda and Fargate. Instead of emulating a full computer, it spins up a tiny, sandboxed virtual machine in milliseconds — like a one-way cabin with only the essentials, standing in seconds and disappearing cleanly after the job.
  1. aws.amazon.com

PACT: Cloudflare and three browsers want to bury CAPTCHA [20]

Cloudflare is developing a protocol with Chrome, Firefox, and Edge (and Shopify) called PACT (Private Access Control Tokens) — a privacy-friendly CAPTCHA replacement meant for W3C standardization.[20] The hook is a number that matters to developers: per Cloudflare Radar, automated systems now account for roughly 58% of global HTTP traffic, versus 42% from real humans. Instead of making every user click traffic lights, the browser supplies an anonymous "I'm real" token. Whether that stays an open standard or de facto lands at the big browsers is the open question.

  1. cloudflare.com

Languages & DB: Python 3.15b3 and PostgreSQL 19 Beta 1 [21][22][23]

Python 3.15.0b3 (June 23) brings 195 bugfixes, stable ABI for the free-threaded build (PEP 803), and a JIT gain of 8–9% per release notes (geometric mean on x86-64 Linux; from beta announcement, not independent measurement).[21][22] The stable line sits at 3.14.6 (June 10, ~179 bugfixes). On the DB side, PostgreSQL 19 Beta 1 (June 4) is the highlight: SQL/PGQ property-graph queries, ON CONFLICT DO SELECT, a REPACK command, parallel autovacuum, and native JSON output for COPY TO. Final expected Sept–Oct 2026.[23]

CRA countdown: Sept 11 brings reporting requirements [24]

A distant deadline, but one with teeth: the EU Cyber Resilience Act activates reporting obligations on September 11, 2026 for manufacturers — 24-hour early warning, 72-hour full report, 14-day closure for actively exploited vulnerabilities, via the ENISA single reporting platform.[24] If you ship software in the EU, set up the reporting process now, not in September. The framework for conformity assessment bodies has been operational since June 11.

  1. europa.eu

It's Wednesday, June twenty-fourth. Today is security day, not a release day. Four serious vulnerabilities collide, and one of them carries the worst possible rating.

The story of the day is PTC Windchill, an engineering platform for industrial corporations. The vulnerability scored a perfect ten out of ten. An attacker needs no password to remotely execute arbitrary code. Germany's BSI called affected companies at two thirty in the morning, and security researchers say backdoors are already being placed on vulnerable servers. The patch has been available since June fifteenth; if you run Windchill, check your logs too.

Second big thing: a worm in the npm supply chain. Thirty-two packages from Red Hat's official namespace were poisoned, with roughly eighty thousand downloads per week. The entry point was a compromised employee account, whose credentials had been in credential-theft logs since April. The malware steals cloud access tokens the instant someone types npm install. The lesson, as Microsoft, Wiz, and Snyk frame it: a valid signature from a big brand no longer guarantees safety.

Then Splunk: a vulnerability rated nine point eight, in the U.S. government's catalog of actively exploited vulnerabilities since June eighteenth, with public exploit code. And Node.js released security fixes on June eighteenth across all version lines, closing a TLS bypass and a WebCrypto crash.

Curiosity item: Squidbleed. The venerable Squid proxy has been leaking other users' plaintext credentials into foreign responses since nineteen ninety-seven. Twenty-nine years, nine release families, and the fix is one line of code.

Not just security: AWS is letting its serverless Lambda function now run up to eight hours instead of fifteen minutes, on real mini virtual machines. And Cloudflare is collaborating with Chrome, Firefox, and Edge on a protocol to replace the annoying CAPTCHA. The backdrop: bots now make up fifty-eight percent of web traffic.

In the background, a clock is ticking. Starting September eleventh, the EU Cyber Resilience Act imposes reporting requirements, with a twenty-four-hour early warning window. If you sell software in Europe, set up the reporting process now.

Tomorrow is expected to be quieter — no fixed vendor deadline. If CISA posts another entry, it will appear in the next brief. For now: patch what's open.
