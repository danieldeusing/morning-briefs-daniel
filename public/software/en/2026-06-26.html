UniFi: three CVSS-10 flaws, exploited — and the deadline is today

The one thing that really matters today. On 06-23, CISA added three Ubiquiti UniFi OS flaws to the KEV catalog — all with max severity 10.0 and all already actively exploited — and gave federal agencies three days to patch. Those three days end today.[1][4]

KEV Catalog: The U.S. agency CISA maintains a list of vulnerabilities that aren't just theoretically dangerous but are actively being used by attackers — like a wanted list for burglary tools that are already in circulation. Once a flaw lands on that list, the "wait or patch?" conversation is over.

The chain: CVE-2026-34908 allows unauthorized remote changes, -34909 is a path traversal flaw (file access + account manipulation), and -34910 is an input validation weakness that leads to remote code execution via command injection. Chained together, they give complete network takeover without authentication. The patch shipped on 05-21 in UniFi OS Server 5.0.8 — if you run a UDM/Cloud Key/UniFi OS instance and aren't on 5.0.8 yet, this is your Friday afternoon.[1][2]

Source Note CISA.gov returned a 403 on direct access; KEV addition date and CVSS 10 are cross-checked via BleepingComputer and GBHackers, individual CVE scores from NVD are not in our primary sources. The thrust — three max-severity flaws, actively exploited, deadline today — is clear.

  2. gbhackers.com — Patch in UniFi OS Server 5.0.8 (05-21).
  3. cisa.gov — KEV alert, three-day deadline.

"Miasma": a Red Hat account poisons 32 npm packages

Exactly the kind of story this brief exists for: no AI headline, but supply chain as an attack surface. On 06-01, Wiz disclosed a campaign called Miasma — a compromised GitHub account belonging to a Red Hat employee was used to inject malware into 32 @redhat-cloud-services npm packages. Together they added up to roughly 80,000 downloads per week.[5]

The trick is what's alarming. The attacker requested OIDC tokens via GitHub Actions and published the trojanned packages with valid SLSA provenance — so the packages carried exactly the supply-chain authenticity credentials that are supposed to build trust.[5]

SLSA Provenance / OIDC: SLSA provenance is like a sealed proof of origin, showing "this package was actually built by this pipeline." OIDC is the ID card the pipeline uses when publishing. Here, the attacker forged the ID card and thus affixed a genuine seal to a fake package — the seal checks out, the contents don't.

On install, a preinstall hook fired a 4.29 MB obfuscated dropper, fetched the Bun runtime, and harvested GitHub, npm, AWS, Azure, and GCP credentials — each infection individually encrypted so one signature doesn't catch all of them. Red Hat has an advisory (RHSB-2026-006) out; remediation is ongoing.[6]

Not alone It's busy season: on 06-17, a typosquatted easy-day-js (dayjs lookalike) poisoned 140+ packages in the Mastra AI framework ecosystem via postinstall dropper. The hook here is also npm supply chain, not the model.[7]

  2. access.redhat.com — Payload details, advisory.
  3. upwind.io — Mastra AI, easy-day-js.

Apple acquires the Swift Package Index

Announced on 06-23: Apple is acquiring the Swift Package Index, built by Dave Verwer and Sven A. Schmidt — the de-facto catalog for Swift packages with over 10,000 indexed packages. It stays open source, and the stated plan is package signing plus its own registry to "move away entirely from the GitHub model".[8]

For non-Swift folks, still relevant: it's the same shift we're seeing everywhere right now — after the npm incidents, ecosystems are moving away from "GitHub account = trust" toward signed, self-hosted registries. Apple can pull this off for Swift because it owns the platform. For everyone else, it's a data point on where package security is heading.

  1. swiftpackageindex.com — Acquisition, 10k+ packages, signing + registry.

Check Point VPN: IKEv1 auth bypass, exploited by Qilin

CVE-2026-50751 (CVSS 9.3) is a logic flaw in certificate validation for the legacy IKEv1 key exchange: an unauthenticated attacker can bypass user authentication and establish a remote-access VPN connection without a valid password. Affected are Quantum Security Gateway (R80.40–R82.10), Spark Firewalls, and various Gaia OS versions.[9]

This isn't theoretical risk: exploitation began on 05-07, and post-compromise activity by a Qilin ransomware affiliate has been confirmed in at least one case. CISA added the flaw to the KEV catalog (deadline was 06-11); the patch shipped on 06-08. If you still have IKEv1 active: disable or patch.[10]

  1. nvd.nist.gov — CVSS 9.3, IKEv1 auth bypass, affected products.
  2. helpnetsecurity.com — Qilin affiliate, exploit from 05-07.

Oracle PeopleSoft: zero-day RCE, ShinyHunters in 100+ orgs

CVE-2026-35273 (CVSS 9.8) hits the PSEMHUB component of PeopleSoft — easily exploitable, no authentication, plain HTTP access is enough for complete compromise. The ShinyHunters group exploited this as a zero-day from 05-27 until the patch on 06-10 and claims 300 instances across 100+ organizations, of which 68% are in higher education.[11]

For Daniel, no direct stack hit (PeopleSoft is enterprise HR/ERP), but an instructive pattern: a weeks-long exploit window before the patch, targeted at a sector with lean IT teams. Anyone running PeopleSoft should have patched long ago and reviewed May logs.[11]

  1. thehackernews.com — ShinyHunters, 300 instances, 68% higher education.

Microsoft's June round on 06-09 set a record: 200 vulnerabilities, of which 33 are critical (28 RCE) and 3–6 zero-days. The most important is CVE-2026-42897 — an Exchange Server XSS: a crafted email, when opened in OWA, executes arbitrary JavaScript in the recipient's browser without the attacker needing privileges. EU CRA: Chapter IV live since 06-11

Quiet but relevant compliance milestone: since 06-11, Chapter IV of the Cyber Resilience Act is in effect — notification of conformity assessment bodies. Member states must now designate notifying authorities; this is the infrastructure through which products will later earn security CE marking. Next milestone: reporting obligations starting 09-11-2026, full enforcement then on 12-11-2027.[14]

Spring Framework 7.0.8: hardening, and 6.2 dies Tuesday

The 7.0.8 release on 06-08 is more security hardening than feature drop, and exactly why it matters: Jackson deserialization in JMS converters can now be restricted to whitelisted packages via setTrustedPackages(...), and SpEL evaluation is now capped at ten thousand operations by default.[17]

Deserialization Trustlist / SpEL Cap: Deserialization turns incoming data back into objects — without a whitelist, an attacker can force classes that execute code. SpEL is Spring's mini-language for dynamic expressions; a limit stops a malicious expression from locking the CPU in an infinite loop. Both make common RCE/DoS classics much harder.

And the calendar note: Spring Framework 6.2.x reaches end-of-life on 06-30 — next Tuesday. If you're still on 6.2, you should be planning the jump to 7.0 now.[18]

  1. endoflife.date — 7.0.8 (06-08), JMS trustlist + SpEL cap.
  2. endoflife.date — 6.2.x EOL 06-30-2026.

PostgreSQL: rollup update, PG19 Beta 1, PG14 winding down

The rollup from 05-14 (18.4 / 17.10 / 16.14 / 15.18 / 14.23) closes 11 CVEs and over 60 bugs — including three CVSS 8.8 flaws: an integer wraparound (CVE-2026-6473), a symlink issue in pg_basebackup (-6475), and a libpq stack overwrite (-6477). If you're not yet on the latest minor for your version, plan for it.[19]

Looking ahead: PG19 Beta 1 is out (final expected Sept/Oct), and PostgreSQL 14 reaches end-of-life on 11-12-2026. For Daniel's PG stack, that means moving 14 instances onto an upgrade path over the summer.[19][20]

Microsoft builds a real server distro: Azure Linux 4.0

Unveiled at the OSS Summit NA (mid-May): Azure Linux 4.0 (Preview) — a Fedora-derived, RPM-based distribution that Microsoft builds and maintains for Azure VMs. The remarkable thing is the "first time": it's Microsoft's first supported Linux distro beyond pure container hosting.[21]

In parallel, Azure Container Linux went GA (announced mid-May at OSS Summit; broader rollout on 06-02 at Build) — an immutable, container-optimized host OS based on the acquired Flatcar project. Together, this is Microsoft's Linux confidence as a platform, not just a guest.[22]
