When the BSI calls at night: PTC Windchill, CVSS 9.8 [1][2]

The story of the week is less the CVE itself than the response to it. CVE-2026-12569 in PTC Windchill and FlexPLM — the product lifecycle management platform used by industrial companies to manage product data and engineering drawings — scored CVSS 9.8 (3.1) or 9.3 (4.0). It's unsafe deserialization that allows unauthenticated remote code execution: reachable over the network, no login required, easily automated.[1]

Unsafe deserialization: A program receives data and "unpacks" it to rebuild objects — like assembling a piece of furniture from a blueprint. If someone sends a malicious blueprint, the program doesn't build a shelf; it executes foreign commands. That's what happens here — and nobody needs to log in first.

What stands out is the escalation level from authorities. The German cyber agency (BSI) called administrators at night, and in several German states, police physically visited affected companies to warn of imminent attacks.[2] You almost never see this for a CVE — the signal is clear: systems are already being scanned, and the target profile (manufacturing, mid-market, often poorly patched) is exactly what keeps BSI up at night.

The patch has been available since 06/15/2026. Patched versions are 13.1.2.8, 13.1.3.4, 13.0.2.12, and 12.1.2.27; affected versions range from 11.0 through various 12.x and 13.x releases. CISA has also flagged the bug.[1][2] If you run Windchill: don't wait for a police cruiser to show up.

Red Hat hit in npm namespace: "Miasma" [3][4]

A supply-chain attack that didn't just hit some hobby package this time — it hit Red Hat itself. In the npm namespace @redhat-cloud-services, researchers at Wiz found 32 package releases tampered with a common malicious payload that didn't match the legitimate source repos. Affected packages include frontend-components, compliance-client, and rbac-client; together they account for roughly 80,000 downloads per week.[3]

Supply-chain attack (npm): Instead of attacking your app directly, the attacker poisons an ingredient you unknowingly cook with — an npm package you pull in via npm install. Here, the malicious code hides in a preinstall script that runs automatically on every install, before you even write a line of your own code.

The breach vector: a compromised GitHub account belonging to a Red Hat employee, through which malicious commits were sneaked in.[4] Wiz attributes the tradecraft to the "Mini Shai-Hulud" cluster of TeamPCP — the same npm worm wave that already hit 84 artifacts across 42 TanStack packages and 170+ packages ecosystem-wide in May.[3][5] This isn't an isolated incident; it's a pattern. If you build: pin your lockfiles, seriously consider --ignore-scripts, and verify releases against the source repo.

Open Source Policy: CRA countdown ticks, few hear it [5][6]

Since 06/11/2026, Chapter IV of the EU Cyber Resilience Act has been in force — conformity assessment bodies and notification procedures are in place, and member states had to designate their notifying authorities.[6] But the real impact comes on 09/11/2026: from then on, manufacturers must report actively exploited vulnerabilities within 24 hours (early warning) and 72 hours (full report).

CRA (Cyber Resilience Act): An EU regulation that treats software and connected devices like toys as if they had safety standards: if you sell it in the EU, you must prove it meets certain security baselines — and report vulnerabilities as soon as they're exploited. Open-source maintainers get special provisions, but the obligation is real for commercial "steward" roles.

The problem: according to OpenSSF, 66% of developers and manufacturers worldwide know "nothing" or "almost nothing" about the CRA — in the US and Canada it's even 72%.[5] The same OpenSSF survey reports a 394% jump in new CVEs in Q1 2026 and an 811% surge in high-and-above severity. So regulation is coming right when supply chains are already on fire.

Ubiquiti UniFi OS: three CVSS 10.0, already in the wild [7][8]

Three vulnerabilities in Ubiquiti's UniFi OS that chain to full root RCE: CVE-2026-34908 (access control bypass), 34909 (path traversal, leaks config files and credentials), and 34910 (command injection). CISA added all three to the KEV catalog on 06/23; an active Mirai/Gaafgyt botnet campaign is already underway.[7]

Source status The CVSS 10.0 ratings come from the CNA (HackerOne) and are confirmed via Bishop Fox / Bleepingcomputer; the official NVD assessment for the three CVEs is still pending (currently "NVD assessment not yet provided"). The "critical" rating is solid; the exact NVD score is coming.

For U.S. federal agencies, CISA set a three-day deadline under BOD 26-04 — patch by 06/26. The fix is UniFi OS 5.0.8 or newer.[8] If you have a UniFi console on the network (UDM, Cloud Key, Dream Machine), don't dismiss it as "just home gear" — Mirai variants scan exactly these devices.

Chrome V8: the fifth 0-day of the year [9][10]

CVE-2026-11645 is an out-of-bounds read-and-write in V8, Chrome's JavaScript engine — CVSS 8.8, actively exploited. A crafted HTML page is enough to execute code within the sandbox. CISA added the bug to KEV on 06/09; it's already the fifth exploited Chrome 0-day in 2026.[9][10]

Fix: Chrome 149.0.7827.103 (Windows/macOS), .102 (Linux); Google paid $55,000 bounty. In the same mid-June cycle, Chrome 149.0.7827.155/.156 closed 33 more defects, including six critical use-after-free bugs. Chrome 150 Stable is targeted for 06/30 — auto-update should handle most, but a browser restart enforces it.

Patch Tuesday: 206 CVEs and an "HTTP/2 Bomb" [11][12]

Microsoft's June Patch Tuesday (06/09) closed 206 vulnerabilities, 33 rated critical, three zero-days. Two stand out. CVE-2026-49160 ("HTTP/2 Bomb", CVSS 7.5) is an unauthenticated DoS in Windows HTTP.sys via uncontrolled resource consumption — testing showed it exhausting 64 GB RAM on IIS in ~45 seconds. The catch: the bug affects not just Windows but also NGINX, Apache, Envoy, and Cloudflare.[11]

CVE-2026-45657 (CVSS 9.8) is a rare, fully unauthenticated, network-reachable RCE straight at the Windows kernel.[11] Plus CVE-2026-50507 (BitLocker bypass, CVSS 6.8) with public PoC available.[12] If you run a reverse-proxy-heavy stack: don't dismiss the HTTP/2 fix as a "Windows thing."

Spring Boot 4.1.0: gRPC out of the box [14]

On 06/10, Spring Boot 4.1.0 landed on Maven Central. Highlights for a Java/Spring stack: gRPC autoconfig (Netty or Servlet), SSRF protection in the HTTP client via an InetAddressFilter, Kotlin 2.3 support, OpenTelemetry updates, Log4j file rotation, lazy datasource connections, and async context propagation for @Async.[14]

SSRF (Server-Side Request Forgery): You ask your server to fetch a URL — and an attacker smuggles in an internal address it should never reach (like a cloud metadata service). The new filter checks the resolved IP before the server makes the request.

In parallel, 4.0.7 came out as a maintenance release with a security fix bundle — if you're still on 4.0.x, grab it. spring.io

PostgreSQL 19 Beta 1: parallel autovacuum finally here [15]

PostgreSQL 19 Beta 1 is available for testing since 06/04 (GA expected Sept./Oct.). The feature list is strong: parallel autovacuum (autovacuum_max_parallel_workers), ON CONFLICT DO SELECT for atomic get-or-create, a REPACK CONCURRENTLY command, SQL/PGQ property-graph queries, 2x faster FK checks, default_toast_compression now lz4, JIT off by default, and WAIT FOR LSN for read-your-writes.[15]

Autovacuum: Postgres doesn't immediately mark deleted rows as free; instead, a background "vacuum" process cleans up periodically. For large tables, this was single-threaded and could lag — now it can spawn multiple workers in parallel.

Especially ON CONFLICT DO SELECT solves an old pain point: previously, "does it exist? if not, create it, then give me the row" required tricks. Now it's atomic in one statement. postgresql.org

Angular 22 & Linux 7.1 [16][17][18]

Angular 22 — Signal Forms become stable

Since 06/03, Angular 22 is out: Signal Forms are stable, as are resource()/rxResource()/httpResource() and the @angular/aria package. OnPush is now the default change-detection strategy; the HTTP client switches to Fetch. Before upgrading, note: Angular 22 requires TypeScript v6 — TypeScript 5.9 is no longer supported.[16][17]

Linux 7.1 — new NTFS driver, 140k lines gone

On 06/14, Linus Torvalds released Linux 7.1. It includes a new in-kernel NTFS driver with full write support (iomap, delayed allocation), Intel FRED enabled by default — and over 140,000 lines of old x86-i486-era code removed (12,996 changesets from 2,011 developers). The 7.2 merge window closes today; RC1 is targeted for the same day; GA is expected mid-to-late August.[18]

It's Sunday, June twenty-eighth. And it was a week where the security department set the tone — three times the highest severity rating, plus an incident showing how serious authorities have gotten.

The story everyone's talking about is PTC Windchill. That's the platform industrial firms use to manage their engineering data. The vulnerability itself is serious enough — code execution over the network, no login needed. But what really stands out is the response: the German cyber agency called admins at night, and police in several states actually drove out to companies in person to warn them. You almost never see that for a security flaw. The patch has been out since June fifteenth.

Second big story: Red Hat got hit itself. In Red Hat's npm namespace, thirty-two package versions were tampered with — around eighty thousand downloads a week. The way in was a compromised employee account on GitHub. Researchers at Wiz link it to the same worm wave that already hit the TanStack ecosystem in May. If you build: lock your dependencies down and treat install scripts with suspicion.

Add Ubiquiti to the list: three vulnerabilities in their UniFi operating system, all three at the highest severity, chainable to full takeover as root. They've been on the active-exploitation list since June twenty-third, and a botnet is already scanning for them. The fix is UniFi OS version five point zero point eight. One honest caveat: the highest severity came from the vendor report; the official government assessment is still coming.

On the brighter side, there were solid releases. Spring Boot four point one brings gRPC with zero configuration and defense against S-S-R-F attacks. PostgreSQL nineteen is out as beta, with parallel autovacuum and a long-awaited command for atomic create-or-fetch. Angular twenty-two makes Signal Forms stable — but demands TypeScript version six. And Linux seven point one has a new NTFS driver and threw out over one hundred forty thousand lines of old code.

In the background, the European cyber-resilience timer is ticking. Chapter four went into effect June eleventh; the hard reporting obligation starts in September. And two-thirds of developers worldwide say they barely know the regulation. That's work coming for many in quarter three.
