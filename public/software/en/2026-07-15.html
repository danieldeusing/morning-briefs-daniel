npm supply chain, round two: AsyncAPI hijacked via CI

On Jul 14, Wiz's red team documented an incident that continues this week's pattern — but strikes at a different spot. The AsyncAPI generator (the code-gen tool built around the API specification standard) was taken over via a so-called "pwn request" in its GitHub Actions pipeline. The attacker opened 37 pull requests to camouflage the actual exploit, let the vulnerable workflow run its malicious code, stole the asyncapi-bot PAT, and used it to publish 4 malicious @asyncapi npm packages (5 versions). The affected packages together add up to over 3M downloads per week. The payload scanned the runner environment for secrets and exfiltrated them to a dead-drop URL on rentry.co.

"pwn request" / pull_request_target: GitHub Actions can trigger workflows when someone opens a pull request. The pull_request_target trigger runs with the target repo's secrets — intended for tasks like labeling PRs from strangers. Imagine a stranger is allowed to drop mail through your slot, and your household robot opens every envelope with your front-door key still in hand. Here the flaw was that the workflow checked out the attacker's code instead of its own base branch — and that code then ran with the full keyring.

The sharp punchline: this is the second npm hit within a week (jscrambler was the first, Jul 11, thread in the Jul 12 brief) — and both escape the defense npm is currently selling as its big move. npm v12 disables install scripts by default (covered in detail on Jul 12). Except: AsyncAPI was compromised on the publish side — malicious versions, uploaded with stolen credentials; no postinstall anywhere in sight. And jscrambler's later payloads executed themselves on import, meaning they survived even --ignore-scripts. The script lockdown is correct and overdue — but the attackers are already a level ahead: CI pipelines, publishing tokens, the import path.

What this means in practice The perimeter question shifts from "what runs during npm install?" to "who's allowed to publish, and with which token?" Concretely: pin the lockfile and run npm ci, verify provenance/signatures, restrict pull_request_target workflows to read-only permissions with no checkout of foreign code, and trim bot PATs down to minimal scopes. npm's planned GAT restrictions (phase 1 in early August) target exactly the publishing side — the right direction.

GitHub as a single point of failure: ten-hour Actions outage

Fitting for the CI theme: on Jul 9, GitHub Actions was down from 03:29 to 13:39 UTC — a good ten hours. Per GitHub's status-page history, for most of the outage roughly 8% of workflow runs on hosted runners were delayed by more than 5 minutes, and about 2% never started at all. The cause was an unhealthy backend service for runner provisioning (data replication). The exact percentages are a secondary aggregate — the status-page detail view couldn't be read directly — so take them with a grain of salt; the order of magnitude (a multi-hour, partial outage) stands.

Less dramatic but annoying: in early July, user reports of GitHub Copilot billing and quota glitches piled up (access wrongly blocked despite a reset or raised budget). Billing glitches, not a security issue — but another reminder of how much dev-day-to-day hangs on a single vendor.

OSS politics: Bambu Lab vs. the OrcaSlicer fork

An AGPLv3 dispute from May 2026 with no confirmed new development this week: developer Pawel Jarczak maintains a fork ("OrcaSlicer-bambulab") of the AGPLv3-licensed Linux client for Bambu Studio. Bambu Lab has threatened legal action, arguing the modified software endangers its servers through manipulated metadata (user-agent spoofing → DDoS/security risk). Repair activist Louis Rossmann offered $10,000 for the defense, and Josef Prusa publicly criticized Bambu Lab. At its core, this is an AGPLv3 copyleft and right-to-repair conflict.

Note on the date: The origin is likely May 2026 (the golem.de article carries a May slug); the topic resurfaced in golem.de's ticker on Jul 14. Whether there's an actual new development this week couldn't be confirmed behind the cookie wall — so it's deliberately framed here as "still simmering," not as a fresh incident.

SAP July Patch Day: a 9.9 in the NetWeaver core [1]

On Jul 14, SAP published its July Patch Day: 16 new security notes plus a GitHub security advisory, three of them rated critical. For Daniel's environment (Java/enterprise stack, many DE clients run SAP in the background), the most important date of the week.

The frontrunner is CVE-2026-44747 at CVSS 9.9: a memory access flaw caused by a logic error in the memory management of the SAP NetWeaver Application Server ABAP — the classic core of many SAP landscapes. Plus two 9.1s:

CVE-2026-27690 (CVSS 9.1) — HTTP request smuggling in SAP Approuter before version 20.10.0.

HTTP request smuggling: When two servers (say, a proxy and the backend) interpret the boundary between two HTTP requests differently, an attacker can "smuggle in" a second, hidden request. Like a letter with two addresses in the same envelope: the mail room reads one, the recipient reads the other.

CVE-2026-44761 (CVSS 9.1) — insecure sample credentials in SAP Commerce Cloud (HY_COM 2205, COM_CLOUD 2211, and 2211-JDK21). The classic: shipped default credentials that should never have been active in production.

Source note: The figures come from the security trade press (securityonline.info, cross-checked against two further outlets); the primary SAP notes page itself wasn't directly accessible. Verify the concrete notes in the SAP Support Portal before a change window.

CISA KEV: Langflow is the first AI agent platform in the catalog [1]

CISA newly added four actively exploited flaws to the KEV catalog (FCEB deadline Jul 10). One of them stands out for our beat:

KEV (Known Exploited Vulnerabilities): CISA's list of vulnerabilities confirmed to be under active exploitation in the wild right now. US federal agencies must patch them by a deadline. For everyone else, the list is a "this is genuinely being attacked" signal — priority, not a panic button.

Langflow — CVE-2026-55255 (CVSS 6.1): an authorization bypass via a user-controlled key that lets an authenticated attacker execute other users' flows. The score is moderate, the symbolism isn't: Langflow is a low-code platform for building AI agents — making it the first AI agent platform ever in the KEV. Exactly the second-order effect this brief cares about: the agent-tooling layer itself becomes an attack surface, regardless of which model runs underneath.

The other three follow the familiar CMS-plugin-upload pattern, but at full force: Joomlack Page Builder — CVE-2026-56290 (CVSS 10.0, unauthenticated file upload → RCE) and JoomShaper SP Page Builder — CVE-2026-48908 (CVSS 10.0, unrestricted upload; fix: update to 6.6.2+). The fourth, Adobe ColdFusion (CVE-2026-48282, 10.0), already ran in the Jul 10 brief — noted here only for completeness.

Source note: The CISA advisory pages themselves returned HTTP 403 on access; the CVE details are documented via The Hacker News (which in turn cites CISA) and a cross-check — solid, but not quoted directly from the primary advisory.

PostgreSQL 19 in beta — and a CNPG addendum

PostgreSQL 19 Beta 1 has been out since Jun 4, with GA targeted for September 2026. No action pressure yet for Daniel (Postgres in the stack), but the right moment to test his own extensions and JDBC paths against the beta before the feature-freeze reality hits in fall.

Addendum, deliberately flagged as such: Current security roundups are again mentioning CVE-2026-44477 — a privilege escalation in the Kubernetes operator CloudNativePG (CVSS 9.4): the metrics exporter could escalate via RESET ROLE → COPY TO PROGRAM to OS-level RCE and superuser. This is not a new incident — the release and advisory are dated May 8, 2026, fixed in 1.29.1 / 1.28.3. If you're running CNPG in a cluster and didn't patch in May: do it now. Affected were ≤ 1.29.0 and ≤ 1.28.2.

It's Wednesday, July 15th. Today's big story isn't a new vulnerability — it's a pattern. For the second time in a week, the npm supply chain got hit, and the latest attack shows the defense is standing in the wrong place.

On July 14th, Wiz's red team documented how the AsyncAPI generator got hijacked. The attacker exploited a broken GitHub Actions pipeline, opened thirty-seven pull requests as camouflage, and stole the project bot's access token. With that, he published four malicious packages under the AsyncAPI name — together pulling over three million downloads a week.

And here's the real punchline. npm is currently rolling out version twelve, which disables install scripts by default to slow down exactly this kind of attack. Except this attack ran through the publishing side, with stolen credentials. No install script would have stopped it. The attackers are already a level ahead — inside CI pipelines and around publishing tokens.

Out of Germany comes the second big date: SAP published its July Patch Day on the fourteenth. Sixteen security notes, three of them critical. The worst one, in the NetWeaver core, scores nine point nine. Plus one nine-point-one flaw each in Approuter and Commerce Cloud.

There's a small piece of history at CISA. The US agency added Langflow to its catalog of actively exploited vulnerabilities — the first AI agent platform ever to land there. The score is a moderate six point one, but the message is clear: the tooling layer around AI agents is becoming an attack target in its own right. Also added: two Joomla page builders with the maximum score of ten.

Fitting the infrastructure theme, GitHub Actions suffered a ten-hour outage on July ninth. About eight percent of runs were delayed more than five minutes, roughly two percent never started at all. The exact numbers come from the status page and should be read with some caution — but they're a reminder of how much developer routine hangs on a single vendor.

To close, two quieter threads. PostgreSQL nineteen is in its first beta, with the finished version due in September. And the dispute between Bambu Lab and an open-source developer over an OrcaSlicer fork dates back to May — a copyleft conflict with no confirmed new update this week.
