The npm Supply Chain Is on Fire — Twice in Four Days

Supply-chain attack (import-time): Instead of breaking down your door, the attacker poisons the ingredients your package supplier ships you. New and nasty this time: the malicious code fires the moment the package is imported, not when it's installed. The classic defense, npm install --ignore-scripts, therefore does not help — the line of code runs as soon as your build loads the module.

On 14.07, the AsyncAPI organization on npm was hijacked: five package versions across four names (@asyncapi/specs, generator, generator-components, generator-helpers) were republished within roughly 90 minutes, each carrying the same injected loader. It delivered the "Miasma modular runtime" — a remote-access trojan with C2 at 85.137.53[.]71. Microsoft's MSTIC named the campaign miasma-train-p1 and classifies it as a successor to the Miasma RAT from the Red Hat incident in June. The first downstream hit (a Yarn cache) showed up at 08:49 UTC. microsoft.com

Just three days earlier, on 11.07, Jscrambler was hit: five jscrambler versions (8.14.0 through 8.20.0) plus the Webpack, Gulp, Grunt, and Metro plugins were poisoned via stolen publish credentials. Hidden native binaries harvested cloud, crypto-wallet, and AI-tool credentials. According to Rescana, 1,479 downloads went through before the packages were pulled (single source, secondhand); clean starts only at 8.22.0. rescana.com

Two independent org takeovers in four days aren't a coincidence — the npm ecosystem's trust model is the attack surface. Anyone still putting off reproducible builds, pinned lockfiles, and an internal registry mirror should start this week at the latest.

npm v12: Install Scripts Finally Off by Default

Right on cue: GitHub announced on 08.07 that npm v12 ships allowScripts=off this month. Lifecycle scripts (preinstall, install, postinstall) and implicit node-gyp builds will then run only after explicit approval; Git dependencies and remote-URL sources also require consent. Granular access tokens (GATs) lose sensitive permissions starting August 2026, and publishing restrictions kick in from January 2027. Warnings already appear as of npm 11.16.0. thehackernews.com

Context Overdue — but no silver bullet. The AsyncAPI attack fired at import, not install; --ignore-scripts or scripts-off would not have stopped this particular incident. MSTIC says so itself. The new default closes the broad postinstall hole but doesn't replace provenance checks.

Gitea Docker Image Trusts Every Proxy Header

The official Gitea Docker image hardcoded the line REVERSE_PROXY_TRUSTED_PROXIES=* in the default app.ini (CVE-2026-20896, CVSS 9.8). With reverse-proxy login enabled, that wildcard trusts every source IP: anyone who reaches the port can send an X-WEBAUTH-USER header and log in as any user, no password or token needed. Fixed in 1.26.3 (late June); the first exploit probes appeared roughly 13 days after disclosure, and about 6,200 Gitea instances are still exposed online. thehackernews.com

CISA KEV: SharePoint Zero-Day and Two FortiSandbox Flaws

Deserialization of untrusted data: A program unpacks stored data back into live objects — like unwrapping a piece of flat-pack furniture. If it trusts that data without checking it, an attacker can "wrap up" working program code instead of harmless furniture parts, and the server runs it the moment it's unpacked.

On 16.07, CISA added CVE-2026-58644 (CVSS 9.8) to its Known Exploited Vulnerabilities catalog: a deserialization flaw (CWE-502) in Microsoft SharePoint that an unauthenticated attacker can use over the network to achieve code execution. Affected are SharePoint Enterprise Server 2016 (< 16.0.5556.1005), Server 2019 (< 16.0.10417.20153), and the Subscription Edition (< 16.0.19725.20384). The deadline for US federal agencies had already expired on 19.07 — meaning the flaw was exploited before the patch. The same KEV batch also added two FortiSandbox flaws (CVE-2026-25089, CVE-2026-39808, both 9.1, OS command injection). thehackernews.com · nvd.nist.gov

Side note on the pattern: back on 07.07 and 10.07, the KEV was filled almost entirely with CMS-plugin flaws (JoomShaper SP Page Builder, Langflow, iCagenda, Balbooa) — the bulk of active exploitation still comes from the poorly maintained CMS zoo. cisa.gov

Linux KVM "Januscape": the Guest Escapes to the Host

Guest-to-host escape: A VM is like an apartment in a building, and the hypervisor is the building superintendent's control room. A guest-to-host escape means the tenant breaks out of their apartment into the control room — and now controls the whole building, including every neighboring apartment. On cloud hardware, those "neighbors" are other customers.

CVE-2026-53359 (plus companion CVE-2026-46113) is a use-after-free in KVM's shadow MMU. The catch: the vulnerable code has been in the kernel since commit 2032a93d66fa from August 2010 — 16 years. Both Intel and AMD hosts are affected. Disclosed on 06.07, the fix was merged into mainline on 19.06 and landed in stable kernels on 04.07 (7.1.3, 6.18.38, 6.12.95, 6.6.144, 6.1.177, 5.15.211, 5.10.260). SUSE rates it 8.8 (v3.1) or 9.3 (v4.0); an official NVD score was missing at publication. It requires root in the guest plus nested virtualization exposed outward — a real risk mainly for cloud and self-hosting setups running KVM. A companion flaw (CVE-2026-63807, reportedly a guest-triggered host crash in the hugepage recovery path) is less firmly established. thehackernews.com

7-Zip: Heap Overflow in the XZ Decoder

CVE-2026-14266 (CVSS 7.0) is a heap buffer overflow when unpacking crafted, chunked XZ data in 7-Zip — usable for code execution, but only with user interaction (someone has to open the archive). Versions 21.07 through 26.01 are vulnerable, fixed in 26.02 (25.06). The ZDI advisory was published on 15.07; as of 20.07 there's no exploitation in the wild and no public PoC, and the flaw isn't in the KEV. No alarm — but update 7-Zip if it sits anywhere in your build or archive path. zerodayinitiative.com

11 Forgotten UEFI Shims Undermine Secure Boot

Secure Boot & shim: Secure Boot is the bouncer at computer startup — it only lets signed boot software through. A "shim" is a small go-between, countersigned by Microsoft, so Linux can get past the bouncer too. Old, vulnerable shims are like valid IDs that were never confiscated: whoever holds one can smuggle malicious code past the bouncer.

ESET found eleven old shim bootloaders (v0.9 and earlier) that are trusted via Microsoft's third-party UEFI CA certificate and can be abused to load bootkits (such as BlackLotus) on virtually any UEFI machine — OS-independent, and possible for over a decade (CVE-2026-8863, CVE-2026-10797). Responsibly disclosed in February 2026, the shims were revoked via Microsoft's June patch day. Anyone who applies the revocations is protected. thehackernews.com

Zimbra: Stored XSS in the Classic Web Client

ZCS 10.1.19 (07.07) closes a critical stored-XSS flaw where a crafted email runs malicious code the moment it's opened and can expose mailbox, session, and account data. Found by Google's Threat Analysis Group; no CVE ID had been assigned at publication. Anyone running a self-hosted Zimbra instance: patch soon. blog.zimbra.com

Browser Emergency Patches: Firefox and Chrome

Firefox 152.0.6 (15.07) closes two critical flaws: CVE-2026-15718 (invalid pointer in the JS/WebAssembly path) and CVE-2026-15719 (site isolation/DOM navigation). Mozilla states openly that exploit code is public for the first one, though it knows of no in-the-wild attacks — the fix just needs a browser restart. In parallel, Google shipped Chrome 150.0.7871.124/.125: CVE-2026-15764 and CVE-2026-15765, both critical use-after-free bugs in the Ozone display layer (the "exploit-public" note applies only to Firefox). A two-click job, but do it today. mozilla.org · thehackernews.com

PostgreSQL 19 Beta 2

PostgreSQL 19 Beta 2 landed on 16.07. Highlights relevant to Daniel's stack: REPACK as a low-lock replacement for VACUUM FULL + CLUSTER (reorganize a table without exclusively locking the whole thing), property-graph queries via SQL/PGQ, FOR PORTION OF UPDATE/DELETE on temporal tables, and inserts up to twice as fast with foreign-key checks. GA is planned around September/October 2026 — now's the window to test it against your own test DB. postgresql.org

Oracle Quarterly CPU: Expected Today

Oracle's quarterly Critical Patch Update follows a "third Tuesday of January/April/July/October" pattern — the April CPU landed on 21.04, and 21.07 is also a Tuesday, so today. Caution on the details: Oracle's advisory page was blocked (403) at deadline, and a second source instead cites 15.07. Early roundups mention roughly 1,455 patches org-wide, of which ~20 are new for Oracle Java SE (18 remotely exploitable without authentication) — these numbers remain unconfirmed until Oracle's page is reachable again. Already active for real: the PeopleSoft chain CVE-2026-35278 × CVE-2026-35273, which ShinyHunters/UNC6240 is using for HR/payroll data theft (breach cluster on 03.07, over 100 orgs notified, 68% in higher education). threat-modeling.com

GitHub: Two Partial Outages in a Week

Double trouble for anyone running GitHub Actions pipelines: on 13.07 (13:11–13:53 UTC), 30% of Actions jobs failed to start and another 2% arrived more than five minutes late — the cause was an autoscaling config with stale capacity thresholds that scaled the service below baseline. Then on 16.07 (22:21–23:50 UTC), around 39% of REST API requests returned HTTP 500 errors because an infrastructure change incorrectly marked most of a region's API backends as "unhealthy." Both were internally caused, both were fixed quickly — but a reminder not to hang your deploy path on a single provider alone. incidenthub.cloud

Tailscale Tightens Up

Tailscale v1.98.9 clears out a batch of bulletins (TS-2026-004 through TS-2026-009). The oddest one: in TS-2026-009, a client that connected over SSH with the username -i got misinterpreted as --no-idn — getent then dumped the entire contents of /etc/passwd (starting with root), and Tailscale opened an interactive root session. The exact release date of 1.98.9 couldn't be firmly confirmed; the bulletin IDs are cross-checked against the official security bulletins. Anyone using Tailscale: update to the patched version. tailscale.com

It's Tuesday, July twenty-first. Today's non-AI software news has one clear theme: the supply chain. Two separate npm organizations were hijacked within four days, on top of a SharePoint zero-day with an expired federal deadline and a sixteen-year-old Linux bug that lets a virtual machine break out to its host.

The lead story: on July fourteenth, the AsyncAPI organization on npm was taken over. Five package versions were republished within about ninety minutes, all carrying the same malicious code that pulls down a remote-access trojan. The nasty part, as Microsoft describes it: the code fires the moment the package is imported, not when it's installed. The usual defense of disabling install scripts doesn't help here at all. And this was already the second incident of the week — three days earlier, the Jscrambler packages were hit via stolen credentials.

Fittingly, there's one piece of good news: npm is pulling the emergency brake. Version twelve turns off install scripts by default starting this month. That closes a wide hole, though not the import-time hole from the AsyncAPI attack. Progress, not a cure-all.

Among the vulnerabilities, the SharePoint zero-day stands out. The US cyber agency CISA added it to its catalog of actively exploited flaws, with the maximum severity score of nine point eight. The deadline for US federal agencies had already expired on July nineteenth. The same batch also added two FortiSandbox flaws scoring nearly as high, at nine point one.

The most striking one for developers is the Linux bug nicknamed Januscape. It's been sitting in the kernel since August two thousand ten — picture a VM as an apartment and the host as the building's control room; here the tenant breaks into the control room and takes over the whole building. On cloud hardware, the neighbors are other customers. The stable kernels have been patched since July fourth.

On top of that came emergency patches for Firefox and Chrome, with public exploit code already circulating for the Firefox flaw. PostgreSQL nineteen is in its second beta and brings, among other things, REPACK, a low-lock replacement for the old VACUUM FULL. And eleven ancient boot go-betweens, countersigned by Microsoft, have been exploitable for over a decade to bypass Secure Boot — found by ESET, revoked via the June patch day.

On the side: GitHub had two partial outages this week, one in Actions and one in the REST API, both caused by internal configuration errors. And Oracle's quarterly patch bundle is expected today — I'll leave the exact fix count open, since Oracle's own page was blocked at the time of writing.

What to watch next: npm twelve's rollout later this month, and, on July twenty-seventh, the European Commission's final ruling on Google's compliance under the Digital Markets Act. Tomorrow, another look at the supply chain and patch status.
