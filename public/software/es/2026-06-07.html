Los lanzamientos de modelos IA de la semana (Claude Opus 4.7, MAI-Code-1 de Microsoft, Gemini 3.5 Flash) pertenecen al brief ai-dev; aquí es plataforma, runtime, lenguajes, infra y ley.

  • Infra EU DE GitHub Enterprise Local: GHES en Azure Local, ahora también air-gapped — para entornos regulados/on-prem.

Microsoft Construye Hardware Dev de Nuevo: Surface RTX Spark Dev Box [1][2]

La mayor sorpresa de Build 2026 no fue una característica de software, sino hardware: Microsoft anuncia con la Surface RTX Spark Dev Box su primer dispositivo construido específicamente para desarrolladores en años.[1] En el núcleo está el superchip "RTX Spark" de NVIDIA — una CPU Grace (ARM) más GPU RTX Blackwell — que según Microsoft entrega 1 petaflop de cómputo IA y 128 GB de Unified Memory. La promesa: ejecutar o hacer fine-tune en modelos de 120 mil millones de parámetros con contexto de 1 millón de tokens localmente, sin factura de GPU en la nube.[1]

Unified Memory: Normalmente CPU y GPU tienen memoria separada y deben copiar datos constantemente de un lado a otro — como dos chefs con despensas separadas pasándose ingredientes. Unified Memory es una despensa compartida: CPU y GPU acceden al mismo pool. Para modelos IA gigantes, ese es el truco crucial, porque un modelo de 120 mil millones de parámetros no cabría en la memoria GPU de otra forma.

El gabinete es un bloque monolítico de aluminio con aproximadamente 1.000 agujeros de ventilación, funcionando simultáneamente como disipador — diseñado para trabajos largos de entrenamiento sin throttling.[2] Está construido con propósito: Windows 11 Pro con PowerShell 7 como shell predeterminada, WSL 2 con passthrough GPU y CUDA, VS Code, Git, Python, Node — utilizable directo de la caja.[1]

La trampa: el precio y el cronograma exacto están faltando. Microsoft solo dice "este año, en EE.UU., exclusivo vía microsoft.com" y una lista de espera — Nadella bromeó que él mismo está en la fila.[2] Los analistas estiman USD 3.000–3.500; para comparación, el DGX Spark de NVIDIA ahora cuesta USD 4.699 (elevado de 3.999 en febrero por escasez de memoria), y el Ryzen AI Halo PC de AMD, USD 3.999.[2] Lo que importa: esta es la respuesta de Microsoft a la tendencia Mac Studio/DGX Spark — estaciones de trabajo IA locales para devs. Menos interesante como objeto de compra que como señal — Windows está siendo posicionado como una plataforma dev de primera clase, incluido hardware ARM con GPU seria.

Coreutils for Windows: Comandos Unix Nativos — y una Declaración de Licencia [3][4]

Desde Build, Coreutils for Windows está disponible en general: 75 comandos Linux familiares (ls, cp, grep, find, xargs, du, uptime …) se ejecutan nativamente en Windows, instalables vía WinGet, agrupados con findutils y un grep compatible GNU.[3] Para quien salta entre servidor Linux y laptop Windows, eso es alivio real de fricción: los scripts de shell se vuelven más portátiles sin necesidad de iniciar WSL primero.

La parte interesante está en la nota de pie: Microsoft no construye sobre los coreutils GNU clásicos, sino en uutils — una reimplementación en Rust multiplataforma — y eso está licenciado MIT, mientras que los originales GNU son GPL.[4]

MIT vs. GPL: Ambas son open source, pero con carácter diferente. GPL es "comparte por igual": si pones código GPL en tu producto y lo distribuyes, debes divulgar tu propio código en los mismos términos — un adhesivo para libertad. MIT es "haz lo que quieras": incluye, modifica, incluso coloca en productos cerrados, sin obligación de devolución. Para una empresa como Microsoft, MIT es mucho más conveniente — sin riesgo de "infección" GPL incrustada en el SO.

Que un proveedor de plataforma gigante intercambie un kit de herramientas GPL por una variante MIT y la meta en el SO es parte de una tendencia OSS más grande (rewrites Rust, licencias permisivas). No impactante para el día a día de Daniel, pero un marcador de hacia dónde van las herramientas de sistema. Tropiezo práctico: comandos como dir, more, paste, whoami están propositalmente afuera (colisión con built-ins Windows), y cosas POSIX como chmod, chown, chroot, nohup faltan completamente.[3]

WSL Containers + Windows Limpio: el Puente Linux Se Hace Más Ancho [5]

Dos anuncios menores pero operacionalmente relevantes: WSL Containers (WSLC) traen una forma integrada, tipo Docker, de iniciar containers Linux vía CLI y API directo en WSL — incluyendo passthrough GPU para cargas de trabajo IA locales. Estado: "pronto" en Preview Público.[5] Esto llena un vacío entre "WSL para una shell" y "Docker Desktop completo" y puede ser una configuración de containers local más ligera.

Junto a eso, Windows Developer Configurations ahora están GA: un dev-config.winget más scripts que hacen un Windows fresco listo para dev en minutos — Explorer limpio, tema oscuro, sin widgets, sin "sugerencias", sin popups.[5] Pequeño, pero cualquiera que haya despojado una máquina Windows para trabajo sabe que ahorra tiempo real.

PostgreSQL 19 Beta 1 Llegó [7]

El momento de base de datos de la semana: El PostgreSQL Global Development Group lanzó PostgreSQL 19 Beta 1 el 04.06.[7] Esto significa que el conjunto de características para la próxima versión major está efectivamente congelado — lo que está en la beta va (excepto bugs) al lanzamiento final, planeado para septiembre de 2026.

Beta / Feature-Freeze: Hasta la beta, nuevas características aún pueden llegar; a partir de beta, es estabilización, pruebas, correcciones de bugs — como un restaurante que fija su menú y ahora solo practica cocinar los platillos perfectamente. Beta va en un entorno de prueba, no tu base de datos de producción: el formato on-disk aún puede cambiar hasta GA.

En la práctica: cualquiera que mantenga extensiones customizadas, ORMs o scripts de migración debe probar contra beta ahora — ese es el punto entero de la fase. En el stack de Daniel (Spring/FastAPI en PostgreSQL), una mirada temprana a mejoras de query planner y logical replication tiene sentido antes del GA de septiembre. No sueltes esto en datos de producción.

Magento RCE en Lista KEV: CVE-2026-45247 [8][9]

La alerta de seguridad única, pero pesada: CISA añadió CVE-2026-45247 al catálogo KEV el 03.06 — una ejecución remota de código sin autenticación activamente explotada con CVSS 9.8 en el Mirasvit Full Page Cache Warmer, extensión de cache ampliamente usada para Magento 2 / Adobe Commerce.[8]

Lista KEV: El catálogo "Known Exploited Vulnerabilities" de la agencia EE.UU. CISA es el rol de brechas para las cuales hay actividad de ataque real confirmada en el campo — no "podría teóricamente ocurrir" sino "está siendo usado ahora". Algo en la lista no es más una situación "parcha cuando tengas tiempo".

El mecanismo es clásico: la extensión mete parte de la cookie CacheWarmer sin validar directo en unserialize() de PHP, sin restricciones en qué clases pueden ser instanciadas — un gadget de inyección de objeto PHP lo hace, una cookie elaborada en cualquier request normal de la tienda es suficiente para ejecución de código, sin login necesario.[9] Versiones afectadas antes de la 1.11.12; el parche está disponible desde 25.05. El plazo para agencias federales EE.UU. fue 06.06. Sansec (que reportó primero) señala como indicación fuerte una cookie que comienza con CacheWarmer: seguida por Tz, Qz o YT; Imperva reporta intentos activos de exploit.[9] Si ejecutas una tienda Magento/Adobe Commerce con esta extensión: actualiza.

Capa Infra: GitHub Air-Gapped, Azure Linux 4.0, HorizonDB [11][12]

Del rincón de infra de Build, fácil de perder en el ruido IA, tres cosas que importan para setups regulados y on-prem:

GitHub Enterprise Local (Preview) trae GitHub Enterprise Server a Azure Local — conectado o air-gapped, con runners auto-hospedados.[11] Para gobierno, bancos y cualquier cosa con reglas estrictas de soberanía de datos (muy relevante para DE/EU) "GitHub sin internet" es un pitch serio.

Air-gapped: Un sistema con cero conexión internet — físicamente separado, como si hubieras cortado los cables de red. Usado para entornos altamente sensibles (militar, infraestructura crítica) donde incluso un camino teórico de ataque online es demasiado.

Azure Linux 4.0 (Public Preview) es la distro propia de Microsoft, nueva con base Fedora; ya alimenta millones de cores en AKS, Azure SQL y Cosmos DB.[12] Y Azure Container Linux (GA), la próxima iteración Flatcar, apunta a hosts mínimos optimizados para container. Ambas son movimientos estándar de plataforma Linux — sin ángulo IA.

En el lado de base de datos, Azure HorizonDB (Preview) flanquea el día Postgres: servicio administrado compatible con PostgreSQL con resiliencia de zona y almacenamiento elástico hasta 128 TB.[12] Significado: Microsoft se posiciona aún más fuerte como casa Postgres — relevante si Daniel alguna vez evalúa Postgres administrado en Azure.

Plataforma Web: WebMCP & HTML-in-Canvas en Origin Trial [13]

Del carryover de web standards de Google I/O (mayo), ahora en implementación: Chrome 149 lanza un Origin Trial para WebMCP, un estándar web abierto propuesto que expone funciones JavaScript y formularios HTML como "tools" estructurados para que agentes de navegador ejecuten tareas con más confiabilidad.[13]

Origin Trial: La forma de Chrome de probar una nueva característica de navegador, aún no estandarizada, en producción real. Un sitio se opt-in y recibe un token que desbloquea la característica solo para sus visitantes — como un beta test para una web API, time-limited y sin rollout global.

Junto con eso, la HTML-in-Canvas API (también Origin Trial): elementos reales del DOM pueden embutir en un Canvas con WebGL/WebGPU y permanecer buscables y accesibles — previamente una elección difícil entre "lindamente renderizado" y "utilizable".[13] Para el trabajo frontend de Daniel (Angular/React), ambos son señales tempranas, no características de producción — pero la dirección es clara: la plataforma navegador se está preparando para agentes y UIs dirigidas por GPU.

Regulación EU: Reloj CRA Corre Hacia 11.06 [14]

Hilo permanente, aún vivo: El Cyber Resilience Act alcanza un hito administrativo en 11.06 — para entonces la autoridad designadora (en Alemania, el BSI) debe haber establecido y publicado los procedimientos por los cuales los evaluadores de conformidad son evaluados, nombrados y notificados.[14]

Cyber Resilience Act (CRA): Ley EU requiriendo todos los productos "con elementos digitales" — desde lámparas smart-home hasta software B2B — sean construidos, mantenidos y reportados como seguros a lo largo de su vida. Una especie de requisito de inspección de seguridad para productos de software conectados.

Para devs solo y casas de software pequeñas, aún no es un plazo de entrega, pero la carrera hacia la navaja real: de 11.09.2026 en adelante, un deber de notificación de 24 horas entra en vigor para vulnerabilidades activamente explotadas e incidentes severos en productos conectados (primera notificación dentro de 24 h, follow-up dentro de 72 h).[14] Si vendes software en la EU, prepara tu proceso de notificación ahora, no en septiembre.
