La API pública de GitHub se convierte en herramienta de espionaje corporativo

Hoy no hay un bug espectacular aislado, sino un patrón: reconocimiento y negligencia. La API pública de GitHub está siendo usada de forma deliberada como herramienta de espionaje corporativo, Nextcloud GmbH dejó una base de datos interna expuesta en internet, y un hallazgo del CCC muestra cómo una porción entera del mercado de microinversores transmite su contraseña de radio en texto plano. A eso se suma que hoy vence un plazo de la CISA, y el frente de lanzamientos está lleno: OpenSSH 10.4, una actualización enorme de Chrome y npm v12, que por fin cierra la cadena de suministro.

  • Sec Enumeración en GitHub: más de 50 cuentas "fantasma" y tokens robados llevan desde octubre mapeando organizaciones y repositorios corporativos — un repositorio privado fue clonado de forma comprobada (Datadog).
  • Sec Nextcloud GmbH: 367,000 registros internos (~8 GB) quedaron expuestos en internet — un Elasticsearch corporativo mal configurado, no un bug de producto ni de cliente.
  • Sec Inversores Hoymiles transmiten su contraseña de radio sin cifrar (CCC) — el fabricante atiende cerca del 20% del mercado europeo de microinversores.
  • Sec RCE en SharePoint on-prem (CVE-2026-45659, CVSS 8.8) está en el catálogo KEV desde el 1 de julio — ejecución de código autenticada vía deserialización.
  • Sec Contraataque de GitHub: Secret Scanning "Public Monitoring" para enterprises — en preview, sin costo adicional, escanea todo github.com en tiempo real.
  • Release OpenSSH 10.4 (6 de julio): ocho correcciones, entre ellas bugs de redirección en SFTP/SCP, más firmas poscuánticas experimentales.
  • Infra npm v12 (julio) bloquea por defecto los scripts de instalación y las dependencias de git y remotas — la respuesta directa al año de la cadena de suministro.

  • Release Git 2.55 y Ubuntu Arm64: repacks más rápidos y el comando `git history fixup`; el live patching de kernel llega también a Arm64.

GitHub: cuando la API pública se convierte en espía corporativo

La historia de seguridad más interesante de la semana no es un zero-day, es un método. Datadog Security Labs documentó una campaña activa desde octubre de 2025 que rastrea sistemáticamente la API pública de GitHub para mapear organizaciones corporativas y sus repositorios. Se confirmaron más de 50 cuentas "fantasma" repartidas en varios user agents; la actividad ocurre en ráfagas de una a tres semanas y golpea a muchas empresas a la vez, sobre todo a través del endpoint de GraphQL. Los user agents usados tienen nombres bastante reveladores, como GitHub-Company-Scraper, GitHubAnalytics/1.5 o GitHarvester.

Reconocimiento con cuentas fantasma: imagina a un ladrón que camina sin llamar la atención por la calle durante meses antes de actuar — nadie recuerda su cara. Eso es exactamente lo que hacen estas cuentas: según Datadog, tienen entre dos y cinco años de antigüedad antes de activarse para el scraping. "Una cuenta con un historial de varios años parece más legítima que una registrada la misma semana en que empieza a hacer scraping", dice Julie Agnes Sparks, ingeniera de Datadog. Los metadatos públicos de un repositorio son inofensivos por sí solos — juntos, arman un mapa completo de tu empresa.

Dos detalles llevan esto más allá de la simple recolección de metadatos. Primero, una subcampaña paralela con tokens robados: entre finales de diciembre de 2025 y principios de enero de 2026, decenas de cuentas legítimas distintas de GitHub (tokens OAuth/PAT comprometidos) golpearon a una sola organización en cuestión de minutos. Segundo — y esta es la parte que duele — un user agent llamado repo-dumper ejecutó clones de git y llamadas a la API dentro de un repositorio privado. Eso ya no es enumeración, es exfiltración confirmada.

Quién está expuesto y qué hacer: cualquier organización con repositorios públicos o metadatos visibles públicamente aporta material de reconocimiento — nada evita eso, es justo el sentido de "público". La palanca real está en los tokens: inventariar los PATs y las concesiones OAuth, reducirlos al mínimo privilegio, rotar los tokens viejos o demasiado amplios, activar el secret scanning y monitorear ráfagas inusuales de llamadas a la API. El análisis de CSO Online describe todo esto como una mezcla de herramientas de escaneo, abuso oportunista de credenciales y redes coordinadas de cuentas desechables — no un solo actor.

Fuentes: securitylabs.datadoghq.com · csoonline.com

Contraataque de GitHub: secret scanning para toda la superficie pública

En sintonía con la historia de la enumeración, GitHub lanzó el 1 de julio una función defensiva en preview público: "Secret Scanning Public Monitoring for Enterprises". En lugar de revisar solo sus propios repositorios, GitHub ahora monitorea "toda la superficie pública de github.com en tiempo real en busca de secretos filtrados" y atribuye cada hallazgo al enterprise afectado. Sin costo adicional para clientes con GitHub Secret Protection. Esto cierra justo el hueco que explota la subcampaña de tokens mencionada arriba: un secreto que se filtra de un contexto privado a cualquier repositorio público se detecta antes de que un repo-dumper pueda usarlo.

Nextcloud GmbH: 367,000 registros internos quedaron expuestos en internet

Importante aclarar de entrada, para que el titular no se malinterprete: esto no es un bug del software Nextcloud ni tampoco una instancia de cliente hackeada. Lo afectado fue la TI interna de la propia Nextcloud GmbH. Según deskmodder, la empresa tenía una base de datos interna con cerca de 367,000 registros temporalmente accesible libremente desde internet. El clúster de Elasticsearch expuesto públicamente sumaba casi ocho gigabytes.

Elasticsearch expuesto en internet: Elasticsearch es una base de datos de búsqueda — práctica precisamente porque se puede consultar por HTTP. Justo eso se vuelve la trampa cuando el puerto queda apuntando a internet sin autenticación: como dejar el archivero en la banqueta con el cajón abierto. Un clásico error de configuración, no una explotación.

Investigadores de seguridad de Cybernews descubrieron el incidente el 18 de mayo de 2026; tras el aviso, Nextcloud GmbH cerró el acceso en dos días, y desde el 27 de mayo el conjunto de datos ya no está disponible públicamente. El contenido era delicado: cerca de 71,000 PDFs, 53,000 PNGs y 23,000 archivos Markdown, entre ellos facturas, contratos, correos internos, datos de empleados y scripts de proyectos de clientes — según Cybernews, algunos con credenciales de base de datos incrustadas. La lección es poco espectacular, y justo por eso resulta molesta: los clústeres de búsqueda deben estar detrás de autenticación y en la red interna, punto. (El reporte original de Cybernews devolvió un error 403 al momento de la consulta; las cifras provienen de la cobertura en alemán.)

CISA KEV: cuatro agregados nuevos — el plazo vence hoy

El punto más urgente de hoy: para cuatro fallas que la CISA agregó al catálogo KEV el 7 de julio, el plazo de parcheo de las agencias federales de EE. UU. termina el 10 de julio — o sea, hoy. El cuarteto lo encabeza CVE-2026-48282, una falla de path traversal en Adobe ColdFusion que, según securityaffairs, fue explotada apenas horas después de darse a conocer. A eso se suman CVE-2026-48908 (JoomShaper SP Page Builder, subida de archivos sin restricción), CVE-2026-55255 (Langflow, bypass de autorización a través de una llave controlada por el usuario) y CVE-2026-56290 (Joomlack Page Builder, control de acceso defectuoso) — todas por explotación activa.

Catálogo KEV: la lista "Known Exploited Vulnerabilities" de la agencia estadounidense CISA — fallas con evidencia de que en este momento están siendo explotadas activamente. Para las agencias federales de EE. UU. es un plazo de parche obligatorio; para todos los demás, un "esta primero" muy claro. Estar en el KEV significa: ya no es teórico.

Langflow es un constructor de flujos de trabajo para LLM — pero la noticia aquí es el CVE, no la función de IA; la capa de herramientas también es superficie de ataque. Quien opere ColdFusion no debería posponer el parche: una falla de path traversal ya explotada en el mundo real es la combinación más incómoda que existe. (La página de alerta de CISA devolvió un error 403 al consultarla directamente; los cuatro CVE y el plazo de hoy se verificaron con securityaffairs.)

Fuente: securityaffairs.com

RCE en SharePoint on-prem (CVSS 8.8) en el catálogo KEV

Una semana antes, la CISA ya había fijado el ancla más dura: CVE-2026-45659, una ejecución remota de código en SharePoint on-prem con puntaje base CVSS de 8.8 (alto). Mecanismo según NVD: la deserialización de datos no confiables permite a un atacante autenticado ejecutar código a través de la red. Solo afecta a las versiones on-prem — SharePoint Server Subscription Edition, 2019 y Enterprise 2016; Microsoft ya había publicado un parche fuera de ciclo en mayo. CISA agregó la falla al catálogo KEV el 1 de julio, con plazo del 4 de julio para las agencias federales. Quien todavía opere una granja on-prem y no haya aplicado el parche de mayo debería ponerse al día — "autenticado" no es un gran obstáculo en un servidor de colaboración interno.

Fuente: nvd.nist.gov

Los inversores Hoymiles transmiten su contraseña en texto plano

Un hallazgo del entorno del CCC que va más allá del mundo puramente del software, pero es un pecado de diseño digno de libro de texto. El investigador del CCC "Hunz" (Benedikt Heinz) mostró que los microinversores Hoymiles de las series HM/HMS/HMT intercambian su contraseña de acceso por radio — los últimos ocho dígitos del número de serie — sin cifrar por las bandas de frecuencia de 868 MHz y 2.4 GHz. Hoymiles asegura que atiende cerca del 20% del mercado europeo de microinversores, así que el problema es amplio.

Microinversor: la cajita detrás de cada panel solar individual que convierte la corriente directa del panel en corriente alterna apta para la red eléctrica. Se configura por radio — y justo ese canal de radio está aquí prácticamente sin candado.

La demostración muestra lo práctico que resulta esto: un escáner portátil modificado bastó para ubicar dos docenas de inversores ajenos, junto con sus IDs únicos, en 20 minutos. Como el alcance de la radio llega a varios cientos de metros, ese hardware de ataque podría montarse sin problema en un dron, según heise — con lo cual un atacante podría interrumpir el suministro de barrios enteros. El contacto con el fabricante no ha ido bien: se dice que el CCC y el BSI (la autoridad alemana de seguridad) llevan desde febrero de 2026 intentando avisar a Hoymiles, sin parche hasta ahora (esta parte de la cronología se apoya en reportes secundarios, no en una fuente primaria verificada directamente). Para quienes están expuestos, hay poco que ajustar en el corto plazo — el endurecimiento de la seguridad de radio no es una actualización de firmware que se instale sin más. En la práctica: conciencia, presión sobre el fabricante y, en caso de duda, mantener los equipos afectados en la lista de vigilancia.

Fuente: heise.de

Chrome: 382 correcciones de un jalón, 15 críticas

A finales de mes (30 de junio), Google lanzó una actualización estable difícil de pasar por alto: 382 correcciones de seguridad, 15 de ellas clasificadas como "Critical", en la versión 150.0.7871.46/.47 (Win/Mac) y .46 (Linux). Lo más destacado es CVE-2026-13789, un use-after-free en el componente de GPU que, mediante HTML preparado, puede llevar a un escape de sandbox.

Escape de sandbox: el navegador encierra cada página web en un proceso aislado — el sandbox — para que no llegue al resto del sistema. Un escape de sandbox es la fuga de esa celda: un código que en teoría solo debería correr dentro de la pestaña de pronto accede al sistema operativo. Por eso los bugs de GPU y de renderizado son tan delicados.

Aquí basta con una simple actualización automática — pero conviene comprobar que sí se aplicó, en lugar de darlo por hecho. El 8 de julio llegó otra actualización estable para escritorio; al cierre de esta edición todavía no había notas detalladas y confiables al respecto.

Fuente: malwarebytes.com

npm v12 bloquea los scripts de instalación — respuesta al año de la cadena de suministro

El lanzamiento más relevante del mes para quien tiene un package.json: npm v12 se espera para julio de 2026 según el changelog de GitHub, y eleva bastante la seguridad por defecto. En concreto: "npm install ya no ejecutará los scripts preinstall, install o postinstall de las dependencias, a menos que estén permitidos explícitamente." Además, las dependencias de git (solo con --allow-git) y las dependencias de URL remota, como los tarballs HTTPS (--allow-remote), quedan bloqueadas por defecto.

Esta es la respuesta directa al patrón detrás de los ataques a la cadena de suministro de npm este año: un hook postinstall que se dispara justo cuando alguien escribe npm install. El daño colateral es limitado — solo una pequeña parte de los paquetes realmente necesita scripts de instalación (típicamente módulos nativos vía node-gyp). Para el CI de Daniel, esto significa: antes de actualizar conviene revisar qué dependencias realmente necesitan scripts y definir una lista de permitidos — así nada se rompe en julio.

OpenSSH 10.4: ocho correcciones y una probadita de poscuántico

OpenSSH 10.4 (también conocido como 10.4p1) salió el 6 de julio y trae ocho correcciones relevantes para la seguridad. Las dos más interesantes tienen que ver con la transferencia de archivos: en una descarga SFTP por línea de comandos (sftp host:/path .), un servidor malicioso podía redirigir el archivo a una ubicación inesperada; en una copia SCP de servidor a servidor, un servidor malicioso podía escribir en el directorio padre del destino. A eso se suma un bug de sshd en el que las líneas de comando largas para internal-sftp se truncaban silenciosamente después del noveno argumento — lo que podía hacer que se perdieran opciones relevantes para la seguridad — y un use-after-free del lado del cliente.

Firma poscuántica: los esquemas de firma actuales se consideran quebrables en cuanto existan computadoras cuánticas potentes. Un esquema compuesto combina uno clásico (Ed25519) con uno resistente a ataques cuánticos (ML-DSA-44): la firma se mantiene válida mientras al menos uno de los dos esquemas sea seguro — un cinturón de seguridad con un segundo broche.

Eso es justo lo que la versión 10.4 ofrece como soporte experimental y opcional: un esquema de firma combinado ML-DSA-44 más Ed25519. Todavía no es para producción, pero es una señal clara de hacia dónde va todo. Para el día a día de Linux/SSH de Daniel, la 10.4 es un "vale la pena actualizar" sin mayor drama.

Fuente: openssh.org

Git 2.55: repacks más rápidos y un `git history fixup`

Un poco más antiguo (29 de junio), pero vale la pena ponerse al día: Git 2.55. Según el post de lanzamiento de GitHub, lo más destacado es el repacking incremental del multi-pack index (mantenimiento más rápido en repositorios grandes), un nuevo git history fixup, ejecución paralela de hooks y agrupamiento (batching) en el partial clone. Son mejoras puramente de rendimiento y ergonomía, sin urgencia de seguridad — pero el git history fixup y los hooks paralelos son el tipo de comodidad que rápidamente ya no querrás perder en el día a día.

Ubuntu: live patching de kernel en Arm64 — retroceso en coreutils en Rust

Canonical extendió su live patching de kernel — correcciones de seguridad aplicadas al kernel en ejecución, sin reiniciar — a Arm64, tanto en el Ubuntu 26.04 regular como en el Ubuntu Core 26 inmutable (The Register, 9 de julio). Una ventaja operativa real para servidores Arm y dispositivos de borde (edge). La reescritura de coreutils en Rust no salió tan bien: el cp en Rust procesaba mal la bandera -L, por lo que el equipo volvió, por ahora, al cp clásico de GNU basado en C. La meta de "100% coreutils en Rust" no está descartada, pero se aplaza al Ubuntu 26.10 — una señal sana de que una regresión de verdad dispara un rollback en lugar de dejarse pasar sin más.

Fuente: theregister.com

Tema recurrente — EU CRA: obligación de reporte a partir del 11 de septiembre

La semana de seguridad también deja claro, de paso, por qué el próximo hito regulatorio no es un tigre de papel: a partir del 11 de septiembre de 2026 entra en vigor la obligación de reporte del Artículo 14 del EU Cyber Resilience Act — alerta temprana en 24 horas, reporte en 72 horas e informe de seguimiento para vulnerabilidades explotadas activamente e incidentes graves. Las multas llegan hasta 15 millones de euros o el 2.5% de los ingresos anuales globales. Cada uno de los eventos de hoy — desde el SharePoint en el KEV hasta el clúster expuesto — caería bajo ese reloj a partir de entonces. La fecha deliberadamente no está en la línea de tiempo de 7 días (todavía faltan dos meses), pero ya debe estar en la hoja de ruta de cualquier producto con mercado en la UE. (Plazo según análisis de un despacho legal, fecha sólida; la interpretación detallada sigue en movimiento.)

Fuente: crowell.com

Hoy es viernes diez de julio. Hoy no hay un solo bug espectacular, sino un patrón: reconocimiento y negligencia. Voy a repasar una semana de seguridad que muestra cuánto se puede averiguar sobre una empresa sin entrar nunca a la fuerza — y cuántas veces la puerta ya estaba abierta de todos modos. Y, de paso, hoy también vence un plazo de parcheo.

La historia central viene de Datadog Security Labs. Desde octubre del año pasado, una campaña coordinada rastrea la interfaz de programación pública de GitHub para mapear organizaciones corporativas y sus repositorios. Se confirmaron más de cincuenta cuentas llamadas "fantasma". Lo perverso es que estas cuentas tienen entre dos y cinco años de antigüedad antes de activarse — un historial largo simplemente parece más legítimo que una cuenta registrada la misma semana en que empieza a hacer scraping.

Dos cosas llevan esto más allá de la simple recolección de datos. Entre finales de diciembre y principios de enero, decenas de cuentas reales, pero robadas, de GitHub golpearon a una sola organización en cuestión de minutos. Y una herramienta llamada repo-dumper clonó, de forma comprobada, datos desde dentro de un repositorio privado. Esto ya no es teoría, es exfiltración confirmada. La palanca de defensa está en los tokens: inventariarlos, reducirlos a lo esencial, rotar los antiguos. En sintonía con esto, GitHub lanzó una contramedida en preview el primero de julio, que rastrea toda la superficie pública de github.com en tiempo real en busca de secretos filtrados.

La segunda gran historia es una fuga de datos en Nextcloud GmbH. Importante: esto no es una falla del software Nextcloud ni una instancia de cliente hackeada, sino la propia base de datos interna de la empresa. Cerca de trescientos sesenta y siete mil registros, casi ocho gigabytes, quedaron expuestos en internet por una base de datos de búsqueda Elasticsearch mal configurada. Facturas, contratos, correos internos, datos de empleados. Descubierto en mayo, cerrado en dos días. La lección es banal, y justo por eso resulta molesta: los clústeres de búsqueda deben estar detrás de autenticación, punto.

Del mundo del hardware llega un hallazgo del Chaos Computer Club que muestra qué tan mala puede ser la seguridad de radio. Los microinversores Hoymiles — las cajitas detrás de los paneles solares — envían su contraseña de radio sin cifrar. Un escáner portátil encontró dos docenas de equipos ajenos en veinte minutos. Un alcance de varios cientos de metros hace pensable incluso un ataque con dron, capaz de interrumpir el suministro de barrios enteros. Hoymiles atiende cerca del veinte por ciento del mercado europeo, y todavía no hay parche.

En los catálogos de vulnerabilidades, hoy lo que destaca es el plazo. Para cuatro fallas que la agencia estadounidense CISA clasificó como explotadas activamente el siete de julio, el plazo de parcheo de las agencias federales vence justo hoy. El cuarteto lo encabeza una falla de path traversal en Adobe ColdFusion, explotada apenas horas después de darse a conocer. Una semana antes ya destacaba una falla de SharePoint: una ejecución de código con una gravedad de ocho punto ocho, presente solo en las versiones on-premise.

El frente de lanzamientos está lleno esta semana. OpenSSH diez punto cuatro trae ocho correcciones, incluidos dos molestos bugs de redirección en SFTP y SCP, y, mirando hacia adelante, un modo experimental de firma poscuántica. Chrome lanzó de un jalón trescientas ochenta y dos correcciones de seguridad, quince de ellas críticas — lo más destacado es un bug de GPU con potencial de escape de sandbox. A eso se suma Git dos punto cincuenta y cinco, con repacks más rápidos y un nuevo history-fixup, y Ubuntu ahora también trae live patching de kernel a Arm64.

Para anotar: la versión doce de npm llega en julio y bloquea por defecto los scripts de instalación y las dependencias de git y remotas — la respuesta directa al año de la cadena de suministro. Quien prepare hoy una lista de permitidos para los pocos paquetes que de verdad necesitan scripts no tendrá sorpresas en julio. Y el Reglamento Europeo de Resiliencia Cibernética trae, a partir del once de septiembre, una obligación de reporte en veinticuatro y setenta y dos horas, con multas de hasta quince millones de euros.

Mañana es fin de semana, así que el flujo de noticias debería calmarse. La semana que viene voy a ver si Hoymiles reacciona y si la versión doce de npm realmente se lanza. Hasta entonces.
