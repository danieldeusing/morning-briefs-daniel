Mini Shai-Hulud: um worm npm que se assina de forma válida [1][2][3]

A maior história de software da semana não é um release, mas uma falha no modelo de confiança. O grupo "TeamPCP" lançou um worm auto-propagável chamado Mini Shai-Hulud através dos registros npm e PyPI em 11 de maio — aproximadamente 169 nomes de pacotes npm com mais de 370 versões maliciosas, incluindo nomes conhecidos como @tanstack/* (TanStack Router serviu ~12 milhões de downloads por semana), @mistralai/*, e mais de 70 pacotes @uipath.[1] Rastreado sob CVE-2026-45321, CVSS 9,6.[2]

SLSA Provenance: Imagine que cada pacote receba um recibo selado provando "este artefato veio exatamente desta pipeline de compilação verificada". O worm emitiu esse recibo para si mesmo — sequestrou uma pipeline real do GitHub Actions usando tokens OIDC roubados e gerou atestados válidos de Build Level 3 com ele. O recibo é genuíno, o conteúdo é veneno. É o primeiro worm npm documentado a produzir pacotes maliciosos validamente atestados.[3]

Mecânica: O worm procura por um token npm com bypass_2fa = true (publicado sem segundo fator), depois consulta todos os pacotes do mesmo mantenedor e os contamina mais. TanStack rastreou a violação inicial até uma cadeia de pull_request_target, envenenamento de cache do GitHub Actions, e extração de memória de um token OIDC do processo do runner — três lacunas, nenhuma das quais sozinha teria sido suficiente.[1]

Atenção Consequência prática para CI de Daniel: assinaturas Provenance indicam qual pipeline compilou algo, não se se comportou corretamente. Se instalar lockfiles npm com --ignore-scripts, manter tokens OIDC de vida curta, e impor 2FA em tokens de publicação, você tira as três alavancas do worm. Tokens que podem ter sido expostos na onda devem ser rotacionados.

Typosquatting ataca segredos CI/CD diretamente [4]

Combinando com a onda de worm, uma segunda história de cadeia de suprimentos que Microsoft analisou em 28 de maio: Um único ator (alias vpmdhaj) publicou 14 pacotes maliciosos em quatro horas, falsificando bibliotecas OpenSearch, ElasticSearch e DevOps (nomes como opensearch-setup, elastic-opensearch-helper).[4]

A volta: O harvester de credenciais ~195-KB roda silenciosamente em install e ataca AWS IAM/STS, tokens HashiCorp Vault, segredos GitHub Actions, e tokens npm de publicação — exatamente as chaves que comprometem a cadeia de suprimentos ainda mais adiante. O conselho da Microsoft espelha o acima: npm install --ignore-scripts, e rotacione todos os tokens de CI que possam ter tocado em um runner afetado ou máquina de dev.[4]

PAN-OS GlobalProtect: prazo CISA encerra hoje [6][7]

GlobalProtect da Palo Alto tem um bypass de autenticação, CVE-2026-0257, CVSS 9,1. Um atacante não-autenticado falsifica cookies "Authentication-Override" e constrói um túnel VPN não autorizado para a rede interna.[6] Rapid7 detectou os primeiros ataques começando em 17 de maio (primeira onda 18 de maio de infraestrutura Vultr, segunda começando 21 de maio). CISA adicionou a lacuna ao catálogo KEV em 29 de maio com prazo de agência 1º de junho — hoje.[7]

Afetado: PAN-OS 10.2.x, 11.1.x, 11.2.x, e 12.1.x, além de deployments Prisma Access. Qualquer um executando gateways GlobalProtect expostos sem patches deve atualizar para a versão corrigida de sua branch (ex., 10.2.13-h18).

Catálogo KEV: Lista CISA de "vulnerabilidades conhecidas exploradas". Agências dos EUA devem corrigir entradas pelo prazo informado — para todos os outros, uma listagem KEV é o sinal mais forte de que "isto está sendo explorado ativamente agora", não apenas teoricamente possível.

Dirty Frag: bug kernel de 9 anos, PoC público [8][9]

Em circulação desde 8 de maio, mas continua uma tarefa em andamento: "Dirty Frag" (CVE-2026-43284 + CVE-2026-43500, mais CVE-2026-46300 em RHSB-2026-003 da Red Hat) é uma escalação de privilégio local no caminho de descriptografia in-place de esp4/esp6 e rxrpc. A porção ESP foi introduzida em 2017, a porção RxRPC em 2023 — a lacuna é antiga.[8]

Crítico: Um PoC funcional é público, e qualquer usuário local não-privilegiado alcança root com um único comando. O exploit precisa apenas de syscalls padrão e módulos (esp4, esp6, rxrpc) que são enviados habilitados em pacotes kernel padrão em todas as principais distros enterprise — Ubuntu, RHEL, Alma, Fedora, openSUSE. Microsoft relatou exploração ativa pós-comprometimento. Patches estão linkados de NVD e disponíveis das distros; KernelCare entregou patches ao vivo em 24 horas.[9]

Avaliação Nenhum vetor remoto — requer um pé na porta já. Mas precisamente por isso é um componente clássico de "escalação de privilégio após acesso inicial" que você corrige em toda caixa Linux (incluindo ddmini/ddStudio) via atualizações regulares de distro.

BSI avisa sobre lacuna 7-Zip [10]

Da cena alemã: O BSI alertou em 26 de maio sobre a vulnerabilidade 7-Zip CVE-2026-48095 (CVSS 8,8, descoberta por GitHub Security Lab). Um contêiner NTFS preparado deixa o deslocamento de tamanho do stream-buffer rodar para expoente 32 — o buffer é mal-alocado para 1 byte, o próximo read dispara um heap overflow, que permite execução de código.[10] Insidioso: A lógica de fallback de assinatura também roteia arquivos com extensões .7z, .zip, ou .rar através do parser NTFS. A correção é 7-Zip 26.01 (fim de abril); tudo através de 26.00 é afetado.

Angular 22 sai esta semana — a era "signal-first" [11][12]

Para o lado frontend de Daniel, o release da semana: Angular 22 está agendado para a semana de 1º de junho (RC.1 está em circulação desde 20 de maio).[11] Consolida o que vem sendo construído desde Angular 17:

  • Zoneless como padrão para novos projetos, novos componentes padrão OnPush.
  • Signal Forms estável — o sistema de formulário baseado em signal experimental em 21 substitui o padrão classic Reactive Forms por uma API mais leve.
  • Selectorless Components: componentes podem ser importados diretamente em templates sem uma string de selector.
  • Vitest é o novo test runner padrão; caminhos experimentais Jest/web-test-runner são removidos.
Zoneless / Signals: Anteriormente, Angular escutava via Zone.js cada evento, depois verificava toda a árvore de componentes para mudanças — como um zelador que caminha por todo o prédio após cada ruído. Signals são pequenos contêineres que notificam Angular quando seu valor muda — Angular atualiza apenas os lugares que realmente dependem daquele signal. Menos trabalho, UI mais rápida, nenhum overhead Zone.js.
Avaliação Se você está em Angular 21, o salto deve ser relativamente suave — o corte duro (signals, standalone, control flow) está atrás de nós. Reactive Forms ficam por enquanto, mas para novo trabalho, Signal Forms vale explorar.

Release de coleta PostgreSQL corrige 11 lacunas [13]

Manutenção de banco de dados: PostgreSQL 18.4, 17.10, 16.14, 15.18, e 14.23 foram lançados em 14 de maio — 11 correções de segurança mais de 60 bugfixes.[13] Três são relevantes para setups PostgreSQL de Daniel: uma verificação de autorização ausente em CREATE TYPE (um criador de objetos pode sequestrar queries estrangeiras que resolvem tipos via search_path); um vazamento de timing encoberto na comparação de senha MD5 (não afeta scram-sha-256, o padrão); e recursão descontrolada em negociação SSL/GSS que permite DoS contínuo via socket AF_UNIX.

Chrome 148 corrige duas lacunas RCE críticas [14]

Higiene de navegador: A atualização Chrome de 19 de maio (versão 148.0.7778.178/179) corrige múltiplas lacunas, duas críticas — um use-after-free em WebRTC (CVE-2026-9111, RCE via visita de página preparada) e uma implementação UI falha em Windows. Nenhuma foi explorada ativamente antes do patch. Qualquer um em sistemas gerenciados ou manualmente atualizados não deveria esperar pelo rollout escalonado.[14]

Contagem regressiva CRA: obrigação de relato entra em vigor em setembro [15]

Fio de regulação em andamento: O EU Cyber Resilience Act ativa obrigações de relato em 11 de setembro de 2026 — lacunas ativamente exploradas e incidentes graves em "produtos com elementos digitais" devem ser relatados via Single-Reporting-Platform em 24 horas (aviso prévio) ou 72 horas, com relato final devido em 14 dias de remediação disponível.[15] Também se aplica retroativamente a produtos já no mercado. Praticamente, para quem distribui software na EU: configure SBOM e um processo de vulnerabilidade/VEX agora, não em setembro. Obrigações de produto completo entram em vigor 11 de dezembro de 2027.

É segunda-feira, primeiro de junho. O noticiário hoje gira em torno de cadeias de suprimentos de software — e de uma linha que foi ultrapassada esta semana.

Um grupo de atacantes chamado TeamPCP lançou um worm auto-propagável chamado Mini Shai-Hulud através dos registros npm e PyPI. Aproximadamente cento e sessenta e nove nomes de pacotes foram contaminados, incluindo nomes bem conhecidos como TanStack, Mistral e UiPath. O que é realmente novo: o worm entregou seus pacotes maliciosos com uma assinatura de autenticidade válida. Esta chamada Provenance deveria provar que um pacote veio de uma pipeline de compilação verificada — e o worm emitiu essa prova para si mesmo ao sequestrar uma pipeline real com tokens roubados. É o primeiro caso documentado desse tipo. Quem pode ter sido apanhado na onda deve rotacionar seus tokens de CI e instalar pacotes com a opção ignore-scripts.

Uma segunda história de cadeia de suprimentos, analisada pela Microsoft no final de maio, se encaixa perfeitamente: catorze pacotes falsos falsificando bibliotecas OpenSearch e Elastic que especificamente direcionam tokens AWS e Vault — exatamente as chaves usadas para comprometer a cadeia ainda mais.

Enquanto isso, um prazo encerra hoje. A agência de cibersegurança dos EUA CISA adicionou uma vulnerabilidade ativamente explorada no GlobalProtect VPN da Palo Alto à sua lista — um bypass de autenticação com severidade de nove ponto um, observado desde dezessete de maio. Agências devem estar corrigidas hoje. E no kernel Linux, Dirty Frag permanece atual: uma escalação de privilégio antiga para a qual existe um proof of concept público. Patches estão disponíveis; distribua-os.

No lado mais brilhante: Angular vinte e dois deve sair esta semana. Torna a arquitetura baseada em signal e zoneless o padrão, Signal Forms tornam-se estáveis, e Vitest substitui o antigo test runner. PostgreSQL fechou onze lacunas de segurança em meados de maio, e Chrome está na versão cento e quarenta e oito com duas correções críticas.

Em background, o relógio está marcando para o Cyber Resilience Act europeu: a partir de onze de setembro, há obrigação de relatar lacunas exploradas. Qualquer um entregando software na EU deveria configurar SBOM e um processo de vulnerabilidade agora.
