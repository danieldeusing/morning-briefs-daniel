Miasma: um verme npm que se propaga via binding.gyp

A história da semana não é um lançamento de modelo, é um incidente de cadeia de suprimentos — e um com mecânica inovadora. Um verme npm autorreplicante chamado Miasma atacou em uma onda no dia 3 de junho e de acordo com StepSecurity comprometeu 57 pacotes npm em mais de 286 versões maliciosas em menos de duas horas. Primeiro atingiu @vapi-ai/server-sdk (mais de 408 mil downloads/mês) às 23:30 UTC, depois incluindo ai-sdk-ollama (mais de 120 mil/mês).[1]

binding.gyp: Um arquivo de configuração que permite pacotes npm compilar componentes C/C++. Pense nele como as instruções de montagem que um móvel executa automaticamente quando desembalado — normalmente inofensivo. Miasma usa um arquivo de 157 bytes com substituição de comando gyp para executar código durante npm install — completamente contornando os scripts de lifecycle package.json usuais que cada scanner observa. O ponto cego é o mecanismo em si.

O payload coleta credenciais multi-nuvem amplamente: chaves AWS e tokens IMDS, contas de serviço GCP, identidades Azure, segredos GitHub Actions (incluindo raspagem de memória do runner), tokens npm, credenciais Vault e SSH. Com tokens roubados, o verme então publica pacotes infectados adicionais — daí "verme" em vez de apenas "roubo".[1]

Linhagem: Miasma é derivado do malware Mini-Shai-Hulud de acordo com análise da Microsoft, que um ator colocou em código aberto em 12 de maio — essencialmente um kit de ferramentas que múltiplos atores podem agora reconstruir.[2] Quem pode ser afetado: verifique seu lockfile, bloqueie scripts de instalação em CI com --ignore-scripts, rotacione tokens.

Red Hat confirma: 32 pacotes npm comprometidos

A primeira grande onda Miasma atingiu a própria Red Hat em 1º de junho: sob o namespace @redhat-cloud-services, 96 versões em 32 pacotes foram comprometidas — totalizando cerca de 116.991 downloads/semana (RHSB-2026-006).[3] São bibliotecas frontend do Hybrid Cloud Console. O ponto de entrada foi uma conta GitHub comprometida usada para injetar código malicioso passando por revisão de código.

O comunicado da Red Hat é instrutivo: nenhuma release do Console foi publicada durante o incidente, e o processo de publicação remove scripts de instalação antes da implantação — a camada protetora se manteve. Para clientes finais "nenhuma ação necessária". Mas quem puxou os pacotes diretamente do npm não está protegido por essa mesma parede.[3]

Contexto Em paralelo, Microsoft relatou uma campanha separada: 33 pacotes npm usando confusão de dependência para espionar ambientes de desenvolvedores.[4] Dois ataques independentes em uma semana — a camada de registro é o ponto quente agora, não o código dentro.

Quando a assinatura mente: provenance SLSA como disfarce

A parte mais desagradável do Miasma não é o roubo, é o disfarce. O verme troca tokens roubados por certificados de assinatura genuínos do Sigstore de Fulcio e cria entradas no log de transparência Rekor — os pacotes reinfectados parecem legitimamente construídos, com provenance SLSA válida.[1]

Provenance SLSA: Um recibo assinado provando "este pacote foi construído neste pipeline a partir deste commit". Projetado como um selo de autenticidade — como um carimbo notarial. Miasma mostra: quem rouba o notário pode forjar cada carimbo. O selo prova origem, não benignidade.

Consequência prática para nós: "tem provenance" não é uma passagem livre. A corrente é apenas tão forte quanto seu detentor de token mais fraco. O que permanece valioso é profundidade — hashes fixados, bloqueio de script de instalação, e ceticismo em relação a pacotes carregando arquivos não-entry-point desproporcionalmente grandes.

Atenção Miasma também descarta arquivos backdoor em repositórios — .claude/setup.mjs, .cursor/rules/setup.mdc, .vscode/tasks.json. Ele envenena a camada de ferramentas através da qual código é gerado, não apenas o código em si. Audite esses diretórios também.[1]

Build 2026, a metade não-IA: Windows se torna amigável para Linux para desenvolvedores

As manchetes do keynote foram para os modelos — eles vivem no brief de ai-dev. Para nós mais interessante é a metade da plataforma, e é substancial desta vez.

Coreutils para Windows (GA)

Microsoft está disponibilizando mais de 75 ferramentas de linha de comando estilo Unix nativamente no Windows — ls, cat, cp e mais sem WSL, sem gambiarras Git Bash. Elas vêm do projeto de código aberto uutils, uma reimplementação em Rust do GNU Coreutils.[5] Para scripts multiplataforma, redução real de atrito.

Containers WSL (Preview)

Containers Linux agora podem ser criados e executados diretamente do WSL com sua própria CLI/API e gerenciamento de políticas empresariais. Isso visa abertamente reduzir a dependência do Docker Desktop e outros.[6] Desde que WSL foi disponibilizado como código aberto na Build 2025, Microsoft relata mais de 200 pull requests por mês sendo recebidos — o projeto está vivo.[5]

Terminal Inteligente (experimental)

Um terminal que detecta comandos falhados, mostra contexto e sugere correções — via um Protocolo de Capacidade de Agente (ACP). Interessante, mas experimental; anotado aqui como contexto de plataforma, a substância do agente é território de ai-dev.[6]

  2. blogs.windows.com

Azure Linux 4.0 & Cobalt 200: A sala de máquinas Linux da Microsoft

Azure Linux 4.0 está em preview público — e é a primeira distro de servidor da Microsoft declarada como de propósito geral, não apenas um host de container mais. Agora é baseada em Fedora, código aberto no GitHub (azurelinux), com footprint de pacote menor e política SELinux de negação padrão mais rigorosa.[7]

No lado do hardware, a geração de VM Cobalt 200 entrou em preview de acesso antecipado: geração Arm de segundo servidor da Microsoft (SoC em Arm Neoverse V3), que de acordo com Microsoft traz até 50% mais desempenho de geração versus Cobalt 100.[8] Para quem executa workloads ARM na nuvem, esse é o número chave da semana.

RTX Spark Dev Box & GitHub Enterprise Local

Dois anúncios de hardware/infraestrutura que importam para o debate local-versus-nuvem. A Surface RTX Spark Dev Box é uma estação de desenvolvimento compacta com silício NVIDIA RTX Spark, 1 petaflop de computação e 128 GB de unified memory — suficiente para executar modelos além de 120 bilhões de parâmetros localmente, sem contas de API. Disponível mais tarde em 2026, inicialmente apenas nos EUA via Microsoft.com; preço ainda a ser determinado.[6]

Mais relevante infraestruturalmente: GitHub Enterprise Local, uma variante GitHub Enterprise que funciona no Azure Local e pode ser operada conectada ou isolada — com inferência on-prem via Foundry Local. Para ambientes regulados ou orientados por soberania (agências da EU, por exemplo) um sinal; preço e timing ainda abertos.[9]

CISA KEV: Falha WebLogic de dois anos fica quente

CISA adicionou CVE-2024-21182 em Oracle WebLogic Server ao catálogo Known-Exploited-Vulnerabilities em 1º de junho — a falha é ativamente explorada. CVSS 7,5 (HIGH), acesso à rede não autenticado via T3/IIOP, afeta versões 12.2.1.4.0 e 14.1.1.0.0. Oracle a corrigiu no CPU de julho 2024; está sendo explorada contra atrasados.[10][11]

Catálogo KEV: Lista da CISA de falhas com exploração ativa comprovada na selvagem. Não "teoricamente arriscada", mas "sendo usada agora mesmo". Agências federais dos EUA devem corrigir dentro do prazo — aqui era ontem, 04.06. Para nós é um bom sinal de priorização, mesmo sem o mandato.

A exposição é real: de acordo com Security Affairs, mais de 1.592 servidores WebLogic estão online e vulneráveis (961 em 12.2.1.4.0, 631 em 14.1.1.0.0).[12] Executando Fusion Middleware: verifique o status do seu CPU.

Mais adições KEV: cgroups Linux & framework Android

CISA adicionou mais dois em 2 de junho: CVE-2022-0492 (kernel Linux, escalação de privilégio cgroups, CVSS 7,8 per NVD) — um atacante local pode escapar de um container — e CVE-2025-48595 (framework Android, integer overflow, CVSS 8,4, Android 14–16). Em 3 de junho veio CVE-2026-45247 (Mirasvit Full Page Cache Warmer, desserialização insegura de dados).[13][14]

A falha cgroups é aquela para nós: se você executa containers em kernels antigos, alinhe seu nível de patch — escape de container não é um risco periférico.

Nota CISA.gov está retornando 403 para crawlers agora; os detalhes de adição vêm dos títulos de páginas de alerta CISA e são verificados em relação a securityaffairs/thehackernews e NVD (CVSS WebLogic direto do NVD).

EU/DE: O aparato CRA é acionado

Thread de regulação em andamento: em 11 de junho as disposições para nomear órgãos de avaliação de conformidade sob o Cyber Resilience Act entram em vigor — na Alemanha, o BSI como autoridade notificadora publica os procedimentos.[15] Parece burocrático, mas é a pista em que as obrigações CRA para fabricantes de produtos conectados ficam concretas a partir de 2027.

O prazo mais importante para desenvolvedores vem no outono: começando em 11 de setembro, o dever de notificação para falhas ativamente exploradas entra em vigor — dentro de 24 horas.[15] Multas por não conformidade: até EUR 15 milhões ou 2,5% da receita anual global; o BSI está contratando 141 posições até 2029 para aplicá-la.[16]

Releases no espelho retrovisor: Node 24 & PostgreSQL 18.4

Tranquilo na frente de linguagem/DB, mas dois releases de maio valem a pena notar. Node.js 24 (21.05) traz a proposta TC39 Explicit Resource Management nativamente via V8 13.6 — a palavra-chave using para limpeza determinística de recursos, similar a try-with-resources em Java.[17]

PostgreSQL 18.4 (14.05) é uma atualização pura de segurança/manutenção da linha 18 — 11 correções de segurança e mais de 60 bugfixes, nenhum novo recurso.[18] Os destaques reais vieram com PostgreSQL 18.0 (set 2025): subsistema I/O assíncrono (até 3× em cargas pesadas de I/O), colunas geradas virtuais, e uuidv7() nativa — a última economiza muitos de geradores personalizados para IDs classificáveis por tempo. Se você está na 18, atualize para 18.4 pelas correções de segurança.

É sexta-feira, cinco de junho. A semana de software gira em torno da cadeia de suprimentos — e de uma conferência Microsoft Build cujos elementos mais interessantes não têm nada a ver com inteligência artificial.

No coração está um verme npm chamado Miasma. Em uma única onda em três de junho, StepSecurity relata que ele comprometeu cinquenta e sete pacotes em menos de duas horas. O que é novo: ele se propaga via um arquivo de compilação chamado binding.gyp e executa seu código durante a instalação — completamente contornando os lugares onde os scanners de segurança normalmente olham. Ele rouba credenciais de nuvem da Amazon, Google, Azure e GitHub e as usa para publicar o próximo lote de pacotes infectados.

A primeira grande onda atingiu a própria Red Hat em primeiro de junho: noventa e seis versões em trinta e dois pacotes, totalizando cerca de cento e dezessete mil downloads por semana. A camada protetora da Red Hat se manteve, mas o incidente mostra como a camada de registro está vulnerável agora.

O mais desagradável é o disfarce. O verme troca tokens roubados por assinaturas genuínas via Sigstore — e dá aos seus pacotes provenance SLSA válida. A lição para nós: uma assinatura prova de onde um pacote vem, não se é benigno.

Na Microsoft Build, a metade de plataforma foi substancial. Windows recebe mais de setenta e cinco comandos Unix nativamente, escritos em Rust. WSL agora pode executar containers Linux diretamente sem Docker Desktop. E Azure Linux quatro ponto zero é a primeira verdadeira distribuição de servidor de propósito geral da Microsoft, agora baseada em Fedora.

Na segurança, CISA adicionou uma falha Oracle WebLogic de dois anos atrás à sua lista de vulnerabilidades ativamente exploradas, com uma gravidade de sete vírgula cinco. Mais de mil e quinhentos servidores estão vulneráveis online. O prazo para agências era ontem, quatro de junho — atrasados devem patchar agora.

E na Europa o aparato do Cyber Resilience Act continua: começando onze de junho, o BSI alemão nomeia os órgãos de avaliação. O prazo mais importante para desenvolvedores chega em setembro, quando o requisito de relatório de vinte e quatro horas para falhas exploradas entra em vigor.

Amanhã será mais tranquilo — sessões de Build permanecem disponíveis sob demanda, e continuamos observando avisos npm para ver se Miasma envia outra onda. De volta à visão geral.
